1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 장바구니에 담긴 상품, 옵션, 혜택, 관심상품을 확인하고 주문 가능한 구매안으로 정리하는 기준을 정의한다.

• 대상 업무는 통합 장바구니 조회, 처리 단위 구분, 상품·세트 변경, 조합 검증, 주문 전환 준비, 비로그인 복원, 로그인 이관, 관심상품 저장·공유, 변동 복구를 포함한다.

• 대상 채널은 앱·웹 FO이며 상품 기준 정보, 고객·회선 조건, 재고, 가격·혜택, 주문 가능성 판정을 위한 주문 연계 시스템 회신을 포함한다.

• 상품 상세의 담기 버튼 기준과 주문서 작성, 결제 승인, 배송·개통 이행, 환불·취소 처리는 인접 정책서에서 상세화한다.

• API 필드, DB 컬럼, 운영자 화면 구성, 추천 알고리즘 상세는 제외하되 고객 판단과 주문 가능 여부에 영향을 주는 기준은 정책 항목으로 남긴다.

• 액세서리, 휴대폰·스마트기기, 인터넷·IPTV 탭의 수량·금액·가격 명세는 고객의 주문 가능성 판단에 필요한 범위까지만 포함한다.

• 함께쓰기 요금제처럼 모회선 인증이 필요한 구성은 주문 전 검증 흐름에 포함하고, 인증 화면 상세와 제출 서식은 후속 설계에서 상세화한다.

• 채널 업무 시스템은 장바구니 표시, 옵션 변경, 만료·삭제 안내, 주문 전환 경로를 책임지고 주문 연계 시스템은 가격·재고·가입 가능성·구성 충돌·BSS 기준 정보 검증을 책임진다.

• 고객 표시 상태는 담기 가능, 구성 확인 필요, 가격 변경, 품절·삭제, 주문 전환 가능으로 안내하고 내부 상태는 재고 검증, 가격 재산정, BSS 자격 확인, 예약 만료 이력으로 분리한다.

• 포함 범위는 장바구니 조회, 옵션·수량 변경, 구성 충돌 확인, 가격·재고 변동 확인, 주문 전환 가능성 판정, 만료·삭제 복구 기준이다.

• 제외 범위는 결제 승인, 배송·개통 이행, 교환·반품, 해지·환불 상세 처리이며 해당 결과가 주문 전환 가능성에 영향을 주는 경우에만 정책 항목으로 연결한다.

• 핵심 정책 질문은 누가 장바구니를 유지할 수 있는가, 언제 주문 전환이 가능한가, 몇 번까지 재검증·복구할 수 있는가, 어떤 조건이면 분리 주문·삭제·상담 전환되는가, 실패하면 어떤 상태와 안내를 남기는가이다.

나. 설계 원칙

• 구매안 통합: 고객 화면은 하나의 장바구니로 제공하되 실제 주문·결제·배송·발급 단위는 명확히 구분한다.

• 주문 전 검증: 조합 가능성, 고객 조건, 재고, 가격·혜택 변동은 주문서 진입 전에 먼저 확인한다.

• 선택 상태 유지: 비로그인, 로그인, 다른 기기, 상담 전환 후에도 고객의 선택 상품과 비교 맥락을 최대한 유지한다.

• 변동 사전 고지: 담은 시점 이후 달라진 가격, 혜택, 재고, 판매 상태, 가입 조건은 고객이 조치하기 전에 알려야 한다.

• 셀프 복구 우선: 구매 불가, 분리 주문, 만료, 충돌 항목은 삭제만 제공하지 않고 대체, 재검증, 단독구매, 상담 전환을 함께 제공한다.

• 운영 통제 가능성: 제공 범위, 한도, 보존, 이관, 조합 정책, 변동 모니터링 기준은 운영자가 이력 기반으로 관리한다.

• 담기 후 이탈 방지: 고객이 상품을 담은 직후 쇼핑 계속, 장바구니 이동, 주문 전환 중 다음 행동을 선택할 수 있게 한다.

상태 전이표에서 사용하는 상태명은 상태 코드 목록의 상태명과 일치해야 한다.

1) 상태 코드

상태 코드상태명정의대표 후속 처리
ST-CRT-001진입 가능고객이 홈, 상품 목록, 상품 상세, MY 등 주요 경로에서 장바구니로 진입할 수 있는 상태다.
고객에게 전체 구매안과 처리 단위 구분을 먼저 보여준다.
ST-CRT-002임시 보관비로그인 또는 식별 전 고객의 선택 상품이 브라우저·디바이스 기준으로 임시 저장된 상태다.
로그인 전에도 조회와 일부 정리는 허용하되 계정 저장 전 제한을 안내한다.
ST-CRT-003계정 보관로그인 고객의 장바구니가 계정 기준으로 저장되어 다른 기기와 재방문에서 복원 가능한 상태다.
비로그인 장바구니가 있으면 이관·병합 기준을 안내한다.
ST-CRT-004구매안 구성 중고객이 상품, 옵션, 세트, 관심상품, 추천 항목을 조정하며 주문 후보를 구성하는 상태다.
구성 변경 시 가격, 혜택, 처리 단위를 다시 계산한다.
ST-CRT-005검증 필요상품 조합, 고객 조건, 재고, 혜택, 선행 가입 조건을 다시 확인해야 하는 상태다.
주문 연계 시스템 판정 결과를 받아 주문 가능 여부를 갱신한다.
ST-CRT-006주문 전환 가능선택한 장바구니 항목이 주문서로 이동 가능한 상태다.
주문으로 이동하기 전 적용 조건과 예상 비용을 확정 안내한다.
ST-CRT-007분리 주문 필요한 화면에서 함께 보이지만 실제 주문이나 결제가 상품 유형별로 분리되어야 하는 상태다.
분리 이유와 각 처리 단위의 다음 행동을 안내한다.
ST-CRT-008단독구매 필요다른 상품과 함께 주문할 수 없고 별도 구매 경로로 진행해야 하는 상품이 포함된 상태다.
현재 구매안을 유지하면서 해당 상품의 단독 구매 경로를 제공한다.
ST-CRT-009변경 확인 필요담은 시점 이후 가격, 혜택, 재고, 판매 상태, 가입 가능 조건이 달라진 상태다.
변경 전후 차이와 대체 행동을 고객에게 알려준다.
ST-CRT-010수량 한도 초과로그인 기준 장바구니 최대 보관 수량을 초과해 추가 담기가 제한된 상태다.
기존 항목 삭제 또는 주문 전환 후 다시 담도록 안내한다.
ST-CRT-011구매 불가단종, 판매중단, 재고 없음, 조건 불충족 때문에 장바구니 항목을 주문할 수 없는 상태다.
구매 버튼을 제한하고 삭제, 대체 상품, 상담 전환을 제공한다.
ST-CRT-012만료 예정장바구니 보존 기간이 임박해 고객에게 삭제 예정 고지가 필요한 상태다.
삭제 예정일과 복원 가능 범위를 안내한다.
ST-CRT-013자동 삭제 완료보존 기간 초과 또는 주문 완료로 장바구니 항목이 자동 삭제된 상태다.
삭제 이력을 저장하고 주문하지 않은 항목은 유지한다.
ST-CRT-014상담 전환 필요고객이 조합 충돌, 고관여 상품, 재고·조건 문제를 스스로 해결하기 어려운 상태다.
장바구니 문맥과 실패 사유를 상담사 또는 AI 지원에 전달한다.
ST-CRT-015주문 완료 반영장바구니 일부 또는 전체 항목이 주문·가입 완료되어 장바구니 정리가 필요한 상태다.
완료된 세트는 삭제하고 남은 세트는 계정 장바구니에 유지한다.
ST-CRT-016인증 필요함께쓰기 요금제 또는 회선 관계 확인이 필요한 장바구니 구성이 주문 전환 전에 모회선 인증을 요구하는 상태다.
고객에게 필요한 인증 사유와 인증 완료 후 이어질 주문 가능성 검증을 안내한다.
ST-CRT-017장바구니 비어 있음고객의 장바구니에 주문 후보 상품이 없어 상품 탐색 또는 로그인 안내가 필요한 상태다.
로그인 상태에 따라 복원 가능 여부와 상품 보러가기 경로를 안내한다.

2) 상태 전이 기준

현재 상태전이 이벤트다음 상태처리 기준 및 후속 처리
진입 가능장바구니 구매안 통합 관리구매안 구성 중고객이 주요 진입점에서 장바구니를 열면 전체 구매안과 처리 단위를 보여주고 구성 중 상태로 전환한다.
진입 가능장바구니 이어보기와 복구임시 보관식별 전 고객의 이전 선택 상태가 브라우저·디바이스에 남아 있으면 임시 보관 상태로 복원한다.
임시 보관장바구니 이어보기와 복구계정 보관고객이 로그인하고 이관·병합 기준을 확인하면 계정 장바구니로 전환한다.
계정 보관장바구니 상품 구성 변경과 정리구매안 구성 중고객이 상품, 옵션, 세트, 관심상품을 변경하면 구매안 구성 중 상태로 갱신한다.
구매안 구성 중조합 검증과 주문 전환 준비검증 필요주문 대상 선택, 세트 변경, 추천 추가, 단독구매 항목 포함 시 조합 검증이 필요하다.
검증 필요상품·고객 조건 판정 회신주문 전환 가능상품 조합, 고객 조건, 재고, 혜택, 선행 가입 조건이 모두 충족되면 주문 전환 가능 상태로 바꾼다.
검증 필요상품·고객 조건 판정 회신분리 주문 필요결제 방식, 배송 유형, 상품 유형이 달라 한 번에 주문할 수 없으면 분리 주문 필요 상태로 바꾼다.
검증 필요상품·고객 조건 판정 회신단독구매 필요다른 상품과 동시 주문이 불가한 항목은 단독구매 필요 상태로 분리한다.
검증 필요상품·고객 조건 판정 회신구매 불가단종, 판매중단, 재고 없음, 가입 불가 조건이면 구매 불가 상태로 전환한다.
구매안 구성 중가격·혜택·재고 변동 회신변경 확인 필요담은 시점과 조회 시점의 가격, 혜택, 재고, 판매 조건이 달라지면 변경 확인 필요 상태로 바꾼다.
계정 보관장바구니 제공 기준 운영수량 한도 초과로그인 기준 보관 상품·세트가 50개를 초과하면 추가 담기를 제한한다.
계정 보관보존·이관·만료 정책 운영만료 예정최초 담은 날짜로부터 30일에 가까워진 항목은 만료 예정으로 고지한다.
만료 예정보존·이관·만료 정책 운영자동 삭제 완료최초 담은 날짜로부터 30일을 초과한 항목은 순차 자동 삭제한다.
주문 전환 가능조합 검증과 주문 전환 준비주문 완료 반영고객이 주문·가입 완료 후 완료 결과를 받으면 완료된 세트만 장바구니에서 삭제한다.
변경 확인 필요장바구니 문제 상담 전환상담 전환 필요고객이 변동, 충돌, 재고 문제를 직접 해결하지 못하면 상담 전환 필요 상태로 바꾼다.
구매안 구성 중장바구니 함께쓰기 요금제 모회선 인증 제공인증 필요함께쓰기 요금제나 회선 관계 확인이 필요한 상품이 선택되면 주문 전환 전에 모회선 인증 필요 상태로 전환한다.
인증 필요장바구니 함께쓰기 요금제 모회선 인증 제공검증 필요모회선 인증이 완료되면 가입유형별 인증 결과를 기준으로 상품·고객 조건 재검증을 수행한다.
구매안 구성 중장바구니 상품 없음 시 로그인/비로그인 안내 제공장바구니 비어 있음조회 결과 주문 후보 상품이 없으면 로그인 고객은 복원 가능성을, 비로그인 고객은 상품 보러가기와 로그인 후 복원 가능성을 안내한다.

나. 정책 상세

1) 함께쓰기 인증·권한 정책 (PG-CRT-AUTH-001)

• 모회선 인증 (PI-CRT-AUTH-001-01)
- 함께쓰기 요금제 또는 회선 관계 확인이 필요한 상품은 주문 전환 전에 모회선 인증을 완료해야 한다.
- 인증 완료 전에는 주문 전환 가능 상태로 변경하지 않는다.
• 가입유형별 인증 (PI-CRT-AUTH-001-02)
- 가입유형별로 필요한 본인확인, 회선 권한 확인, 대리 권한 확인 결과가 충족된 경우에만 상품·고객 조건 재검증을 수행한다.
• 인증 실패 후속 (PI-CRT-AUTH-001-03)
- 모회선 인증이 실패하거나 권한 확인이 불충족되면 인증 필요 상태를 유지하고 재시도, 상품 제외, 상담 전환 중 가능한 후속 행동을 안내한다.
- 실패 사유와 재시도 이력은 저장한다.

2) BSS·연계 판정 정책 (PG-CRT-BSS-001)

• 판정 항목 (PI-CRT-BSS-001-01)
- 주문 연계 시스템은 고객 상태, 회선 상태, 상품 상태, 판매 가능 기간, 재고, 혜택 적용 조건, 선행 가입 조건을 판정한다.
• 응답 지연 (PI-CRT-BSS-001-02)
- 연계 판정이 3초 안에 완료되지 않으면 장바구니 항목을 검증 필요 상태로 유지하고 고객에게 일시 지연 안내와 재시도 버튼을 제공한다.
• 연계 실패 (PI-CRT-BSS-001-03)
- BSS 또는 외부 연계 실패가 2회 연속 발생하면 주문 전환을 제한하고 상담 전환 또는 나중에 다시 시도 경로를 안내한다.

3) 장바구니 한도·보존 정책 (PG-CRT-CAPACITY-001)

• 최대 수량 (PI-CRT-CAPACITY-001-01)
- 로그인 고객 기준 장바구니에 담을 수 있는 전시 상품 또는 마케팅프로그램 세트는 최대 50개로 제한한다.
- 50개 초과 시 기존 항목 삭제 또는 주문 전환 후 추가하도록 안내한다.
• 보존 기간 (PI-CRT-CAPACITY-001-02)
- 장바구니 세트는 최초 담은 날짜부터 30일 동안 보존한다.
- 30일을 초과하면 오래된 세트부터 순차 자동 삭제한다.
• 만료 안내 (PI-CRT-CAPACITY-001-03)
- 자동 삭제 예정 세트는 삭제 전 고객에게 만료 예정일, 삭제 대상, 복구 가능 여부를 안내한다.
- 복구 불가 세트는 대체 상품 탐색 또는 다시 담기 경로를 제공한다.

4) 가격·혜택·재고 변동 정책 (PG-CRT-CHANGE-001)

• 변동 고지 (PI-CRT-CHANGE-001-01)
- 담은 시점 이후 가격, 혜택, 재고, 판매 상태, 가입 가능 조건이 달라지면 변경 전 값과 변경 후 값을 구분해 안내한다.
• 대체 행동 (PI-CRT-CHANGE-001-02)
- 변동 항목에는 재검증, 대체 상품 보기, 삭제, 상담 전환 중 최소 1개 이상의 후속 행동을 제공한다.
• 자동 제거 (PI-CRT-CHANGE-001-03)
- 판매중단, 단종, 재고 없음으로 구매 불가가 확정된 세트는 주문 전환을 제한한다.
- 자동 제거는 고객 고지 후 수행하고 삭제 이력을 저장한다.

5) 상담·AI 지원 전환 정책 (PG-CRT-CONSULT-001)

• 전환 조건 (PI-CRT-CONSULT-001-01)
- 고관여 상품, 조건 충돌, 결합 판단, 재고 이슈, 반복 검증 실패가 있으면 장바구니 문맥을 유지한 상담 또는 AI 지원 전환을 허용한다.
• 문맥 전달 (PI-CRT-CONSULT-001-02)
- 상담 전환 시 선택 세트, 충돌 사유, 검증 결과, 가격·혜택 요약, 고객이 마지막으로 선택한 행동을 전달한다.
• 상담 후 복귀 (PI-CRT-CONSULT-001-03)
- 상담이나 AI 지원 후 고객이 복귀하면 기존 장바구니 상태를 유지하고 상담에서 제안한 대체 상품 또는 분리 주문 경로를 함께 표시한다.

6) 삭제·자동삭제 정책 (PG-CRT-DELETE-001)

• 직접 삭제 (PI-CRT-DELETE-001-01)
- 고객은 장바구니 상품 또는 세트를 직접 삭제할 수 있다.
- 삭제 전에는 세트 전체 삭제인지 일부 구성 삭제인지 구분해 확인한다.
• 완료 세트 삭제 (PI-CRT-DELETE-001-02)
- 주문 또는 가입이 완료된 세트는 장바구니에서 자동 삭제한다.
- 주문하지 않은 세트와 관심상품은 그대로 유지한다.
• 삭제 이력 (PI-CRT-DELETE-001-03)
- 고객 직접 삭제, 자동삭제, 주문 완료 삭제는 삭제 사유, 대상 세트, 처리 시각, 복구 가능 여부를 이력으로 저장한다.

7) 관심상품·공유 정책 (PG-CRT-FAV-001)

• 관심상품 (PI-CRT-FAV-001-01)
- 관심상품은 주문 후보와 구분해 저장한다.
- 고객이 주문 후보로 전환할 때만 조합 검증과 주문 전환 검증 대상에 포함한다.
• 그룹 공유 (PI-CRT-FAV-001-02)
- 그룹 상품 공유는 공유 대상이 볼 수 있는 상품명, 세트 요약, 예상 비용, 제한 사유만 노출한다.
- 개인정보, 회선 상세, 결제정보는 공유하지 않는다.
• 공유 만료 (PI-CRT-FAV-001-03)
- 공유 링크에는 만료 기한을 둔다.
- 만료 기한은 기본 7일로 하고 운영자가 주제별로 단축할 수 있다.

8) 세트·패키지 구성 정책 (PG-CRT-GROUP-001)

• 세트 단위 (PI-CRT-GROUP-001-01)
- 단말, 요금제, 부가서비스, 구독, 혜택이 함께 주문되어야 하는 경우 하나의 세트로 묶는다.
- 세트 안 구성은 주문 전 검증과 가격 산정의 기본 단위로 사용한다.
• 구성 변경 (PI-CRT-GROUP-001-02)
- 세트 구성 변경 시 총 비용, 월 납부액, 할인, 혜택, 재고, 가입 가능 상태를 즉시 재계산한다.
- 변경 전 결과와 변경 후 결과를 구분해 표시한다.
• 충돌 표시 (PI-CRT-GROUP-001-03)
- 세트 안에서 동시 선택할 수 없는 옵션이나 혜택이 있으면 충돌 항목을 표시하고 고객이 삭제, 교체, 분리 주문 중 하나를 선택하게 한다.

9) 이력·고지·보관 정책 (PG-CRT-HISTORY-001)

• 이력 저장 (PI-CRT-HISTORY-001-01)
- 담기, 삭제, 자동삭제, 이관, 병합, 조합 검증, 변동 고지, 상담 전환, 주문 전환 결과는 고객 식별 가능 범위를 최소화해 저장한다.
• 고지 기준 (PI-CRT-HISTORY-001-02)
- 고객에게 영향을 주는 가격·혜택·재고 변동, 만료 예정, 자동삭제, 구매 불가, 분리 주문 필요 상태는 화면 고지와 이력 저장을 함께 수행한다.
• 보관 기준 (PI-CRT-HISTORY-001-03)
- 장바구니 처리 이력은 고객 문의, 장애 추적, 품질 개선 목적에 필요한 기간만 보관한다.
- 개인정보와 결제정보는 장바구니 이력에 저장하지 않는다.

10) 담기 후 동선 정책 (PG-CRT-NAV-001)

• 동선 선택 (PI-CRT-NAV-001-01)
- 장바구니 담기 완료 후 고객에게 쇼핑 계속하기와 장바구니 이동을 제공한다.
- 주문 대상이 바로 확정 가능한 경우에는 장바구니 이동 후 주문 가능 여부 검증을 먼저 수행한다.
• 결과 고지 (PI-CRT-NAV-001-02)
- 담기 완료 고지는 상품명, 선택 옵션, 수량, 장바구니 보관 상태를 포함한다.
- 담기 실패 또는 일부 담기 성공이면 실패 사유와 다시 시도 가능한 행동을 함께 안내한다.
• 이탈 복귀 (PI-CRT-NAV-001-03)
- 고객이 쇼핑 계속하기를 선택해도 담긴 상품의 선택 상태와 담기 시점 이력은 유지한다.
- 재방문 시 가격, 재고, 혜택 변동이 있으면 변경 확인 필요 상태로 전환한다.

11) 운영 변경·모니터링 정책 (PG-CRT-OPS-001)

• 변경 승인 (PI-CRT-OPS-001-01)
- 조합 가능성, 분리 주문, 보존 기간, 자동삭제 기준 변경은 운영자 승인 후 배포한다.
- 변경 시 영향받는 장바구니 세트 수와 고객 안내 필요 여부를 확인한다.
• 모니터링 (PI-CRT-OPS-001-02)
- 운영자는 장바구니 검증 실패율, 주문 전환율, 자동삭제 건수, 변동 고지 클릭률, 상담 전환 건수를 매일 확인한다.
• 롤백 기준 (PI-CRT-OPS-001-03)
- 운영 기준 변경 후 검증 실패율이나 구매 불가 전환이 전일 대비 20% 이상 증가하면 변경 기준을 롤백하거나 특정 상품군만 차단한다.

12) 주문 전환·분리 주문 정책 (PG-CRT-ORDER-001)

• 전환 가능 세트 (PI-CRT-ORDER-001-01)
- 주문 전환 가능 세트는 조합 검증, 가격·혜택 산정, 고객 조건 판정, 필수 안내 확인이 완료된 항목으로 제한한다.
• 분리 주문 (PI-CRT-ORDER-001-02)
- 결제 방식, 배송 유형, 발급 방식, 이행 주체가 다른 항목은 주문 실행 단위를 분리한다.
- 고객에게 무엇이 함께 처리되고 무엇이 별도 처리되는지 표시한다.
• 단독구매 (PI-CRT-ORDER-001-03)
- 단독구매 전용 상품은 현재 구매안을 유지한 채 별도 구매 경로로 이동한다.
- 단독구매 전환 시 장바구니의 다른 세트는 삭제하지 않는다.

13) 가격·혜택 요약 정책 (PG-CRT-PRICE-001)

• 총 비용 (PI-CRT-PRICE-001-01)
- 장바구니 요약에는 세트별 상품 가격, 할인, 포인트·혜택 적용 전후 금액, 예상 총 비용을 표시한다.
• 월 납부액 (PI-CRT-PRICE-001-02)
- 통신 상품이나 구독 상품은 월 예상 납부액, 최초 청구 예상액, 추가 비용 발생 여부를 구분해 보여준다.
• 혜택 영향 (PI-CRT-PRICE-001-03)
- 함께 구매 추천이나 구성 변경으로 할인·적립·혜택이 달라지면 고객이 이해할 수 있도록 증가분, 감소분, 적용 조건을 함께 표시한다.
• 액세서리 수량·금액 (PI-CRT-PRICE-001-04)
- 액세서리 수량 변경 시 총 결제 예상 금액, 품절 여부, 보관 가능 개수를 즉시 재산정한다.
- 수량 변경으로 최대 보관 수량을 초과하거나 품절 상태가 되면 주문 전환을 제한하고 사유를 고지한다.

14) 함께 구매 추천 정책 (PG-CRT-RECO-001)

• 추천 대상 (PI-CRT-RECO-001-01)
- 함께 구매 추천은 현재 장바구니 상품과 호환되는 추가 상품, 결합 상품, 구독, 혜택 연계 상품으로 제한한다.
• 추천 설명 (PI-CRT-RECO-001-02)
- 추천 항목에는 단순 노출 사유가 아니라 추가 시 비용, 할인, 혜택, 주문 가능성 변화 중 하나 이상을 설명한다.
• 추천 추가 검증 (PI-CRT-RECO-001-03)
- 추천 상품을 장바구니에 추가하면 기존 세트와 동일한 조합 검증, 가격·혜택 산정, 재고 검증을 다시 수행한다.

15) 장바구니 범위·경계 정책 (PG-CRT-SCOPE-001)

• 범위 경계 (PI-CRT-SCOPE-001-01)
- 장바구니는 담긴 상품과 관심상품을 주문 전 구매안으로 정리하고 검증하는 업무까지 다룬다.
- 주문서 작성, 결제 승인, 배송·개통 이행은 후속 정책서에서 다룬다.
• 예외 상품 (PI-CRT-SCOPE-001-02)
- 바로구매만 가능한 상품, 외부 제휴 결제가 필요한 상품, 단독구매 전용 상품은 장바구니에 표시할 수 있으나 주문 전환 경로를 별도 표시한다.
• 관심상품 구분 (PI-CRT-SCOPE-001-03)
- 관심상품은 탐색·재방문 목적 저장이며 주문 전환 전에는 장바구니 세트로 보지 않는다.
- 고객이 주문 후보로 선택하면 장바구니 검증 대상에 포함한다.

16) 비로그인·로그인 저장 이관 정책 (PG-CRT-SESSION-001)

• 임시 저장 (PI-CRT-SESSION-001-01)
- 비로그인 고객은 브라우저·디바이스 기준으로 장바구니를 임시 저장할 수 있다.
- 계정 간 동기화와 장기 보관은 로그인 후에만 허용한다.
• 이관 기준 (PI-CRT-SESSION-001-02)
- 로그인 시 임시 장바구니는 계정 장바구니로 이관한다.
- 같은 상품·옵션·세트가 있으면 중복 세트를 만들지 않고 최근 수정 정보를 우선 적용한다.
• 병합 충돌 (PI-CRT-SESSION-001-03)
- 임시 장바구니와 계정 장바구니의 상품 조합이 충돌하면 고객에게 충돌 세트와 선택 가능한 유지·삭제·분리 주문 행동을 안내한다.

17) 통합·분리 장바구니 구조 정책 (PG-CRT-STRUCT-001)

• 통합 조회 (PI-CRT-STRUCT-001-01)
- 고객 화면은 단말, 요금제, 부가서비스, 액세서리, 구독, 혜택 상품을 하나의 구매안으로 조회하게 하되 실제 처리 단위는 그룹으로 구분한다.
• 처리 단위 (PI-CRT-STRUCT-001-02)
- 상품 유형, 결제 방식, 배송 유형, 발급 방식, 가입/구매 이행 주체가 다르면 같은 화면에 있어도 주문 실행 단위를 분리한다.
• 다중 탭 (PI-CRT-STRUCT-001-03)
- 휴대폰·스마트기기, 인터넷·IPTV, 액세서리처럼 처리 기준이 다른 탭은 탭별 상품 개수와 가능한 행동을 표시한다.
- 탭 구분은 고객에게 하나의 구매안으로 보이되 주문 실행 단위가 다를 때만 사용한다.

18) 조합 가능성 검증 정책 (PG-CRT-VALID-001)

• 호환성 기준 (PI-CRT-VALID-001-01)
- 상품 조합 검증은 상품 유형, 선행 가입, 고객 상태, 회선 상태, 재고, 판매 기간, 혜택 배타 조건을 함께 확인한다.
• 선행 조건 (PI-CRT-VALID-001-02)
- 선행 가입이나 필수 상품이 필요한 조합은 주문 전환 전에 필수 조건과 충족 방법을 안내한다.
- 충족 전에는 주문 전환을 제한한다.
• 중복 제한 (PI-CRT-VALID-001-03)
- 동일 상품, 동일 혜택, 동일 회선 기준으로 중복 가입이 불가한 항목은 장바구니 단계에서 중복 사유를 표시하고 추가 선택을 제한한다.
• 실시간 재검증 (PI-CRT-VALID-001-04)
- 장바구니 진입, 수량·옵션 변경, 주문하기 선택 시 재고, 가격, 혜택, 고객 조건을 다시 검증한다.
- 검증 결과가 달라지면 변경 확인 필요 상태로 전환하고 변경 사유를 고지한다.

최종 점검 기준

카트/장바구니 정책서 제출 전 점검
1. 범위 정합성
담기, 장바구니, 관심상품, 주문서의 경계가 문서에서 구분되어야 한다.
2. 구매안 완결성
고객이 전체 구매안, 처리 단위, 주문 가능 항목, 제외 항목을 이해할 수 있어야 한다.
3. 액터 적정성
로그인·비로그인·개인·그룹 고객을 액터로 분리하지 않고 상태와 정책 조건으로 관리해야 한다.
4. 유즈케이스-프로세스 연결성
사람 액터 유즈케이스는 2개 이상의 절차 프로세스로 분해되어야 한다.
5. 상태 전이 정합성
보관, 검증, 주문 가능, 분리 주문, 변동, 만료, 삭제 상태가 상태 전이와 연결되어야 한다.
6. 조합 검증 구체성
호환성, 선행 조건, 중복 제한, 재고, 혜택 배타 기준이 정책 항목으로 정의되어야 한다.
7. 비로그인/로그인 이관 기준
임시 저장, 계정 이관, 병합, 중복·충돌 처리 기준이 명확해야 한다.
8. 보존·삭제 기준
최대 수량, 30일 보존, 자동삭제, 완료 세트 삭제, 삭제 이력이 정의되어야 한다.
9. BSS·연계 포함성
상품·고객 조건, 가격·혜택·재고 변동, 연계 실패 처리 기준이 포함되어야 한다.
10. 개발/QA 전환성
상세 요구사항 43건이 유즈케이스, 상태, 프로세스, 기능, 정책, 정책 항목 중 하나 이상으로 추적 가능해야 한다.
11. 요구사항 추적성
상세 요구사항 43건은 구매안 통합, 처리 단위, 상품·세트 변경, 가격·재고 변동, 모회선 인증, 주문 전환 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했는지 확인한다.
