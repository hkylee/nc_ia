1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 기존 상품 조건을 변경하는 업무 기준을 정의한다.

• 포함 범위는 요금제 변경, 선택형 혜택·옵션 변경, 구독 해지·일시정지·재개, 예약 변경, USIM/eSIM 재발급, 상품 일시정지, 약정 동시 처리, 변경 실패 복구를 포함한다.

• 변경 가능 여부, 불가 사유, 비용·혜택·약정 영향, 적용 시점, 예약·철회 기준, BSS 반영 결과는 고객에게 사전에 설명되어야 한다.

• 신규 가입, 주문 생성, 배송·교환·반품, 결제 승인 자체, 멤버십 혜택 사용은 인접 정책서에서 다루며 본 문서는 변경으로 인한 영향과 연결 기준만 정의한다.

• API 필드, DB 컬럼, 운영자 화면 상세, 배치 설계는 후속 상세 설계에서 다루고 본 문서는 정책 판단에 필요한 기준만 포함한다.

• 채널 업무 시스템은 변경 조건 안내, 영향 비교, 신청 접수, 고객 표시 상태를 책임지고 상품 변경 연계 시스템은 변경 가능성, 약정·혜택 영향, BSS 반영 결과를 책임진다.

• 고객 표시 상태는 변경 가능, 영향 확인 필요, 예약 변경 대기, 변경 처리 중, 변경 완료, 보완·복구 필요로 안내하고 내부 상태는 조건 판정, 예약 충돌, 원장 반영, 실패 이력으로 분리한다.

• 통합 리스크는 상품 변경 연계 시스템 응답 지연, 예약 변경 충돌, 약정·혜택 계산 불일치, 채널 간 상태 캐시 차이이며 고객에게는 재확인 필요 또는 복구 진행 상태로 표시한다.

• 상세 요구사항 39건은 변경 가능성과 영향 비교, 상품 변경 신청, 예약 변경, 구독·옵션 변경, 진행 조회와 실패 복구 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했다.

나. 설계 원칙

• 고객 영향 선고지: 변경으로 잃고 얻는 혜택, 비용, 약정, 적용 시점을 고객이 신청 전에 이해하도록 구성한다.

• 셀프 처리 우선: 고객이 앱·웹에서 직접 완료할 수 있는 변경은 상담 전환보다 셀프 경로를 우선 제공한다.

• 변경 가능성 명확화: 가능 여부뿐 아니라 불가 사유, 다음 가능일, 대체 변경안을 함께 제공한다.

• BSS 판정 통합: 요금·혜택·약정·구독·일시정지 상태처럼 BSS 판정이 필요한 기준은 프로세스와 정책에 함께 반영한다.

• 실패 복구 내장: 인증 만료, 예약 충돌, 연계 실패, 보류 상태에서 고객이 다음 행동을 선택할 수 있게 한다.

• 운영 기준 분리: 예외 승인, 보류, 반려, 실패 모니터링 기준은 운영자가 관리하되 고객에게는 이해 가능한 사유로 노출한다.

• 대상 채널 (PI-CHG-SCOPE-003)

2) 변경 가능 조건 정책 (PG-CHG-ELG-001)

• 셀프 허용 기준 (PI-CHG-ELG-001)
- 고객 상태, 현재 상품, 처리 중 주문·배송·설치 상태, 제휴사 제한, 미납 여부가 모두 허용이면 셀프 변경을 허용한다.
• 불가 사유 표시 (PI-CHG-ELG-002)
- 변경 제한 사유는 약정 제한, 미납, 배송·설치 진행, 제휴사 확인 필요, 횟수 초과, 인증 불가 중 하나 이상으로 표시한다.
• 변경 횟수 제한 (PI-CHG-ELG-003)
- 동일 회선의 동일 상품군 변경은 운영자가 등록한 월 단위 제한 횟수를 초과하면 제한하고 다음 가능일을 함께 표시한다.
• 휴일 처리 (PI-CHG-ELG-004)
- 휴일에 즉시 BSS 반영이 불가능한 상품은 예약 변경만 허용하고 적용 예정일을 첫 영업일로 계산한다.
• 대체 요금제 제안 (PI-CHG-ELG-005)
- 고객이 선택한 요금제로 변경할 수 없으면 제한 사유, 유사 요금제 후보, 무위약금 여부, 예상 비용·혜택 차이를 함께 표시한다.
- 대체 후보가 없으면 상담 전환 또는 변경 가능일 알림을 제공한다.
• 24시간 셀프 처리 (PI-CHG-ELG-006)
- 우주패스 해지, 환불 접수, 요금제 연계 변경처럼 고객 영향이 큰 핵심 업무는 주말·휴일에도 셀프 접수를 우선 허용한다.
- 단 BSS 즉시 반영이 불가하면 예약 변경 또는 처리 보류 상태로 접수하고 예상 반영 시점을 안내한다.

3) 영향도 비교·고지 정책 (PG-CHG-IMPACT-001)

• 비용 영향 (PI-CHG-IMPACT-001)
- 추가 비용, 감면액, 차액 결제 또는 부분 취소 금액은 변경 전 확인 화면에서 기존 조건과 변경 후 조건을 나란히 표시한다.
• 혜택 영향 (PI-CHG-IMPACT-002)
- 소멸 혜택, 유지 혜택, 새로 시작되는 혜택, 자동선물 해제 여부는 고객 확인 전 필수 고지로 표시한다.
• 적용 시점 (PI-CHG-IMPACT-003)
- 즉시 변경은 완료 즉시 적용 시각을 표시하고 예약 변경은 적용 예정일과 철회 가능 시간을 함께 표시한다.
• 되돌림 가능 범위 (PI-CHG-IMPACT-004)
- 고객이 변경 완료 후 취소할 수 없는 항목은 완료 전 단계에서 취소 불가 문구를 별도 확인으로 받는다.
• 통합 변경 예상 절감액 (PI-CHG-IMPACT-005)
- 여러 회선이나 여러 서비스의 변경을 함께 검토하는 경우 변경 조합별 예상 절감액, 추가 비용, 할인 변동 사유를 한 화면에서 비교한다.
- 계산 근거가 부족하면 예상값 대신 산정 불가 사유와 개별 변경 경로를 안내한다.
• 패키지 상품 교체 금액 비교 (PI-CHG-IMPACT-006)
- 패키지 상품 교체 시 현재 고객이 결제 중인 금액, 변경 후 결제 예정 금액, 추가·감면 금액, 혜택 소멸 여부를 교체 확정 전에 표시한다.
- 금액 산정이 지연되면 확정 전 제출을 제한한다.
• 요금 시뮬레이션 변경 (PI-CHG-IMPACT-007)
- 회원정보 포함 또는 비포함 기준의 변경 후 요금 시뮬레이션은 고객이 직접 전환해 비교할 수 있게 한다.
- 시뮬레이션 결과에는 적용 시점, 할인 반영 여부, 실제 청구 금액과 차이가 날 수 있는 조건을 함께 표시한다.

4) 요금제·약정 변경 정책 (PG-CHG-PLAN-001)

• 요금제 단독 변경 (PI-CHG-PLAN-001)
- 요금제만 변경하는 경우 옵션·구독·약정 영향이 없는지 산정한 뒤 단독 변경 전용 흐름으로 접수한다.
• 약정 동시 처리 (PI-CHG-PLAN-002)
- 약정 변경이 동반되면 약정 기간, 위약금 발생 가능성, 할인 유지 여부를 변경 확인 전 필수 영향으로 표시한다.
• 무위약금 후보 (PI-CHG-PLAN-003)
- 위약금이 발생하는 요청에는 무위약금 또는 낮은 위약금 대체 요금제를 함께 표시한다.
• 연령 기준 (PI-CHG-PLAN-004)
- 연령 초과 전후 조건이 달라지는 상품은 현재 적용 기준과 변경 후 적용 기준을 비교 표시한다.
• 다회선 변경 일괄 처리 (PI-CHG-PLAN-005)
- 다회선 변경은 지원 대상 업무와 회선별 가능 조건이 모두 확인된 경우에만 일괄 처리한다.
- 일부 회선이 실패하면 성공·실패 회선을 분리해 표시하고 실패 회선은 개별 재시도 또는 상담 전환을 제공한다.

5) 혜택·옵션 변경 정책 (PG-CHG-BEN-001)

• 선택형 혜택 조합 (PI-CHG-BEN-001)
- 기본 요금제 변경 없이 선택형 혜택 모듈을 추가·해제·조합할 수 있는 경우에는 혜택 단위 변경으로 처리한다.
• 연동 혜택 경고 (PI-CHG-BEN-002)
- 상품 변경으로 멤버십, 자동선물, 우주패스, 제휴 혜택이 중단되면 변경 전 경고와 고객 확인을 필수로 둔다.
• 혜택 시작일 (PI-CHG-BEN-003)
- 신규 혜택은 변경 적용 시점부터 시작하고 기존 혜택 소멸일과 중복 적용 여부를 함께 표시한다.

6) 구독 변경·해지·일시정지 정책 (PG-CHG-SUB-001)

• 구독 해지 (PI-CHG-SUB-001)
- 구독 해지는 현재 결제 주기, 잔여 혜택, 다음 결제 예정일, 해지 후 복구 가능 여부를 확인한 뒤 접수한다.
• 일시정지·재개 (PI-CHG-SUB-002)
- 구독 일시정지와 재개는 시작일, 종료일, 남은 기간, 자동 재개 조건을 고객이 확인한 경우에만 접수한다.
• 유료 전환 (PI-CHG-SUB-003)
- 무료 또는 프로모션 구독이 상품 변경으로 유료 전환되면 전환 예정일과 첫 결제 예상 금액을 필수 고지한다.
• 결제수단 안내 (PI-CHG-SUB-004)
- 구독 변경으로 결제수단 재확인이 필요한 경우 PAY 연결 경로와 현재 적용 중인 결제수단을 표시한다.
• 로밍 무료충전 신청 경로 (PI-CHG-SUB-005)
- 로밍 신청 완료 후 클럽 T로밍 무료충전 대상이면 완료 화면에서 무료충전 신청 버튼 또는 딥링크를 제공한다.
- 대상이 아니면 미제공 사유를 안내하고 로밍 신청 완료 상태는 유지한다.
• 클럽형 혜택 신청 조건 (PI-CHG-SUB-006)
- 클럽형 혜택 상품은 가입 조건, 제공 혜택, 유지 조건, 해지 영향이 모두 확인된 경우에만 신청 단계로 진입한다.
- 조건 미충족 시 부족 조건과 대체 혜택을 안내한다.
• 옵션상품 변경·해지 통합 (PI-CHG-SUB-007)
- 옵션상품의 추가, 변경, 해지는 같은 구조에서 처리하되 현재 가입 상태, 변경 후 상태, 해지 후 영향, 즉시·예약 적용 가능 여부를 함께 표시한다.

7) 예약 변경 정책 (PG-CHG-RESV-001)

• 예약 가능일 (PI-CHG-RESV-001)
- 예약 가능일은 청구 주기, 구독 갱신일, 배송·설치 상태, 기존 예약 상태를 반영해 고객별로 산정한다.
• 철회 가능 시간 (PI-CHG-RESV-002)
- 예약 변경은 적용 예정일 전날 23:59까지 철회 또는 수정할 수 있으며 이후에는 취소 불가로 전환한다.
• 예약 충돌 (PI-CHG-RESV-003)
- 동일 상품군에 예약이 2건 이상 있으면 가장 최근 고객 확정 요청을 우선하고 이전 예약은 철회 또는 유지 선택을 요구한다.

8) 인증·동의 정책 (PG-CHG-AUTH-001)

• 본인확인 (PI-CHG-AUTH-001)
- 요금, 약정, 결제, 고객 권한에 영향이 있는 변경은 본인확인을 완료해야 접수할 수 있다.
• 인증 유효 시간 (PI-CHG-AUTH-002)
- 인증 유효 시간은 10분이며 만료되면 변경 초안은 유지하고 인증만 다시 요구한다.
• 동의 재확인 (PI-CHG-AUTH-003)
- 변경 초안의 비용 또는 약정 영향이 바뀌면 기존 동의는 무효화하고 변경된 영향 기준으로 다시 동의받는다.

9) 미성년 회선 변경 정책 (PG-CHG-MINOR-001)

• 미성년 회선 (PI-CHG-MINOR-001)
- 미성년 회선 변경은 고객 연령, 명의, 법정대리인 관계가 확인된 경우에만 진행한다.
• 대리 동의 (PI-CHG-MINOR-002)
- 법정대리인 동의가 필요한 변경은 동의 성공, 거절, 만료 상태를 고객에게 표시하고 거절 시 변경 접수를 차단한다.
• 보호자 고지 (PI-CHG-MINOR-003)
- 요금 또는 약정 영향이 있는 미성년 회선 변경은 법정대리인에게 변경 전후 영향 요약을 함께 고지한다.

10) USIM/eSIM 재발급 정책 (PG-CHG-USIM-001)

• 재발급 신청 (PI-CHG-USIM-001)
- USIM/eSIM 재발급은 본인확인, 회선 상태, 배송 또는 다운로드 가능 조건이 모두 충족되면 신청할 수 있다.
• 진행 상태 (PI-CHG-USIM-002)
- 재발급 요청은 접수, 처리 중, 발급 완료, 발급 실패, 상담 전환 상태로 고객에게 표시한다.
• 실패 복구 (PI-CHG-USIM-003)
- eSIM 발급 실패는 동일 요청 3회까지 재시도하고 이후에는 상담 전환 또는 오프라인 발급 경로를 안내한다.

11) 일시정지·해제 정책 (PG-CHG-PAUSE-001)

• 일시정지 예약 (PI-CHG-PAUSE-001)
- 상품 일시정지는 시작일과 종료일을 함께 지정해야 예약할 수 있으며 종료일이 없으면 접수를 제한한다.
• 자동 해제 (PI-CHG-PAUSE-002)
- 예약 종료일이 도래하면 자동 해제를 시도하고 실패하면 고객에게 실패 사유와 재시도 경로를 안내한다.
• 조기 해제 (PI-CHG-PAUSE-003)
- 고객은 일시정지 중 상태에서 조기 해제를 요청할 수 있으며 BSS 반영 성공 후 즉시 사용 가능 상태로 전환한다.

12) 변경 요청 접수·중복 정책 (PG-CHG-REQ-001)

• 임시 저장 (PI-CHG-REQ-001)
- 변경 초안은 고객이 이탈해도 24시간 보관하며 인증 완료 후 제출 전 초안은 30분 동안 유지한다.
• 중복 요청 (PI-CHG-REQ-002)
- 동일 회선, 동일 상품군, 동일 변경 항목 요청이 5초 안에 반복되면 중복 요청으로 분류하고 추가 접수를 차단한다.
• 접수 결과 (PI-CHG-REQ-003)
- 변경 접수 결과는 접수 번호, 처리 상태, 예상 완료 시각, 실패 시 다음 행동과 함께 표시한다.

13) BSS 반영·연계 정책 (PG-CHG-BSS-001)

• 원장 반영 (PI-CHG-BSS-001)
- BSS 반영 성공은 상품 상태, 요금 상태, 혜택 상태, 약정 상태, 고객 표시 상태가 모두 변경 후 기준과 일치해야 인정한다.
• 연계 결과 (PI-CHG-BSS-002)
- 제휴사, 결제, 주문, 구독 연계 응답이 지연되면 고객에게 처리 중 상태를 유지하고 예상 재확인 시점을 안내한다.
• 부분 성공 (PI-CHG-BSS-003)
- 일부 연계만 성공한 경우 고객 상태는 보류로 표시하고 성공·실패 항목을 분리해 운영자 확인 대상으로 남긴다.

14) 실패·보류·복구 정책 (PG-CHG-FAIL-001)

• 재시도 기준 (PI-CHG-FAIL-001)
- 인증 만료, 일시 연계 오류, 입력 보정 오류는 동일 세션에서 최대 3회까지 재시도할 수 있다.
• 복구 불가 (PI-CHG-FAIL-002)
- 약정 제한, 제휴사 제한, 미납, 미성년 동의 거절은 고객 재시도 대상에서 제외하고 제한 사유와 대체 경로를 안내한다.
• 보류 처리 (PI-CHG-FAIL-003)
- 운영 승인 또는 원장 정합성 확인이 필요한 요청은 처리 보류로 전환하고 담당자, 사유, 등록 시각을 저장한다.

15) 상담 전환 정책 (PG-CHG-CS-001)

• 상담 전환 (PI-CHG-CS-001)
- 셀프 처리 불가 상태에서는 고객이 선택한 변경 항목, 실패 사유, 영향 비교 결과를 상담 문맥으로 전달한다.
• 상담 재처리 (PI-CHG-CS-002)
- 상담사가 대체 처리나 예외 승인 요청을 등록하면 고객에게 처리 예정 상태와 후속 알림 기준을 안내한다.
• 문맥 보존 (PI-CHG-CS-003)
- 고객이 중단한 변경 초안과 실패 이력은 상담 전환 후 24시간 동안 조회 가능하게 유지한다.

16) 알림·고지 정책 (PG-CHG-NOTI-001)

• 변경 가능일 알림 (PI-CHG-NOTI-001)
- 변경 가능일이 도래하는 요청은 고객이 알림 수신에 동의한 경우 1회 이상 사전 알림을 발송한다.
• 완료 안내 (PI-CHG-NOTI-002)
- 적용 완료 안내에는 변경 후 상품명, 적용 시각, 비용 영향, 후속 확인 경로를 포함한다.
• 실패 안내 (PI-CHG-NOTI-003)
- 실패 안내에는 실패 사유, 재시도 가능 여부, 상담 전환 가능 여부, 고객이 해야 할 다음 행동을 포함한다.

17) 이력·모니터링 정책 (PG-CHG-HIST-001)

• 변경 이력 (PI-CHG-HIST-001)
- 변경 요청, 영향 확인, 인증·동의, 접수, 반영, 실패, 상담 전환 이력은 요청 단위로 저장한다.
• 운영 이력 (PI-CHG-HIST-002)
- 운영자 승인, 보류, 반려, 재처리 이력은 담당자, 사유, 시각, 변경 전후 상태와 함께 저장한다.
• 모니터링 (PI-CHG-HIST-003)
- 실패율, 지연 건수, 상담 전환 사유, 반복 불가 사유는 일 단위로 집계해 운영자가 확인할 수 있게 한다.

18) 운영 예외·승인 정책 (PG-CHG-OPR-001)

• 기준 관리 (PI-CHG-OPR-001)
- 운영자는 상품 유형, 처리 상태, 제휴 조건, 횟수 제한, 휴일 처리 가능 여부별 셀프 허용 기준을 등록한다.
• 예외 승인 (PI-CHG-OPR-002)
- 예외 승인은 고객 피해 가능성, 비용 영향, 원장 정합성, 제휴사 확인 결과가 함께 확인된 경우에만 허용한다.
• 반려 사유 (PI-CHG-OPR-003)
- 반려된 변경 요청은 고객 안내 문구와 운영 사유를 분리해 저장하고 고객에게는 이해 가능한 사유만 표시한다.

최종 점검 기준

상품변경 정책서 제출 전 점검
1. 범위 정합성
요금제·옵션·구독·일시정지·USIM/eSIM·예약 변경은 포함하고 주문·배송·결제 승인 자체는 인접 정책서로 분리했는지 확인한다.
2. 고객 완결성
고객이 변경 가능 여부, 영향 비교, 신청, 예약, 결과 확인, 실패 복구까지 앱·웹에서 이해하고 완료할 수 있는지 확인한다.
3. 액터 입자도
로그인 고객, 제한 고객, 미성년 고객처럼 상태 조건을 액터로 분리하지 않았는지 확인한다.
4. 유즈케이스 입자도
유즈케이스가 버튼·검증 단계가 아니라 고객 또는 운영자의 완결 업무 단위로 작성되었는지 확인한다.
5. 프로세스 연결성
각 고객·운영자 유즈케이스에 두 개 이상의 프로세스가 연결되고 프로세스가 기능·정책과 연결되는지 확인한다.
6. 상태 전이 정합성
상태 전이표의 현재 상태와 다음 상태가 상태 코드 목록에 존재하고 정상·제한·보류·실패 흐름이 포함되는지 확인한다.
7. 정책 구체성
정책 상세에 조건, 시간, 횟수, 상태, 고지, 이력, BSS 반영 기준 중 하나 이상이 포함되는지 확인한다.
8. BSS 포함성
변경 요청 접수, 원장 반영, 연계 결과 회신, 부분 성공, 보류, 실패 처리 기준이 정책에 포함되는지 확인한다.
9. 요구사항 반영
상품변경 상세 요구사항 39건이 유즈케이스, 프로세스, 기능, 정책 중 하나 이상에 추적 가능하게 반영되었는지 확인한다.
10. 상담 전환 기준
셀프 불가와 실패 상황에서 고객 선택값과 실패 사유가 상담 문맥으로 유지되는지 확인한다.
11. 요구사항 추적성
상세 요구사항 39건은 변경 가능성과 영향 비교, 상품 변경 신청, 예약 변경, 구독·옵션 변경, 진행 조회와 실패 복구 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했는지 확인한다.
12. 개발·QA 전환성
개발·QA가 변경 전 상태, 변경 후 상태, 적용 시점, 비용·혜택 영향, 예약 충돌, BSS 반영 실패와 복구 기대 결과를 테스트 케이스로 전환할 수 있는지 확인한다.
