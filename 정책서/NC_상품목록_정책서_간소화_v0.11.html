1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 상품 후보를 탐색하고 비교·상세·담기·주문으로 이동하기 위한 상품 목록 처리 기준을 정의한다.

• 대상 업무는 카테고리 진입, 목록 카드 확인, 필터·정렬 적용, 비교 후보 유지, 찜·최근본 관리, 가입 가능 여부 확인, 목록 복귀 맥락 유지다.

• 대상 채널은 통합채널 앱·웹 FO이며, 운영자가 관리하는 카테고리·필터·정렬·노출 기준과 BSS와 상품 기준 시스템의 가입 가능 판정 결과를 포함한다.

• 포함 범위는 상품 속성 기반 통합 분류, 카테고리 경로, 목록 카드 표준, 목적형 정렬, 개인화 빠른 필터, 가입 불가 구분, 빈 화면·오류 폴백, 목록 품질 운영 기준이다.

• 제외 범위는 상품 상세의 세부 설명, 담기 이후 주문·계약 절차, 결제, 할인 확정 계산, 전시 전체 운영 체계, API 필드, DB 컬럼, 화면 상세 설계다.

• 후속 산출물에서는 상품 상세, 장바구니, 주문·계약, 할인·시뮬레이션 정책서가 목록에서 전달된 상품 후보와 고객 조건을 이어받아 상세 기준을 확정한다.

나. 설계 원칙

• 목적 기반 탐색: 상품 목록은 조직·브랜드 기준이 아니라 고객의 이용 목적, 상품 속성, 현재 보유 상태를 기준으로 후보를 좁히게 한다.

• 선택 전 이해 가능성: 고객은 상세 진입 전에도 가격, 혜택, 가입 가능성, 제한 사유, 현재 상품 대비 차이를 목록 단계에서 이해할 수 있어야 한다.

• 맥락 유지: 탭, 필터, 정렬, 스크롤, 비교 후보, 최근본 맥락은 상세·검색·주문 진입 후 복귀해도 유지되어야 한다.

• 가입 가능성 우선: BSS와 상품 기준 시스템이 판단한 가입 가능 상품을 우선 제시하되, 가입 불가 상품은 숨김 또는 비활성화와 함께 사유를 안내한다.

• 운영 통제 가능성: 카테고리, 필터, 정렬, 전시 단위, 품절·단종 제외 기준은 운영자가 관리하되 고객 혼선을 막는 품질 Gate를 거친다.

• 후속 전환 연결: 목록은 탐색으로 끝나지 않고 비교, 상세, 담기, 주문, 상담, 검색, 추천 카테고리 같은 다음 행동으로 자연스럽게 이어져야 한다.

ST-PRD-013운영 배포 완료검수된 목록 운영 기준이 배포되어 고객 목록 조회에 반영된 상태다.
배포 이력, 적용 채널, 영향 카테고리, 롤백 기준을 기록한다.
ST-PRD-014운영 회수 필요배포된 목록 기준에서 오노출, 품질 오류, 정책 충돌이 발견되어 회수가 필요한 상태다.
노출 중지, 이전 기준 복구, 영향 고객 안내, 수정 재검수 절차를 진행한다.

2) 상태 전이 기준

현재 상태전이 이벤트다음 상태처리 기준 및 후속 처리
탐색 전상품 카테고리와 목록 후보 탐색카테고리 진입고객이 상품 목록 진입 경로를 선택하면 카테고리 경로와 기본 목록 조건을 설정한다.
카테고리 진입상품 속성 기반 통합 분류 적용목록 조회 완료상품 기준 시스템과 전시 단위가 유효하고 노출 가능 조건을 만족하면 목록 후보를 표시한다.
목록 조회 완료목록 조건 조정과 상품 후보 선별조건 적용고객이 필터, 정렬, 검색, 빠른 조건을 적용하면 결과 수와 적용 조건 요약을 갱신한다.
조건 적용후보 기준 충족후보 선별 완료필터 의존성과 상호배타 조건을 통과하고 1개 이상 후보가 있으면 후보 선별 완료로 둔다.
후보 선별 완료고객별 가입 가능 조건 판정가입 가능 확인BSS가 고객 회선, 연령, 약정, 단말, 제한 상태 기준으로 가입 가능 결과를 회신하면 가입 가능 확인으로 전이한다.
후보 선별 완료가입 제한 조건 발견제한 사유 확인가입 불가 또는 조건 미충족 상품은 숨김 또는 비활성화하고 제한 사유를 표준 문구로 표시한다.
가입 가능 확인상품 후보 비교·찜·최근본 이어가기비교·보관 중고객이 비교 후보, 찜, 최근본을 선택하면 후보 상태와 목록 맥락을 보관한다.
비교·보관 중상세·담기·주문 진입후속 진입고객이 상세, 비교, 담기, 주문 중 하나를 선택하면 목록 맥락을 유지하고 후속 정책서로 전달한다.
조건 적용검색 결과 없음결과 없음적용 조건 결과가 0건이면 조건 초기화, 추천 카테고리, 검색, 상담 경로를 제공한다.
목록 조회 완료목록 조회 실패오류·복구 필요상품 기준 시스템, BSS, 채널 조회 중 하나가 실패하면 고객에게 재시도와 대체 경로를 안내한다.
오류·복구 필요재시도 성공목록 조회 완료동일 조건 재조회가 성공하면 기존 카테고리와 필터 맥락을 복원한다.
운영 검수 대기목록 품질 Gate 통과운영 배포 완료카테고리, 필터, 목록 카드, 가입 가능 사유, 폴백 기준이 검수 기준을 통과하면 배포 완료로 둔다.
운영 배포 완료오노출·품질 이슈 발견운영 회수 필요오노출, 잘못된 가입 가능 사유, 단종 상품 노출, 필터 오류가 발견되면 회수 필요 상태로 전환한다.
운영 회수 필요수정 기준 재검수 요청운영 검수 대기회수된 카테고리, 필터, 카드, 전시 그룹 기준의 수정본이 등록되면 재검수 대기 상태로 전환한다.

3) 상태 전이 다이어그램

상태 전이 다이어그램은 대표 상태와 전이 이벤트를 개념적으로 보여주며, 상세 판정 기준은 상태 전이표를 따른다.

나. 정책 상세

1) 상품 목록 범위·경계 정책 (PG-PRD-SCP-001)

• 직접 범위 (PI-PRD-SCP-001)
- 상품 목록은 카테고리, 필터, 정렬, 목록 카드, 가입 가능 표시, 비교 후보, 찜, 최근본, 폴백 안내를 직접 범위로 둔다.
- 상품 상세 설명, 담기 이후 주문, 결제, 할인 확정 계산은 후속 정책서 범위로 분리한다.
• 인접 정책서 경계 (PI-PRD-SCP-002)
- 전시/관리 기능의 전체 슬롯 운영, 상품상세의 상세 정보, 장바구니의 구성 검증, 주문의 계약 확정은 본 문서에 세부 절차로 작성하지 않는다.
- 목록은 후속 업무로 넘길 카테고리, 필터, 정렬, 비교 후보, 고객별 가입 가능 여부, 제한 사유를 전달 맥락으로 정의하고 전달 이력을 저장한다.

2) 카테고리·분류 정책 (PG-PRD-CAT-001)

• 통합 분류 기준 (PI-PRD-CAT-001)
- 카테고리는 상품 속성, 이용 목적, 고객 보유 상태를 우선 기준으로 구성한다.
- 동일 성격 상품이 브랜드, 조직, 채널 이름 차이로 2개 이상 분리되면 표준 분류 기준으로 통합하거나 중복 사유를 운영 검수 대상으로 둔다.
• 경로 표시 (PI-PRD-CAT-002)
- 카테고리는 1단계 이상 계층을 가질 수 있으며 고객에게 현재 경로를 표시한다.
- 딥링크로 진입해도 상위 경로와 현재 목록 기준을 함께 표시한다.
• 전체 보기 진입 (PI-PRD-CAT-003)
- 상품군별 전체 보기는 2Depth 대표 목록으로 진입시키고, 3Depth 하위 카테고리는 목록 내 탭 또는 필터로 전환한다.
- 목록 복귀 시 이전 탭, 필터, 정렬, 스크롤을 유지한다.

3) 목록 카드 정보 정책 (PG-PRD-CARD-001)

• 목록 카드 필수 정보 (PI-PRD-CARD-001)
- 목록 카드는 이미지 또는 식별 정보, 상품명, 핵심 가치, 가격 또는 예상 비용, 주요 혜택, 상태 배지, 주요 조건, 후속 CTA 중 5개 이상을 포함한다.
- 가격이나 혜택이 확정값이 아니면 예상 또는 조건부 표시를 함께 둔다.
• 현재 상품 대비 요약 (PI-PRD-CARD-002)
- 로그인 고객에게는 현재 사용 상품 대비 월 요금 차이, 혜택 변화, 약정 또는 조건 변화 중 확인 가능한 항목을 목록 단계에서 요약한다.
- BSS 판정이 없으면 확정 비교가 아니라 상세 확인 필요로 표시한다.
• 유무료 구분 (PI-PRD-CARD-003)
- 부가서비스나 혜택형 상품은 유료, 무료, 조건부 무료를 구분하는 배지를 제공한다.
- 유료 전환 조건이 있는 경우 목록 카드 또는 상세 진입 전 고지에 제한 조건을 남긴다.
• 상품 목록 공통 정보 규격 (PI-PRD-CARD-004)
- 상품 목록 카드는 가격, 혜택, 구성요소, 주요 특징, 가입 가능 여부, 현재 사용 상품 대비 차이를 공통 항목 순서로 제공한다.
- 로그인 고객에게만 제공 가능한 정보는 권한 확인 후 노출한다.
• 탐색 보조 배너·연관 메뉴 (PI-PRD-CARD-005)
- 상품 탐색 화면과 목록 화면은 탐색 보조형 배너와 연관 메뉴를 제공해 다음 탐색 경로를 이어갈 수 있게 한다.
- 보조 배너는 주문 CTA보다 우선하지 않으며 고객 목적과 관련된 경우에만 노출하고, 동일 세션 반복 노출은 1회로 제한한다.
• 인터넷/BTV 신청 비교 카드 (PI-PRD-CARD-006)
- 인터넷/BTV 셋톱박스 선택 정보는 주요 기능, 가격, 제공 조건, 차이점을 목록 카드 또는 비교 영역에서 함께 제공해 상세 진입 전 선택 판단이 가능해야 한다.

4) 필터·정렬 정책 (PG-PRD-FLT-001)

• 필터 유형 (PI-PRD-FLT-001)
- 필터는 체크, 라디오, 범위, 칩 중 카테고리 특성에 맞는 유형으로 제공한다.
- 기본 필터는 고객이 3초 안에 이해할 수 있는 핵심 조건을 우선 배치한다.
• 필터 의존성 (PI-PRD-FLT-002)
- A 조건 선택 시 B 조건이 무효가 되는 경우 상호배타로 처리하고 고객에게 제외 사유를 안내한다.
- 의존 필터는 선행 조건이 해제되면 함께 초기화한다.
• 조건 초기화 (PI-PRD-FLT-003)
- 고객은 적용된 필터와 정렬을 한 번에 초기화할 수 있어야 한다.
- 초기화 후에는 카테고리 기본 정렬과 고객이 진입한 경로 기준을 복원하며, 초기화 전 적용 조건 수와 결과 변화 가능성을 안내한다.
• 상품 목록 공통 필터·정렬 (PI-PRD-FLT-004)
- 상품 목록은 카테고리와 상품군 특성에 맞는 필터와 정렬 기준을 공통 구조로 제공한다.
- 고객 선택 빈도와 가입 가능성에 영향을 주는 핵심 기준을 우선 배치하고 과도한 옵션은 접힘 또는 보조 영역으로 분리하며, 변경 시 적용 채널과 기준 버전 이력을 저장한다.

5) 목록 우선순위 정책 (PG-PRD-RNK-001)

• 기본 정렬 우선순위 (PI-PRD-RNK-001)
- 기본 정렬은 가입 가능, 고객 목적 적합성, 혜택 가치, 가격 영향, 최신성, 인기 순으로 적용한다.
- 운영자가 별도 캠페인을 우선 노출하더라도 가입 불가 또는 단종 상품은 우선 노출하지 않는다.
• 목적형 정렬 (PI-PRD-RNK-002)
- 카테고리별로 가격 낮은 순, 혜택 많은 순, 절감 가능성 높은 순, 최근 출시 순, 추천 순 중 2개 이상 정렬을 제공할 수 있다.
- 정렬 결과에는 현재 적용 기준을 고객이 확인할 수 있게 한다.

6) 개인화 목록 정책 (PG-PRD-PER-001)

• 개인화 적용 조건 (PI-PRD-PER-001)
- 개인화 빠른 필터는 로그인 상태, 고객 동의, 보유 서비스, 최근 탐색 이력 중 필요한 조건이 충족된 경우에만 제공한다.
- 고객은 개인화 조건을 해제하고 일반 목록으로 전환할 수 있다.
• 피로도 제한 (PI-PRD-PER-002)
- 동일 유형의 추천 조건이나 혜택 브랜드는 동일 세션에서 반복 노출을 제한한다.
- 고객이 관심없음 또는 숨김을 선택한 항목은 해당 세션에서 다시 우선 노출하지 않는다.

7) 비교 후보 정책 (PG-PRD-CMP-001)

• 비교 후보 수 (PI-PRD-CMP-001)
- 비교 후보는 최소 2개 이상 선택 시 비교 진입을 활성화한다.
- 최대 선택 수는 카테고리별 비교 가능 컬럼 수를 기준으로 운영 정책에서 정하며, 초과 시 기존 후보 해제 또는 추가 제한을 안내한다.
• 비교 후보 유지 (PI-PRD-CMP-002)
- 고객이 상세에 진입했다가 목록으로 복귀해도 비교 후보와 목록 맥락은 유지한다.
- 후보 상품이 품절, 단종, 가입 불가로 변경되면 비교 후보에는 남기되 상태 배지와 대체 행동을 표시한다.

8) 찜·최근본 정책 (PG-PRD-SAV-001)

• 찜 저장 (PI-PRD-SAV-001)
- 찜은 상품, 기획전, 이벤트, 콘텐츠, 메뉴 중 정책에서 허용한 대상에 적용할 수 있다.
- 비로그인 임시 찜은 로그인 시 고객 확인 후 이관하고, 이관 실패 시 임시 보관 상태와 삭제 가능성을 안내한다.
• 최근본 보관 (PI-PRD-SAV-002)
- 최근본은 고객별 개인화 이력으로 관리하고 기본 보관 개수와 기간을 운영 설정값으로 둔다.
- 정책 초안 기준은 최대 50개, 최대 90일을 기본값으로 하되 실제 운영 값은 개인정보·서비스 정책 확정 후 조정한다.
• 삭제·끄기 (PI-PRD-SAV-003)
- 고객은 찜과 최근본을 개별 삭제, 전체 삭제, 기능 끄기로 관리할 수 있다.
- 기능 끄기 상태에서는 신규 최근본을 저장하지 않고 기존 이력은 고객 선택에 따라 유지 또는 삭제한다.
• 찜 대상 확장·통합 관리 (PI-PRD-SAV-004)
- 찜 기능은 상품, 기획전, 이벤트, 콘텐츠, 메뉴 등 주요 탐색 대상에 적용할 수 있다.
- 비로그인 임시 찜은 로그인 후 이관하고, 품절·가격변동·혜택종료 같은 상태 변화는 찜 목록에서 확인할 수 있게 한다.

9) 상품 상태·가입 가능 정책 (PG-PRD-AVL-001)

• 가입 가능 표시 (PI-PRD-AVL-001)
- 가입 가능 여부는 고객 회선, 연령, 보유 상품, 단말, 약정, 제한 상태 기준으로 판정한다.
- 가입 가능 상품은 우선 노출할 수 있고 가입 제한 상품은 숨김 또는 비활성화하되 사유를 함께 제공한다.
• 품절·단종 제외 (PI-PRD-AVL-002)
- 품절, 판매 중단, 단종, 주문 불가 상품은 목록 기본 노출에서 제외한다.
- 고객이 직접 조건을 해제하거나 과거 주문·최근본에서 접근한 경우에는 주문 불가 상태와 대체 상품 경로를 표시한다.
• 외국인 가입 제한 (PI-PRD-AVL-003)
- 외국인 고객의 가입 가능 여부는 상품 정책과 BSS 판정 결과를 우선한다.
- 번역 페이지에서도 가입 불가 상품은 비활성화하고 제한 사유를 고객 언어 기준으로 안내한다.
• 공시지원금 기준 일관성 (PI-PRD-AVL-004)
- 단말과 요금제 조합에 따른 공시지원금은 상품 탐색 단계와 주문 단계에서 동일한 기준일과 산정 조건을 사용한다.
- 기준 금액이 변경되면 고객에게 적용 시점과 변경 사유를 안내한다.

10) BSS 판정 연계 정책 (PG-PRD-BSS-001)

• BSS 판정 요청 (PI-PRD-BSS-001)
- 목록 단계에서 고객별 가입 가능 여부가 필요한 경우 채널은 고객 식별 상태와 선택한 상품군 조건을 BSS 판정 요청에 포함한다.
- BSS 회신이 없으면 가입 가능 확정값으로 표시하지 않는다.
• 회신 지연 처리 (PI-PRD-BSS-002)
- BSS 판정이 3초 안에 완료되지 않으면 목록은 일반 후보를 먼저 표시하고 가입 가능 판정은 확인 중으로 표시할 수 있다.
- 동일 조건 재조회가 2회 실패하면 상세 확인 또는 상담 전환 경로를 안내한다.
• 통합 장애 영향 처리 (PI-PRD-BSS-003)
- 통합 연동 장애가 여러 채널 목록에 동시에 영향을 주면 가입 가능 배지는 보수적으로 숨기고 장애 이력과 영향 카테고리를 저장한다.
• 고객 ID와 선택 프로그램 판정 (PI-PRD-BSS-004)
- 채널은 고객 식별 정보와 선택한 마케팅 프로그램 정보를 BSS에 전달하고, BSS는 고객 상태와 가입 가능 조건을 기준으로 가입 가능한 마케팅 프로그램 목록을 회신한다.

11) 빈 결과·오류 폴백 정책 (PG-PRD-ERR-001)

• 빈 결과 안내 (PI-PRD-ERR-001)
- 적용 조건 결과가 0건이면 조건 과다, 가입 제한, 데이터 없음 중 가능한 원인을 구분해 안내한다.
- 고객에게 조건 초기화, 추천 카테고리, 검색, 상담 중 1개 이상의 다음 행동을 제공한다.
• 조회 오류 복구 (PI-PRD-ERR-002)
- 상품 기준 시스템, BSS, 채널 조회 오류가 발생하면 동일 조건 재시도를 제공한다.
- 동일 조건의 중복 요청은 하나의 멱등 요청으로 묶어 중복 처리와 반복 고지를 방지한다.
- 재시도 실패 시 기존 목록 맥락을 보존하고 나중에 다시 시도 또는 상담 전환을 안내한다.
• 빈 화면·오류 폴백 (PI-PRD-ERR-004)
- 전시 데이터 없음, 상품 속성 조회 실패, 가입 가능 판정 지연은 서로 다른 오류 유형으로 구분한다.
- 빈 화면으로 종료하지 않고 추천 카테고리, 비교 진입, 검색, 상담, 최근본 중 고객이 선택 가능한 다음 행동 경로를 1개 이상 제공하며 오류 유형과 선택한 폴백 이력을 저장한다.

12) 후속 진입 정책 (PG-PRD-CTA-001)

• 후속 CTA 기준 (PI-PRD-CTA-001)
- 목록 카드에는 상세, 비교, 찜, 담기, 주문, 상담, 검색 중 상품 상태와 고객 조건에 맞는 CTA만 노출한다.
- 가입 불가 상품에는 주문 CTA를 제공하지 않고 제한 사유와 대체 경로를 제공한다.
• CTA 우선순위 (PI-PRD-CTA-002)
- 목록 카드의 Primary CTA는 고객이 다음에 완료할 가능성이 가장 높은 행동 1개로 제한한다.
- 가입 가능 상품은 상세 또는 주문 진입을 우선하고, 조건 확인이 필요한 상품은 상세 확인을 우선한다.
• 제한 상태 대체 CTA (PI-PRD-CTA-003)
- 품절, 판매 중지, 가입 불가, 회선 조건 미충족 상태에서는 주문 CTA를 노출하지 않는다.
- 이 경우 제한 사유, 대체 상품, 상담 또는 조건 확인 CTA 중 1개 이상을 제공한다.
• 후속 진입 이력 (PI-PRD-CTA-004)
- 상세, 비교, 찜, 담기, 주문, 상담 CTA 선택 이력은 상품 ID, 고객 조건, 노출 위치, 선택 일시와 함께 저장한다.
- 동일 상품 반복 진입은 최근 선택 기준으로 이어보기 또는 비교 후보에 반영한다.

13) 탐색 맥락 유지 정책 (PG-PRD-CTX-001)

• 목록 맥락 저장 (PI-PRD-CTX-001)
- 탭, 필터, 정렬, 검색어, 스크롤, 비교 후보는 동일 세션에서 유지한다.
- 고객이 다른 상품군으로 이동하면 카테고리와 직접 관련 없는 조건은 초기화한다.
• 맥락 만료 (PI-PRD-CTX-002)
- 목록 맥락은 동일 세션 기준으로 우선 유지하고, 고객이 로그아웃하거나 보관 동의를 철회하면 개인화 맥락 저장을 중단한다.
- 저장된 비교 후보와 최근본은 고객 삭제 요청 시 즉시 삭제한다.
• 전시-상세-복귀 상태 유지 (PI-PRD-CTX-004)
- 목록에서 상세로 진입한 뒤 복귀하면 탭, 필터, 정렬, 스크롤, 비교 후보를 유지한다.
- 유지 시간이 만료되거나 목록 기준이 변경된 경우에는 변경 사유와 초기화 여부를 안내한다.

14) 카테고리 운영 정책 (PG-PRD-OPR-001)

• 카테고리 운영 항목 (PI-PRD-OPR-001)
- 운영자는 카테고리명, 사용 여부, 사용 채널, 상위 카테고리, 전시 순서, 전시 단위 매핑, 채널별 수정 가능 범위를 관리한다.
- 카테고리 ID는 생성 후 변경하지 않는다.
• 전시 단위 유형 (PI-PRD-OPR-002)
- 전시 단위 유형은 Product Offering, 번들 구조, 전시 그룹으로 구분한다.
- 번들 구조는 기본 포함 상품과 선택 옵션 상품 수를 구분하고, 전시 그룹은 고객 세그먼트, 가격대, 키워드 기준으로 묶을 수 있다.
• 전용관 운영 기준 (PI-PRD-OPR-003)
- 전용관은 브랜드, 대상 고객군, 프로모션, 멤버십 제공 조건, 알림 가능 여부를 묶어 관리한다.
- 전용관 대상 고객과 노출 기간은 운영 승인 후 적용하며 변경 이력을 저장한다.
• 정책상품 중복 조합 처리 (PI-PRD-OPR-004)
- 판매정책 등록 시 동일한 채널, 가입유형, 요금제 조합의 정책상품이 존재하면 신규 등록을 제한하거나 기존 정책상품의 변경 절차로 전환한다.
- 중복 판단 결과와 처리자는 이력으로 저장한다.

15) 목록 품질 Gate 정책 (PG-PRD-GAT-001)

• 목록 품질 검수 (PI-PRD-GAT-001)
- 목록 변경은 배포 전 카테고리 경로, 필터 의존성, 목록 카드 필수 정보, 가입 가능 사유, 폴백 안내, 개인정보 노출 여부를 검수해야 한다.
- P1 오류가 1건 이상 있으면 배포 승인 상태로 전환하지 않는다.
• 긴급 회수 (PI-PRD-GAT-002)
- 오노출, 단종 상품 주문 가능 표시, 잘못된 가입 가능 사유, 가격·혜택 조건 오표시가 발견되면 긴급 회수 대상으로 둔다.
- 회수 후에는 이전 정상 기준으로 복구하고 변경 이력을 남긴다.

16) 이력·감사 정책 (PG-PRD-AUD-001)

• 운영 변경 이력 (PI-PRD-AUD-001)
- 카테고리, 필터, 정렬, 목록 카드, 전시 단위, 가입 가능 표시 기준이 변경되면 변경 전후 값, 승인자, 적용 시점, 회수 여부를 이력으로 저장한다.
• 고객 행동 이력 (PI-PRD-AUD-002)
- 목록 노출, 필터 적용, 정렬 변경, 비교 후보 선택, 찜, 최근본, 후속 CTA 클릭은 고객 경험 개선과 오류 복구 목적의 이력으로 관리한다.
- 개인정보 활용 동의 범위를 벗어난 개인화 분석에는 사용하지 않는다.

17) 개인정보·보관 정책 (PG-PRD-PRV-001)

• 접근 제어 (PI-PRD-PRV-001)
- 찜, 최근본, 비교 후보, 개인화 빠른 필터는 고객 개인 탐색 정보로 관리한다.
- 타인 계정, 공유 화면, 비로그인 상태에서 개인 탐색 이력이 노출되지 않도록 접근 제어를 적용한다.
• 보관·삭제 (PI-PRD-PRV-002)
- 개인화 탐색 이력은 고객 삭제 요청, 기능 끄기, 탈퇴, 보관 기간 만료 중 하나가 발생하면 삭제 또는 분리 보관 기준을 적용한다.
- 삭제 후에는 동일 이력을 추천이나 정렬 기준으로 다시 사용하지 않는다.

최종 점검 기준

상품 목록 정책서 제출 전 점검
1. 범위 정합성
상품 목록이 카테고리, 목록 카드, 필터·정렬, 비교·찜·최근본, 가입 가능 표시, 폴백 안내 중심으로 작성되었는지 확인한다.
상품 상세, 장바구니, 주문, 결제, 전시 전체 운영의 세부 절차가 본 문서에 과도하게 들어가지 않았는지 확인한다.
2. 고객 탐색 완결성
고객이 어떤 카테고리에 있고 어떤 조건으로 목록을 보고 있는지 이해할 수 있어야 한다.
상세 진입 전에도 가격, 혜택, 가입 가능성, 제한 사유, 현재 상품 대비 차이를 판단할 수 있어야 한다.
3. 필터·정렬 품질
목적형 필터와 정렬이 고객 후보 선별에 필요한 최소 기준으로 제공되는지 확인한다.
필터 의존성, 상호배타, 초기화, 결과 요약 기준이 정책 항목으로 분리되어 있는지 확인한다.
4. 맥락 유지와 후속 진입
목록에서 상세, 비교, 담기, 주문, 상담으로 이동해도 탭, 필터, 정렬, 스크롤, 비교 후보 맥락이 유지되는지 확인한다.
후속 정책서로 넘겨야 할 조건이 기능과 정책 ID로 연결되는지 확인한다.
5. 가입 가능성과 제한 사유
BSS와 상품 기준 시스템의 판정 결과가 목록 카드와 상태 전이에 반영되는지 확인한다.
가입 불가, 품절, 단종, 조건 미충족 상품은 숨김 또는 비활성화와 함께 사유와 대체 경로를 제공해야 한다.
6. 찜·최근본 개인정보 보호
찜, 최근본, 비교 후보, 개인화 빠른 필터가 고객 개인 탐색 정보로 취급되는지 확인한다.
보관 기간, 삭제, 기능 끄기, 접근 제어, 동의 범위 기준이 정책으로 정의되어 있는지 확인한다.
7. 운영 관리 정합성
카테고리, 전시 단위, 필터, 정렬, 목록 카드 기준의 등록, 검수, 배포, 회수 기준이 연결되어 있는지 확인한다.
운영 관리 내용이 화면·DB 상세로 내려가지 않고 정책 판단 기준 수준에 머무르는지 확인한다.
8. 요구사항 반영
상세 요구사항명과 상세 요구사항 설명에서 확인된 72건 요구사항이 유즈케이스, 프로세스, 기능, 정책 중 하나 이상에 매핑되었는지 확인한다.
요구사항 문구를 그대로 복사하지 않고 고객 과업과 판단 기준으로 재구성했는지 확인한다.
