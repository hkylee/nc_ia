1. 핵심 신호

휴대폰 해지 / 핸드폰 해지 / 번호 해지 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

번호 · 회선 해지 셀프 처리 불가

" 서류 없이 빠르게 처리하고 싶다 " — 복잡한 오프라인 절차에 대한 피로감 + 디지털 셀프 처리 기대

02

미성년자 자녀 회선 셀프 채널 처리 불가

"자녀 요금을 부모가 직접 관리하고 싶다 " — 통제 욕구 및 편의 욕구

03

모르게 가입된 외부 제휴 부가서비스 과금 및 즉시 해지 경로 부재

" 동의 없이 요금이 빠져나가는 것에 대한 불안 " — 즉각적인 해지 · 환불 처리 및 재발 방지 욕구

04

요금제 · 약정 · 청구 내역 이해 부족

" 실제 사용량에 맞는 최적 요금제를 선택하고 싶다 " — 합리적 소비 욕구 및 청구 내역 명확 이해 욕구

05

유심 보호 서비스 해지 및 기기 변경 프로세스 단절

" 기기 변경 즉시 완료 원함, 반복 시도 없이 " — 기기 변경 시 자동으로 안내받고 즉시 해결하고 싶은 즉각성 욕구

2. Pain Point 포트폴리오

상담 유형은 단순 문의가 아니라 탐색 실패, 이해 부족, 권한 제약, 시스템 단절, 처리 결과 불안으로 반복된다.

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

모바일 T 월드 번호 해지 셀프 기능 + 전자서명 연동 대응 필요

즉시

보안 불안 해소 욕구 (3 위): " 동의 없이 요금이 빠져나가는 것에 대한 불안 "

외부 제휴 부가서비스 자동 감지 알림 + 원터치 해지 대응 필요

유심 보호 서비스 자동 감지 + 원스톱 해지 안내 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01번호 해지 의사 결정 후 절차 문의 — 전 기간 1 위, 서류 제출 필요 안내로 이탈
  2. 02자녀 요금 절감 목적 요금제 변경 시도 (미성년자 셀프 채널 불가) — 전 기간 2 위
  3. 03청구서 · 요금 내역 확인 중 모르게 가입된 외부 제휴 부가서비스 발견 — 전 기간 3 위
  4. 04기기 변경 후 유심 보호 서비스로 인한 통신 불가 — 설 연휴 · 기기변경 시즌 급증
  5. 05분실 신고 후 폰 습득하여 정지 해제 필요 — 연휴 기간 집중 발생
  6. 06데이터 소진 후 속도 저하 발생 (리필 쿠폰 존재 미인지 포함) — 전 기간 지속
  7. 07해외 체류 중 로밍 데이터 소진 후 재가입 필요 — 연휴 기간 집중
  8. 08약정 만료 문자 수신 → 재약정 필요 여부 및 불이익 확인 — 전 기간 지속
  9. 09번호 이동 후 이전 통신사 요금 포함 청구 → 이중 청구 혼란 — 전 기간 지속
  10. 10연휴 · 야간 해지 업무 불가 → 처리 불가 안내 후 재문의 예정 — 연휴 기간 집중
  11. 11유료 서비스 가입 완료 문자 수신 후 잘못 가입 인지 → 즉시 해지 요청 — 전 기간 지속
  12. 12자녀 번호 정리 필요성 인지 → 보호자 서류 제출 절차 복잡 — 전 기간 지속
  13. 13요금제 변경 후 우주패스 · 가족모아 자동 해지 문자 수신 → 혼란 — 3/3 이후 급증
  14. 14STT 인식 오류로 상담 내용 파악 불가 — 전 기간 지속
  15. 15해외 출국 일정 확정으로 로밍 신청 필요 — 연휴 기간 집중
  16. 16사망 · 거동 불편 등 특수 상황 발생 → 대리인 해지 셀프 처리 불가 — 전 기간 지속
  17. 17자동이체 해지 후 대체 납부 수단 선택 경로 불명확 — 전 기간 지속
  18. 18기기 변경 시 유심 이동 절차 불명확 (유심 보호 서비스 해지 포함) — 전 기간 지속
  19. 19할인 종료 (영히어로 등) 후 요금 증가 원인 불명 → 전화 확인 — 전 기간 지속
  20. 20구독 서비스 (우주패스 · 티티피디 등) 해지 경로 불명확 — 전 기간 지속

5. Quick Win

즉시 개선 가능한 항목은 진입점 노출, 완료 안내, 상태 조회, 권한 처리, 실패 복구를 중심으로 정리된다.

Immediate actions
순위 개선 항목 등장 빈도 기대 효과
1 모바일 T 월드 번호 해지 셀프 기능 구현 (서류 업로드 방식 + 전자서명) 전 기간 전체 번호 해지 전화 상담 20~30% 즉시 감소
2 외부 제휴 부가서비스 자동 감지 알림 + 원터치 해지 + 환불 신청 자동 처리 전 기간 전체 오가입 해지 전화 상담 15~20% 즉시 감소
3 유심 보호 서비스 해지 → 기기 변경 → 재부팅 가이드 원스톱 처리 화면 전 기간 전체 유심 보호 관련 전화 상담 18~32% 즉시 감소
4 분실 정지 해제 셀프 처리 (접수번호 자동 조회 + 간편 인증 + 즉시 해제) 전 기간 전체 분실 정지 해제 전화 상담 15~22% 즉시 감소
5 데이터 잔량 위젯 + 리필 쿠폰 자동 알림 + 원터치 충전 기능 전 기간 전체 데이터 관련 전화 상담 15% 감소
6 해지 예약 기능 (연휴 · 야간 신청 후 평일 자동 처리 + 결과 알림) 연휴 기간 집중 연휴 해지 불가 전화 상담 10% 감소
7 청구서 항목별 설명 팝업 + 번호 이동 후 이전 통신사 요금 분리 표시 전 기간 전체 청구 관련 전화 상담 10% 감소
8 로밍 신청 원스톱 플로우 (출국일 · 국가 입력 → 할인 적용 요금제 추천 → 원터치 신청) 연휴 기간 집중 로밍 관련 전화 상담 30% 감소

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 휴대폰 해지 / 핸드폰 해지 / 번호 해지 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

효율

반복 입력과 재문의를 어떻게 줄이는가?

반복 입력, 회선별 재처리, 동일 문의 재접수를 줄이도록 일괄 처리와 저장된 정보 재사용 기준을 설계해야 한다

진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

휴대폰 해지 · 핸드폰 해지 · 번호 해지 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
