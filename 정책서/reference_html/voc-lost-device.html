VOC analysis readout

분실 VoC 분석

분실 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 분실 신고 · 정지 처리 경로 불명 및 원스톱 처리 부재이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

1. 핵심 신호

분실 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

분실 신고 · 정지 처리 경로 불명 및 원스톱 처리 부재

" 분실 직후 패닉 상태에서 무엇을 해야 할지 한눈에 보고 싶다 " — 즉각적인 보호 조치 + 단계별 명확한 안내 욕구

02

분실 해제 · 습득 처리 셀프 기능 부재 및 접수번호 의존 구조

" 찾았으니 빨리 정상 사용하고 싶다 " — 최소한의 단계로 즉시 복구 욕구

03

가족 · 대리인 분실 신고 · 해제 처리 불가 및 인증 장벽

" 가족 폰을 내가 빠르게 처리해줄 수 있어야 한다 " — 대리 처리 권한 + 인증 간소화 욕구

04

분실 신고 후 후속 서비스 경로 불명

" 분실 신고 한 번으로 모든 후속 처리가 완료되길 원한다 " — 분실 이후 전체 여정 원스톱 처리 욕구

05

분실 관련 정보 · 정책 안내 부재

"정지하면 어떻게 되는지 미리 알고 싶다 " — 정지 옵션별 영향 시뮬레이션 + 명확한 정책 안내 욕구

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

즉각 처리 욕구 (1 위): " 분실 직후 패닉 상태에서 무엇을 해야 할지 한눈에 보고 싶다 "

분실 신고 원스톱 플로우 + 단계별 가이드 대응 필요

통제

빠른 복구 욕구 (2 위): " 찾았으니 빨리 정상 사용하고 싶다 "

습득 처리 셀프 기능 + 접수번호 재조회 대응 필요

즉시

대리 처리 욕구 (3 위): " 가족 폰을 내가 빠르게 처리해줄 수 있어야 한다 "

가족 대리 인증 체계 + 다중 인증 수단 대응 필요

이해

정보 통제권 욕구 (4 위): "정지하면 어떻게 되는지 미리 알고 싶다 "

정지 옵션 영향 시뮬레이션 + 결합 혜택 안내 대응 필요

연계

원스톱 처리 욕구 (5 위): " 분실 신고 한 번으로 모든 후속 처리가 완료되어야 한다 "

분실 신고 완료 후 후속 서비스 원스톱 연계 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01폰 분실 인지 직후 신고 방법 몰라 전화 — 전체 Trigger 의 30~40%, 전 기간 1 위
  2. 02폰 찾은 후 습득 처리 (정지 해제) 방법 몰라 전화 — 전 기간 2 위, 약 15~20 건 / 일
  3. 03가족 (배우자 · 부모 · 자녀) 폰 분실 후 대리 신고 시도 — 전 기간 3 위, 약 12~18 건 / 일
  4. 04분실 신고 후 위치조회 (친구찾기) 방법 문의 — 전 기간 4 위, 약 8~10 건 / 일
  5. 05분실 접수번호 없어 습득 처리 · 정지 해제 불가 — 전 기간 5 위
  6. 06정지 시 결합 혜택 영향 불안으로 전화 — 전 기간 지속, 동일 패턴 6 건 이상 / 일
  7. 07납부카드 · 주소 불일치로 본인확인 실패 — 전 기간 지속
  8. 08분실 신고 후 보험 가입 여부 확인 경로 불명 — 전 기간 지속, 약 5~8 건 / 일
  9. 09정지 유형 (수신가능 / 전면정지 / 발신만) 차이 이해 어려움 — 전 기간 지속
  10. 10신분증 없어 위치조회 불가 → 재방문 필요 안내 — 전 기간 지속, 약 7 건 / 일
  11. 11해외 (일본 · 미국 · 필리핀 등) 에서 폰 · 유심 분실 후 신고 방법 문의 — 전 기간 지속
  12. 12분실 신고 후 임대폰 신청 방법 불명 — 전 기간 지속
  13. 13분실 후 소액결제 악용 우려 및 차단 방법 불명 — 전 기간 지속
  14. 14분실 신고 후 개인정보 · 금융 보안 조치 안내 없음 — 전 기간 지속
  15. 15접수번호 문자 미수신 또는 배터리 방전으로 접수번호 확인 불가 — 전 기간 지속
  16. 16분실 신고 후 약정 기간 영향 안내 없음 — 전 기간 지속
  17. 17연휴 중 분실 외 업무 처리 불가 안내 → 셀프 처리 유도 없음 — 연휴 기간 급증
  18. 18분실 신고자와 해제 요청자 불일치로 처리 거절 — 전 기간 지속
  19. 19타 통신사 분실 해제 후 SKT 미반영 → 유심 꽂아도 전화 불통 — 전 기간 지속
  20. 20분실 보험 vs 파손 보험 차이 몰라 혜택 확인 전화 — 전 기간 지속

Policy conversion
권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

분실 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다

알림

어떤 조건에서 고객에게 먼저 알려야 하는가?

분실 관련 변경, 실패, 혜택 영향, 추가 조치 필요 시점을 알림 기준으로 전환해야 한다
