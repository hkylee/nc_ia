1. 핵심 신호

자동 납부 해지 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

01

납부수단 변경 · 해지 경로 불명확

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

납부 통제권 확보 욕구 (1 위): " 내가 원하는 방식으로, 원하는 시점에 납부하고 싶다 "

납부수단 변경 원스톱 메뉴 + 자동납부 현황 대시보드 대응 필요

통제

즉각적 서비스 복구 욕구 (2 위): " 지금 당장 정지를 풀고 싶다 "

정지 해제 셀프 메뉴 + 자동납부 카드 · 은행 마스킹 힌트 표시 대응 필요

즉시

이중납부 방지 욕구 (3 위): "이중납부 없이 안전하게 처리하고 싶음 "

납부수단 변경 시 이중납부 방지 안내 팝업 + 당월 결제 처리 안내 대응 필요

이해

가족 회선 통합 관리 욕구 (4 위): " 가족 모두의 납부수단을 한 번에 변경하고 싶음 "

패밀리 계정 납부수단 일괄 변경 + 명의자 간편동의 대응 필요

연계

정보 투명성 욕구 (5 위): " 요금 구성과 자동납부 상태를 명확히 알고 싶다 "

청구서 항목별 상세 설명 + 자동납부 현황 실시간 위젯 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01카드 해지 · 만료 · 분실 후 납부수단 변경 필요 인지 — 전체 Trigger 의 40~50%, 전 기간 1 위
  2. 02정지 상태에서 즉시 해제 필요 (장기 / 일시 / 해외 / 분실 / 미납 정지) — 설 연휴 기간 1 위, 평일 2~3 위
  3. 03가족 명의 번호 납부수단 변경 필요 인지 (자녀 · 배우자 · 부모님) — 전 기간 3~4 위
  4. 04자동납부 해지 후 납부방법 전환 필요 인지 (지로 · 가상계좌 · 직접납부) — 전 기간 지속
  5. 05귀국 후 로밍 자동 해지 여부 불안 — 설 연휴 기간 집중 발생
  6. 06자동납부 예정 안내 문자 수신 후 내역 확인 및 해지 요청 — 전 기간 지속
  7. 07자동납부 당일 카드 변경 원하나 처리 불가 안내 — 전 기간 지속
  8. 08번호이동 후 이관 요금 포함 고액 청구 문자 수신 — 번호이동 시즌 집중
  9. 09미납 · 결제 실패 문자 수신 후 원인 파악 및 납부 처리 — 전 기간 지속
  10. 10연휴 중 납부 방법 변경 필요 인지 → 처리 불가 안내 — 공휴일 · 주말 집중
  11. 11자동납부 해지 후 청구 문자 재수신으로 해지 여부 혼란 — 전 기간 지속
  12. 12카드 유효기간 만료로 납부수단 변경 필요 인지 — 전 기간 지속
  13. 13자동납부 해지 신청 후 당월 결제 강행에 불만 — 전 기간 지속
  14. 14유심보호서비스로 인해 유심 이동 · 기기 변경 불가 — 3 월 이후 증가
  15. 15ARS 오류 · 입력 불가 상황에서 자동납부 등록 · 변경 필요 — 전 기간 지속
  16. 16카드사와 통신사 안내 불일치로 혼란 — 전 기간 지속
  17. 17자동납부 해지 후 미납 발생 알림 및 재등록 필요 — 전 기간 지속
  18. 18인터넷 ·TV 자동납부 변경 시 무선 부서에서 처리 불가 — 전 기간 지속
  19. 19자동납부 할인 미적용 확인 후 원인 파악 위해 전화 — 전 기간 지속
  20. 20고령 고객 앱 사용 어려워 가족이 대리 전화 → 명의자 확인 절차 필요 — 전 기간 지속

5. Quick Win

즉시 개선 가능한 항목은 진입점 노출, 완료 안내, 상태 조회, 권한 처리, 실패 복구를 중심으로 정리된다.

Immediate actions
순위 개선 항목 등장 빈도 기대 효과
1 앱 내 자동납부 해지 원스톱 메뉴 신설 (3 클릭 이내 해지 완료) 전 기간 전체 단순 해지 전화 30~40% 즉시 감소
2 자동납부 해지 완료 화면에서 당월 결제 처리 여부 + 대체 납부수단 즉시 안내 팝업 전 기간 전체 해지 후 재문의 전화 50% 감소
3 정지 해제 셀프 메뉴에 자동납부 카드 · 은행 마스킹 힌트 (앞 2 자리) 표시 설 연휴 기간 집중 인증 실패 전화 대폭 감소
4 카드 유효기간 만료 30 일 전 앱 푸시 알림 + 납부수단 변경 딥링크 전 기간 전체 카드 만료 관련 전화 선제 차단
5 자동납부 해지 완료 후 " 청구 문자는 정상 발송됩니다 " 안내 팝업 전 기간 전체 청구 문자 혼란 전화 즉시 감소
6 앱 메인 자동납부 현황 실시간 위젯 (등록 상태 · 이용 정지 상태 통합 표시) 전 기간 전체 현황 확인 전화 20~30% 감소
7 카드 변경 시 이중납부 방지 안내 팝업 + 이번 달 결제 처리 방법 명확 안내 전 기간 전체 이중납부 우려 전화 대폭 감소
8 로밍 현황 메뉴에 자동 해지 예정일 + 현재 상태 실시간 표시 설 연휴 기간 집중 귀국 후 로밍 확인 전화 즉시 감소

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 자동 납부 해지 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

자동 납부 해지 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
