Signal summary
01

단일 진입점

기능 단위 통합

브랜드별 화면을 붙이는 방식보다 기능을 해체하고 고객 여정 기준으로 재그룹핑해야 한다.

03

AI 실행 보조

4. 현재 채널 Gap 진단

현재 채널의 문제는 화면이 부족한 것이 아니라 기준이 분산되어 있다는 점이다.
Gap은 고객 혼란, 처리 단절, 운영 복잡도로 이어진다.

Gap diagnosis
링크 집합

채널 간 연결은 있으나 고객 여정이 하나로 이어지지 않는다.

Critical
정보 과다

반복 배너와 정책성 설명이 목록 단계부터 과도하게 노출되고, 작은 폰트와 낮은 대비로 가독성이 떨어진다.

High
용어 불일치

혜택과 쿠폰의 의미가 채널별로 달라 고객 해석이 흔들리고, 사업자 중심 워딩으로 제공 가치가 즉시 전달되지 않는다.

High
인증·결제

인증 수단, 인증 강도, 결제수단, 주문 순서가 서비스마다 달라 채널 이동 시 반복 인증과 결제 경험 단절이 발생한다.

Critical
MY·장바구니

T월드, T멤버십, T다이렉트, T우주의 MY와 장바구니가 분리되어 구매 맥락과 번들 판단이 끊긴다.

High
셀프 처리

요금 납부 전용 계좌 신청, 결합상품 해지, 유선 주문취소, T멤버십 카드 일시정지 취소처럼 앱 내 처리가 제한되는 업무가 남아 있다.

High
우선 전환 방향

고객 목적을 기준으로 홈, MY, 혜택, 주문, 고객지원, 설정을 재배치한다.

운영 기준

부서별 운영은 유지하되 디자인, 용어, 데이터, 배포, 롤백 기준을 공통화한다.

정책서 반영 방식

각 정책서에는 AI 연결, 공통 인증, 공통 결제, 셀프 처리, 예외 이관 기준을 남긴다.

6. 가입 프로세스 재설계 관점

가입 편의성은 입력 항목의 절대 개수보다 고객이 한 번에 처리해야 하는 판단과 행동의 양에 더 크게 좌우된다.

Signup friction
01생각할 양을 줄인다.

한 화면에 여러 질문과 설명을 동시에 쏟아내면 실제 항목 수가 비슷해도 고객은 더 어렵게 느낀다.
통신 3사 방식의 높은 시각 밀도와 롱폼 세로 스크롤보다, 한 화면당 하나의 질문을 제시하는 단계형 흐름이 체감 속도를 높인다.

02행동할 양을 줄인다.

약관 개별 체크, 선택 약관 해제, 입력 영역 직접 선택, 세로 스크롤, 요금제 제약사항 확인, 동일 정보 재확인이 누적되면 가입은 길고 무거운 작업으로 인식된다.
고지 의무는 지키되 터치와 재확인을 줄여야 한다.

03설득 논리를 제공한다.

요금제 추천은 단순 나열이 아니라 현재 사용 조건, 절감액, 혜택 차이, 경쟁사 대비 장점을 함께 보여줘야 신뢰가 생긴다.

7. 통합채널 설계 원칙

벤치마킹 결과를 통합채널 정책서 작성 기준으로 바꾸면 아래 원칙이 된다.

Design principles
01
목적 먼저, 브랜드는 나중.

고객이 무엇을 하려는지 먼저 드러나고 브랜드나 서비스 경계는 필요한 순간에만 보인다.

02
기능은 공통화하고 예외는 정책화한다.

인증, 결제, 주문, 장바구니는 공통 모듈을 기본으로 하고 상품별 차이는 예외 기준으로 관리한다.

03
AI는 탐색이 아니라 실행 보조다.

AI는 메뉴 안내에서 끝나지 않고 조건 확인, 비교, 처리 시작, 상담 이관까지 책임 범위를 가져야 한다.

04
혜택은 루프로 설계한다.

고객이 혜택을 확인하고 실제 사용하며 다시 방문하도록 미션, 보상, 사용 경로, 소진 기준을 연결한다.

05
제휴는 독립 운영, 경험은 통합 통제.
