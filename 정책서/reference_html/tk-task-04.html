01

As-Is 문제

데이터, 통화, 쿠폰, 할인바코드, 이용권이 채널과 유형별로 분산되어 고객은 보유 자산과 사용 가능한 행동을 한눈에 파악하기 어렵다.

  • 자산별 정보 수준, 표현, 순서, UI가 다르다.
  • 데이터 선물·충전 정책과 제한 조건을 실행 전에 이해하기 어렵다.
  • 혜택은 다양하지만 오늘 바로 쓸 수 있는 것 중심으로 재구성되지 않는다.
02

전략 전환

모든 자산을 내 자산 기준으로 통합하고, 보유 현황, 사용 조건, 만료일, 사용 방법, 다음 행동을 공통 모델로 표준화한다.

  • 자산 상태와 실행 가능 CTA를 같은 구조로 제공한다.
  • 조건을 사전 체크해 가능한 수단만 노출한다.
  • 미사용·만료 임박·위치·행동 맥락을 기반으로 혜택 사용을 제안한다.
03

To-Be 결과

고객은 여러 메뉴를 오가지 않고 지금 쓸 수 있는 자산을 확인하고, 충전·선물·사용·공유·자동화를 빠르게 완료한다.

  • 탐색 시간과 상담 의존이 줄어든다.
  • 혜택 사용률과 체감 가치가 높아진다.
  • 가족·다회선과 헤비 유저의 반복 업무 처리 시간이 감소한다.

이 문서는 통합채널의 상품·서비스 자산 이용 경험을 고객이 가입 상품에서 발생한 혜택과 사용 가능 자산을 이해하고 바로 실행할 수 있는 자산 관리 허브로 정의한다.
동시에 데이터 충전·선물, 쿠폰·이용권 사용, 가족·다회선 관리, 미사용 혜택 케어, 24시간 셀프 처리 기준을 포함한다.

2. 과제 목적

  • 01

    요금제, 부가서비스, 구독상품, 로밍 등 가입 상품·서비스에서 발생하는 자산을 고객 기준의 통합 관리 구조로 재설계한다.

  • 02

    데이터, 통화, 쿠폰, 할인바코드, 이용권의 상태, 조건, 만료, 사용 방법, 실행 가능 액션을 같은 원리로 이해하게 한다.

  • 03

    조회 화면에서 그치지 않고 사용, 선물, 충전, 공유, 자동화, 알림, 후속 업무까지 빠르게 완료하도록 한다.

  • 04

    데이터 선물·충전 정책과 제한 조건을 실행 전에 이해하고 가능한 수단만 선택하게 해 실패 경험을 줄인다.

  • 05

    가족·다회선, 미성년, 해외, 헤비 유저, 주말·야간 상황에서도 필요한 자산 업무를 셀프로 처리할 수 있게 한다.

3. 문제 정의

현 상태

가입상품에서 발생하는 다양한 자산이 채널별·유형별로 다른 구조와 이용 방식으로 제공된다.
고객은 자신의 자산을 이용하기 위해 여러 메뉴와 서비스 영역을 오가야 한다.

Pain Point

자산별 특화 기능과 UI가 제각각이라 고객은 유형별로 정보를 다시 해석해야 한다.
조회 중심 구조에 머물러 사용, 충전, 선물, 공유 같은 다음 행동으로 바로 이어지지 않는다.

Root Cause

자산 관리 과정 전반에서 접근 복잡성, 이용 방식의 이질성, 정보 전달 부족, 시스템 제약이 겹쳐 있다.
자산 속성, 사용 맥락, 공유 맥락을 반영한 공통 정보 구조가 부족하다.

미해결 Risk

통합채널 내 자산 관리 복잡도가 지속되어 고객 혼선이 심화된다.
혜택 인지와 이용률이 낮아지고, 핵심 업무 사용성이 떨어지며, 이용·충전·공유 전환이 감소한다.

4. As-Is 문제점

01 정보 분산
현상

데이터, 통화, 쿠폰, 할인바코드, 이용권 정보가 분산되어 있다

문제점

고객이 자신의 자산을 이용하려면 여러 메뉴와 서비스 영역을 오가야 한다.

02 자산별 학습 부담
현상

자산 유형별 정보 수준과 UI가 다르다

문제점

고객이 자산 유형별로 정보를 매번 새롭게 해석해야 하며 학습 부담이 발생한다.

03 조회 중심 구조
현상

사용 경로가 직관적이지 않고 조회 중심 구조에 머문다

문제점

고객은 정보 조회 이후 필요한 업무를 바로 처리하지 못하고 다시 탐색하거나 상담으로 이탈한다.

04 정책 복잡성
현상

데이터 관련 정책이 복잡하고 한눈에 보이지 않는다

문제점

고객은 데이터 선물, 리필, 공유, 충전 방식의 차이와 한도, 횟수를 이해하기 어렵고 시도 후 실패를 경험한다.

05 반복 처리 제약
현상

대용량 선물과 다수 수신자 동시 처리가 어렵다

문제점

고객은 한 명씩, 1GB씩 반복 처리하며 조작 피로를 경험하고 전화가 더 빠르다고 느낀다.

06 즉시 사용성 부족
현상

혜택이 오늘 바로 쓸 수 있는 것 중심으로 정리되지 않는다

문제점

고객이 이용할 혜택을 직접 찾고 해석해야 해 실제 사용 전환이 낮아진다.

07 혜택 케어 부족
현상

혜택 사용을 유도·케어하는 체계가 부족하다

문제점

고객이 혜택을 사용하지 못한 채 놓치고 상품·서비스 체감 가치가 낮아진다.

08 가족 관리 한계
현상

가족 단위 실사용 시나리오를 충분히 지원하지 못한다

문제점

보호자가 자녀, 배우자, 부모 회선의 구독, 데이터, 혜택, 쿠폰을 대신 관리하기 어렵다.

09 운영시간 제약
현상

공휴일·주말·야간 핵심 업무 제한이 잦다

문제점

고객은 필요한 시점에 앱이 무력하다고 느끼고 채널 신뢰도를 잃는다.

5. To-Be 흐름

전환 흐름은 보유 자산을 조회하는 데서 끝나지 않고, 사용 가능 여부를 먼저 판단하고 바로 실행·공유·케어까지 이어지는 구조여야 한다.

01
Asset View 자산 통합 조회

데이터, 통화, 쿠폰, 할인바코드, 이용권을 내 자산 기준 단일 화면에서 확인한다.

02
Condition 상태/조건 이해

사용 가능 상태, 만료 예정, 사용 조건, 한도, 횟수, 다음 행동을 고객 언어로 설명한다.

03
Eligibility 실행 가능성 체크

고객 조건과 자산 정책을 사전 확인해 사용 가능한 충전, 선물, 공유, 등록 수단만 노출한다.

04
Action 즉시 실행/일괄 처리

사용하기, 선물하기, 충전하기, 공유하기, 자동 설정을 단일 또는 일괄 처리로 수행한다.

05
Follow-up 완료/후속 안내

처리 완료 후 적용 결과, 남은 자산, 후속 처리, 대안, 결과 알림을 이어서 안내한다.

06
Care 혜택 케어/자동화

미사용, 저활용, 만료 임박, 위치, 시간대, 최근 행동을 기준으로 혜택 사용을 먼저 제안한다.

Benefit Wallet

  1. 01

    혜택 사용처와 관련 서비스 바로가기를 연결하고 원터치 사용을 제공한다.

Proactive Care

고객이 혜택을 찾아가기 전에 만료, 위치, 행동, 시간 맥락에 맞춰 사용 기회를 알려준다.

1자산은 데이터, 통화, 쿠폰, 할인바코드, 이용권처럼 유형이 달라도 보유 현황, 사용 조건, 만료일, 사용 방법, 다음 행동 기준으로 관리해야 한다.

6혜택 알림은 수신 동의, 빈도, 피로도, 위치·시간·행동 맥락, 왜 지금 사용해야 하는지의 설명 기준을 가져야 한다.
7공휴일, 주말, 야간에는 즉시 처리 가능, 접수 가능, 처리 예정, 결과 알림, 대안 제공 상태를 구분해야 한다.
8고객 보유 자산 상태는 MY, 멤버십, 이벤트, 주문 사후관리, 고객지원에서 같은 상태명과 후속 행동으로 제공해야 한다.
9자산 사용 실패와 만료 임박은 알림, 복구, 대체 혜택, 상담 연결 기준을 가져야 한다.
