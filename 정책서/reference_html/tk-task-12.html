서비스별 계정 관리에서 신뢰 기반 통합 회원 운영 체계로 전환한다.

핵심은 로그인 화면을 통일하는 것이 아니라, 고객의 신원, 권한, 동의, 알림, 개인화 설정을 하나의 기준으로 관리하고 필요한 순간에만 안전하게 인증하는 것이다.

전략 전환 맵

이 과제는 계정, 인증, 약관, 알림, 설정을 각각 개선하는 작업이 아니라, 통합채널의 모든 고객 경험이 의존하는 신뢰 기반 운영 레이어를 만드는 일이다.
고객은 하나의 계정으로 서비스를 이용하고, 시스템은 동의와 권한, 인증 수준, 수신 설정, 대표값을 일관되게 해석해야 한다.

01

As-Is 문제

서비스마다 가입, 로그인, 인증, 약관, 알림, 설정 체계가 다르고 고객 정보도 각각 관리된다.

  • 고객은 같은 정보를 여러 곳에서 수정해야 한다.
  • 동일한 인증을 반복해 이용 흐름이 끊긴다.
  • 약관과 알림 상태를 한 번에 관리하기 어렵다.
02

전략 전환

CI 기반 통합 계정, 공통 인증 모듈, 통합 동의·알림·설정 허브를 만들어 고객 정보와 권한을 하나의 운영 기준으로 관리한다.

  • 로그인 수단과 세션 정책을 표준화한다.
  • 위험도에 따라 인증을 차등 적용한다.
  • 동의·수신·개인화 설정을 고객 통제형으로 제공한다.
03

To-Be 결과

고객은 하나의 계정으로 서비스를 이동하고 필요한 순간에만 인증하며, 회원정보와 동의·알림·설정을 스스로 관리한다.

  • 통합 회원 전환율이 높아진다.
  • 재인증과 설정 관련 문의가 줄어든다.
  • 개인화와 알림 기반 행동 전환이 안정적으로 확장된다.
Customer 가입·로그인·설정 제어

고객은 하나의 계정으로 서비스를 이용하고 정보, 동의, 알림, 개인화 설정을 직접 관리한다.

Identity Layer CI·계정·세션

통합 계정 레이어는 가입, 로그인, 탈퇴, 계정 연동, 세션 유지와 전환을 담당한다.

Trust Layer 인증·권한·동의

인증 레이어는 업무 위험도와 고객 상태에 따라 필요한 인증과 권한 확인, 약관 동의를 판단한다.

Preference Layer 알림·대표값·개인화

설정 레이어는 알림 수신, 홈, 대표값, 접근성, 언어, 서비스 권한을 고객 기준으로 관리한다.

1. 이 문서가 다루는 과제

이 문서는 회원 가입, 로그인, 탈퇴, 인증, 회원정보 관리, 약관 동의, 알림, 설정을 통합 회원 체계 안에서 관리하는 기준을 정의한다.
고객은 서비스마다 별도 계정을 만들거나 같은 인증을 반복하지 않고, 필요한 정보와 권한을 한 곳에서 확인하고 변경할 수 있어야 한다.
동시에 개인정보, 인증, 약관, 알림 수신 권한은 편의성만이 아니라 고객 통제와 보안 안정성을 함께 만족해야 한다.

통합채널 연결 관점

회원·계정·인증·동의 체계는 12개 과제 전체의 기반 레이어이므로 각 서비스가 동일한 신원, 권한, 동의, 알림 기준을 사용해야 한다.

3. 문제 정의

현 상태

서비스별로 회원가입, 로그인, 인증, 회원정보, 약관, 알림, 설정 기능이 분산 운영된다.
같은 고객이라도 채널별로 다른 계정 경험과 설정 구조를 만나게 된다.

Pain Point

고객은 서비스마다 로그인 방식과 인증 절차를 다시 익히고, 약관·알림·회원정보 변경 상태를 한 번에 확인하기 어렵다.
로그인 전에는 제공 가치가 약해 가입 전환 동기도 낮다.

Root Cause

채널별 계정·인증 체계가 분리되어 있고, 회원정보·약관·알림·설정 데이터를 통합 관리하는 구조가 부족하다.
인증과 설정 정책도 표준화되어 있지 않다.

미해결 Risk

계정 관리 피로와 기능 탐색 시간이 늘고 인증·설정 문의가 증가한다.
개인화 서비스 확장과 데이터 활용이 제한되며, 채널 간 경험 불일치로 고객 불만과 이탈 가능성이 커진다.

4. As-Is 문제점

01 가입·로그인 분산
현상

회원가입 절차와 로그인 방식이 서비스마다 다르게 제공된다

문제점

채널별 개별 회원정보 관리로 다회선·다서비스 고객이 한 번에 통합 관리하기 어렵다.

02 회원정보 분산
현상

회원정보가 서비스별로 따로 관리된다

문제점

프로필, 배송지, 연락처 같은 정보를 변경하려면 각 서비스에 접속해 개별 수정해야 하며 UX도 일관되지 않다.

03 반복 인증
현상

인증이 유지되지 않아 동일한 인증을 반복해야 한다

문제점

채널별 인증 유지가 미흡해 반복 인증이 발생하고 이용 피로도와 이탈 가능성이 높아진다.

04 약관 관리 복잡
현상

약관이 많고 서비스별로 나뉘어 상태 파악이 어렵다

문제점

고객은 어떤 약관에 동의했는지 확인하기 어렵고 변경, 철회, 재동의 과정도 번거롭다.

05 전환 가치 부족
현상

로그인 전에 기능과 혜택을 확인하기 어렵다

문제점

로그인 전 제공 가치가 부족해 가입과 로그인 전환 동기가 약해진다.

06 설정 학습 비용
현상

설정 기능이 서비스마다 달라 사용 방법을 다시 익혀야 한다

문제점

채널별 설정 구조와 방식이 달라 고객 혼란과 학습 비용이 증가한다.

5. To-Be 흐름

회원 관리 흐름은 비로그인 가치 확인에서 시작해 가입·로그인, 세션 유지, 상황별 인증, 회원정보 변경, 약관·알림·설정 제어, 탈퇴·철회·이력 확인까지 이어져야 한다.

01
Pre-login 비로그인 가치 확인

고객은 로그인 전에도 혜택, 정보, 멤버십 가치, 주요 기능 미리보기를 확인한다.

02
Unified ID 통합 가입·로그인

고객은 CI 기반 통합 계정으로 가입하거나 기존 계정을 전환하고 SSO, PASS, 생체, 소셜 로그인 중 적합한 수단을 선택한다.

03
Step-up Auth 세션 유지·상황 인증

로그인 이후 세션을 유지하고 가입, 변경, 결제, 민감정보 변경 같은 고위험 행동에서 필요한 인증만 수행한다.

04
Profile 회원정보 통합 관리

프로필, 연락처, 배송지, 결제수단, 대표 회선, 서비스 연결 정보를 조회·수정하고 변경 이력을 확인한다.

05
Control 동의·알림·설정 제어

약관 동의, 철회, 재동의, 알림 채널·유형별 수신 권한, 홈·대표값·개인화 설정을 조정한다.

06
Govern 이력·탈퇴·보정

탈퇴, 계정 해제, 동의 이력, 알림 수신 이력, 설정 변경 결과를 확인하고 운영 품질 보정에 반영한다.

↺
Feedback Loop

가입 전환, 로그인 실패, 재인증 발생, 약관 철회, 알림 수신 변경, 설정 변경, 탈퇴 사유 데이터는 다시 로그인 수단 우선순위, 인증 단계, 동의 문구, 알림 피로도, 개인화 설정 기본값과 보안 정책 보정에 반영된다.

6. 핵심 지향점·기대 효과·KPI

회원 통합 과제는 CI 기반 계정 일원화, 로그인·세션 연속성, 인증 모듈 공통화, 회원정보 통합 관리, 약관·알림·설정의 고객 통제, 비로그인 전환 경험을 함께 달성해야 한다.

01

CI 기반 통합 계정 관리

통합 회원 전환율계정 전환 후 서비스 이용률계정 관련 문의율
To-do
  • 가입, 로그인, 탈퇴, 계정 변경, 서비스 연동을 CI 기반 통합 계정으로 관리한다.
  • 기존 서비스별 회원을 통합 회원 체계로 전환하는 기준과 전환 안내를 마련한다.
  • 계정 상태, 연결 서비스, 대표값, 탈퇴·해제 조건을 한 곳에서 확인하게 한다.
기대 효과
  • 고객이 서비스마다 가입하지 않고 하나의 계정으로 이용한다.
  • 계정 전환 없이 서비스 이동과 주요 업무 수행이 가능해진다.
02

통합 로그인과 세션 연속성

To-do

  • 로그인 상태 유지와 서비스 간 세션 연속 이용 기준을 정의한다.
  • 비로그인 상태에서 혜택과 정보를 보여주고 로그인 필요 시점에는 선택값을 유지해 전환한다.
기대 효과
  • 서비스 이동 중 재로그인 피로가 줄어든다.
  • 로그인 전 탐색과 로그인 후 업무 실행이 자연스럽게 이어진다.
03

인증 모듈 공통화와 상황 기반 인증

주요 업무 내 인증 횟수인증 이후 업무 완료율인증 이후 이탈률
To-do
  • 본인인증, 회선 확인, 명의 검증, 결제 인증을 공통 인증 모듈로 표준화한다.
  • 가입, 변경, 결제, 민감정보 변경 등 업무 위험도에 따라 인증 수준을 차등 적용한다.
  • 업무 시작 시 필요한 인증을 한 번에 수행하고 동일 업무 흐름 내 반복 인증을 방지한다.
기대 효과
  • 주요 업무 수행 중 인증으로 인한 중단이 줄어든다.
  • 보안이 필요한 행동은 안전하게 보호하면서 일반 탐색과 설정 변경의 편의성은 유지된다.

회원정보 통합 관리

  • 서비스별 보유 정보와 통합 회원정보의 우선순위, 동기화, 반영 시점을 정의한다.
  • 변경 이력과 최신 반영 상태를 고객이 확인하게 한다.
기대 효과
  • 회원정보 변경을 여러 서비스에서 반복하지 않아도 된다.
  • 고객 정보의 최신성과 운영 데이터 품질이 높아진다.

  • 약관 동의, 거부, 철회, 재동의, 변경 이력을 하나의 화면에서 관리한다.
  • 알림 채널과 유형별 수신 권한을 통합 관리하고 알림에서 관련 화면과 후속 행동으로 연결한다.
기대 효과
  • 고객이 동의 상태와 수신 권한을 한 번에 이해하고 변경한다.
  • 불필요한 알림 수신과 동의 상태 관련 혼란이 줄어든다.

  • 고객이 알림, 개인정보, 대표 서비스, 개인화 수준을 직접 조정할 수 있게 한다.

7. 주요 프로세스 및 기능 설명

01
Convert Flow

비로그인 탐색과 통합 회원 전환

로그인 전에도 가치를 보여주고 필요한 순간에 통합 회원 전환으로 자연스럽게 연결한다.

Outcome

비로그인 탐색 이탈 감소와 통합 회원 전환율 향상

  1. 01

    고객은 비로그인 상태에서 혜택, 정보, 멤버십 가치, 주요 기능의 일부를 확인한다.

  2. 02

    로그인이 필요한 시점에는 필요한 이유와 로그인 후 추가로 가능한 기능을 안내받는다.

  3. 03

    고객은 기존 계정을 통합 회원으로 전환하거나 신규 통합 계정을 생성한다.

  4. 04

    시스템은 CI를 기준으로 기존 회원 정보와 연결 가능한 서비스를 식별하고 전환 결과를 안내한다.

02
Auth Flow

로그인·세션 유지와 상황별 인증

로그인 경험은 가볍게 유지하고 위험 행동만 필요한 인증으로 보호한다.

Outcome

재로그인·반복 인증 감소와 주요 업무 완료율 향상

  1. 01

    고객은 SSO, PASS 인앱, 생체인증, 소셜 로그인 중 가능한 수단으로 로그인한다.

  2. 02

    로그인 후 서비스 이동과 일반 탐색에서는 세션을 유지해 재로그인을 줄인다.

  3. 03

    가입, 변경, 결제, 민감정보 수정, 탈퇴 같은 고위험 행동에서는 업무 시작 시 필요한 인증을 수행한다.

  4. 04

    인증 결과는 동일 업무 흐름 안에서 재사용하고 실패 시 원인과 복구 경로를 안내한다.

03
Profile Flow

회원정보와 대표 설정 관리

회원정보와 대표값을 한 곳에서 관리하고 변경 영향과 반영 상태를 명확히 보여준다.

Outcome

중복 수정 감소와 고객 정보 품질 향상

  1. 01

    고객은 프로필, 연락처, 배송지, 결제수단, 대표 회선, 연결 서비스를 통합 조회한다.

  2. 02

    정보 수정 시 서비스별 영향, 반영 범위, 반영 예정 시점, 변경 불가 사유를 확인한다.

  3. 03

    고객은 홈, 대표값, 개인화 수준, 접근성, 언어, 서비스 권한 설정을 변경한다.

  4. 04

    시스템은 변경 이력과 최신 상태를 저장하고 고객에게 반영 결과와 후속 행동을 안내한다.

04
Consent Flow

약관·알림 수신 권한 제어

동의와 수신 권한을 고객이 이해하고 직접 조정하는 통제 경험으로 만든다.

Outcome

동의·알림 관련 혼란 감소와 알림 행동 전환 향상

  1. 01

    고객은 서비스별 약관을 목적, 필수·선택, 동의 상태, 변경일, 철회 가능 여부 기준으로 확인한다.

  2. 02

    필요한 약관에 동의, 거부, 철회, 재동의를 수행하고 이력을 조회한다.

  3. 03

    알림 센터에서 채널별·유형별 알림을 확인하고 수신 권한을 설정한다.

  4. 04

    알림은 관련 화면과 후속 행동으로 연결되고, 고객은 과다 수신이나 누락을 줄이도록 수신 범위를 조정한다.

8. 정책서 작성 시 남겨야 할 판단 기준

1통합 회원은 하나의 로그인 UI가 아니라 CI 기반 계정, 연결 서비스, 계정 상태, 탈퇴·해제 조건을 함께 관리하는 운영 모델이어야 한다.
2로그인 수단은 SSO, PASS, 생체, 소셜 등 고객 선택지를 제공하되 서비스 이동과 일반 탐색에서는 세션 연속성을 보장해야 한다.
3인증은 모든 기능에 반복 요구하는 방식이 아니라 업무 위험도, 민감정보 여부, 결제·변경 영향, 고객 상태에 따라 차등 적용해야 한다.
4회원정보는 프로필, 연락처, 배송지, 결제수단, 대표 회선, 연결 서비스의 원천 우선순위와 반영 시점을 명확히 정의해야 한다.
5약관은 서비스별 나열이 아니라 목적, 필수·선택, 철회 가능 여부, 재동의 필요, 변경 이력 기준으로 구조화해야 한다.
6알림은 채널과 유형별 수신 권한을 고객이 직접 제어하되 필수 고지, 마케팅, 처리 결과, 보안 알림의 우선순위를 구분해야 한다.
7통합 설정은 홈, 대표값, 개인화, 접근성, 언어, 서비스 권한을 포함하고 변경 시 영향 범위와 즉시·지연 반영 여부를 안내해야 한다.
8비로그인 상태에서도 고객이 혜택과 정보를 이해할 수 있어야 하며 로그인 필요 시점에는 선택값과 탐색 맥락을 유지해야 한다.
9계정, 인증, 동의, 알림, 설정 변경 이력은 고객 문의, 보안 확인, 운영 품질 개선을 위해 추적 가능하게 관리해야 한다.
10통합 계정, 인증, 약관, 알림, 설정 기준은 상품, 주문, 결제, 청구, 고객지원, MY에서 공통 참조하는 기반 정책이어야 한다.
11개인화, AI 추천, 알림, 혜택 제안은 동의 상태와 철회 이력, 정보 변경 이력을 즉시 반영해야 한다.
