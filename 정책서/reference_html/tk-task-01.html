Customer · Agent · Data · Admin
01

As-Is 문제

탐색, 비교, 추천, 실행, 데이터가 서로 다른 채널 구조와 운영 기준으로 분리되어 있다.

  • 검색은 정보 탐색에 머문다.
  • 추천과 챗봇은 고객 통제와 업무 실행으로 충분히 연결되지 않는다.
  • 행동 데이터가 추천 정책과 운영 보정으로 환류되지 않는다.
02

전략 전환

AI Agent를 고객 과업의 실행형 허브로 두고, 추천·비교·업무 실행·상담 전환·데이터 학습을 하나의 체계로 묶는다.

  • 발화 의도를 과업 단위로 분해한다.
  • 추천 근거와 직접 비교 경로를 함께 제공한다.
  • 고위험 업무는 고객 승인과 상담 전환 기준으로 보호한다.
03

To-Be 결과

고객은 한 흐름 안에서 찾고 비교하고 실행하며, 운영자는 결과 데이터를 근거로 정책과 추천 품질을 보정한다.

  • 탐색 이탈과 재문의가 줄어든다.
  • 추천 수용성과 업무 완결률이 개선된다.
  • 트래킹 데이터가 다음 추천과 운영 룰로 되돌아간다.
Customer 발화·비교·선택·실행

운영자가 AI 제안과 추천 룰을 검수하고 빠르게 보정한다.

1. 이 문서가 다루는 과제

이 문서는 통합채널의 AI Agent를 단순 챗봇이 아니라, 고객의 탐색·비교·추천·업무 실행·문제 해결을 연결하는 실행형 진입점으로 정의한다.
동시에 추천 성과와 행동 데이터를 수집해 운영자가 정책을 보정하는 데이터 루프까지 포함한다.

통합채널 연결 관점

AI 탐색·추천 체계는 단독 서비스가 아니라 상품 지식, 전시, 주문, 고객지원, 회원 동의·인증이 같은 여정 데이터로 연결될 때 정책서 작성 기준이 명확해진다.

2. 과제 목적

  • 01

    정보 탐색부터 의사결정, 업무처리, 문제해결까지 이어지는 AI Agent 기반 원스톱 고객경험을 구현한다.

  • 02

    고객 상태와 맥락에 맞는 추천과 행동 제안을 제공하되, 추천 근거와 직접 비교·숨김·비개인화 폴백을 함께 제공하는 고객 통제형 추천 체계를 구축한다.

  • 03

    고객이 최종 승인하기 전에는 가입·변경·해지·결제 같은 고위험 업무가 확정되지 않도록 safe-action 기준을 둔다.

  • 04

    처리 불가, 고위험, 반복 실패 업무는 상담 전환과 맥락 전달 기준을 표준화한다.

  • 05

    검색어, 의도, 추천, 노출, 실행, 실패, 상담 전환 데이터를 같은 여정 기준으로 수집하여 AI 서비스와 채널 고도화의 데이터 기반을 확보한다.

3. 문제 정의

현 상태

검색과 AI 챗봇이 분리되어 있고, 탐색·비교·판단·실행이 서로 다른 채널 구조로 흩어져 있다.
기존 IA는 메뉴와 화면 단위로 깊어져 고객이 과업 기준으로 찾고 실행하기 어렵다.

Pain Point

고객은 탐색 후 다시 비교하고, 비교 후 다시 상담이나 가입 메뉴로 이동한다.
챗봇 답변은 형식적이라고 느끼기 쉽고, AI 자동 추천보다 고객이 직접 비교하고 선택하려는 니즈가 강하다.

Root Cause

통합 AI Agent, Next Action 체계, 추천 가드레일, 이벤트 수집/트래킹 표준이 하나의 운영 구조로 묶이지 않았다.
실행 전 승인, 상담 전환, 추천 통제, AI readable data 기준도 분산되어 있다.

미해결 Risk

고객 피로와 이탈이 증가하고, 전환 기회와 수익화 기회를 놓친다.
AI 추천·실행이 잘못 동작하면 고객 불만, 상담 증가, 운영 롤백, 데이터 신뢰도 하락으로 이어진다.

4. As-Is 문제점

01 탐색 단절
현상

검색이 정보 탐색에 머문다

문제점

후속 상담, 가입, 변경 같은 업무로 바로 연결되지 않아 화면 이탈과 경험 단절이 발생한다.

02 해결 단절
현상

챗봇의 문제 해결 완결성이 낮다

문제점

의도와 맞지 않는 답변이나 형식적 응답으로 상담원 전환과 재탐색이 반복된다.

03 판단 지원 부족
현상

비교·분석·추천 기반 의사결정 지원이 약하다

문제점

고객이 직접 비교하고 판단해야 하므로 선택 피로와 의사결정 지연이 발생한다.

04 맥락 부재
현상

개인화 없는 획일 전시 구조다

문제점

고객 맥락과 무관한 상품·혜택이 노출되어 탐색 부담이 증가한다.

05 추천 신뢰 약화
현상

사업자 중심 업셀 추천이 강하다

문제점

고객이 납득하기 어려운 추천은 수용성을 낮추고 거부감을 만든다.

06 행동 전환 부재
현상

Next Action 유도 체계가 없다

문제점

약정 만료, 데이터 초과 위험, 여정 이탈 같은 순간에 다음 행동 제안이 약하다.

07 데이터 분산
현상

데이터/트래킹 체계가 분산되어 있다

문제점

구조화되지 않은 이벤트 정의, 비식별 데이터 중심 수집, GA 등 트래킹 도구 분산으로 데이터 일관성과 활용도가 낮아진다.

08 고객 통제 부족
현상

AI 추천 신뢰와 고객 통제 기준이 약하다

문제점

고객은 편의성 개인화는 선호하지만 구매·변경 결정은 직접 비교하고 선택하려는 경향이 강하므로, 추천이 일방적 제안으로 보이면 신뢰가 낮아진다.

09 실행 안전 부족
현상

AI 실행 안전장치와 데이터 준비도가 부족하다

문제점

AI가 업무 처리까지 개입하려면 slot filling, 고객 최종 승인, 데이터 시차, API 품질, 할루시네이션 리스크를 통제해야 하지만 현재는 운영 주체와 데이터 형태가 분산되어 있다.

5. To-Be 흐름

7. 주요 프로세스 및 기능 설명

01
Experience Hub

AI Agent 기반 End-to-End 경험

고객 발화에서 업무 실행까지 이어지는 단일 진입점을 만들고, 실패 시 상담 전환까지 끊기지 않게 설계한다.

Outcome

탐색 이탈 감소와 업무 완결률 향상

  1. 01

    공통 Agent 진입점과 텍스트/음성 입력, 예시 질문, 추천 질문을 제공한다.

  2. 02

    발화 의도를 탐색, 비교, 추천, 처리, 문제해결로 분류하고 복합 요청을 분해한다.

  3. 03

    정보형, 비교형, 추천형, 실행형 응답을 고객 상태와 상품 이력에 맞게 구성한다.

  4. 04

    검색 결과에서 상담, 가입, 변경, 신청 등 실행 플로우로 연결하되 고위험 업무는 고객 최종 승인 전 확정하지 않는다.

  5. 05

    처리 불가, 반복 실패, 고위험 업무는 상담 전환 사유와 대화 맥락을 전달한다.

  6. 06

    세션 간 맥락과 이전 이력을 이어주고, 중단 업무 재개와 후속 액션을 제안한다.

  7. 07

    운영자는 검색어, 응답 정책, 사전, 처리 시나리오, 품질 지표를 관리한다.

02
Decision Support

AI 의사결정 지원 및 개인화 추천

추천을 고객 대신 결정하는 장치가 아니라, 비교와 근거를 통해 납득 가능한 선택을 돕는 체계로 만든다.

Outcome

추천 신뢰도와 전환 기여도 향상

  1. 01

    상품/혜택/옵션 상세 접점에서 비교 진입 CTA와 비교 후보를 제공한다.

  2. 02

    가격, 할인, 혜택, 데이터, 부가서비스, 약정/제약 등 비교 기준을 구조화한다.

  3. 03

    고객 상태, 이용 이력, 방문 맥락, 비교 결과와 사업 가드레일을 반영해 추천안을 생성한다.

  4. 04

    최적안, 차선안, 대안형 추천을 구분하고 현재 조건 대비 차이, 예상 혜택, 제한 조건, 미충족 사유를 함께 제공한다.

  5. 05

    AI 추천, 운영자 큐레이션, 인기·테마 추천을 구분하고 개인화 불가 시 비개인화 폴백과 직접 비교 경로를 제공한다.

  6. 06

    추천 우선순위, 비노출, 예외 노출, 숨김·관심 없음, 반복 노출 방지 기준을 운영한다.

03
Journey Recovery

맥락 기반 Next Action

중단, 실패, 만료 임박, 혜택 미사용 같은 여정 신호를 포착해 다음 행동으로 복구한다.

Outcome

고객 행동 재개와 End-to-End 완료율 향상

  1. 01

    가입, 변경, 해지, 중단 재개, 실패 복구, 만료 임박, 혜택 미사용 등 Next Action 유형과 후보 생성 트리거를 정의한다.

  2. 02

    현재 맥락, 고객 상태, 최근 행동, 완료 여부, 수신 동의, 노출 피로도를 반영해 대표 오퍼를 선정한다.

  3. 03

    홈, MY, 상세 등 접점별 전시 위치와 카드, 배너, 팝업, CTA 포맷을 정한다.

  4. 04

    제안된 액션에서 즉시 실행 플로우로 진입하고 저장, 숨김, 리마인드, 만료 같은 후속 상태를 제공한다.

  5. 05

    노출, 클릭, 실행, 전환 성과를 같은 여정 ID로 측정해 오퍼와 전시 기준을 보정한다.

04
Learning Loop

채널 데이터 수집/트래킹

검색, 추천, 실행, 실패, 상담 전환 데이터를 같은 여정 기준으로 연결해 운영 보정의 근거로 쓴다.

Outcome

AI 품질 개선과 운영 의사결정 고도화

  1. 01

    노출, 클릭, 검색, 비교 시작/완료, 추천 조회/수용, 실행 시작/완료, 이탈, 오류, 상담 전환 이벤트를 정의한다.

  2. 02

    세션, 화면, 컴포넌트, 프로세스 단위의 측정 구간과 수집 지점을 설계한다.

  3. 03

    화면, 상품, 고객, 추천 시나리오, Action 맥락 정보를 AI가 읽을 수 있는 공통 파라미터로 표준화한다.

  4. 04

    검색어, 의도, 추천 후보, 실행, 실패, 상담 전환을 동일 여정 ID로 연결한다.

  5. 05

    데이터 시차, 누락, API 실패, 상품 기준정보 변경을 AI 결과 신뢰도에 영향을 주는 품질 이슈로 관리한다.

  6. 06

    전환, 매출, 이탈, 추천 반응, 상담 전환 지표를 리포팅으로 연결한다.

  7. 07

    반복 탐색, 추천 미수용, Action 미실행 같은 문제 구간을 개선 인사이트로 전환한다.

8. 정책서 작성 시 남겨야 할 판단 기준

1AI Agent가 직접 실행 가능한 업무, 안내만 가능한 업무, 상담 전환해야 하는 업무를 구분하고, 고위험 업무는 고객 최종 승인 전 확정하지 않는다.
2추천은 고객 적합도, 사업 우선순위, 제한 상품, 프로모션, 반복 노출 방지 기준을 함께 적용하되 추천 출처, 근거, 제한 조건, 미충족 사유를 고객에게 설명해야 한다.
3개인화 근거가 부족하거나 동의가 없으면 비개인화 추천과 직접 비교 경로를 제공하고, 숨김·관심 없음 반응은 즉시 재노출 제한에 반영해야 한다.
4Next Action은 트리거, 후보 선정, 전시 위치, 문구, CTA, 저장/숨김, 리마인드, 만료, 노출 피로도 기준이 필요하다.
5트래킹은 이벤트명, 파라미터, 수집 위치, 보관 기준, 민감정보 제외 기준뿐 아니라 동일 여정 ID, 데이터 시차, API 실패, 상품 기준정보 변경의 품질 기준이 정책서에 남아야 한다.
6운영 Admin은 정책 변경, 긴급 보정, 실험, 승인, 품질 점검, 배포, 회수, 롤백, 효과 추적 이력을 관리해야 한다.
7AI 추천과 실행은 회원 동의, 인증 수준, 상품·주문 가능 조건, 상담 전환 기준을 같은 여정 ID로 추적해야 한다.
8추천 실패와 상담 전환 데이터는 상품 지식, 전시 우선순위, 고객지원 FAQ와 함께 보정되어야 한다.
