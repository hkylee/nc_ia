VOC analysis readout

부가서비스 가입 VoC 분석

부가서비스 가입 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 의도치 않은 부가서비스 자동 가입이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

1. 핵심 신호

부가서비스 가입 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

의도치 않은 부가서비스 자동 가입

" 내가 동의하지 않은 서비스에 돈이 빠져나가는 것을 원천 차단하고 싶다 " — 과금 공포 + 통제감 상실 + 재발 방지 욕구

02

기기변경 후 유지기간 만료 부가서비스 해지 불가

" 유지기간 만료 즉시 해지하고 싶다 " + " 기기변경 시 가입된 모든 서비스를 한번에 정리하고 싶다 "

03

부가서비스 현황 탐색 실패

" 내 계정에 뭐가 가입됐는지 한눈에 보고 싶다 " — 정보 통제권 + 불필요 비용 절감 욕구

04

해지 후 요금 청구 혼란

" 해지했는데 왜 요금이 나오는지 이해하고 싶다 " — 완전한 취소 확신 욕구 + 예측 가능성 요구

05

공휴일 · 주말 상담 운영 정책 제약

" 공휴일 · 주말에도 내 서비스를 즉시 처리할 수 있어야 한다 " — 즉각 처리 욕구 + 정책 불신

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

과금 통제 욕구 (1 위): " 내가 모르는 사이 돈이 빠져나가는 것을 막고 싶다 "

즉시 취소 딥링크 + 가입 차단 토글 대응 필요

통제

정보 통제권 욕구 (2 위): " 내 계정에 어떤 서비스가 가입됐는지 한눈에 파악하고 직접 관리하고 싶다 "

부가서비스 현황 대시보드 대응 필요

즉시

즉각 처리 욕구 (3 위): " 지금 당장 해지 / 취소해야 하는데 방법이 없다 "

24 시간 셀프 해지 시스템 대응 필요

이해

정보 이해 욕구 (4 위): " 해지했는데 왜 요금이 나오는지, 청구서 항목이 무엇인지 이해하고 싶다 "

청구서 항목 설명 팝업 + 해지 후 잔여 청구 안내 대응 필요

연계

가족 보호 욕구 (5 위): " 고령 부모 · 자녀 명의 서비스를 내가 직접 관리하고 싶다 "

가족 계정 통합 관리 권한 체계 대응 필요

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 부가서비스 가입 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

부가서비스 가입 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다

알림

어떤 조건에서 고객에게 먼저 알려야 하는가?

부가서비스 가입 관련 변경, 실패, 혜택 영향, 추가 조치 필요 시점을 알림 기준으로 전환해야 한다

이력

고객과 운영자는 무엇을 추적해야 하는가?

부가서비스 가입 처리 이력과 고객 고지 이력을 남겨 재문의와 책임 공백을 줄여야 한다
