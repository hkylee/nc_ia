# IA 멘탈 모델

> 출처: ia-import-spec.md §1 + §4

---

## 1. 기본 구조

- **홈에서 시작**하고, **각 모듈의 진입 페이지**가 그 모듈 트리의 시작점(`depth1` 뿌리)이 된다.
  - 예: 회원(MBR) 모듈은 `로그인`이 진입 페이지 → 거기서 회원가입·T-ID·소셜·계정찾기 등이 펼쳐짐.
- 계층은 트리(nested)가 아니라 **`depth1 > depth2 > depth3 …` 경로**로 표현한다. 작업은 depth 컬럼을 왼쪽부터 채우는 방식.

---

## 2. 두 종류의 연결

| 연결 종류 | 컬럼 | 의미 |
|----------|------|------|
| **depth** | `depth1~N` | 포함관계 (메뉴 트리상 부모-자식) — 자동으로 계층 엣지가 됨 |
| **from** | `from` | 화면 전환 — 이 화면으로 들어오는 출발 화면. **depth로 안 잡히는 전환만** 적는다 |

- **모듈 경계를 넘는 흐름은 `from`으로** 이어진다 (예: 로그인(MBR)의 from에 마이(MY)).

---

## 3. 플로우 맵 앵커 = 화면 이름

플로우 맵(노드·화살표)은 **화면 이름을 조인 키(join key)**로 그려진다.

| 역할 | 컬럼 | 맵에서 |
|------|------|--------|
| 노드(점) | `page_id` (표시명 = 마지막 depth 값) | 같은 page_id = 같은 점 1개 |
| 계층 엣지 | `depth1 > depth2 > …` | 부모→자식 포함 선 |
| 전환 엣지 | `from` | 이동 화살표 (들어옴, 이름으로 매칭) |
| 흐름 스타일 | `flow_type` | 기본=실선 / 케이스=색 구분 / 예외=점선·흐림 등 구분 렌더 |
| 그룹/레인 | `domain`, `uc_code` | 모듈·UC 단위 묶음 (배치 기준) |
| 노드 정체성 | `page_id` | 같은 page_id = 같은 페이지(병합). 이름 아닌 ID로 식별 |
| 원본 표시 | `owner_page`, `owner_uc` | 마스터 페이지·원본 UC 체크(✅) — 병합 시 대표 인스턴스 |
| 배치 순서 | 행 순서 | 그룹 내 정렬 |

---

## 4. 노드 식별 규칙

- **같은 `page_id` = 같은 페이지로 병합** (사방에서 화살표 수렴). page_id가 다르면 이름이 같아도 별개 노드.
- **`from`은 화면 이름으로 매칭**하므로 같은 UC 안에선 이름이 유일해야 함. (page_id가 중복 페이지를 안전히 구분)
- `from`의 값은 반드시 실재하는 화면 — 오타·미존재는 끊긴 화살표(dangling).

---

## 5. Import 전 검증 패스

전달받은 IA 데이터를 import하기 전 반드시 아래 4개를 확인한다:

1. 부모 행 누락 / depth 좌측 공백
2. `from` 값 중 실재하지 않는 화면 이름 (dangling)
3. `page_id` 누락·불일치 (같은 페이지인데 다른 id / 다른 페이지인데 같은 id), 각 page_id에 `owner_page`·`owner_uc` ✅가 각각 1개인지
4. 채널-도메인 매핑, 정책서엔 있는데 IA에 없는 화면(또는 반대)
