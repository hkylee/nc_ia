# dev_type 분류 가이드

> page_id 형식: `NOVA-{모듈}-{구현유형}-{순번}-{분기}`
> 예: `NOVA-MBR-PG-001-0`, `NOVA-MBR-CP-001-0`

---

## 코드 목록

| 코드 | 유형 | 판단 기준 |
|------|------|----------|
| `PG` | 페이지 | 독립 전체 화면. 뒤로가기 존재. 고유 라우트를 가짐 |
| `PU` | 풀스크린 모달 | 화면 전체를 덮는 모달. 독립 라우트. 뒤로가기로 닫힘 |
| `BS` | 바텀시트 | 화면 하단에서 올라오는 시트. 반모달형 |
| `CP` | 센터팝업 | 화면 중앙에 뜨는 팝업. 확인/취소 버튼 구성이 일반적 |
| `AP` | 액션팝업 | 하단 미니 팝업 또는 스낵바형. 짧은 피드백·선택 제공 |
| `CT` | 커스텀 | 위 유형으로 분류되지 않는 특수 UI |
| `MD` | 모달 | PU가 아닌 인라인 모달 (화면 일부를 덮음) |

---

## 판단 흐름

```
화면이 독립 URL/라우트를 갖는가?
  ├── 예 → 전체 화면 점유? → 예 → PG 또는 PU
  │         └── PU: 이전 화면 위에 레이어로 쌓임
  │             PG: 화면 자체가 교체됨
  └── 아니오 → 위치/크기로 구분
        ├── 화면 하단에서 슬라이드 → BS
        ├── 화면 중앙 오버레이 → CP
        ├── 최소 UI (토스트·미니팝업) → AP
        └── 기타 → CT 또는 MD
```

---

## 유형별 예시

| dev_type | 예시 화면 |
|----------|----------|
| `PG` | 로그인, 상품 목록, 마이 홈, 주문 내역, 설정 |
| `PU` | 카카오 로그인 웹뷰, 결제 인증 웹뷰, 약관 전체 보기 |
| `BS` | 정렬 선택, 필터, 배송지 선택, 쿠폰 선택 |
| `CP` | 본인인증, 탈퇴 확인, 오류 안내, 로그아웃 확인 |
| `AP` | 담기 완료 토스트, 복사 완료, 짧은 경고 |
| `CT` | 커스텀 트랜지션 화면, 온보딩 슬라이드 |
| `MD` | 인라인 이미지 확대, 툴팁형 오버레이 |

---

## 주의사항

- `dev_type`이 불명확하면 **비워둔다** (ia-column-spec.md §1 기준)
- 같은 page_id는 dev_type도 반드시 같아야 한다 (동일 화면의 정체성)
- T4S 레거시 화면의 구현 유형은 참고용 — NOVA에서 재분류 가능
