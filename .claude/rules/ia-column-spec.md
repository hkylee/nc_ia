# IA 컬럼 규격

> 출처: ia-import-spec.md §2

---

## 1. 컬럼 목록

| 컬럼 | 필수 | 값 | 설명 |
|------|------|----|------|
| `domain` | ✅ | `채널-모듈코드` | 화면이 **속한 모듈** 코드 (보이는 위치 아님). 채널 프리픽스 포함. 예: `앱-DSP`, `앱-MBR` |
| `uc_code` | ✅ | `US-…` | UC 코드(§2 참조). 형식 `US-[도메인]-[카테고리]-[순번]`. 예: `US-MBR-LGN-001` |
| `uc_name` | ✅ | 텍스트 | UC명. 예: `회원가입` |
| `page_id` | ✅ | `NOVA-…` | **스크린 ID**. `NOVA-[도메인]-[타입]-[순번]-[변형]`. 같은 페이지면 어디서든 같은 값 |
| `depth1` | ✅ | 텍스트 | 계층 1단계 (모듈 진입페이지명 또는 `홈`) |
| `depth2`, `depth3`, … | ⬜ | 텍스트 | 하위 계층. 필요한 만큼 컬럼 확장. **왼쪽부터 빈칸 없이** 채움 |
| `dev_type` | ⬜ | `PG\|PU\|CP\|AP\|BS` | 구현유형. 모르면 비움 |
| `description` | ⬜ | 텍스트 | 화면 설명·진입조건·정책 |
| `from` | ⬜ | 화면명(들) | 이 화면으로 **들어오는 출발 화면 이름**. depth로 안 잡히는 전환만. 여러 개면 `;`로 구분 |
| `flow_type` | ⬜ | `기본`/`케이스 N`/`예외 N` | 플로우 구분. 비우면 `기본`. 번호는 **같은 owner UC 내** 기준 |
| `owner_page` | ⬜ | `✅`/공백 | **원본 페이지 여부**. 같은 `page_id` 행들 중 마스터면 ✅. 단일 UC 페이지는 ✅ |
| `owner_uc` | ⬜ | `✅`/공백 | **원본 UC 여부**. 이 UC가 페이지의 소속(원본) UC면 ✅ |
| `legacy_screen_id` | ⬜ | `TW-…; TM-…` | 레거시 채널 화면 ID. `;`로 구분 |
| `출처서비스` | ⬜ | `TW, TM, TU, TD` | 참조한 채널 약어. `,`로 구분 |
| `통합여부` | ⬜ | `●`/공백 | 여러 레거시 채널 화면이 통합된 경우 `●` |
| `비고` | ⬜ | 텍스트 | 특이사항. fadeout 검토대상이면 `fadeout 검토대상 (P1)` 형식 기재. `T4S_IA_fadeout.tsv` 또는 `T4S_IA.tsv` `fadeout 권고 대상` 컬럼 기준 |

> **행 순서 = 화면 순서.** 별도 순서 컬럼 없음.

## 3. page_id 채번 규칙

형식: **`NOVA-[도메인]-[개발타입]-[순번]-[변형]`**

예: `NOVA-MBR-PU-005-0`, `NOVA-PRDD-PG-004-1`

| 세그먼트 | 값 | 설명 |
|----------|-----|------|
| prefix | `NOVA` | 고정 (legacy 파일엔 `NC-`도 존재) |
| 도메인 | `MBR`·`DSP`… | 모듈 코드(2~6 대문자). **채널 프리픽스 제외** — `domain`의 `앱-` 뺀 값 |
| 개발타입 | `PG·PU·BS·CP·AP·CT·MD` | = `dev_type` |
| 순번 | `001` | 3자리 zero-pad, **도메인+개발타입별**로 1부터 |
| 변형 | `0` | 동일 화면 변형/상태, 기본 `0` |

- **수동 채번** (자동 생성 함수 없음). 라우팅 slug라 중복 불가.
- 같은 페이지를 여러 UC에서 참조할 때도 **항상 동일한 page_id** 사용.

## 4. UC 코드 채번 규칙

> ⚠️ 카테고리 코드·순번 스코프는 정책서 확인 전까지 추론값

형식(추론): **`US-[도메인]-[카테고리]-[순번]`**

예: `US-MBR-LGN-001`

| 세그먼트 | 값 | 설명 |
|----------|-----|------|
| prefix | `US` | 고정 |
| 도메인 | `MBR`·`DSP`… | 모듈 코드(채널 프리픽스 제외) |
| 카테고리 | `LGN`·`JON`… | **프로세스 분류 — 정책서 정의값** (예제 코드는 추론 placeholder) |
| 순번 | `001` | 3자리 zero-pad |

- 시스템 자동 생성은 카테고리 자리에 `NEW` 고정 (`US-MBR-NEW-001`).
- 사용자 입력은 자유형 (`^[A-Z0-9][A-Z0-9_-]*$`, 도메인 내 unique).
- 연습장은 `DRAFT-[YYYYMMDD]-[순번]`.
