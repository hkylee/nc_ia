1. 개요

가. 범위

• 본 정책서는 NC 통합채널에서 고객이 하나의 계정으로 서비스를 시작하고 종료하는 회원 가입·탈퇴 업무 기준을 정의한다.

• 대상 업무는 회원가입·통합회원 전환, 본인확인·재인증, 로그인·계정 복구, 소셜·간편 로그인 연결, 중복 계정 병합, 외국인·미성년 예외 가입, 그룹·법인 회원 전환, 회원 탈퇴·재가입 제한을 포함한다.

• 대상 채널은 앱·웹 FO이며 고객이 직접 진입, 신청, 인증, 영향 확인, 처리 요청, 결과 확인을 수행하는 셀프 처리 흐름을 포함한다.

• BSS, 회원계정 시스템, 외부 인증기관, 약관·동의 관리 시스템이 수행하는 자격 판정, 권한 확인, 상태 변경, 이력 저장, 결과 회신은 정책 판단 기준에 포함한다.

• 회선 가입, 주문 계약, 결제, 상품 해지, 약관 원문 관리, 회원정보 세부 변경은 별도 정책서에서 상세화하되 회원 가입·탈퇴 가능 여부에 영향을 주는 선행 조건과 안내 기준은 본 문서에 포함한다.

• 화면 UI 상세, API 필드, DB 컬럼, 인증기관별 상세 전문, 운영자 화면 상세는 제외하되 고객 판단과 개발·QA 테스트에 필요한 정책값은 정책 항목으로 남긴다.

나. 설계 원칙

• 통합 시작점: 회원 가입은 서비스별 반복 가입을 줄이고 통합회원 상태에서 필요한 서비스만 추가 동의로 확장하는 시작점으로 설계한다.

• 가치 선제 안내: 고객에게 가입 또는 로그인 요구 전에 얻는 가치, 필요한 정보, 제한 사유, 다음 행동을 먼저 설명한다.

• 점진 수집: 가입 완료에 필수인 정보만 먼저 수집하고 추가 정보는 서비스 이용 시점에 목적과 필수 여부를 밝힌 뒤 수집한다.

• 인증 최소화: 동일 세션에서 유효한 본인확인 결과는 재사용하되 민감 행위, 위험 신호, 탈퇴 확정은 재인증 예외로 관리한다.

• 상태 기반 탈퇴: 탈퇴는 처리 중 주문, 잔여 혜택, 그룹 대표자 권한, 법정 보관 대상 등 선행 조건을 확인한 뒤 상태 기반으로 처리한다.

• 복구 가능성 고지: 중단, 실패, 탈퇴, 계정 병합처럼 되돌리기 어려운 행위는 실행 전에 영향과 복구 가능성을 명확히 안내한다.

2) 상태 전이 기준

가입 진행 중통합회원 가입·전환 신청인증 완료필수 입력, 약관 동의, 본인확인이 완료되면 인증 완료 상태로 전환한다.
가입 진행 중외국인·미성년 예외 가입추가 확인 필요외국인 대체 인증, 법정대리인 동의, 연령 제한, 증빙 확인이 필요한 경우 추가 확인 상태로 전환한다.
추가 확인 필요본인확인·재인증 수행인증 완료대체 인증 또는 법정대리인 동의가 완료되면 기존 가입 흐름으로 복귀한다.
인증 완료통합회원 가입·전환 신청가입 완료중복 계정, 제한 조건, 필수 동의, 정보 저장이 모두 통과되면 가입 완료로 전환한다.
인증 완료중복 계정 병합 신청추가 확인 필요동일 식별정보 계정이 발견되면 병합 영향도 확인 또는 별도 유지 선택 전까지 추가 확인 상태로 둔다.
가입 완료소셜·간편 로그인 연결 관리가입 완료소셜 또는 간편 로그인 연결·해제 결과를 저장하고 로그인 가능 수단 목록을 갱신한다.
가입 완료계정 찾기·복구인증 완료계정 복구가 필요한 경우 본인확인 후 로그인 가능 상태로 복구한다.
가입 완료이상 로그인 감지이용 제한새 기기, 반복 실패, 위험 위치 등 위험 신호가 확인되면 접근을 제한하고 재인증 또는 비밀번호 변경을 요구한다.
가입 완료회원 탈퇴 요청탈퇴 요청고객이 탈퇴 메뉴에 진입하면 영향도, 선행 조치, 최종 인증 필요 여부를 표시한다.
탈퇴 요청회원 탈퇴 요청탈퇴 보류진행 중 주문, 미정산, 이용 중 서비스, 그룹 대표자 권한이 남아 있으면 탈퇴 완료를 보류한다.
탈퇴 요청회원 탈퇴 요청탈퇴 완료선행 조건이 없고 최종 본인확인이 완료되면 회원 상태를 탈퇴 완료로 전환한다.
탈퇴 완료탈퇴 후 재가입 가능성 확인재가입 제한재가입 제한 기간, 부정 이용 이력, 동일 식별정보 제한이 있으면 재가입 제한 상태를 안내한다.
탈퇴 완료탈퇴 후 재가입 가능성 확인가입 진행 중재가입 제한이 없으면 신규 가입 또는 계정 재생성 기준에 따라 가입 진행 상태로 전환한다.

나. 정책 상세

1) 가입 자격·회원 유형 정책 (PG-MBR-ELG-001)

• 회원 유형 구분 (PI-MBR-ELG-001-01)
- 회원 유형은 비회원, 라이트회원, 부분가입, 통합회원, 그룹회원, 법인회원으로 구분한다.- 각 유형별 허용 기능과 추가 인증 필요 업무를 가입 진입 전 안내한다.
• 가입 가능 판정 (PI-MBR-ELG-001-02)
- 가입 가능 여부는 연령, 실명 여부, 기존 계정, 제한 이력, 필수 동의, 본인확인 결과가 모두 충족된 경우로 둔다.
• 라이트회원 제한 (PI-MBR-ELG-001-03)
- 라이트회원은 탐색, 혜택 안내, 마케팅 수신처럼 낮은 위험 업무만 허용하고 주문, 결제, 개인정보 변경, 탈퇴 확정은 통합회원 전환 후 허용한다.
• 비통합 가입 처리 (PI-MBR-ELG-001-04)
- 통합회원 전환을 거부한 고객은 정책상 허용된 제한 기능만 이용하게 하고 미지원 기능은 통합 전환 또는 대체 경로를 안내한다.

2) 본인확인·인증 정책 (PG-MBR-IDN-001)

• 인증수단 허용 (PI-MBR-IDN-001-01)
- 가입 본인확인은 휴대폰 본인확인, PASS, 간편인증, 외국인 대체 인증 중 해당 고객 유형에 허용된 수단으로 수행한다.
• 인증 유효 시간 (PI-MBR-IDN-001-02)
- 인증 결과의 기본 유효 시간은 동일 세션 10분으로 둔다.- 만료되거나 민감 행위가 발생하면 재인증을 요구한다.
• 재인증 대상 (PI-MBR-IDN-001-03)
- 비밀번호 변경, 결제수단 변경, 계정 병합, 탈퇴 확정, 새 기기 위험 로그인은 재인증 대상이다.
• 인증 실패 제한 (PI-MBR-IDN-001-04)
- 동일 인증 단계에서 5회 연속 실패하면 해당 세션의 인증 시도를 중지하고 대체 인증 또는 고객센터 경로를 안내한다.

3) 약관·동의 정책 (PG-MBR-AGR-001)

• 필수 약관 동의 (PI-MBR-AGR-001-01)
- 통합회원 가입에 필요한 필수 약관은 모두 동의해야 가입 완료를 허용한다.
• 선택 동의 처리 (PI-MBR-AGR-001-02)
- 선택 동의 미동의는 회원가입 자체를 제한하지 않으며 맞춤 혜택, 마케팅 수신, 위치 기반 안내 등 해당 기능만 제한한다.
• 서비스별 추가 동의 (PI-MBR-AGR-001-03)
- 개별 서비스 이용 시 필요한 약관은 가입 시 일괄 강제하지 않고 해당 서비스 진입 시 추가 동의로 수집한다.
• 동의 이력 (PI-MBR-AGR-001-04)
- 동의 결과는 약관 버전, 동의 일시, 채널, 철회 여부, 대리 동의 여부와 함께 이력으로 저장한다.
• 대리 동의 이력 (PI-MBR-AGR-001-05)
- 대리 동의는 약관 버전, 동의 항목, 대리인 식별값, 처리 일시, 철회 여부를 이력으로 저장한다.- 고객이 성년이 된 이후에는 대리 동의 유지 여부와 직접 동의 전환 필요 여부를 안내한다.

4) 입력·정보 재사용 정책 (PG-MBR-INP-001)

• 필수 입력 최소화 (PI-MBR-INP-001-01)
- 가입 완료에 필요한 입력은 이름, 연락처, 식별정보, 로그인 수단, 필수 동의처럼 필수 항목으로 제한한다.
• 정보 재사용 (PI-MBR-INP-001-02)
- 본인확인 결과, 기존 고객 정보, 오프라인 가입 정보, 기기 정보는 고객 확인 후 재사용하고 고객이 수정 가능한 항목을 구분한다.
• 진행 상태 저장 (PI-MBR-INP-001-03)
- 가입 진행 단계, 입력값, 인증 성공 여부, 동의 결과는 재진입 복원을 위해 임시 저장하되 민감 인증 원문은 저장하지 않는다.
• 복원 불가 안내 (PI-MBR-INP-001-04)
- 세션 만료, 인증 만료, 민감 정보 보호 사유로 복원이 불가하면 손실 범위와 다시 입력해야 하는 항목을 안내한다.

5) 상태·세션 정책 (PG-MBR-STS-001)

• 상태 표시 (PI-MBR-STS-001-01)
- 고객에게는 가입 진행 중, 추가 인증 필요, 가입 완료, 이용 제한, 탈퇴 보류, 탈퇴 완료 상태를 이해 가능한 문구로 표시한다.
• 자동 로그인 해제 (PI-MBR-STS-001-02)
- 로그아웃, 탈퇴 완료, 비밀번호 변경, 기기 변경, 위험 로그인 감지 시 자동 로그인과 간편 로그인 세션을 해제한다.
• 세션 만료 복귀 (PI-MBR-STS-001-03)
- 세션 만료 후 재로그인하면 직전 목적 화면으로 복귀시키되 민감 업무는 재인증 후 진행한다.
• 상태 변경 이력 (PI-MBR-STS-001-04)
- 회원 상태 변경은 변경 전 상태, 변경 후 상태, 사유, 처리 주체, 처리 일시를 이력으로 저장한다.

6) 로그인·계정 복구 정책 (PG-MBR-LGN-001)

• 아이디 찾기 결과 (PI-MBR-LGN-001-01)
- 아이디 찾기 결과는 일부 마스킹하여 제공하고, 고객이 바로 로그인하거나 비밀번호 재설정으로 이동할 수 있게 한다.
• 비밀번호 정책 (PI-MBR-LGN-001-02)
- 새 비밀번호는 길이, 문자 조합, 이전 비밀번호 재사용 제한 기준을 충족해야 하며 실패 사유를 구체적으로 안내한다.
• 식별자 불일치 (PI-MBR-LGN-001-03)
- 휴대폰 번호, 이메일, CI가 기존 계정과 불일치하면 계정 찾기, 정정, 통합 절차 중 가능한 경로를 제시한다.
• 복구 성공 처리 (PI-MBR-LGN-001-04)
- 계정 복구 성공 후에는 최근 로그인 이력, 등록 인증수단, 추가 인증 필요 업무를 요약해 안내한다.

7) 소셜·간편 로그인 정책 (PG-MBR-SOC-001)

• OAuth 연결 분기 (PI-MBR-SOC-001-01)
- 소셜 최초 로그인은 신규 가입, 기존 계정 연결, 중복 계정 확인으로 분기하고 민감 업무는 별도 본인확인을 요구한다.
• 소셜 연결 해제 (PI-MBR-SOC-001-02)
- 소셜 계정 해제는 대체 로그인 수단이 1개 이상 등록된 경우에만 허용한다.
• 간편 로그인 등록 (PI-MBR-SOC-001-03)
- 간편 로그인 등록은 해당 기기에서 본인확인이 완료된 세션에서만 허용한다.
• 외부 계정 오류 (PI-MBR-SOC-001-04)
- 외부 계정 삭제, 제공자 장애, 권한 철회로 로그인 실패 시 비밀번호 로그인 또는 본인확인 복구 경로를 안내한다.

8) 외국인·미성년 예외 정책 (PG-MBR-FGN-001)

• 외국인 가입 대상 (PI-MBR-FGN-001-01)
- 외국인 가입 대상은 국내 거주 외국인, 단기 체류자, 해외 거주자 중 정책상 지원 대상만 허용하고 미지원 대상은 대체 경로를 안내한다.
• 외국인 입력 기준 (PI-MBR-FGN-001-02)
- 외국인 가입 정보는 외국인등록번호, 영문명, 생년월일, 국적, 체류정보 중 필수·선택 항목과 수집 목적을 구분한다.
• 미성년 동의 (PI-MBR-FGN-001-03)
- 만 14세 미만 또는 정책상 대리 동의가 필요한 고객은 법정대리인 인증과 동의가 완료된 경우에만 가입을 허용한다.
• 예외 인증 실패 (PI-MBR-FGN-001-04)
- 외국인·미성년 인증 실패는 재시도, 다른 인증수단, 서류 접수, 고객센터 연결 순서로 복구 경로를 제공한다.
• 대리권 확인 (PI-MBR-FGN-001-05)
- 법정대리인 동의는 대리인 본인확인, 고객과의 관계 확인, 동의 대상 업무가 모두 충족된 경우에만 유효하다.- 관계 확인에 실패하면 가입 또는 변경을 완료하지 않고 대체 인증 또는 고객센터 경로를 안내한다.

9) 그룹·법인 회원 정책 (PG-MBR-GRP-001)

• 그룹 가입 게이트 (PI-MBR-GRP-001-01)
- 그룹 가입은 로그인 상태, 실명 여부, 연령, 대표자 권한, 법인 이용 가능 범위가 충족된 경우에만 진행한다.
• 초대 상태 제어 (PI-MBR-GRP-001-02)
- 그룹 초대는 발송, 대기, 수락, 만료, 취소, 재발송 상태별로 허용 행동과 안내 문구를 다르게 적용한다.
• 역할별 권한 (PI-MBR-GRP-001-03)
- 대표자, 관리자, 구성원은 조회 가능한 정보, 수정 가능한 항목, 탈퇴 가능 여부를 역할별로 분리한다.
• 대표자 탈퇴 제한 (PI-MBR-GRP-001-04)
- 대표자는 대표자 변경 또는 그룹 해산이 완료되기 전 구성원 탈퇴나 회원 탈퇴를 완료할 수 없다.

10) 중복 계정·병합 정책 (PG-MBR-MRG-001)

• 중복 감지 기준 (PI-MBR-MRG-001-01)
- CI, 휴대폰 번호, 이메일, 소셜 식별자 중 둘 이상이 기존 계정과 일치하면 중복 계정 후보로 안내한다.
• 병합 전 영향 안내 (PI-MBR-MRG-001-02)
- 병합 전에는 승계 정보, 삭제 또는 정리 정보, 복구 불가 항목, 서비스 연결 영향, 상담 필요 사유를 안내한다.
• 병합 제한 (PI-MBR-MRG-001-03)
- 명의 불일치, 법인·개인 혼재, 미성년 대리 동의 미확보, 데이터 충돌이 있으면 셀프 병합을 제한한다.
• 병합 이력 (PI-MBR-MRG-001-04)
- 병합 결과는 병합 대상 계정, 고객 확인 결과, 처리 일시, 처리 후 대표 계정, 실패 사유와 함께 저장한다.

11) 회원 탈퇴 정책 (PG-MBR-OUT-001)

• 탈퇴 선행 조건 (PI-MBR-OUT-001-01)
- 회원 탈퇴 전 미처리 주문, 미정산 금액, 이용 중 서비스, 잔여 혜택, 그룹 대표자 권한, 법정 보관 사유를 확인한다.
• 탈퇴 영향도 안내 (PI-MBR-OUT-001-02)
- 탈퇴 전 소멸 혜택, 쿠폰, 구독, 이용 이력, 로그인 권한, 재가입 제한, 보관 예외를 요약과 상세로 안내한다.
• 탈퇴 최종 확인 (PI-MBR-OUT-001-03)
- 탈퇴 최종 단계는 비밀번호 재입력 또는 본인확인을 요구하고 복구 불가 항목을 확인한 뒤 처리한다.
• 탈퇴 보류 처리 (PI-MBR-OUT-001-04)
- 선행 조치가 남아 있으면 탈퇴 완료 대신 탈퇴 보류 상태로 관리하고 해결 경로와 재시도 가능 시점을 안내한다.

12) 재가입·복구 정책 (PG-MBR-RET-001)

• 재가입 가능 시점 (PI-MBR-RET-001-01)
- 재가입은 탈퇴 완료 상태, 제한 이력, 동일 식별정보 재사용 제한, 보관 예외가 모두 확인된 경우에만 허용한다.
• 재이용 제한 안내 (PI-MBR-RET-001-02)
- 재가입 후에도 일부 서비스, 혜택, 쿠폰, 이력은 복구되지 않을 수 있음을 가입 전 안내한다.
• 복구 가능 범위 (PI-MBR-RET-001-03)
- 탈퇴 유예 또는 보류 상태에서는 본인확인 후 철회 또는 복구를 허용하되 탈퇴 완료 후에는 신규 가입 기준을 따른다.
• 복구 불가 고지 (PI-MBR-RET-001-04)
- 법정 보관 외 파기 완료, 병합 확정, 탈퇴 완료 후 유예 종료 항목은 고객 셀프 복구 불가로 고지한다.

13) 보안·위험 알림 정책 (PG-MBR-SEC-001)

• 이상 로그인 알림 (PI-MBR-SEC-001-01)
- 새 기기, 이상 위치, 반복 실패, 위험 세션이 감지되면 고객에게 알림을 발송하고 본인 아님 신고 경로를 제공한다.
• 계정 잠금 (PI-MBR-SEC-001-02)
- 동일 계정의 인증 또는 로그인 실패가 정책 기준을 초과하면 계정을 잠금 처리하고 해제 방법을 안내한다.
• 민감 화면 보호 (PI-MBR-SEC-001-03)
- 바코드, 쿠폰코드, 개인정보, 결제 관련 화면은 설정 또는 정책에 따라 생체·간편비밀번호 재인증 후 노출한다.
• 원격 세션 종료 (PI-MBR-SEC-001-04)
- 고객은 로그인 기기·세션 목록에서 선택 또는 전체 세션을 원격 로그아웃할 수 있다.

14) 이력·보관 정책 (PG-MBR-HIS-001)

• 가입 이력 (PI-MBR-HIS-001-01)
- 가입과 전환 처리는 신청 경로, 인증 결과, 동의 버전, 회원 유형, 완료 여부를 이력으로 저장한다.
• 인증 로그 (PI-MBR-HIS-001-02)
- 인증기관 결과 로그는 성공 여부, 인증수단, 일시, 유효 범위, 실패 사유를 저장하고 신분증 이미지 직접 보관은 지양한다.
• 탈퇴 보관 예외 (PI-MBR-HIS-001-03)
- 탈퇴 후에도 법령, 정산, 분쟁, 부정 이용 방지 목적의 정보는 별도 보관 대상으로 분리한다.
• 마스킹 및 접근 권한 (PI-MBR-HIS-001-04)
- 회원 식별정보와 인증 이력은 조회 권한을 제한하고 고객 화면과 운영자 화면에서 필요한 범위만 마스킹해 노출한다.

최종 점검 기준

회원 가입·탈퇴 정책서 제출 전 점검
1. 회원 가입·탈퇴 범위가 계정 생성, 인증, 동의, 그룹·법인, 탈퇴, 재가입까지 고객 과업 기준으로 명확한지 확인한다.
2. 로그인 고객, 비로그인 고객, 정상 고객, 제한 고객을 액터로 분리하지 않고 상태와 정책 조건으로 관리했는지 확인한다.
3. 사람 액터 유즈케이스는 프로세스 정의 대상 Y, 시스템 액터 유즈케이스는 N으로 관리했는지 확인한다.
4. 상태 전이 이벤트가 유즈케이스 또는 고객 업무 이벤트로 작성되고 상태명은 상태 코드 목록과 일치하는지 확인한다.
5. 가입·탈퇴 프로세스마다 관련 기능과 관련 정책이 실제 ID와 정책명으로 연결되는지 확인한다.
6. 정책 항목이 허용 기준, 제한 조건, 시간, 횟수, 상태, 예외, 고지, 이력 중 하나 이상을 포함하는지 확인한다.
7. 탈퇴 전 선행 조치, 영향도, 보류, 재가입 제한, 복구 불가 기준이 고객에게 사전 안내되는지 확인한다.
8. 외국인, 미성년, 법정대리인, 그룹·법인, 중복 계정 같은 예외 흐름이 정책과 상태로 추적 가능한지 확인한다.
9. 요구사항 상세명과 설명에서 확인된 가입, 인증, 로그인, 탈퇴, 보안, 그룹·법인 요구가 프로세스·기능·정책 중 하나 이상에 반영되었는지 확인한다.
10. 개발자가 기능 설계 항목으로, QA가 테스트 조건과 기대 결과로 전환할 수 있는 수준인지 확인한다.
