1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 통합회원 가입, 인증, 계정 연결, 보안 관리, 회원탈퇴를 수행하기 위한 정책 기준을 정의한다.

• 포함 범위는 가입 진입·유형 분기, 가입 자격, 본인인증, 약관 동의, 계정 생성·연결, 소셜·간편 로그인, 그룹·대리 권한, 보안·세션, 탈퇴·복구, 운영 감사다.

• 제외 범위는 주문·계약·가입의 상품 주문 절차와 나의 가입 정보의 계약 조회 상세이며, 계정·인증·회원 상태와 연결되는 조건만 포함한다.

• 대상 채널은 앱·웹 FO와 운영자 관리 경로이며 고객이 가입부터 탈퇴까지 가능한 범위 안에서 셀프 처리하도록 설계한다.

• 인터페이스 항목명, 저장소 물리 구조, 화면 UI 상세, 개별 인증기관 연동 전문은 후속 상세 설계 산출물에서 정의한다.

• 핵심 정책 질문은 누가 이 업무를 시작할 수 있는지, 어떤 조건에서 제한되는지, 중복 요청을 어떻게 처리하는지, 실패 시 어떤 상태와 후속 조치로 복구하는지를 기준으로 둔다.

• 고객 표시 상태는 고객이 이해해야 하는 진행·완료·제한 상태로, 시스템 내부 상태는 BSS·연계 시스템의 판정·저장·회신 상태로 구분한다.

• 개발·QA는 유즈케이스 ID, 프로세스 ID, 기능 ID, 정책/정책항목 ID를 기준으로 정상·제한·실패·복구 테스트 조건과 기대 결과를 도출한다.

• 동일 가입·인증·탈퇴 요청이 반복 접수되면 기존 처리 세션과 이력을 우선 조회하고, 5분 이내 동일 요청은 중복 요청으로 보아 추가 계정 생성·탈퇴 처리를 금지한다.

• 통합 계정 장애, 인증기관 지연, 소셜 계정 연동 실패가 다른 채널 로그인·탈퇴 경로에 영향을 줄 수 있으므로 장애 영향과 채널 간 통합 리스크를 상태·고지·상담 전환 기준으로 관리한다.

• 현황 분석에서 확인된 고객 목적 기반 진입, 인증 전환 마찰 최소화, 상담 의존 감소 신호를 가입과 탈퇴 전 과정의 판단 기준으로 반영한다.

• IA 분석의 회원·전체메뉴 영역은 통합 채널에서 Core화·공통화하되, 가입 유형, 법정대리인 동의, 탈퇴 영향처럼 특화 판단이 필요한 흐름은 상태와 정책으로 유지한다.

나. 설계 원칙

• 통합 계정 중심: 서비스별 반복 가입이 아니라 CI 기반 통합 계정으로 가입, 로그인, 탈퇴 상태를 일관되게 관리한다.
IA와 탐색 경로의 일관성을 유지해 고객이 서비스별 메뉴 위치를 기억하지 않아도 가입, 로그인, 탈퇴 목적 기준으로 진입할 수 있게 한다.

• 입력 최소화: 가입 완료에 필요한 정보만 먼저 받고 기존 정보와 인증 결과를 재사용해 중복 입력을 줄인다.
고객 조사와 VoC에서 반복 입력과 인증 이탈이 상담 의존으로 이어지는 점을 고려해 기존 정보 재사용, 중단 복원, 다음 행동 안내를 우선한다.

• 위험 기반 인증: 민감행위, 탈퇴, 권한 변경, 이상 접속은 재인증과 추가 확인을 적용한다.
인증·전환 단계의 마찰을 최소화하되 법정대리인, 명의자 동의, 탈퇴처럼 위험이 큰 행위는 사유를 고지하고 필요한 재인증만 요구한다.

• 역할·권한 명확화: 고객, 법정대리인, 명의자, 그룹 대표자, 운영자 권한을 상태와 정책 조건으로 분리한다.

• 탈퇴 영향 보호: 탈퇴 전 삭제, 보관, 복구 가능 여부를 충분히 고지하고 고객 확인 이력을 남긴다.
상담 의존을 줄이기 위해 탈퇴 제한, 유예, 복구 가능성, 남은 주문·혜택 영향을 고객이 셀프로 이해하고 완료할 수 있게 한다.

• 감사 가능성: 가입, 인증, 동의, 계정 연결, 보안 이벤트, 탈퇴 이력을 감사 가능한 구조로 저장한다.

나. 정책 상세

1) 가입 진입·유형 분기 정책 (PG-MBR-REG-001)

• 진입 표준화 (PI-MBR-REG-001-01)
- 회원가입과 로그인 진입은 앱·웹 주요 목적형 접점에서 동일 명칭과 동일 우선순위로 제공한다.
- 가입 전에는 가입 후 가능한 업무 가치를 함께 안내한다.
• 가입 유형 분기 (PI-MBR-REG-001-02)
- 진입 시 통합회원 여부, 보유 회선, 모회선 인증 필요 여부, 신규·전환·그룹·초대 참여 여부를 판정해 이후 단계와 필수 입력 항목을 분기한다.
• 입력 최소화 (PI-MBR-REG-001-03)
- 가입 완료에 필요한 필수 항목만 먼저 수집한다.
- 기존 고객정보와 최근 인증 결과로 채울 수 있는 항목은 자동 반영하고 선택 항목은 가입 후 점진 수집으로 전환한다.
• 중단 복원 (PI-MBR-REG-001-04)
- 가입 중 이탈 후 재진입하면 마지막 완료 단계와 입력값을 복원한다.
- 복원이 불가하면 복원 불가 사유와 다시 입력해야 할 항목을 안내해 고객이 메뉴를 다시 찾거나 상담을 요청하지 않게 한다.

2) 가입 자격·제한 정책 (PG-MBR-ELG-001)

• 가입 자격 판정 (PI-MBR-ELG-001-01)
- 가입 시작 전 또는 해당 단계에서 연령, 고객 상태, 가입 제한, 실명 여부, 법인 여부, 외국인 지원 범위를 판정한다.
- 제한 조건이 있으면 진행 차단 사유와 대체 경로를 제공한다.
• 미성년·법정대리인 기준 (PI-MBR-ELG-001-02)
- 미성년자 또는 법정대리인 동의가 필요한 가입은 주민번호 또는 대체 판별 결과를 기준으로 분기한다.
- 법정대리인 동의 완료 전에는 계정 생성 또는 민감 기능 진입을 허용하지 않는다.
• 외국인 가입 기준 (PI-MBR-ELG-001-03)
- 외국인 가입은 지원 대상, 가능한 본인확인 수단, 필수 입력 항목, 대체 인증 경로를 가입 전에 안내한다.
- 인증 실패 시 재시도, 다른 인증수단, 고객센터 연결 순서로 복구 경로를 제공한다.
• 부정가입 제한 (PI-MBR-ELG-001-04)
- 동일 CI, 동일 휴대폰번호, 반복 실패, 위험 IP, 도용 의심 이력이 기준값을 초과하면 가입을 보류한다.
- 보류 건은 운영 확인 또는 추가 인증 대상으로 전환한다.

3) 본인인증·로그인 수단 정책 (PG-MBR-AUT-001)

• 인증 수단 선택 (PI-MBR-AUT-001-01)
- 가입과 민감행위 인증은 PASS, 토스, 네이버, 카카오 등 지원 인증수단을 선택할 수 있게 한다.
- 정책상 지원하지 않는 수단은 노출하지 않는다.
• 인증 재사용 (PI-MBR-AUT-001-02)
- 최근 유효한 인증 결과는 동일 고객, 동일 기기, 동일 위험 수준, 유효 시간 안에서만 재사용한다.
- 민감 정보 변경, 탈퇴, 권한 변경은 재인증을 요구한다.
• 인증번호 유효시간 (PI-MBR-AUT-001-03)
- 인증 유효 시간은 10분이며 만료되면 재인증을 요구한다.
- 인증 요청과 실패 횟수는 채널별 차단 시간 정책을 적용한다.
• 소셜 로그인 연결 (PI-MBR-AUT-001-04)
- 소셜 로그인 최초 진입 시 신규 가입과 기존 계정 연결을 분기한다.
- 소셜 계정이 변경되거나 삭제되면 대체 로그인 또는 본인확인 복구 수단을 제공한다.

4) 약관·동의 정책 (PG-MBR-CNS-001)

• 필수 동의 기준 (PI-MBR-CNS-001-01)
- 서비스 제공에 필요한 필수 약관과 개인정보 처리 동의가 모두 완료되어야 가입 완료 상태로 전환한다.
- 선택 동의 거부는 가입 제한 사유로 사용하지 않는다.
• 동의 변경·철회 (PI-MBR-CNS-001-02)
- 마케팅, 알림, 선택 동의는 가입 후 고객이 직접 변경 또는 철회할 수 있어야 한다.
- 철회 결과는 동의 이력과 알림 수신 기준에 즉시 반영한다.

5) 계정 생성·연결 정책 (PG-MBR-ACC-001)

• CI 기반 계정 생성 (PI-MBR-ACC-001-01)
- 통합 계정은 CI 또는 이에 준하는 고유 식별 기준으로 생성한다.
- 동일 고객의 중복 계정이 발견되면 신규 생성보다 계정 연결 또는 전환 경로를 우선 제공한다.
• ID 생성 기준 (PI-MBR-ACC-001-02)
- 가입 유형별 ID 형식, 사용 가능 문자, 중복 검사, 예약어 제한을 가입 전에 안내한다.
- 사용할 수 없는 ID는 사유와 대체 입력 기준을 함께 안내한다.
• 기존 정보 재사용 (PI-MBR-ACC-001-03)
- 전환 가입은 기존 고객정보를 우선 불러오고 누락된 필수 정보만 추가 입력하게 한다.
- 현행화가 필요한 정보는 고객이 직접 수정할 수 있게 한다.

6) 계정 보안·세션 정책 (PG-MBR-SEC-001)

• 비밀번호 기준 (PI-MBR-SEC-001-01)
- 비밀번호 변경은 현재 인증 또는 2차 인증을 통과한 경우에만 허용한다.
- 만료 예정 또는 보안 위험이 있는 경우 로그인 이전 단계에서 사전 안내를 제공한다.
• 세션 만료 기준 (PI-MBR-SEC-001-02)
- 세션 만료 전에는 남은 시간을 안내하고 연장 선택을 제공한다.
- 만료 후 민감 기능 진입은 재로그인 또는 재인증을 요구하되, 인증·전환 마찰을 줄이기 위해 이전 입력값과 진행 단계는 보존한다.
• 기기·세션 관리 (PI-MBR-SEC-001-03)
- 고객은 로그인 기기와 세션 목록을 조회하고 선택 기기를 원격 로그아웃할 수 있다.
- 비정상 접속 의심 시 보안 알림과 계정 잠금 기준을 적용한다.

7) 그룹·대리 권한 정책 (PG-MBR-GRP-001)

• 그룹 가입 분기 (PI-MBR-GRP-001-01)
- 그룹 가입은 최초 생성과 초대 참여를 구분한다.
- 각 유형에는 필요한 입력, 역할, 동의, 승인 조건만 노출한다.
• 역할별 권한 (PI-MBR-GRP-001-02)
- 대표자, 관리자, 구성원은 조회 가능 정보와 수정 가능 항목을 다르게 적용한다.
- 비허용 기능에는 사유와 승인 요청 경로를 안내한다.
• 대리 처리 동의 (PI-MBR-GRP-001-03)
- 가족 대리 처리나 명의자 동의가 필요한 경우 문자 링크 기반 동의 또는 대체 동의 절차를 완료해야 한다.
- 동의 완료 전에는 정지 해제나 권한 변경을 허용하지 않는다.

8) 회원 탈퇴·유예 정책 (PG-MBR-WDR-001)

• 탈퇴 영향 안내 (PI-MBR-WDR-001-01)
- 탈퇴 전에는 이용 중 서비스, 멤버십, 주문, 쿠폰, 포인트, 알림, 데이터 삭제·보관 영향을 요약해 안내한다.
- 삭제 또는 복구 불가 항목은 별도 확인을 받아야 한다.
• 탈퇴 신청 기준 (PI-MBR-WDR-001-02)
- 미완료 주문, 정산, 분쟁, 법정 보관 대상이 있으면 즉시 탈퇴를 제한하거나 유예 상태로 전환한다.
- 제한 사유, 고객이 완료해야 할 후속 행동, 상담 없이 확인 가능한 복구 경로를 함께 안내한다.
• 유예·복구 기준 (PI-MBR-WDR-001-03)
- 탈퇴 유예 기간 안에는 본인확인 후 복구 가능으로 표시한다.
- 유예 종료 후에는 계정 상태를 탈퇴 완료로 전환하고 복구 불가와 재가입 기준을 안내한다.

9) 운영 승인·감사 정책 (PG-MBR-OPR-001)

• 승인 지연 관리 (PI-MBR-OPR-001-01)
- 운영 승인이 필요한 가입 또는 권한 변경이 기준 기간을 초과하면 신청자와 승인 담당자에게 지연 알림을 제공한다.
- 지연 안내에는 현재 상태, 담당 조직, 고객이 기다릴지 상담으로 전환할지 선택할 수 있는 다음 행동을 포함해 상담 의존을 줄인다.
• 관리자 접근 감사 (PI-MBR-OPR-001-02)
- 운영자는 접근 로그 조회 시 관리자ID, IP, 관리자명 입력값 형식을 검증해야 한다.
- 조회 결과는 개인정보 노출 범위와 감사 목적 기준으로 제한한다.
