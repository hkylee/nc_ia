1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 하나의 검색·질문 진입점으로 정보 탐색, 비교, 업무 실행, 문제 해결을 시작하고 완료 경로를 선택할 수 있도록 하는 AI 검색 기준을 정의한다.

• 포함 범위는 전역 검색 진입, 전시 내 검색 진입, 검색 입력 표준, 질의 정규화, 자동완성, 연관 검색어, 최근·인기 검색어, 통합 결과 탭, 필터·정렬, 결과 내 재검색, 상세 복귀 상태 유지, 무결과 폴백이다.

• AI 확장 범위는 자연어·질문형 의도 해석, 조건·속성 파라미터 추출, 고객 상태 기반 Answer Card, 비교·추천 요약, 실행 경로 안내, 상담 전환 맥락 전달이다.

• 운영 범위는 통합 검색 대상, 대표 엔티티, 동의어·축약어·오타·금칙어 사전, 랭킹 가중치, AI 추천어·응답 후보 검수, 배포·롤백, 품질·성과 지표, 실패 케이스 분석 기준을 포함한다.

• BSS는 고객 상태, 가입 상태, 보유 상품, 권한, 대상 조건, 실행 가능 여부의 기준 정보를 제공하고, 검색 연계 시스템은 해당 기준 정보를 검색 결과와 실행 연결 판정에 반영한다.

• 책임 경계는 통합채널 FO가 입력·표시·실행 경로를 담당하고, 검색 연계 시스템이 의도 해석·후보 생성·권한 판정을 담당하며, BSS는 고객 상태·보유 상품·업무 가능 여부의 기준 정보를 제공하는 구조로 구분한다.

• 제외 범위는 검색 엔진 알고리즘 상세, 모델 파라미터, 프롬프트 원문, API 필드, DB 컬럼, 화면 상세 디자인, 개별 상품·요금·혜택의 상세 정책값이다.

• 검색 결과는 앱·웹에서 동일한 기준으로 정렬·제공하되 화면 특성에 따른 표시 차이는 정책 판단이 아닌 표현 차이로 관리한다.

• 상세 요구사항 101건은 요구사항 Trace 기준으로 유즈케이스, 상태, 프로세스, 기능, 정책 또는 정책 항목 중 하나 이상에 매핑한다.

• 상품·전시·콘텐츠 메타데이터, 페이지네이션·무한스크롤, 응답 시간·렌더링 지연·호출 실패율은 고객 탐색 완결성과 운영 품질 판단 기준으로 포함한다.

나. 설계 원칙

• Intent to Action: AI 검색은 단순 결과 목록이 아니라 고객 의도를 해석해 비교, 실행, 상담, 도움말 중 다음 행동으로 이어지게 한다.

• Customer Control: AI는 고객 대신 가입, 변경, 해지, 결제 결정을 내리지 않으며 주요 실행은 고객의 명시적 선택과 최종 확인 후에만 연결한다.

• Grounded Result: 검색·추천 결과에는 출처, 적용 조건, 미충족 사유, 한계, 대체 경로를 함께 제공해 고객이 스스로 판단할 수 있게 한다.

• Unified Search Scope: 상품, 혜택, 메뉴, FAQ, 공지, 콘텐츠, 상담 진입은 하나의 검색 범위에서 다루되 대표 결과와 유형별 결과를 구분한다.

• Recoverable Search Failure: 무결과, 근거 부족, 오류, 권한 제한은 고객이 재검색, 조건 보완, 상담 전환, 대체 경로를 선택할 수 있는 상태로 전환한다.

• Quality Loop: 검색 정확도, 실행 완료율, 무결과율, 상담 전환율, 피드백, 정책 위반 의심 항목을 운영 루프로 관리한다.

2) 상태 전이 기준

현재 상태전이 이벤트다음 상태처리 기준 및 후속 처리
검색 전통합 검색 진입 및 질의 입력질의 입력 중고객이 공통 검색 진입점을 선택하면 현재 화면 맥락을 유지한 채 질의 입력 상태로 전환한다.
질의 입력 중검색어 입력 검증과 정규화의도 해석 중공백·특수문자·오타·동의어를 정규화하고 최소 글자 수와 금칙어 기준을 통과하면 의도 해석을 시작한다.
의도 해석 중추가 질문 기반 조건 보완조건 보완 필요대상, 기간, 금액, 지역, 상태 등 필수 조건이 부족하면 고객에게 최소 질문만 제시한다.
조건 보완 필요조건 보완 후 재해석결과 구성 중고객이 추가 조건을 선택하거나 입력하면 기존 검색 컨텍스트와 함께 재해석한다.
의도 해석 중AI 검색 결과 이해 및 다음 행동 선택결과 구성 중의도, 대상 유형, 고객 상태, 권한 조건이 충분히 확인되면 결과 후보를 구성한다.
결과 구성 중고객 상태와 적용 조건 반영결과 제공 완료정확일치, 대표 결과, Answer Card, 유형별 결과, 제한 사유, 대체 추천을 반영해 결과를 제공한다.
결과 제공 완료근거 부족 또는 신뢰도 부족 확인근거 부족출처가 없거나 고객 조건을 확인할 수 없어 추천 근거를 명확히 제시할 수 없으면 근거 부족으로 전환한다.
근거 부족검색 실패·불확실 결과 복구조건 보완 필요고객이 원본 확인 대신 조건 보완 또는 재검색을 선택하면 기존 맥락을 유지해 조건 보완으로 전환한다.
결과 구성 중제로결과 폴백결과 없음직접 결과와 연관 결과가 없으면 빈 화면이 아니라 연관어, 인기 카테고리, FAQ, 상담 경로를 제공한다.
결과 없음대체 경로 선택질의 입력 중고객이 연관 검색어, 인기 키워드, 재검색을 선택하면 질의 입력 중 상태로 돌아간다.
결과 제공 완료비교·추천 결과 검토 및 실행 전 확인비교 검토 중고객이 비교 가능한 상품, 혜택, 업무 경로를 2개 이상 선택하면 비교 검토 상태로 전환한다.
비교 검토 중추천 근거와 한계 확인실행 확인 대기추천 사유, 적용 조건, 유의사항, 대체 선택지를 확인한 뒤 고객이 실행을 선택하면 최종 확인을 대기한다.
실행 확인 대기고객 통제 기반 최종 실행 확인실행 전환 완료고객이 필수 인증, 동의, 유의사항, 최종 실행 버튼을 명시적으로 완료한 경우에만 실행 화면으로 전환한다.
결과 제공 완료검색 상담 전환 및 맥락 이어가기상담 전환 대기검색 결과로 해결되지 않아 상담 전환을 선택하면 전달할 검색 컨텍스트를 고객에게 고지한다.
상담 전환 대기상담 컨텍스트 전달 동의실행 전환 완료고객이 검색어, 선택 결과, 실패 이력 전달에 동의하면 상담 채널로 전환한다.
결과 제공 완료오류·실패·중단 복구오류·복구 필요검색 호출 실패, 결과 로딩 실패, 권한 제한, 외부 연동 실패가 발생하면 오류 유형별 복구 상태로 전환한다.
운영 검수 대기AI 응답 후보 검수·배포·롤백운영 반영 완료운영자가 AI 후보의 금칙어, 근거, 브랜드, 법무, 편향 위험을 확인하고 승인하면 반영 완료로 전환한다.
운영 반영 완료품질 이상 징후 감지운영 회수 필요오답률, 정책 위반률, 오류율, 상담 전환율이 기준치를 초과하면 회수 필요로 전환한다.
운영 회수 필요롤백 및 대체 기준 반영운영 검수 대기문제 후보를 차단하거나 이전 버전으로 롤백한 뒤 개선 후보를 운영 검수 큐에 등록한다.
상담 전환 대기검색 맥락 기반 상담 접수실행 전환 완료상담사가 검색어, 선택 결과, 실패 이력, 고객 상태 요약을 확인하면 상담이 중복 질문 없이 시작된다.
실행 전환 완료상담 처리 결과 회신 및 복귀 안내피드백 접수상담사가 해결 결과와 후속 행동을 안내하고 고객 피드백 또는 추가 개선 필요 이력을 남기면 피드백 접수로 전환한다.

1) 통합 검색 범위 정책 (PG-AIS-SCOPE-001)

• 검색 대상 범위 (PI-AIS-SCP-001)
- 통합 검색 대상은 상품, 혜택, 제휴처, 이벤트, 전시·콘텐츠, 메뉴·기능, FAQ·공지·가이드, 상담 진입을 기본 범위로 둔다.
- 단, 개별 상품 가격이나 약관 상세는 원천 상세 화면으로 연결한다.
• 대표 결과 통합 (PI-AIS-SCP-002)
- 동일 목적의 메뉴·상품·혜택이 여러 채널과 명칭으로 존재하면 대표 엔티티 1개를 우선 노출하고 채널별 실행 경로는 하위 선택지로 묶어 제공한다.
• 인접 업무 경계 (PI-AIS-SCP-003)
- 상품 주문, 결제, 해지, 환불의 세부 정책값은 각 정책서가 우선하며 AI 검색 문서는 검색 결과 노출, 실행 연결, 최종 확인 조건까지만 정의한다.
• 앱·웹 결과 일관성 (PI-AIS-SCP-004)
- 동일 검색어는 앱과 웹에서 같은 대표 결과, 유형별 우선순위, 노출 제한 기준을 적용한다.
- 화면 폭이나 컴포넌트 차이로 표시 순서가 달라질 수는 있으나 정책 판단 기준은 동일해야 한다.
• 상품·전시 메타데이터 (PI-AIS-SCP-005)
- 상품, 단말, 전시, 콘텐츠 결과는 검색 결과 단계에서 고객 판단에 필요한 핵심 속성과 전시 가능 메타데이터를 포함한다.
- 상품 상세 정책값은 원천 상세 화면으로 연결하되 제조사, 모델명, 출시일, 주요 속성, 전시 상태, 실행 경로는 검색 결과에서 확인 가능해야 한다.
• 기획전 배너 노출 기준 (PI-AIS-SCP-006)
- 검색 결과의 기획전 탭과 초기 검색 화면은 현재 진행 중인 기획전, 전시 상태, 고객 대상 조건, 운영 우선순위 기준으로 배너 카드를 노출한다.
- 종료·중지·대상 제외 기획전은 추천 검색어와 결과 카드에서 제외한다.

2) 입력·정규화 정책 (PG-AIS-INPUT-001)

• 입력 최소 기준 (PI-AIS-INP-001)
- 검색 질의는 공백 제거 후 1자 이상이면 접수한다.
- 50자를 초과하는 질의는 앞 50자 기준으로 해석하되 고객에게 긴 질의가 요약 처리될 수 있음을 안내한다.
• 질의 정규화 (PI-AIS-INP-002)
- 공백, 반복 특수문자, 이모지는 의미 보존 범위에서 정규화한다.
- 동의어, 축약어, 오타는 사전 기준으로 원 질의와 매칭 질의를 함께 보관한다.
• 제한 입력 안내 (PI-AIS-INP-003)
- 금칙어, 민감정보, 법무·브랜드 제한어가 포함되면 차단 사유와 대체 입력 예시를 안내한다.
- 차단 이력은 운영자 품질 분석 대상에 포함한다.
• 검색어 삭제·초기화 (PI-AIS-INP-004)
- 검색어가 1자 이상 입력되면 삭제 동작을 제공하고, 삭제 시 입력값, 자동완성 후보, 오류 안내를 초기화한다.
- 다만 최근 검색어와 개인화 추천 영역은 고객 삭제 요청 전까지 보관 기준을 유지한다.

3) 검색어 보조 정책 (PG-AIS-SUGGEST-001)

• 자동완성 제공 (PI-AIS-SGG-001)
- 자동완성은 입력 시작 후 1자 이상에서 제공할 수 있으며 메뉴·기능 바로가기, 추천 검색어, 연관 키워드를 구분해 최대 8개까지 우선 노출한다.
• 최근 검색어 관리 (PI-AIS-SGG-002)
- 최근 검색어는 고객 단말 또는 계정 기준으로 보관하며 개별 삭제와 전체 삭제를 제공한다.
- 저장 기간은 기본 30일로 두고 비로그인 상태 검색은 세션 종료 시 삭제한다.
• 인기 검색어 갱신 (PI-AIS-SGG-003)
- 인기·트렌드 검색어는 운영 기준 집계 주기별로 갱신하며 금칙어, 민감어, 법무 제한어는 노출 대상에서 제외한다.
• 개인화 추천검색어 (PI-AIS-SGG-004)
- 추천 검색어는 고객의 동의, 이용 맥락, 보유 상품, 진행 중 혜택, 현재 화면을 기준으로 노출할 수 있다.
- 개인화 조건이 없거나 동의가 없으면 공통 추천어와 인기 검색어로 대체한다.
• 동의어·오타 인식 (PI-AIS-SGG-005)
- 동의어, 축약어, 브랜드 별칭, 상품 약칭, 빈번한 오타는 사전 기준으로 동일하거나 유사한 의도로 매핑한다.
- 운영자가 승인하지 않은 신규 후보는 고객 노출 전 운영 검수 큐를 거친다.
• 검색창 초기화면 개인화 (PI-AIS-SGG-006)
- 검색창 초기화면은 고객 맥락, 진행 중인 기획전, 최근 탐색, 이용 가능 혜택을 기준으로 추천 후보를 제공하되 인기 검색어만 단독 노출하지 않는다.
• 개인화된 추천검색어 노출 (PI-AIS-SGG-007)
- 추천검색어는 고객 상태, 가입 상품, 대상 혜택, 최근 이용 맥락을 충족하는 경우에만 노출한다.
- 대상자가 아닌 혜택 키워드는 추천검색어에서 제외한다.

4) 결과 랭킹·우선순위 정책 (PG-AIS-RANK-001)

• 정확일치 우선 (PI-AIS-RNK-001)
- 전체 탭은 정확일치, 고객 업무 완결 가능성, 고객 상태 적합성, 최신성, 이용 빈도 순으로 기본 우선순위를 적용한다.
• 유형별 우선순위 (PI-AIS-RNK-002)
- 질의가 메뉴·업무 실행 의도이면 Quick Action을 상품·콘텐츠보다 우선한다.
- 질의가 비교·구매 의도이면 상품, 혜택, 비교 가능 결과를 우선한다.
• 상태 기반 가중치 (PI-AIS-RNK-003)
- 고객 가입 상태, 보유 상품, 멤버십 조건, 위치, 시간 맥락으로 이용 불가한 결과는 숨기지 않고 제한 사유와 대체 결과를 함께 제공할 수 있다.
• 의도별 랭킹 가중치 (PI-AIS-RNK-004)
- 리필, 요금제, 혜택, 상담, 주문, 변경, 해지처럼 의도가 특정되면 메뉴, FAQ, 상품, 이벤트, 콘텐츠, 실행 경로의 가중치를 다르게 적용한다.
- 고객이 바로 완료할 수 있는 업무 실행 결과는 설명 콘텐츠보다 우선하며, 운영자는 의도별 기본 가중치와 변경 이력을 저장한다.
• 정확·연관 결과 구분 (PI-AIS-RNK-005)
- 검색어가 직접 포함되거나 동일 의미로 매칭된 결과와 연관 추천 결과는 구분해 표시한다.
- 연관 추천은 정확 결과를 대체하지 않으며 고객이 추천 근거를 확인할 수 있어야 한다.
• 검색 매칭 기준 안내 (PI-AIS-RNK-006)
- 운영자는 검색 입력 또는 결과 관리 화면에서 부분일치, 완전일치, 동의어 매칭, 연관 추천 중 어떤 기준이 적용되는지 확인할 수 있어야 한다.
- 고객 화면에는 필요한 경우 직접 매칭과 연관 추천 라벨로 단순화해 표시한다.
• 필터 구간값 갱신 (PI-AIS-RNK-007)
- 화면크기, 배터리용량, 저장공간, 가격대처럼 조건 검색에 쓰이는 구간값은 신규 상품 스펙과 고객 선택 분포를 기준으로 운영자가 갱신할 수 있다.
- 구간값 변경은 검색 결과 수, 필터 선택 이력, 변경 일시를 함께 저장한다.
• 사용자 행위 중심 검색 결과 (PI-AIS-RNK-008)
- 검색 결과 명칭은 내부 메뉴명보다 고객이 수행하려는 행동과 목적을 우선한다.
- 내부 용어가 필요한 경우 고객 언어로 변환한 라벨과 함께 제공하고, 동일 의미의 결과명은 운영 승인 후 하나의 대표 라벨로 통합한다.

5) 결과 카드·출처 고지 정책 (PG-AIS-CARD-001)

• 출처·근거 표시 (PI-AIS-CRD-001)
- AI 요약과 추천에는 공식 정보 출처, 적용 조건, 생성 기준 시점, 한계 또는 확인 필요 범위를 함께 표시한다.
• Answer Card 우선 (PI-AIS-CRD-002)
- 요금, 잔여 데이터, 주문 상태, 멤버십 사용 가능 여부처럼 고객 상태 기반 즉답이 가능한 질의는 일반 목록보다 Answer Card를 우선 제공한다.
• 카드 필수 정보 (PI-AIS-CRD-003)
- 결과 카드는 제목, 요약, 유형, 적용 조건, 상태 배지, 주요 CTA, 원본 상세 이동 경로를 포함한다.
- 기간이나 제한이 있는 결과는 만료 또는 제한 사유를 함께 표시한다.
• 매칭 근거·하이라이트 (PI-AIS-CRD-004)
- 검색어와 결과가 매칭된 제목, 속성, FAQ 문장, 상품 속성은 가능한 경우 하이라이트하거나 근거 문구로 표시한다.
- 매칭 근거가 없으면 AI 요약만으로 결과를 상단 노출하지 않는다.
• 상품 핵심 정보 요약 (PI-AIS-CRD-005)
- 상품 탐색 결과는 고객이 상세 진입 전 판단할 수 있도록 제조사, 모델, 출시일, 주요 사양, 혜택·가격 표시 가능 여부, 재고 또는 전시 상태 중 해당되는 핵심 정보를 요약한다.
- 표시할 수 없는 항목은 숨기지 말고 상세 확인 필요로 안내한다.
• 유형별 카드 템플릿 (PI-AIS-CRD-006)
- 상품, 이벤트, 콘텐츠, 메뉴, FAQ 결과는 각 유형별 필수 필드와 CTA를 가진 카드 템플릿으로 표시한다.
- 전체 탭에서도 같은 템플릿 기준을 재사용해 고객이 결과 유형을 혼동하지 않게 한다.

6) 고객 상태·노출 제한 정책 (PG-AIS-CONDITION-001)

• 대상 조건 반영 (PI-AIS-CND-001)
- 연령, 회선, 가입 상태, 보유 상품, 멤버십 등급, 위치 조건이 있는 결과는 고객 상태를 확인한 뒤 즉시 이용 가능, 조건 필요, 이용 불가 중 하나로 표시한다.
• 미충족 사유 표시 (PI-AIS-CND-002)
- 조건 미충족 결과는 단순 미노출하지 않고 고객이 이해할 수 있는 사유와 충족 방법 또는 대체 경로를 함께 제공한다.
• 일부 결과 제외 안내 (PI-AIS-CND-003)
- 권한·대상 제한으로 결과 일부를 제외하면 전체 결과 영역 상단 또는 해당 섹션에 일부 결과 제외 안내와 대표 사유를 표시한다.
• 선택동의 분기 (PI-AIS-CND-004)
- AI 개인화 또는 고객 데이터 결합이 필요한 결과는 선택동의 상태를 먼저 확인한다.
- 동의 전에는 공통 결과를 제공하고, 동의 완료 후에는 개인화 결과 또는 AI 서비스 활성화 흐름으로 분기한다.
• 개인화 데이터 결합 제한 (PI-AIS-CND-005)
- 멤버십 등급, 쿠폰, 포인트, 최근 이용내역, 결제 이력 등 고객 데이터는 고객 동의와 업무 목적이 충족된 경우에만 AI 응답에 결합한다.
- 민감정보와 인증값은 RAG 컨텍스트에 저장하지 않는다.

7) 비교·추천 통제 정책 (PG-AIS-COMPARE-001)

• 비교 대상 기준 (PI-AIS-CMP-001)
- 고객은 비교 가능한 상품, 요금제, 혜택을 2개 이상 선택해 비교 화면으로 이동할 수 있다.
- 비교 진입 시 검색어, 필터, 정렬 상태를 유지한다.
• 추천 근거 고지 (PI-AIS-CMP-002)
- AI 추천안은 추천 사유, 조건, 유의사항, 대체 선택지를 함께 제공한다.
- 추천 사유가 고객 상태에 기반하면 사용한 상태 항목을 고객이 이해 가능한 표현으로 표시한다.
• AI 결정 금지 (PI-AIS-CMP-003)
- AI는 가입, 변경, 해지, 결제, 환불을 고객 대신 확정하지 않는다.
- 주요 실행은 고객의 명시적 선택과 최종 확인 이후에만 연결한다.

8) 실행 연결·최종 확인 정책 (PG-AIS-ACTION-001)

• 실행 전 최종 확인 (PI-AIS-ACT-001)
- 가입, 변경, 해지, 납부, 결제, 예약, 신청은 실행 전 대상, 영향, 적용 시점, 취소 가능 여부를 고객에게 확인시킨다.
• 필수 절차 연계 (PI-AIS-ACT-002)
- 본인 확인, 약관 동의, 추가 인증, 유의사항 확인이 필요한 업무는 실행 연결 전 또는 실행 화면 진입 직후 필수 절차로 연계한다.
• 실행 컨텍스트 전달 (PI-AIS-ACT-003)
- 실행 화면에는 검색어, 선택 결과, 비교 기준, 고객이 선택한 조건을 전달한다.
- 고객이 같은 조건을 반복 입력하지 않도록 하며, 전달된 컨텍스트는 실행 결과 이력과 연결 가능한 요청 키로 관리한다.
• 크로스 도메인 실행 번들링 (PI-AIS-ACT-004)
- 고객 의도가 여러 도메인에 걸쳐 있으면 관련 과업 후보를 묶어 제안할 수 있다.
- 단, 가입, 주문, 결제, 해지, 쿠폰 사용처럼 실제 실행이 필요한 항목은 도메인별 최종 확인과 필수 인증을 분리해 처리한다.
• 액션 위젯 실행 (PI-AIS-ACT-005)
- AI 응답 안의 쿠폰 재발급, 포인트 복구, 바코드 열기, 결제수단 등록, 혜택 적용 같은 액션 위젯은 실행 전 대상, 영향, 취소 가능 여부를 표시한다.
- 원터치 실행이 가능한 경우에도 고객 명시 선택과 결과 이력 저장을 필수로 둔다.
• 중복 실행 방지 (PI-AIS-ACT-006)
- 동일 고객, 동일 검색 컨텍스트, 동일 대상의 실행 연결 요청이 2회 이상 반복되면 하나의 요청 키로 묶어 후속 업무 요청은 1건만 생성한다.
- 이미 실행 전환 완료 상태이면 중복 요청을 차단하고 고객에게 기존 처리 상태와 확인 경로를 안내한다.

9) 실패·무결과 복구 정책 (PG-AIS-FAIL-001)

• 무결과 폴백 (PI-AIS-ERR-001)
- 무결과 상태에서는 빈 화면만 표시하지 않는다.
- 연관 검색어, 인기 카테고리, FAQ, 메뉴 바로가기, 상담 연결 중 2개 이상의 대체 경로를 제공한다.
• 추가 질문 기준 (PI-AIS-ERR-002)
- 질의가 모호하면 추가 질문은 최대 2단계까지 제공한다.
- 2단계 이후에도 의도가 특정되지 않으면 검색 결과 목록 또는 상담 전환을 제안한다.
• 오류 유형별 복구 (PI-AIS-ERR-003)
- 네트워크 오류, 검색 호출 실패, 권한 제한, 결과 로딩 실패는 오류 유형별로 재시도, 홈 이동, 상담 연결, 나중에 다시 시도 중 가능한 복구 경로를 안내한다.
• 검색 이탈 방지 (PI-AIS-ERR-004)
- 무결과 또는 낮은 신뢰도 상태에서는 유사어, 제휴처 후보, 인기 카테고리, 현재 화면 관련 도움, 상담 연결 중 최소 2개 이상의 다음 행동을 제공한다.
- 동일 무결과가 2회 반복되면 재검색보다 조건 보완 또는 상담 전환을 우선 제안한다.
• 추가 로딩 실패 복구 (PI-AIS-ERR-005)
- 페이지네이션 또는 무한스크롤 추가 로딩이 실패하면 현재까지 표시된 결과를 유지하고 재시도, 필터 축소, 상담 또는 나중에 다시 시도 중 가능한 복구 경로를 안내한다.
- 실패 횟수와 구간은 품질 지표에 저장한다.
• 0건 상태 구분 (PI-AIS-ERR-006)
- 검색 결과 0건은 조건 불일치(NO_MATCH)와 실제 데이터 부재(NO_DATA)를 구분한다.
- 조건 불일치이면 필터 완화와 유사어를 우선 제안하고, 데이터 부재이면 대체 카테고리, FAQ, 상담 또는 운영 개선 후보 등록을 안내한다.

10) 검색 이력·상태 유지 정책 (PG-AIS-HISTORY-001)

• 최근 검색어 보관 (PI-AIS-HIS-001)
- 계정 기준 최근 검색어는 기본 30일 보관하고 고객이 개별 삭제 또는 전체 삭제하면 즉시 검색 화면에서 제외한다.
- 비로그인 상태 검색 이력은 세션 종료 시 보관하지 않는다.
• 상세 복귀 유지 (PI-AIS-HIS-002)
- 검색 결과에서 상세 화면으로 이동한 뒤 복귀하면 검색어, 탭, 필터, 정렬, 스크롤 위치를 유지한다.
- 단, 세션이 만료되면 검색어와 기본 탭만 유지한다.
• 검색 컨텍스트 재사용 (PI-AIS-HIS-003)
- 검색 컨텍스트는 후속 실행과 상담 전환에 필요한 범위로만 재사용한다.
- 민감정보나 인증값은 컨텍스트에 저장하지 않는다.

11) 상담 전환 정책 (PG-AIS-CS-001)

• 상담 전환 조건 (PI-AIS-CS-001)
- 검색 실패, 근거 부족, 고객 불만 피드백, 실행 제한, 복구 실패가 발생하면 상담 전환을 제공한다.
- 자동 전환은 하지 않고 고객 선택을 받아야 한다.
• 전달 정보 고지 (PI-AIS-CS-002)
- 상담 전환 전 검색어, 선택 결과, 실패 이력, 고객 상태 요약이 전달될 수 있음을 고지한다.
- 고객이 동의하지 않으면 최소 정보만 전달한다.
• 상담 후 복귀 안내 (PI-AIS-CS-003)
- 상담 연결 후 고객이 검색 결과로 돌아올 수 있도록 이전 검색어와 선택 결과 복귀 경로를 제공한다.
• 상황 인지형 도움 전환 (PI-AIS-CS-004)
- 고객이 특정 화면이나 단계에서 기준 시간 이상 머무르거나 반복 실패하면 현재 화면 맥락에 맞는 도움말, 추가 질문, AI Assist, 상담 전환을 제안할 수 있다.
- 자동 전환은 하지 않고 고객 선택을 우선한다.
• LLM 상담봇 연결 (PI-AIS-CS-005)
- LLM 상담봇은 전역 진입과 상황별 진입을 모두 제공할 수 있으나, 가입·변경·결제·해지 같은 주요 업무 확정은 고객 최종 확인과 필수 인증 절차 이후 업무 화면에서 처리한다.
- 봇이 해결하지 못한 경우 검색 컨텍스트를 상담사에게 전달한다.
• 인앱 자가 해결 연결 (PI-AIS-CS-006)
- 쿠폰 사용 실패, 혜택 조건 확인, 포인트 소멸 조회처럼 반복 문의가 많은 항목은 AI 상담에서 자가 해결 경로와 액션 위젯을 우선 제공한다.
- AI로 해결하지 못하면 대화 이력, 고객 상태, 시도한 조치를 상담사에게 승계한다.

12) 운영 사전·금칙어 정책 (PG-AIS-OPS-001)

• 사전 변경 관리 (PI-AIS-OPS-001)
- 동의어, 축약어, 오타, 대표 엔티티 변경은 버전, 변경자, 적용 시점, 회수 가능 여부를 이력으로 저장한다.
• 금칙어 차단 (PI-AIS-OPS-002)
- 금칙어와 정책 제한어는 차단, 치환, 안내, 운영 검수 큐 전환 중 하나의 처리 기준을 가져야 한다.
- 고객에게는 사유와 대체 입력 예시를 제공한다.
• 대표 엔티티 관리 (PI-AIS-OPS-003)
- 동일 목적 결과가 여러 채널에 있으면 대표 엔티티, 채널별 실행 경로, 예외 노출 조건을 운영자가 관리한다.

13) AI 후보 검수·배포 정책 (PG-AIS-QA-001)

• 운영 검수 큐 (PI-AIS-QA-001)
- AI가 자동 생성한 추천어, 자동완성, 응답 후보 중 근거 누락, 정책 위반 의심, 민감 표현, 법무 위험이 있는 항목은 운영 검수 큐로 보낸다.
• 승인·반려 (PI-AIS-QA-002)
- 운영 검수 큐 항목은 승인, 수정 승인, 반려, 차단 중 하나로 처리한다.
- 승인 전 항목은 고객에게 직접 노출하지 않는다.
• 배포·롤백 (PI-AIS-QA-003)
- 배포된 후보는 버전 단위로 관리한다.
- 오답률, 신고, 정책 위반이 기준치를 넘으면 즉시 회수하고 이전 정상 버전으로 롤백한다.

14) 품질·성과 운영 정책 (PG-AIS-METRIC-001)

• 품질 지표 관리 (PI-AIS-MET-001)
- 검색 성공률, 즉답 해결률, 실행 완료율, 재검색률, 무결과율, 검색 이탈률, 상담 전환율을 공통 지표로 관리한다.
- 지표는 일 단위로 집계하고 운영 기준치를 벗어나면 개선 후보로 등록한다.
• 이상 징후 알림 (PI-AIS-MET-002)
- 오답률, 정책 위반률, 오류율, 상담 전환율이 운영 기준치를 초과하면 운영자에게 알림과 대응 가이드를 제공한다.
• 실패 케이스 개선 (PI-AIS-MET-003)
- 무결과, 반복 재검색, 상담 전환, 신고, 실행 실패 케이스는 주 1회 이상 개선 후보로 분류하고 사전, 랭킹, 폴백 기준 개선에 반영한다.
• 성능 기준 관리 (PI-AIS-MET-004)
- 검색 응답 시간, 첫 결과 렌더링 지연, 검색 API 호출 실패율, 추가 로딩 실패율, 캐시·프리페치 적용 여부를 품질 지표로 관리한다.
- 운영 기준치를 초과하면 검색 품질 이상 징후로 분류하고 개선 후보를 등록한다.
• 모델·룰 성과 비교 (PI-AIS-MET-005)
- AI 응답, 룰 기반 결과, 기존 검색 결과의 정확도, 해결률, 전환율, 상담 전환율을 비교 분석한다.
- 특정 방식이 기준 이하로 떨어지면 운영 검수 큐에 개선 후보로 등록한다.

15) 로그·개인정보 보호 정책 (PG-AIS-DATA-001)

• 최소 로그 수집 (PI-AIS-DAT-001)
- 검색 로그는 질의, 의도, 결과 유형, 선택 결과, 성공·실패, 상담 전환, 피드백 등 품질 개선에 필요한 최소 항목만 수집한다.
• 민감정보 마스킹 (PI-AIS-DAT-002)
- 주민번호, 계좌, 카드번호, 인증값, 비밀번호 등 민감정보가 질의에 포함되면 저장 전 마스킹하거나 저장 대상에서 제외한다.
• 보관·파기 기준 (PI-AIS-DAT-003)
- 운영 분석용 검색 로그는 기본 13개월 이내로 보관하고 법적·감사 목적이 없는 원문 질의는 목적 달성 후 비식별화하거나 파기한다.

최종 점검 기준

AI 검색 정책서 제출 전 점검
1. 범위 정합성
AI 검색이 검색 진입, 결과 이해, 실행 연결, 실패 복구, 운영 품질 기준으로 작성되었고 인접 정책서 상세 업무를 침범하지 않았는지 확인한다.
2. 고객 완결성
고객이 검색어 입력부터 결과 확인, 비교, 실행, 상담 전환, 실패 복구까지 다음 행동을 선택할 수 있는지 확인한다.
3. 의도 해석 정합성
키워드 검색, 질문형 검색, 업무 실행 요청, 상담 요청이 하나의 입력 구조에서 분기되도록 작성되었는지 확인한다.
4. 상태 전이 정합성
검색 전, 질의 입력 중, 의도 해석 중, 결과 제공, 근거 부족, 결과 없음, 오류·복구, 상담 전환 상태가 전이표와 일치하는지 확인한다.
5. 프로세스-기능-정책 연결성
모든 프로세스가 관련 기능과 관련 정책을 가지고, 기능과 정책이 검색 흐름의 판단 기준으로 연결되는지 확인한다.
6. 정책 구체성
정책 항목에 입력 길이, 보관 기간, 추가 질문 단계, 랭킹 우선순위, 실행 전 확인, 로그 보관 기준 등 판단값이 있는지 확인한다.
7. AI 통제 원칙
AI가 고객 대신 계약·변경·해지·결제를 확정하지 않고 고객 최종 확인과 원본 확인 경로를 제공하는지 확인한다.
8. BSS 기준 정보 포함성
BSS가 제공하는 고객 상태, 보유 상품, 권한, 대상 조건, 실행 가능 여부 기준과 검색 연계 시스템의 반영 책임이 분리되어 있는지 확인한다.
9. 액터 책임 연결성
고객, 상담사, 운영자, 채널 업무 시스템이 각각 유즈케이스와 프로세스에 직접 연결되어 있는지 확인한다.
10. 실패·복구 가능성
무결과, 근거 부족, 오류, 제한, 상담 전환에서 고객이 재검색 또는 대체 경로를 선택할 수 있는지 확인한다.
11. 운영 관리 가능성
사전, 금칙어, 대표 엔티티, 랭킹, AI 후보 검수, 배포·롤백, 품질 지표 개선 루프가 운영 가능한 수준인지 확인한다.
12. 요구사항 반영 여부
PM-05 상세 요구사항 101건이 검색 진입, 결과 구성, 비교·실행, 복구, 운영 품질 기준 중 하나 이상에 매핑되었는지 확인한다.
