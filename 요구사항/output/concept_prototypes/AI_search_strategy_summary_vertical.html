통합채널 정책 전략 요약 2026.05

AI Search Strategy

AI 검색은 고객 의도에서 실행까지 이어지는 전략 인터페이스다

검색은 결과 목록을 보여주는 기능이 아니라, 고객이 원하는 업무를 이해하고 비교·판단·실행·복구까지 연결하는 통합채널의 업무 시작점이다.

정책서 IDPOL-AIS
버전v0.11
상태작성중
기준일2026.05.11

01작성 지향점

1
Intent-to-Action 전환

검색을 단순 정보 조회가 아니라 고객 의도를 해석해 비교·요약·업무 실행까지 연결하는 통합 진입점으로 정의한다.

2
PI 기준 통합

메뉴 검색, 상품 검색, FAQ, 상담 탐색, 업무 진입을 분리 운영하지 않고 자연어 의도 기반 단일 프로세스로 통합한다.

3
고객 통제권 보장

AI가 결정을 대신하지 않고 출처, 조건, 미충족 사유, 실행 전 확인을 제공해 고객이 직접 최종 판단하게 한다.

02설계 원칙

1고객 목적 중심

상품·메뉴명이 아니라 고객이 끝내려는 업무 목적 기준으로 검색 결과와 실행 경로를 구성한다.

2근거 기반 결과

추천·요약·Answer Card에는 출처, 조건, 미충족 사유를 함께 표시해 신뢰 가능한 결과로 만든다.

3실행 전 고객 확인

가입·변경·납부·해지 등 실제 업무 실행은 AI가 확정하지 않고 고객 확인과 인증 뒤 연결한다.

4복구 가능한 실패

무결과, 근거 부족, 제한, 오류 상황에서도 재검색·조건 보완·대체 경로·상담 전환 기준을 둔다.

5운영 품질 루프

검색 성공률, 즉답 해결률, 실행 완료율, 무결과율, 상담 전환율을 개선 지표로 관리하고 결과 품질을 운영 루프로 되돌린다.

03고객 여정 구조

1
검색 진입홈·전시·상품·CS에서 공통 검색 진입과 입력 표준을 적용한다.
2
의도 해석질문을 정규화하고 고객 목적, 조건, 속성, 실행 가능성을 추출한다.
3
결과 구성Answer Card, 대표 결과, 유형별 탭, 비교 후보를 고객 상태 기준으로 구성한다.
4
판단 지원출처·근거, 적용 조건, 미충족 사유, 비교 차이를 함께 제시한다.
5
실행·복구업무 화면 연결, 재검색, 조건 보완, 상담 전환으로 다음 행동을 보장한다.

04주요 프로세스와 기능·정책

핵심 프로세스

  • PR-AIS-CUS-001공통 검색 진입 및 입력 표준 적용
  • PR-AIS-CUS-003의도 기반 통합 결과 구성
  • PR-AIS-CUS-005비교 대상 선택 및 차이 요약 확인
  • PR-AIS-CUS-008재검색·조건 보완·대체 경로 선택

기능·정책 묶음

  • FN-AIS-ENT / PG-AIS-SCOPE검색 진입점과 통합 검색 범위
  • FN-AIS-INP / PG-AIS-INPUT질의 입력·정규화와 제한 입력 안내
  • FN-AIS-ANS / PG-AIS-CARDAnswer Card·결과 카드·출처 고지
  • FN-AIS-MON / PG-AIS-METRIC품질·성과·로그 모니터링

05주요 액터

고객 운영자 상담사 채널 업무 시스템

06운영 KPI

검색 성공률 즉답 해결률 실행 완료율 무결과율 재검색률 상담 전환율 오류율 피드백률
결론

AI 검색은 검색 결과를 더 잘 보여주는 기능이 아니라, 고객 의도를 공식 근거와 고객 통제로 실행까지 연결하는 통합채널의 전략 인터페이스다.
