As-Is 문제

전시는 링크, 이미지, 채널별 페이지 단위로 흩어져 고객이 목적에 맞는 시작점과 다음 행동을 찾기 어렵다.

  • 사업자 관점 정보가 반복 노출된다.

  • 고정 영역 위에 개인화 슬롯을 얹어 운영 가능한 개인화를 구현한다.

  • 외부 제휴 화면도 하나의 연속된 서비스처럼 인지된다.

2. 과제 목적

  • 01

    기존 채널별 분산 전시를 고객 여정의 확인, 이해, 실행, 완료 흐름에 맞춘 하나의 전시 체계로 수립한다.

  • 02

    전시 요소를 최소 단위로 구조화하고 고객 맥락과 세그먼트에 따라 유연하게 조합·노출할 수 있는 기반을 마련한다.

  • 03

    통합채널 전반에 공통 UX 체계와 운영 관리 체계를 구축해 고객 탐색 부담과 운영 리소스를 줄인다.

  • 04

    전시를 처리 전환으로 연결해 고객이 원하는 상품, 혜택, 관리 기능까지 더 짧게 도달하게 한다.

  • 05

    홈은 공통 구조 위에 고객별 우선순위와 직접 실행 경로를 얹는 개인화된 실행 허브로 전환한다.

3. 문제 정의

현 상태

현재 전시는 채널별 정보와 기능이 분산되어 제공된다.
메뉴와 링크 중심 구조로 인해 고객은 어디서 시작해야 하는지 직관적으로 이해하기 어렵고 서비스 간 흐름도 단절되어 있다.

Pain Point

고객 맥락과 무관한 정보가 반복 노출되고, 자주 쓰는 기능과 현재 필요한 처리 경로가 전면에 배치되지 않아 탐색과 판단 부담이 크다.
채널·제휴 영역별 UX와 운영 방식도 달라 빠른 대응이 어렵다.

Root Cause

서비스별 분절 구조, 개별 운영 체계, 고정형 전시 방식 때문에 통합 관점의 전시 설계, 재조합, 운영이 어렵다.
고객 상황에 따른 우선 노출과 즉시 처리 흐름도 충분히 반영되지 않는다.

미해결 Risk

고객 혼란과 탐색 피로가 커지고 전시에서 처리로 이어지는 전환 효율이 낮아진다.
고객, 정책, 상품 변화에 대한 대응 속도가 떨어져 통합채널의 체감 가치와 Engagement 효과도 제한된다.

4. As-Is 문제점

01 링크 중심 구조
현상

단순 링크 집합 방식의 통합 구조다

문제점

고객은 어디서 무엇을 해야 하는지 직관적으로 이해하기 어렵고, 서비스 간 이동이 하나의 처리 흐름으로 이어지지 않는다.

02 정보 과다
현상

사업자 관점 반복 노출과 정보 과다가 크다

문제점

상품과 혜택의 핵심 가치를 빠르게 파악하기 어렵고 자주 쓰는 기능과 지금 필요한 행동이 전면에 보이지 않는다.

03 이미지 의존
현상

이미지 중심 전시 구조에 운영 리소스가 많이 든다

문제점

제작과 수정 비용이 크고 텍스트·구조화 정보 기반의 검색, 추천, AI 처리에 불리하다.

04 외부 화면 단절
현상

iFrame과 외부 BP 화면의 UX 기준이 일치하지 않는다

문제점

외부 제휴 화면은 톤앤매너와 이용·복귀 경험이 달라 고객이 하나의 연속 서비스로 인지하기 어렵다.

05 관리 분산
현상

전시 관리 BO와 상품·전시 정보 관리 체계가 채널별로 분산되어 있다

문제점

화면마다 노출 방식과 정보 구성이 달라 같은 정보를 다시 확인해야 하고 전시 신뢰도가 낮아진다.

06 수작업 운영
현상

전시 운영 자동화가 부족하고 수작업 의존도가 높다

문제점

신규 상품, 정책, 혜택 정보가 늦게 반영되거나 누락될 수 있어 고객은 최신 정보인지 확신하기 어렵다.

07 개인화 부족
현상

개인화 기반이 부족하다

문제점

고객 맥락에 맞는 추천과 전시를 체감하기 어렵고 필요한 정보와 행동 경로를 직접 찾고 비교해야 한다.

5. To-Be 흐름

전시 흐름은 고객이 어디서 시작할지 이해하는 순간부터 핵심 메시지를 파악하고, 조건과 선택지를 확인한 뒤 바로 실행하고 완료 상태까지 확인하는 구조여야 한다.

01
Check 상태 확인

홈에서 요금, 데이터, 혜택, 신청 상태, 진행 중 업무, 긴급 공지 등 현재 필요한 정보를 확인한다.

02
Understand 목적 이해

핵심 메시지와 조건을 먼저 보고 상품, 콘텐츠, 혜택, 관리 기능의 목적을 빠르게 파악한다.

03
Choose 탐색/선택

목록, 필터, 정렬, 탭, 추천 컬렉션, 개인화 모듈을 통해 필요한 대상과 경로를 좁힌다.

04
Act 실행 진입

배너, 카드, 상세, 퀵메뉴에서 신청, 변경, 해지, 이용, 외부 제휴 진입 같은 다음 행동으로 연결한다.

05
Return 완료/복귀

외부 화면 이용, 처리 성공, 실패, 중단 후에도 이전 탐색 상태를 유지하고 완료·복귀 흐름을 안내한다.

06
Improve 운영 보정

노출, 클릭, 처리 시작, 완료, 숨김, 관심없음 데이터를 기반으로 슬롯과 템플릿을 개선한다.

↺
Feedback Loop

전시 노출, 클릭, 숨김, 관심없음, 경로 진입, 처리 시작, 완료, 외부 복귀 데이터는 다시 슬롯 우선순위, 템플릿 구성, 개인화 규칙, 배너 빈도, 운영 변경 기준에 반영된다.

7. 주요 프로세스 및 기능 설명

01
Home Action

홈 탐색과 실행 진입

홈을 단순 관문이 아니라 고객 상태와 즉시 처리 업무를 보여주는 실행 허브로 운영한다.

Outcome

홈 주요 기능 클릭률과 처리 시작률 향상

  1. 01

    고객은 홈에서 내 상태 요약, 혜택·상품·관리 정보, 진행 중 업무, 최근 이용 이력을 확인한다.

  2. 02

    개인화 모듈과 퀵메뉴를 통해 자주 쓰는 기능과 현재 필요한 업무에 바로 진입한다.

  3. 03

    요금, 데이터, 혜택, 신청 상태 같은 액션형 요약 정보를 제공한다.

  4. 04

    긴급 공지와 필수 알림은 일반 전시보다 우선 노출한다.

02
Surface Control

목록·배너 탐색과 전시 제어

목록, 추천, 배너, 카드, 기획전을 노출 정책과 고객 반응 기준으로 제어한다.

Outcome

전시 피로 감소와 목적 경로 진입률 향상

  1. 01

    고객은 상품, 콘텐츠, 혜택 목록을 필터, 정렬, 탭, 카테고리, 테마, 추천 컬렉션으로 탐색한다.

  2. 02

    배너, 카드, 기획전 섹션은 세그먼트와 고객 맥락에 따라 차등 노출한다.

  3. 03

    배너 클릭과 랜딩 연결, 노출 우선순위, 중복 노출, 빈도 제어, 숨김·관심없음·N일간 비노출을 처리한다.

  4. 04

    상세 진입 이후 다음 행동이 끊기지 않도록 신청, 변경, 이용, 상담, 외부 제휴 진입을 연결한다.

03
Partner Continuity

외부 제휴 화면 진입·이용·복귀

외부 제휴 이용도 통합채널의 연속 경험처럼 인지되도록 안내, 인증, 결과, 복귀를 표준화한다.

Outcome

외부 화면 이탈과 복귀 실패 감소

  1. 01

    외부 제휴 화면 진입 전 고객에게 서비스 성격, 인증 필요 여부, 이동 후 처리 결과를 안내한다.

  2. 02

    SSO와 인증 연계를 수행하고 외부 BP 화면을 호출한다.

  3. 03

    이용 상태와 결과를 수신해 완료, 실패, 중단 상태를 구분한다.

  4. 04

    복귀 시 이전 탐색 상태를 유지하고 필요한 후속 안내와 재시도 경로를 제공한다.

04
Design Studio

전시 관리와 운영 배포

전시 요소를 공통 템플릿과 컴포넌트로 관리해 운영자가 빠르고 안전하게 변경한다.

Outcome

신규 화면 구성 시간과 운영 변경 대응 시간 단축

  1. 01

    운영자는 홈, 목록, 페이지, 컨테이너, 메뉴, 탭, 서브탭, 카테고리 구조를 관리한다.

  2. 02

    레이아웃, 섹션, 슬롯, 컴포넌트 배치를 등록·수정·중지·삭제한다.

  3. 03

    배너는 슬롯별 배치, 세그먼트·조건별 노출, 검수·승인·배포 기준으로 운영한다.

  4. 04

    외부 제휴 화면에는 공통 UX·UI 가이드와 디자인 시스템 자산 적용 기준을 제공한다.

8. 정책서 작성 시 남겨야 할 판단 기준

1전시는 화면 단위가 아니라 목적, 슬롯, 섹션, 컴포넌트, 랜딩, 다음 행동 기준으로 관리해야 한다.
2상품, 혜택, 관리 전시는 확인, 이해, 실행, 완료 흐름 중 어느 단계의 역할을 하는지 명확해야 한다.
3홈 개인화는 완전 자유 노출이 아니라 공통 구조, 고정 슬롯, 개인화 슬롯, 고객 고정 퀵메뉴, 긴급 공지 우선순위 안에서 운영해야 한다.
4배너와 카드 노출은 우선순위, 중복 제한, 빈도, 숨김, 관심없음, N일간 비노출, 메시지-랜딩 일치 기준을 가져야 한다.
5외부 제휴 화면은 진입 전 안내, SSO·인증, 이용 상태 수신, 완료·실패·중단 후 복귀 위치, 이전 탐색 상태 유지 기준을 가져야 한다.
6전시 관리 BO는 페이지, 컨테이너, 메뉴, 탭, 카테고리, 레이아웃, 섹션, 슬롯, 컴포넌트의 등록·수정·중지·삭제 권한과 승인 이력을 관리해야 한다.
7전시 정보는 텍스트·구조화 정보 중심으로 관리해 검색, 추천, AI 응답, 성과 분석에 재사용 가능해야 한다.
8전시 성과는 노출, 클릭, 경로 진입, 처리 시작, 처리 완료, 이탈, 숨김·관심없음 반응을 같은 퍼널 기준으로 추적해야 한다.
9전시 우선순위는 고객 목적, 현재 상태, 긴급도, 동의 상태, 사업 우선순위를 함께 반영해야 한다.
10전시 모듈은 추천, 알림, 주문, 고객지원, MY 상태 데이터를 받아도 일관된 고객 언어와 CTA 기준을 유지해야 한다.
