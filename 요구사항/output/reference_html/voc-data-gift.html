데이터 선물 VoC 분석

데이터 선물 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 앱 내 데이터 선물 메뉴 접근성 부재 및 UX 복잡성이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

1. 핵심 신호

데이터 선물 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

앱 내 데이터 선물 메뉴 접근성 부재 및 UX 복잡성

" 복잡한 앱 조작 없이 빠르고 간단하게 처리하고 싶다 " — 전화가 더 빠르고 확실하다는 인식 · 스스로 처리할 수 있는 자율성 욕구

02

데이터 선물 정책 복잡성 및 사전 안내 부재

" 복잡한 정책 구조를 명확히 이해하고 싶다 " — 정책 명확화 욕구 · 내 데이터 현황을 정확히 파악하고 싶은 욕구

03

자동선물 · 결합 정책 제약 및 선물 불가 사전 안내 부재

" 매달 반복 전화 없이 자동화하고 싶다 " + " 선물 불가 이유를 미리 알고 싶다 " — 반복 작업 자동화 욕구 및 리스크 회피 욕구

04

다수 수신자 동시 선물 및 대용량 처리 불편

" 가족 모두에게 한 번에 처리하고 싶은 욕구 " — 반복 조작 없이 일괄 처리 원함 · 빠른 처리 욕구

05

자동선물 미작동 및 앱 시스템 오류

" 매달 자동으로 처리되길 원함 + 수동 처리 번거로움 " — 안정성 욕구 · " 앱에서 직접 처리 안 되는 상황에 당혹감 " — 시스템 신뢰 회복 욕구

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

편의성 욕구 (1 위): " 복잡한 앱 조작 없이 빠르고 간단하게 처리하고 싶다 "

앱 홈 원터치 바로가기 + 대용량 자동 분할 발송 + 다중 수신자 동시 선물 대응 필요

통제

정책 이해 욕구 (2 위): "데이터선물 · 리필쿠폰 · 자동선물 · 가족선물의 차이와 한도를 명확히 알고 싶다 "

선물 정책 통합 대시보드 + 선물 불가 시 원인 명확 표시 대응 필요

즉시

자동화 욕구 (3 위): " 매달 반복 전화 없이 자동으로 처리되길 원한다 "

자동선물 앱 내 직접 신청 + 정기 선물 스케줄러 대응 필요

이해

즉각 처리 욕구 (4 위): " 로밍 데이터 소진 시 지금 당장 충전하고 싶다 "

로밍 데이터 잔량 실시간 위젯 + 소진 알림 딥링크 + 원터치 충전 대응 필요

연계

가족 케어 욕구 (5 위): " 가족 모두에게 한 번에 데이터를 주고 싶다 "

다중 수신자 동시 선물 + 가족 데이터 통합 대시보드 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01가족 · 자녀 · 배우자 데이터 부족 인지 후 선물 필요 → 앱 메뉴 못 찾아 전화 — 전체 Trigger 의 35~52%, 전 기간 1 위
  2. 02로밍 데이터 소진 문자 / 알림 수신 후 추가 충전 방법 문의 — 공휴일 · 주말 기간 2 위, 전 기간 지속
  3. 03앱에서 선물 시도 후 불가 메시지 확인 → 원인 파악 위해 전화 — 전 기간 3 위
  4. 04매달 전화 신청 번거로움 인지 후 자동선물 요청 — 전 기간 지속
  5. 05리필쿠폰 선물 시도 후 불가 안내 수신 (결합 미가입) — 전 기간 지속
  6. 06출국 전 로밍 요금제 신청 및 에이닷 전화 설정 필요 — 공휴일 · 주말 기간 집중
  7. 07선물 횟수 · 한도 기준 혼동으로 재문의 — 전 기간 지속
  8. 08대용량 (20GB 이상) 선물 필요 → 앱 1GB 단위 반복 발송 불편 — 3/3 이후 급증
  9. 09자동선물 미작동 인지 (요금제 변경 후 해제) — 전 기간 지속
  10. 10공유데이터 소진 후 선물 불가 메시지 확인 — 전 기간 지속
  11. 11데이터 소진 문자 수신 후 리필쿠폰 사용 방법 문의 — 전 기간 지속
  12. 12가족 여러 명에게 동시 선물 필요 → 앱 내 다중 선물 기능 없어 전화 — 3/3 이후 급증
  13. 13해외 도착 후 로밍 데이터 연결 불가 체감 — 공휴일 · 주말 기간 집중
  14. 14앱 UI 변경 후 선물 메뉴 경로 변경으로 혼란 — 전 기간 지속
  15. 15선물 데이터 소진 순서 불명 (선물 → 기본 순서 이해 불가) — 전 기간 지속
  16. 16공휴일 · 야간 데이터 선물 처리 필요 → 상담 불가 안내 — 공휴일 기간 집중
  17. 17리필쿠폰 선물 후 작동 원리 불명확 인지 — 전 기간 지속
  18. 18자동선물 수신자 변경 필요 인지 — 전 기간 지속
  19. 19요금제 변경 후 선물 정책 변화 궁금증 발생 — 전 기간 지속
  20. 20고령 고객 앱 사용 어려움으로 전화 의존 — 전 기간 지속

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 데이터 선물 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

사전 고지

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

효율

반복 입력과 재문의를 어떻게 줄이는가?

반복 입력, 회선별 재처리, 동일 문의 재접수를 줄이도록 일괄 처리와 저장된 정보 재사용 기준을 설계해야 한다

상태·오류

실패와 오류를 어떻게 복구하는가?

접수, 검증, 반영, 실패, 취소 상태를 고객에게 보여주고 재시도, 대체 경로, 상담 전환 기준을 연결해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

데이터 선물 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
