Consulting Variant · BCG Style

전시 구조 및 노출 정책 기획 운영

채널별로 흩어진 링크·이미지 중심 전시를 고객의 확인, 이해, 실행, 완료 흐름에 맞춘 모듈형·운영형·개인화형 전시 체계로 바꾸는 과제다.

메뉴와 링크의 집합에서 고객 목적 기반 실행 허브로 전환한다.

핵심은 화면을 더 많이 꾸미는 것이 아니라, 고객이 지금 해야 할 일을 더 빨리 이해하고 바로 처리하도록 전시 단위, 노출 규칙, 운영 체계를 재설계하는 것이다.

01

Purpose Flow

상품, 혜택, 관리 화면을 메뉴 나열이 아니라 확인, 이해, 실행, 완료 흐름으로 재구성한다.

02

Modular Display

전시 요소를 최소 단위로 구조화해 고객 맥락과 세그먼트에 따라 조합·노출한다.

03

Consistent UX

내부 화면과 외부 제휴 화면에 공통 디자인 모듈과 복귀 기준을 적용해 단절을 줄인다.

04

Personal Home

홈을 조회형 첫 화면이 아니라 상태 확인 후 신청, 변경, 해지, 이용으로 이어지는 실행 허브로 만든다.

Exhibit 01

Strategic Shift Map

As-Is 문제

전시는 링크, 이미지, 채널별 페이지 단위로 흩어져 고객이 목적에 맞는 시작점과 다음 행동을 찾기 어렵다.

사업자 관점 정보가 반복 노출된다.외부 제휴 화면과 내부 화면의 UX 기준이 다르다.전시 정보와 운영 관리 체계가 채널별로 나뉘어 있다.
전략 전환

전시 단위를 구조화하고 공통 UX·디자인 시스템·운영 BO를 적용해 고객 목적과 세그먼트에 따라 재조합한다.

확인, 이해, 실행, 완료 흐름으로 화면을 구성한다.텍스트·구조화 정보 중심으로 검색·추천·AI 연계를 준비한다.고정 영역 위에 개인화 슬롯을 얹어 운영 가능한 개인화를 구현한다.
To-Be 결과

고객은 홈과 목록에서 필요한 기능을 더 빨리 찾고, 전시는 처리 시작과 완료로 자연스럽게 이어진다.

전시→처리 시작률과 처리 완료율이 높아진다.신규 화면 구성과 운영 변경 시간이 줄어든다.외부 제휴 화면도 하나의 연속된 서비스처럼 인지된다.
Exhibit 02

Display Portfolio Logic

Customer Pull

고객이 바로 체감하는 전시 개선

하나의 채널로 완전한 통합 - 고객이 어디서 시작해야 하는지 바로 이해한다.

이해하고 결정하기 쉬운 전시 - 상품과 혜택의 핵심 가치를 더 빠르게 파악한다.

Operating Leverage

운영 효율과 변경 대응력을 키우는 구조

유연하게 재조합되는 모듈형 전시 - 작은 전시 단위를 재사용해 고객·환경 변화 대응이 쉬워진다.

디자인 스튜디오 기반 공통 전시 관리 - 채널별 관리 부담이 줄고 공통 요소를 같은 기준으로 관리한다.

Experience Continuity

내부·외부 화면을 하나의 경험으로 묶는 기준

일관된 UX 체계 구현 - 채널 간 이질감이 줄고 고객이 하나의 연속된 서비스처럼 인지한다.

Personalized Action

홈을 상태 기반 실행 허브로 전환

개인화된 홈 실행 허브 - 고객은 자주 쓰는 기능과 지금 필요한 업무를 더 빠르게 찾는다.

Exhibit 03

Friction to Policy Response

01 단순 링크 집합 방식의 통합 구조다 고객은 어디서 무엇을 해야 하는지 직관적으로 이해하기 어렵고, 서비스 간 이동이 하나의 처리 흐름으로 이어지지 않는다.
 전시는 화면 단위가 아니라 목적, 슬롯, 섹션, 컴포넌트, 랜딩, 다음 행동 기준으로 관리해야 한다.
02 사업자 관점 반복 노출과 정보 과다가 크다 상품과 혜택의 핵심 가치를 빠르게 파악하기 어렵고 자주 쓰는 기능과 지금 필요한 행동이 전면에 보이지 않는다.
 상품, 혜택, 관리 전시는 확인, 이해, 실행, 완료 흐름 중 어느 단계의 역할을 하는지 명확해야 한다.
03 이미지 중심 전시 구조에 운영 리소스가 많이 든다 제작과 수정 비용이 크고 텍스트·구조화 정보 기반의 검색, 추천, AI 처리에 불리하다.
 홈 개인화는 완전 자유 노출이 아니라 공통 구조, 고정 슬롯, 개인화 슬롯, 고객 고정 퀵메뉴, 긴급 공지 우선순위 안에서 운영해야 한다.
04 iFrame과 외부 BP 화면의 UX 기준이 일치하지 않는다 외부 제휴 화면은 톤앤매너와 이용·복귀 경험이 달라 고객이 하나의 연속 서비스로 인지하기 어렵다.
 배너와 카드 노출은 우선순위, 중복 제한, 빈도, 숨김, 관심없음, N일간 비노출, 메시지-랜딩 일치 기준을 가져야 한다.
05 전시 관리 BO와 상품·전시 정보 관리 체계가 채널별로 분산되어 있다 화면마다 노출 방식과 정보 구성이 달라 같은 정보를 다시 확인해야 하고 전시 신뢰도가 낮아진다.
 외부 제휴 화면은 진입 전 안내, SSO·인증, 이용 상태 수신, 완료·실패·중단 후 복귀 위치, 이전 탐색 상태 유지 기준을 가져야 한다.
06 전시 운영 자동화가 부족하고 수작업 의존도가 높다 신규 상품, 정책, 혜택 정보가 늦게 반영되거나 누락될 수 있어 고객은 최신 정보인지 확신하기 어렵다.
 전시 관리 BO는 페이지, 컨테이너, 메뉴, 탭, 카테고리, 레이아웃, 섹션, 슬롯, 컴포넌트의 등록·수정·중지·삭제 권한과 승인 이력을 관리해야 한다.
07 개인화 기반이 부족하다 고객 맥락에 맞는 추천과 전시를 체감하기 어렵고 필요한 정보와 행동 경로를 직접 찾고 비교해야 한다.
 전시 정보는 텍스트·구조화 정보 중심으로 관리해 검색, 추천, AI 응답, 성과 분석에 재사용 가능해야 한다.
Exhibit 04

Operating Flywheel

01 Check

상태 확인

홈에서 요금, 데이터, 혜택, 신청 상태, 진행 중 업무, 긴급 공지 등 현재 필요한 정보를 확인한다.

02 Understand

목적 이해

핵심 메시지와 조건을 먼저 보고 상품, 콘텐츠, 혜택, 관리 기능의 목적을 빠르게 파악한다.

03 Choose

탐색/선택

목록, 필터, 정렬, 탭, 추천 컬렉션, 개인화 모듈을 통해 필요한 대상과 경로를 좁힌다.

04 Act

실행 진입

배너, 카드, 상세, 퀵메뉴에서 신청, 변경, 해지, 이용, 외부 제휴 진입 같은 다음 행동으로 연결한다.

05 Return

완료/복귀

외부 화면 이용, 처리 성공, 실패, 중단 후에도 이전 탐색 상태를 유지하고 완료·복귀 흐름을 안내한다.

06 Improve

운영 보정

노출, 클릭, 처리 시작, 완료, 숨김, 관심없음 데이터를 기반으로 슬롯과 템플릿을 개선한다.
