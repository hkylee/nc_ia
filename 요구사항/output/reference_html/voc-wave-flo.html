웨이브 / 플로 VoC 분석

웨이브 · 플로 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 OTT 가입 후 연결 · 계정 불일치 오류이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

1. 핵심 신호

웨이브 / 플로 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

OTT 가입 후 연결 · 계정 불일치 오류

" 가입했으면 바로 써야 한다 " → 가입 완료 즉시 서비스 이용 가능 상태를 기대하는 즉시성 욕구 · 서비스 연속성 보장 욕구

02

요금제 변경 시 OTT 혜택 변경 사전 안내 부재

" 내가 받던 혜택이 유지되어야 한다 " → 요금제 변경 전 혜택 변동 사항을 미리 알고 싶은 정보 통제 욕구 · 손실 회피 욕구

03

구독 서비스 해지 · 변경 경로 불명

" 불필요한 비용을 없애고 싶다 " → 구독 현황을 한눈에 보고 원클릭으로 정리하고 싶은 자율성 욕구

04

미성년자 · 자녀 회선 보호자 관리 기능 부재

"자녀 · 가족의 데이터 사용을 내가 직접 안정적으로 관리하고 싶다 " → 보호자가 앱에서 직접 처리하지 못하는 구조적 한계로 반복 전화 발생하는 통제 · 배려 욕구

05

로밍 신청 · 할인 조건 · 에이닷 설정 불명

" 출국 전 빠르고 정확하게 로밍 신청을 완료하고 싶다 " → 앱 신뢰 부족 및 UI 복잡성으로 전화 선택 · " 해외에서 예상치 못한 요금이 청구될까봐 두렵다 " → 과금 공포해소 욕구

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

즉시성 욕구 (1 위): " 가입했으면 바로 써야 한다 "

가입 완료 즉시 앱 내 이용권 발급 자동 딥링크 + 계정 연동 원스톱 대응 필요

통제

정보 통제 욕구 (2 위): " 내가 받던 혜택이 유지되어야 한다 "

요금제 변경 전 혜택 변동 사전 안내 팝업 + 비교 UI 대응 필요

즉시

자율성 욕구 (3 위): " 불필요한 비용을 없애고 싶다 "

구독 관리 통합 대시보드 + 원클릭 해지 대응 필요

이해

통제 / 배려 욕구 (4 위): "자녀 / 가족의 데이터 사용을 내가 직접 관리하고 싶다 "

보호자 전용 자녀 폰 관리 메뉴 + 법정대리인 비대면 동의 대응 필요

연계

과금 공포 해소 욕구 (5 위): " 해외에서 예상치 못한 요금이 청구될까봐 두렵다 "

로밍 요금 시뮬레이터 + 에이닷 설정 가이드 자동 발송 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01요금제 변경 목적으로 전화 (앱에서 변경 · 비교 경로 불명) — 전 기간 1 위
  2. 02해외 출국 전 로밍 신청 필요성 인지 (일본 · 미국 · 중국 · 베트남 등) — 연휴 기간 1 위
  3. 03웨이브 · 플로 가입 후 연결 방법 불명 (이용권 발급 링크 문자에만 존재) — 전 기간 지속
  4. 04요금제 변경 후 OTT 혜택 변경 여부 확인 목적 — 전 기간 지속
  5. 05웨이브 · 플로 · 우주패스 해지 경로를 앱에서 찾지 못해 전화 — 전 기간 지속
  6. 06자녀 요금제 변경 목적 (앱 내 자녀 관리 경로 불명) — 전 기간 지속, 3/3 이후 증가
  7. 07웨이브 앱 실행 시 이용권 없음 표시 (계정 불일치) — 전 기간 지속
  8. 08청구서에서 모르는 부가서비스 요금 발견 — 전 기간 지속
  9. 09선택약정 만료 후 연장 방법 불명 (앱 내 알림 · 연장 경로 부재) — 전 기간 지속
  10. 10요금제 변경 후 OTT 연결 오류 발생 (넷플릭스 중복 구독, 플로 1 분 제한 등) — 전 기간 지속
  11. 11VIP 혜택 유지 가능 최저 요금제 확인 목적 — 전 기간 지속
  12. 12로밍 요금 체계 이해 어려움 (해외 여행 전 요금 확인) — 연휴 기간 집중
  13. 13자녀로부터 데이터 연결 안 된다는 연락 수신 — 전 기간 지속
  14. 14기기 변경 후 웨이브 · 플로 이용권 소멸 경험 — 3/3 이후 증가
  15. 15연휴 · 야간 중 담당 부서 미운영으로 처리 불가 상황 발생 — 연휴 기간 집중
  16. 16미성년자 자녀 로밍 신청 시 앱에서 처리 불가 오류 발생 — 연휴 기간 집중
  17. 17플로 · 웨이브 계정 연결 1 시간 이상 실패 후 해지 요청 — 전 기간 지속
  18. 18웨이브 콘텐츠팩 할인 미적용 발생 후 재가입 경로 불명 — 전 기간 지속
  19. 19자녀 요금제 변경 시 약정 조건 · 법정대리인 동의 절차 불명 — 전 기간 지속
  20. 20웨이브 이용 불가 상태에서 요금 청구 발생 (환불 신청 경로 없음) — 전 기간 지속

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

웨이브 · 플로 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
