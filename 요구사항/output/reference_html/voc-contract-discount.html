1. 핵심 신호

약정 할인 / 선택 약정 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

약정 만료 후 재신청 · 연장 경로 불명

" 할인이 끊기지 않고 즉시 재신청 완료하고 싶다 " — 번거로움 회피 + 할인 중단 공포 + 자동화 욕구

02

약정 조건 · 기간 차이 · 할인 구조 이해 부족

" 처리가 정상적으로 됐는지, 내가 손해 보는 건 아닌지 확인하고 싶다 " — 정보 비대칭 해소 + 최적 선택 확신 욕구 + 과금 불안 해소

03

미성년 자녀 · 가족 명의 약정 대리 처리 불가

"자녀 · 가족 약정을 한 번에 간편하게 처리하고 싶다 " — 가족 회선 일괄 처리 편의 욕구 + 고령 부모 대리 처리 욕구

04

약정 처리 완료 후 상태 확인 불가

" 처리가 됐는지 앱에서 직접 눈으로 확인하고 싶다 " — 이중 확인 욕구 + 과금 리스크 회피 욕구

05

주말 · 휴일 약정 처리 불가

" 지금 당장 처리해서 할인 공백 없이 이어가고 싶다 " — 날짜 지연 불이익 공포 + 원스톱 완결 욕구

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?
