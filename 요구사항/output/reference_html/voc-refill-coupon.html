리필 쿠폰 VoC 분석

리필 쿠폰 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 앱 내 리필 쿠폰 사용 경로 불명 및 탐색 실패이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

분석 기간 2026-02-11 ~ 2026-03-11 (29 일) 보고서 29 건 (2/11~2/28, 3/1~3/11) 상담 약 100,000 건 + (일별 최소 48 건 ~ 최대 7,894 건) 샘플 약 2,700 건 + (일별 48~100 건 샘플)
Executive readout 인지 · 탐색 · 권한 · 복구
리필 쿠폰 상담은 고객이 직접 끝내지 못하는 확인과 예외 처리 지점에서 반복된다.

설 연휴 (2/14~2/18, 5 일), 삼일절 연휴 (2/28~3/2), 주말 포함.
공휴일 · 주말 기간 상담 건수 급감 (48~516 건) 후 평일 복귀 시 폭증 (최대 7,894 건) 패턴 반복.
설 연휴 기간 (2/14, 2/17) 은 당직 부서만 운영되어 처리 불가 상황이 전체 상담의 약 20~30% 를 차지하는 특수 상황이었음

1. 핵심 신호

리필 쿠폰 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

앱 내 리필 쿠폰 사용 경로 불명 및 탐색 실패

"전화 없이 스스로 처리하고 싶다 " → 매달 반복 전화에 대한 피로감 + 셀프 서비스 강한 수요 · " 내 쿠폰이 있는지 없는지조차 모르는 답답함 " → 즉각적 가시성 욕구

02

리필 쿠폰 정책 · 개념 이해 부족

" 복잡한 설명 없이 쉽게 이해하고 싶다 " → 약관 · 정책 설명 최소화, 직관적 안내 욕구 · " 내가 받는 혜택이 정확히 무엇인지 알고 싶다 " → 정보 통제 욕구

03

공휴일 · 야간 당직 운영으로 인한 채널 제약

" 연휴 중 데이터 없이 고립될 것에 대한 불안 " → 24 시간 언제든 스스로 해결할 수 있는 채널 욕구 · " 지금 당장 데이터가 필요한데 왜 못 해주냐 " → 즉각 처리 욕구

04

리필 쿠폰 관련 과금 · 손실 불안

" 혜택을 놓치지 않고 싶은 불안 " → 쿠폰 소멸 · 과금 · 취소 불가에 대한 리스크 회피 심리 · " 무료라는 것을 확실히 믿고 싶다 " → 신뢰 확인 욕구

05

가족 회선 대리 처리 및 통합 관리 불가

" 가족 모두의 데이터를 한 번에 관리하고 싶다 " → 가족 통합 관리 욕구 · " 자녀 데이터 문제를 부모가 빠르게 해결해주고 싶은 책임감 "

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01데이터 소진 알림 (앱 / 문자) 수신 후 리필 방법 몰라 전화 — 전체 Trigger 의 40~50%, 전 기간 1 위
  2. 02리필 쿠폰 발급 안내 문자 수신 후 사용 방법 몰라 전화 — 전 기간 2 위
  3. 03데이터 소진 후 속도 저하 직접 체감 후 전화 — 전 기간 3 위
  4. 04데이터 50%·80% 소진 경고 문자 수신 후 리필 쿠폰 존재 여부 확인 — 전 기간 지속
  5. 05설 연휴 · 삼일절 연휴 당직 운영으로 처리 불가 안내 수신 — 공휴일 기간 2 위
  6. 06리필 쿠폰 잔여 장수 · 유효기간 확인 필요로 전화 — 전 기간 지속
  7. 07가족 (자녀 · 배우자 · 부모) 데이터 소진으로 부모가 대신 문의 — 전 기간 지속
  8. 08이미 사용한 리필 쿠폰 재사용 시도 후 실패 — 전 기간 지속
  9. 09리필 쿠폰 사용 시도 중 무료 여부 불안으로 전화 — 전 기간 지속
  10. 10요금제 변경 계획 중 리필 쿠폰 · 데이터 소멸 우려로 전화 — 전 기간 지속
  11. 11매달 자동 신청 가능 여부 문의 후 불가 안내 수신 — 전 기간 지속
  12. 12리필 쿠폰 소멸 예정 인지 후 활용 방법 문의 — 전 기간 지속
  13. 13가족 결합 미등록 번호에 선물 시도 후 불가 안내 수신 — 전 기간 지속
  14. 14데이터 · 음성 리필 선택 방법 몰라 전화 — 전 기간 지속
  15. 15앱 로그인 오류 또는 보호자 인증 오류 발생 — 전 기간 지속
  16. 16리필 완료 후 앱 잔여량 미반영으로 불안 발생 — 전 기간 지속
  17. 17공항 · 긴급 상황에서 데이터 소진으로 즉각 처리 필요 — 전 기간 지속
  18. 18핫스팟 사용 중 리필 쿠폰 적용 여부 혼란으로 전화 — 전 기간 지속

5. Quick Win

즉시 개선 가능한 항목은 진입점 노출, 완료 안내, 상태 조회, 권한 처리, 실패 복구를 중심으로 정리된다.

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 리필 쿠폰 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

확인

처리 결과를 어떻게 확인시키는가?

처리 결과, 이력, 알림, 예상 청구 또는 사용 가능 상태를 고객이 직접 확인할 수 있어야 한다

권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

상태·오류

실패와 오류를 어떻게 복구하는가?

접수, 검증, 반영, 실패, 취소 상태를 고객에게 보여주고 재시도, 대체 경로, 상담 전환 기준을 연결해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

리필 쿠폰 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
