1. IA 분석에서 바로 보이는 핵심 신호

채널별 기능 비중은 각 채널의 역할 차이를 보여주며, 통합 설계에서 무엇을 공통화하고 무엇을 남겨야 하는지의 근거가 된다.

Core signals
01

채널별 역할이 다르다.

T월드는 고객관리, T다이렉트샵은 주문, T멤버십은 혜택 이용, T우주는 구독관리에 기능 비중이 높다.
통합은 역할 삭제가 아니라 역할 재배치다.

02

ORD와 MY는 Core 후보다.

주문·가입·결제와 고객·회원·요금 영역은 유사도가 높고 전 채널 공통성이 크다.
Core 공통 모듈 기준을 먼저 정의해야 한다.

03

주문·가입과 이용안내가 크다.

최신 FO 엑셀에서 주문&가입 계열은 1,155행, 이용안내는 597행으로 확인된다.
콘텐츠 관리와 가입 정보 구조의 중복이 강하게 보인다.

04

전시·혜택은 채널 특화가 강하다.

외부 BP, 혜택 프로그램, 배너, 이벤트, 쿠폰은 T멤버십 중심으로 편중되어 있다.
무조건 공통화보다 특화 기능 분석 후 통합 범위를 정해야 한다.

05

경험 단절이 빈번하다.

채널 간 이동은 아웃링크, 인앱 브라우저, 인증 반복, 입력 반복으로 끊어진다.
통합채널은 이동보다 업무완결 기준으로 IA를 재정렬해야 한다.

4. Core와 Channel 특화 분리

IA 통합의 기준은 채널명을 없애는 것이 아니라, 공통 처리 모듈과 채널 특화 경험을 분리하는 것이다.

Core vs specialty
ORD

주문·가입·결제

TD 중심이지만 일부 TW에도 존재한다.
계약, 인증, 결제, 배송, 변경은 Core 공통화 대상이다.

전 채널 공통성이 매우 높다.

TM 중심으로 특화성이 높다.

구독

전체 통합보다 선택적 통합과 관리 경험 흡수 기준이 필요하다.

5. 채널별 Pain Point와 공통 구조 이슈

정량 비중과 실제 서비스 체험을 합치면 채널별 문제와 여러 채널에 반복되는 구조 문제가 분리되어 보인다.

Pain point
T월드

IA Depth 과다, 메뉴 구조 복잡, 비표준 UX/UI, 회선 중심 정보 제공으로 통합적 사용자 경험이 저해된다.
추천 구조가 분리되어 경험이 단절되고, 설정 경로와 액션도 비표준화되어 있다.

T다이렉트샵

앱과 웹의 주문·입력 방식이 다르고, 상품·고객 유형별 인증 방식과 UX 방식이 다르다.
고객정보 반복 입력, 가입정보 반복 입력, 가입 단계 과다, 가격 정보 후반 노출이 피로를 만든다.

T멤버십

다수 혜택 브랜드가 일괄 노출되고, 지금 가능한 혜택과 예정 혜택을 직관적으로 구분하기 어렵다.
