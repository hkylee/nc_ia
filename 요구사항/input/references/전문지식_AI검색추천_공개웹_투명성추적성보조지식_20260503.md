# 전문지식 보조자료: AI 검색·추천·데이터 트래킹

작성일: 2026-05-03
자료 성격: 공개웹 보조지식
적용 주제: AI 검색, 추천, 데이터 트래킹 체계, 통합 알림, 설정, 상품 목록, 상품 상세

## 1. 사용 원칙

이 문서는 AI 검색, 추천, 개인화, 데이터 트래킹 정책서 작성 시 참고할 수 있는 국제 공개 가이드의 공통 원칙을 정리한 보조자료다.
NC 첨부 요구사항, 참고자료, 템플릿, 샘플 정책서와 상충하면 첨부 자료를 우선한다.
이 문서는 규제 준수 문서가 아니라 정책서의 판단축을 보강하는 참고자료이므로, 정책 항목은 NC 업무 범위 안에서만 작성한다.

## 2. 공개 출처

- NIST AI Risk Management Framework: https://www.nist.gov/itl/ai-risk-management-framework
- OECD AI Principles: https://www.oecd.org/en/topics/ai-principles.html
- European Centre for Algorithmic Transparency: https://algorithmic-transparency.ec.europa.eu/about_en

## 3. 정책서 작성 시 참고할 업무 관점

AI 검색과 추천은 검색어 처리, 후보 집합 구성, 랭킹, 개인화, 필터링, 노출 제한, 피드백 수집, 로그 저장이 연결된 업무다.
정책서는 모델 구조를 설명하는 문서가 아니므로 알고리즘 상세나 학습 파라미터를 쓰지 않는다.
대신 고객에게 어떤 결과가 왜 노출되는지, 어떤 결과는 제외되는지, 어떤 신호가 추천에 쓰이는지, 고객이 제어할 수 있는 값이 무엇인지, 오류·부적절 결과를 어떻게 처리하는지를 정의한다.

NIST와 OECD의 AI 원칙은 AI 시스템의 위험 식별, 투명성, 책임성, 추적성, 안전성, 생애주기 관리를 강조한다.
ECAT는 검색엔진과 추천 시스템에서 정보의 식별, 분류, 순위화, 제안, 표시가 사용자 의사결정에 영향을 줄 수 있음을 전제로 투명성과 감독 가능성을 다룬다.
NC 정책서에서는 이를 `AI가 어떤 판단을 했는가`가 아니라 `채널이 어떤 기준으로 노출·추천·제한·고지·로그를 관리할 것인가`로 바꿔 써야 한다.

## 4. 후보 판단축

### 검색

- 검색 대상 범위, 제외 대상, 금칙어, 개인정보 포함 검색어 처리, 오탈자·동의어 처리, 무결과 처리 기준을 분리한다.
- 고객 상태, 회선 상태, 상품 판매 상태, 지역, 재고, 연령, 법정대리인 동의 여부에 따라 검색 결과 노출을 제한할 수 있다.
- AI 요약 또는 답변형 검색은 출처, 최신성, 고객 행동 연결, 오류 신고, 상담 전환 기준을 둔다.

### 추천

- 추천 입력 신호는 고객 명시 선택, 이용 이력, 조회 이력, 가입 상태, 혜택 보유, 위치 또는 채널 맥락 등으로 분리한다.
- 민감하거나 고객이 예상하기 어려운 신호는 사용 제한 또는 별도 동의가 필요할 수 있다.
- 추천 결과에는 추천 사유, 제외 기준, 중복 노출 제한, 과도한 광고성 노출 제한, 고객 제어 기준을 둔다.

### 데이터 트래킹

- 수집 이벤트, 수집 목적, 보관 기간, 익명·가명 처리 여부, 원본 로그 접근 권한, 재처리 가능 여부를 분리한다.
- 정책 판단에 필요한 로그와 분석 목적 로그를 구분한다.
- 고객 피드백, 클릭, 전환, 실패, 신고, 상담 전환 이력은 품질 개선과 감사 가능성을 위해 추적 기준을 둔다.

### 운영·검수

- 검색·추천 품질은 정확도만이 아니라 고객 과업 완료, 부적절 결과 차단, 최신성, 설명 가능성, 이의제기 처리로 점검한다.
- 모델 또는 룰 변경 시 영향 범위, 롤백 가능성, 변경 이력, 샘플 검수 기준을 둔다.
- AI 결과가 계약, 과금, 혜택 사용, 가입 제한을 직접 결정하지 않도록 최종 정책 판단은 BSS 또는 확정 정책 기준과 연결한다.

## 5. Agent 사용 제한

이 문서의 AI 원칙을 이유로 정책서 범위를 보안 감사, 모델 카드, 알고리즘 설계서 수준까지 확장하지 않는다.
정책서에는 검색·추천 결과를 고객 업무와 정책 판단에 연결하는 데 필요한 기준만 작성한다.
Inspector는 AI 일반 원칙 위반을 포괄적으로 지적하지 말고, 문서 안의 검색·추천·트래킹 정책 항목이 개발/QA가 판단하기 어려울 때만 보완 요청을 낸다.
