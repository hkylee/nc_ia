# SKT 공개웹 보조지식: 개인정보·동의·보관·파기

작성일: 2026-05-03
자료 성격: 공개웹 보조지식
적용 주제: 회원 가입/탈퇴, 회원정보 조회/변경, 통합 약관, 설정, 통합 알림, 나의 가입 정보, 상담/문의/CS, 데이터 트래킹 체계

## 1. 사용 원칙

이 문서는 SK텔레콤 프라이버시센터와 개인정보 처리방침에서 확인되는 개인정보 처리 원칙을 NC 정책서 작성에 참고하기 위한 보조자료다.
요구사항, 첨부 참고자료, 템플릿, 샘플 정책서와 상충하면 첨부 자료를 우선하고 이 문서의 상충 내용은 사용하지 않는다.
정책서에는 법률 문구를 그대로 옮기지 않고, 고객 과업 수행에 필요한 동의, 고지, 열람, 보관, 파기, 이력 기준으로 재구성한다.

## 3. 정책서 작성 시 참고할 업무 관점

통합채널의 회원·설정·알림·상담 업무는 개인정보 수집, 이용 동의, 제3자 제공, 위탁, 보관, 파기, 고객 권리 행사가 함께 연결된다.
개인정보 정책은 화면 동의 체크박스가 아니라 고객이 어떤 업무를 수행하기 위해 어떤 정보 처리가 필요한지 판단하는 기준이다.
필수 동의와 선택 동의는 업무 허용 여부가 다르므로 같은 정책 항목에 섞지 않는다.
선택 동의 철회는 서비스 이용 자체를 제한하지 않는 것이 원칙이나, 맞춤 혜택, 개인화 추천, 광고성 알림, 위치 기반 혜택 등 일부 기능은 제한될 수 있다.

## 4. 후보 판단축

### 동의

- 필수 동의, 선택 동의, 마케팅 수신 동의, 위치정보 동의, 제3자 제공 동의, 처리 위탁 고지를 분리한다.
- 동의 철회 시 즉시 중단되는 처리와 법령·분쟁·정산 목적상 보관되는 처리를 분리한다.
- 미성년자 또는 법정대리인 관련 업무는 법정대리인 고지·동의·철회 기준을 별도 정책으로 둔다.

### 개인정보 열람·정정·삭제

- 고객이 직접 조회 가능한 정보와 본인확인 후 열람 가능한 정보를 분리한다.
- 정정 가능한 정보와 계약·청구·가입 원장 기준으로 변경 불가한 정보를 구분한다.
- 삭제 요청이 들어와도 법령상 보관, 요금 정산, 분쟁 대응, 부정 이용 방지 목적의 보관 예외를 정의한다.

### 보관·파기·이력

- 회원 탈퇴, 동의 철회, 서비스 해지, 회선 해지, 상담 완료, 거래 완료는 서로 다른 보관 기산점을 가질 수 있다.
- 이용내역, 동의 이력, 알림 발송 이력, 고객 요청 이력은 감사와 분쟁 대응을 위해 별도 보관 기준을 둔다.
- 개인정보가 포함된 로그는 접근 권한, 보관 기간, 마스킹 기준, 파기 기준을 정책 항목으로 분리한다.
