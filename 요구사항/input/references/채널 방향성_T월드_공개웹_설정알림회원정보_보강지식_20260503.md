# T월드 공개웹 보조지식: 설정·알림·회원정보

작성일: 2026-05-03
자료 성격: 공개웹 보조지식
적용 주제: 설정, 통합 알림, 회원정보 조회/변경, 회원 가입/탈퇴, 나의 가입 정보, 통합 약관

## 1. 사용 원칙

이 문서는 NC 정책서 작성 시 공개웹에서 확인되는 T월드 업무 단서를 보조적으로 활용하기 위한 자료다.
요구사항, 첨부 참고자료, 템플릿, 샘플 정책서와 상충하면 첨부 자료를 우선하고 이 문서의 상충 내용은 사용하지 않는다.
공개웹 내용은 정책값을 확정하는 근거가 아니라 정책 판단축 후보를 넓히는 참고자료로만 사용한다.

## 3. 정책서 작성 시 참고할 업무 관점

T월드 계정과 회원정보 업무는 로그인 계정, 서비스 약관 동의, 회선 권한, 개인정보 제공 동의, 고객 고지 수단이 분리되어야 한다.
NC 정책서에서는 계정 자체의 가입 상태와 특정 회선에 대한 조회·변경 권한을 같은 것으로 보지 않는다.
회원정보 변경이나 알림 설정은 고객이 직접 변경하는 과업처럼 보이지만, BSS 또는 회원계정 시스템의 권한 판정과 이력 저장이 함께 필요하다.

알림 정책은 단순 Push 수신 여부만으로 정의하지 않는다.
알림 유형, 법정 고지 여부, 고객 선택 가능 여부, 수신 채널, 재발송 가능 여부, 미성년·법정대리인 고지 여부, 실패 시 대체 안내 기준을 나누어야 한다.
요금안내서처럼 법정대리인 동시 통보가 필요한 업무는 고객 본인 알림과 대리인 알림의 적용 조건, 종료 조건, 변경 이력을 함께 정의해야 한다.

모바일 전자증명서 제출 안내는 서류 제출, 본인확인, 대리권 확인, 상담 연계 흐름에서 참고할 수 있다.
다만 정책서에는 제출 서류 전체 목록이나 UI 절차를 과도하게 쓰지 않고, 권한 확인 및 후속 처리 기준만 반영한다.

## 4. 주제별 후보 판단축

### 설정

- 고객이 직접 변경 가능한 설정과 시스템·운영자 승인 후 변경되는 설정을 구분한다.
- 설정 변경은 즉시 반영, 다음 청구 주기 반영, 운영 검토 후 반영 중 하나로 상태를 나눈다.
- 설정 변경 전후 이력은 고객 안내, 감사 로그, 롤백 가능 여부 판단에 활용한다.
- 서비스 필수 고지, 약관 고지, 과금 관련 고지는 고객이 임의로 차단할 수 없는 예외 알림으로 분리한다.

### 통합 알림

- 알림은 정보성, 혜택성, 거래성, 법정 고지성, 운영 장애성으로 분류한다.
- 고객 선택 가능 알림과 필수 발송 알림을 분리한다.
- Push, SMS/MMS, 이메일, 앱 내 알림, 카카오/외부 채널 등 채널별 발송 우선순위와 실패 대체 기준을 둔다.
- 미성년, 법정대리인, 법인 담당자처럼 수신 주체가 여러 명인 경우 주 수신자와 보조 수신자의 기준을 분리한다.

### 회원정보 조회/변경

- 조회 가능 정보, 수정 가능 정보, 본인확인 필요 정보, 상담 전환 필요 정보를 분리한다.
- 연락처, 이메일, 주소, 마케팅 수신 동의는 변경 이력과 최종 변경 시점을 저장한다.
- 회선 명의자와 계정 사용자가 다를 수 있으므로 회선 정보 변경은 계정 정보 변경과 별도 권한 판정을 둔다.

### 회원 가입/탈퇴

- 가입은 계정 생성, 약관 동의, 본인확인, 서비스 연결, 회선 권한 확인을 분리해 정의한다.
- 탈퇴는 계정 탈퇴, 서비스 이용 해지, 개인정보 보관·분리 보관, 미완료 거래 처리 기준을 분리한다.
- 탈퇴 후에도 법령상 보관, 청구·분쟁 대응, 이용내역 보존이 필요한 정보는 삭제 대상에서 제외할 수 있다.

## 5. Agent 사용 제한

이 문서의 항목을 그대로 정책 항목으로 확정하지 않는다.
정책서 작성 Agent는 요구사항과 첨부 참고자료에서 확인된 업무 범위 안에서만 이 후보 판단축을 사용한다.
Inspector는 공개웹 보조지식과 다른 내용이 있다는 이유만으로 오류로 판단하지 말고, 첨부 자료 또는 승인된 블루프린트와 충돌할 때만 오류로 판단한다.
