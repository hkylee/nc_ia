# 34개 정책서 작성 지향점

본 문서는 34개 정책서 작성 시 각 문서가 향해야 할 기준점을 정리한 내부 작성 이정표다.
요구사항, 템플릿, 샘플, 채널 방향성 PDF, TK 과제정의 PDF를 우선 근거로 삼되, 아래 문구는 정책값이 아니라 작성 방향과 누락 점검 기준으로 사용한다.

## 1. 가이드라인/공통/품질/적응형
- 지향점) 통합채널 전반에 적용되는 설계 원칙, 품질 기준, 공통 UX·정책·컴포넌트 기준을 하나의 상위 기준으로 정립한다.
- PI 방향) 채널별로 다른 설계 기준, 품질 기준, 용어, 화면 구성 원칙을 공통화하고 정책서·요구사항·화면 설계 간 불일치를 줄인다.
- 고객 경험 혁신) 고객이 어느 업무를 이용하더라도 동일한 구조, 용어, 안내, 품질을 경험하게 하고, 고객 상황에 따라 필요한 정보만 적응형으로 제공한다.

## 2. 전시/관리 기능
- 지향점) 홈·탭·배너·전시 영역을 서비스 묶음이 아니라 고객 목적과 업무 진입 기준으로 재구성한다.
- PI 방향) 채널별 전시 운영, 배너 관리, 노출 우선순위, 운영 권한을 표준화하고 중복 전시·수작업 편성을 줄인다.
- 고객 경험 혁신) 고객이 첫 화면에서 “내가 할 일”을 바로 찾고, 자주 쓰는 업무와 필요한 혜택을 과도한 탐색 없이 확인하게 한다.

## 3. 상품 목록
- 지향점) 상품 탐색, 필터, 정렬, 비교, 노출 제외 기준을 통합해 고객이 가입 가능한 상품을 쉽게 찾게 한다.
- PI 방향) 채널별 상품 목록 구조와 필터 기준을 공통화하고, 상품 유형별 중복 목록과 상이한 정렬 기준을 줄인다.
- 고객 경험 혁신) 고객이 본인 조건에 맞는 상품을 우선 확인하고, 가격·혜택·가입 조건을 한 화면에서 비교하게 한다.

## 4. 외부 BP 서비스 관리 체계
- 지향점) 외부 제휴 서비스를 통합채널 안에서 일관된 UX와 책임 경계로 제공하되, 운영 책임과 장애 대응 기준을 명확히 한다.
- PI 방향) 외부 BP별 연동, 진입, 종료, 장애, 고객문의 처리 기준을 표준화하고 제휴사별 예외 운영을 줄인다.
- 고객 경험 혁신) 고객이 외부 서비스로 이동해도 이탈감 없이 이용하고, 문제 발생 시 어느 주체가 처리하는지 명확히 알게 한다.

## 5. AI 검색
- 지향점) 검색을 단순 정보 조회가 아니라 고객 의도를 파악해 비교·요약·업무 실행까지 연결하는 통합 진입점으로 전환한다.
- PI 방향) 메뉴 검색, 상품 검색, FAQ, 상담 탐색, 업무 진입을 분리 운영하지 않고 자연어 의도 기반 단일 프로세스로 통합한다.
- 고객 경험 혁신) 고객이 상품/메뉴명을 몰라도 원하는 상품/업무를 말로 찾고, 검색 결과에서 바로 신청·변경·납부·해지 등 처리 흐름으로 이동하게 한다.

## 6. 추천
- 지향점) 추천을 단순 노출·판매가 아니라 고객 맥락, 보유 상품, 이용 상황, 제한 조건을 반영한 납득 가능한 제안 체계로 설계한다.
- PI 방향) 추천 대상, 제외 조건, 우선순위, 개인화 로직, 성과 측정 기준을 표준화하고 채널별 임의 추천 운영을 줄인다.
- 고객 경험 혁신) 고객이 추천 이유와 적용 조건을 이해하고, 원치 않는 추천은 통제할 수 있게 한다.

## 7. 데이터 트래킹 체계
- 지향점) 고객 행동, 업무 상태, 전환, 실패, 이탈 데이터를 표준화해 채널 개선의 근거 체계를 만든다.
- PI 방향) 화면·이벤트·업무·상태·오류 로그 ID를 공통화하고, 채널별로 다른 분석 기준을 하나의 추적 체계로 통합한다.
- 고객 경험 혁신) 고객이 어디서 막히는지, 어떤 안내가 부족한지, 어떤 업무가 상담으로 이탈하는지를 데이터로 파악해 경험을 지속 개선한다.

## 8. 이벤트/미션 프로그램
- 지향점) 이벤트와 미션을 단기 프로모션이 아니라 채널 재방문, 혜택 사용, 서비스 전환을 연결하는 참여 프로그램으로 설계한다.
- PI 방향) 이벤트 생성, 참여 조건, 보상 지급, 당첨, 중복 참여, 부정 이용, 회수 기준을 공통화한다.
- 고객 경험 혁신) 고객이 참여 조건과 보상 결과를 명확히 알고, 미션 참여가 실제 혜택·서비스 이용으로 자연스럽게 이어지게 한다.

## 9. 외부 쿠폰
- 지향점) 외부 제휴 쿠폰의 발급·등록·사용·취소·만료·복원 상태를 표준화한다.
- PI 방향) 제휴사별 쿠폰 상태, 사용 결과, 연동 실패, 복원 기준을 공통화하고 수작업 정산·예외 처리를 줄인다.
- 고객 경험 혁신) 고객이 쿠폰 사용 가능 여부, 사용처, 만료일, 실패 사유를 즉시 이해하고 사용할 수 있게 한다.

## 10. 멤버십 혜택/T 플러스포인트
- 지향점) 멤버십 혜택과 포인트를 고객이 받을 수 있는 대표 혜택 자산으로 통합 관리한다.
- PI 방향) 멤버십 등급, 혜택, 포인트 적립·사용·소멸, 제휴 혜택 기준을 표준화하고 채널별 혜택 운영을 줄인다.
- 고객 경험 혁신) 고객이 지금 쓸 수 있는 혜택, 등급 기준, 포인트 사용·소멸 조건을 쉽게 이해하고 즉시 사용하게 한다.

## 11. 상품상세/담기
- 지향점) 상품 상세에서 고객이 가입 가능 여부, 가격, 혜택, 옵션, 다음 행동을 명확히 판단하게 한다.
- PI 방향) 상품 상세 정보, 옵션 선택, 담기 조건, 담기 후 상태, 가격·혜택 표시 기준을 공통화한다.
- 고객 경험 혁신) 고객이 상세 화면에서 상품을 이해하고, 담기 후 주문·비교·상담 등 다음 행동을 끊김 없이 수행하게 한다.

## 12. 카트/장바구니
- 지향점) 주문 전 상품 구성, 옵션, 수량, 가격, 혜택을 검토하고 안정적으로 주문으로 전환하는 공간으로 설계한다.
- PI 방향) 상품 조합 가능 여부, 옵션 변경, 유효기간, 가격 변경, 품절·삭제 기준을 표준화한다.
- 고객 경험 혁신) 고객이 주문 전 최종 구성과 비용 변화를 예측하고, 오류나 품절이 발생해도 대안을 쉽게 확인하게 한다.

## 13. 할인/시뮬레이션
- 지향점) 고객이 가입·변경·구매 전에 할인과 예상 청구액을 스스로 예측할 수 있게 한다.
- PI 방향) 할인 산식, 적용 순서, 중복 할인, 시뮬레이션 기준, 실제 적용 차이 안내 기준을 표준화한다.
- 고객 경험 혁신) 고객이 “바꾸면 얼마가 되는지”, “어떤 할인이 사라지는지”를 사전에 확인해 손해 불안을 줄인다.

## 14. 주문/계약/가입
- 지향점) 주문 생성부터 계약 확정, 가입 신청, 개통 전 상태까지 하나의 표준 프로세스로 정의한다.
- PI 방향) 주문, 계약, 가입 신청, 신청서, 약관, BSS 반영 전후 상태를 표준화하고 채널별 주문 흐름 차이를 줄인다.
- 고객 경험 혁신) 고객이 신청 후 현재 상태, 남은 절차, 실패 사유, 다음 행동을 명확히 알고 가입을 완료하게 한다.

## 15. 선물주문
- 지향점) 발신자·수신자·상품·결제·수락·취소·만료·환불을 포함한 선물 업무 전체 상태를 명확히 정의한다.
- PI 방향) 상품 유형별 선물 주문, 수신자 수락, 미수락 만료, 취소·환불, 알림·이력 기준을 표준화한다.
- 고객 경험 혁신) 고객이 선물 가능 여부, 수신자 상태, 취소 가능 시점, 만료 조건을 명확히 알고 안심하고 선물하게 한다.

## 16. 상품변경
- 지향점) 요금제·부가서비스·구독 등 변경 업무의 가능 조건, 예약, 반영 시점, 영향 범위를 표준화한다.
- PI 방향) 상품 유형별 변경 가능 조건, 제한, 예약 변경, 변경 후 영향, 원복 기준을 통합 관리한다.
- 고객 경험 혁신) 고객이 언제 변경 가능한지, 변경하면 요금·혜택·약정·부가서비스가 어떻게 달라지는지 사전에 이해하게 한다.

## 17. 결제
- 지향점) 결제 처리, 결제수단, 할인 적용, 포인트 사용, 실패, 취소, 환불을 하나의 결제 정책으로 통합한다.
- PI 방향) 결제 승인, 실패, 재시도, 할인·포인트 적용, 취소·환불·복원, BSS 반영 기준을 표준화한다.
- 고객 경험 혁신) 고객이 최종 결제 금액, 실패 사유, 환불 시점, 포인트 복원 결과를 명확히 확인하게 한다.

## 18. 주문 상태/사후 관리
- 지향점) 주문 이후 배송, 교환, 반품, 상품 연동, 인증, 상태 조회를 하나의 사후 관리 흐름으로 통합한다.
- PI 방향) 주문 상태값, 배송·교환·반품·인증·상품 연동 상태를 표준화하고 상태별 후속 처리를 일관화한다.
- 고객 경험 혁신) 고객이 주문 후 현재 상태와 필요한 조치를 직접 확인하고, 보류·실패·반품 상황에서도 다음 행동을 알게 한다.

## 19. 해지/환불/취소
- 지향점) 취소, 해지, 환불, 철회, 회수 기준을 상태와 시점 중심으로 명확히 정의한다.
- PI 방향) 업무별로 분산된 취소·해지·환불 기준을 통합하고 환불 산식, 혜택 회수, 위약금, 이력 기준을 표준화한다.
- 고객 경험 혁신) 고객이 취소 가능 여부, 환불 금액, 처리 완료 시점, 손실·제한 조건을 사전에 예측하게 한다.

## 20. 나의 가입 정보
- 지향점) 고객이 현재 가입 상품, 약정, 결합, 부가서비스, 이용 상태를 한눈에 이해하는 통합 조회 체계를 만든다.
- PI 방향) 상품·약정·결합·부가서비스 정보를 분산 조회하지 않고 고객 단위로 통합 조회하도록 구조화한다.
- 고객 경험 혁신) 고객이 “내가 무엇에 가입돼 있고, 무엇이 연결돼 있는지”를 쉽게 이해하고 변경·해지·상담으로 연결하게 한다.

## 21. 회선 변경/관리
- 지향점) 대표회선, 가족회선, 회선 권한, 대리 처리, 회선 추가·삭제를 통합 관리한다.
- PI 방향) 회선 단위 권한, 고객 단위 권한, 가족·대리 처리 기준을 표준화하고 반복 인증·개별 처리를 줄인다.
- 고객 경험 혁신) 고객이 본인·가족 회선을 한 번에 보고, 권한 범위 안에서 직접 관리하게 한다.

## 22. 멤버십 카드 관리
- 지향점) 멤버십 카드 발급, 바코드, 재발급, 정지, 사용 제한을 실시간 사용 중심으로 정의한다.
- PI 방향) 카드 상태, 바코드 노출, 재발급, 오류, 정지·해제 기준을 표준화한다.
- 고객 경험 혁신) 고객이 오프라인 현장에서 즉시 멤버십을 사용하고, 오류 발생 시 빠르게 복구하게 한다.

## 23. 청구 및 수납 관리
- 지향점) 청구서, 납부, 납부수단, 납부내역, 실시간 요금, 포인트 수납을 하나의 요금 관리 흐름으로 통합한다.
- PI 방향) 청구→납부→수납확인→미납·이중납부 처리 기준을 표준화하고, 회선·가족·유무선 분산 납부 관리를 줄인다.
- 고객 경험 혁신) 고객이 요금 발생 이유와 납부 상태를 명확히 이해하고, 미납·이중납부·납부 실패 불안을 줄이게 한다.

## 24. 나의 데이터·통화
- 지향점) 데이터·통화 사용량 조회, 설정, 선물, 충전, 리필을 사용량 기반 즉시 조치 흐름으로 통합한다.
- PI 방향) 잔여량 조회, 소진 알림, 충전·선물·리필·설정 변경 프로세스를 하나의 흐름으로 연결한다.
