1. 개요

가. 범위

본 정책서는 통합채널에서 제공하는 AI 검색 업무에 대한 처리 기준을 정의한다.
AI 검색은 고객이 자연어, 키워드, 음성 또는 추천 질의를 통해 원하는 정보, 상품, 혜택, 업무 처리 경로를 탐색하고, 필요한 경우 후속 업무로 진입할 수 있도록 지원하는 기능이다.

대상 채널은 앱·웹 FO를 기준으로 한다.
대상 고객은 로그인 고객, 비로그인 고객, SKT 고객, 비SKT 고객을 포함한다.
다만 고객 상태, 인증 여부, 보유 회선, 가입 상품, 약관 동의 여부에 따라 검색 결과의 노출 범위와 후속 처리 가능 범위는 다르게 적용한다.

본 문서는 AI 검색의 유즈케이스, 상태 전이, 프로세스, 주요 기능, 정책 목록, 정책 상세를 정의한다.
AI 검색 결과의 화면 UI 상세, 검색 모델 상세 알고리즘, LLM 프롬프트 상세, 검색 인덱스 설계, API 필드, DB 컬럼, 운영자 관리 화면 상세는 본 문서에서 다루지 않는다.
다만 검색 결과 노출 기준, 개인화 적용 기준, 업무 진입 기준, 제한 업무 기준, 신뢰도 부족 시 처리 기준, 고객 데이터 활용 기준처럼 정책 판단이 필요한 항목은 본 문서에 포함한다.

AI 검색은 단순 검색창 기능이 아니라 통합채널의 목적 기반 탐색 진입점이다.
따라서 AI 검색은 고객이 “어디서 해야 하는지”를 찾는 단계를 줄이고, 고객 의도를 이해하여 적절한 정보·기능·업무 처리 경로로 연결하는 것을 목표로 한다.

나. 설계 원칙

• 목적 기반 탐색: AI 검색은 메뉴명이나 기존 채널 구조가 아니라 고객이 해결하려는 목적을 기준으로 검색 결과를 제공한다.

• 업무 연결 우선: 단순 안내로 끝내지 않고, 고객이 처리 가능한 업무는 신청·변경·조회·해지 등 후속 프로세스로 바로 연결한다.

• 개인화 제한 적용: 고객 정보 기반 검색 결과는 로그인, 동의, 권한, 보안 기준을 충족한 경우에만 제공한다.

• 신뢰도 기반 응답 제어: 검색 의도나 답변 신뢰도가 낮은 경우에는 확정 답변을 제공하지 않고, 재질의 유도, 후보 결과 제시, 상담 또는 메뉴 탐색으로 전환한다.

• 정책·고위험 업무 보호: 요금제 변경, 해지, 결제, 개인정보 변경, 인증수단 변경 등 고객 피해 가능성이 있는 업무는 AI 검색에서 바로 실행하지 않고, 별도 확인·인증·고지 절차를 거친다.

• BSS·채널 통합 조회: AI 검색 결과는 채널 콘텐츠만 기준으로 구성하지 않고, 고객 상태, 가입 상품, 청구, 혜택, 약정, 구독, 멤버십 등 BSS 및 연계 시스템의 주요 정보를 함께 고려한다.

따라서 검색 방식, 고객 상태, 결과 유형, 신뢰도, 데이터 활용, 후속 업무 연결 기준을 함께 정의한다.

3. 유즈케이스 정의

가. 액터

본 장은 AI 검색 업무에 참여하는 핵심 액터만 정의한다.
액터는 고객 업무를 시작하거나, 검색 결과 생성·후속 업무 처리·운영 관리에 독립적인 책임을 갖는 주체로 한정한다.
로그인 여부는 액터가 아니라 고객 상태와 권한 조건으로 관리한다.
지식 베이스, 채널 업무 시스템, 외부 연계 시스템, 상담 시스템은 개별 액터로 분리하지 않고 연계 시스템에 포함한다.

나. 유즈케이스

본 장은 AI 검색에서 각 액터가 수행하는 유즈케이스를 정의한다.
프로세스 정의 대상은 액터가 사람인지 시스템인지에 따라 1차 판단한다.
고객, 운영자, 법정대리인, 대리인처럼 사람이 수행하는 유즈케이스는 별도 흐름과 판단 기준이 필요하므로 원칙적으로 프로세스 정의 대상을 Y로 지정한다.
AI 검색 시스템과 연계 시스템처럼 시스템이 수행하는 검증·생성·조회·저장 처리는 사람 유즈케이스를 지원하는 보조 유즈케이스로 보고 원칙적으로 N으로 지정한다.

검색 진입 전 고객 질의 입력 질의 접수 검색 질의를 접수하고 검색 세션을 생성한다.
음성 입력은 텍스트 변환 후 질의로 접수한다.
질의 접수 질의 유효성 확인 완료 의도 분석 중 빈 질의, 입력 불가 문자, 금칙어 포함 여부를 확인한다.
금칙어 또는 위험 질의는 제한 업무 안내 또는 상담 전환으로 분기한다.
의도 분석 중 검색 의도 명확 결과 생성 중 정보 탐색, 상품 비교, 업무 처리, 오류 해결, 상담 연결 등 의도 유형을 확정하고 결과 생성을 시작한다.
의도 분석 중 검색 의도 불명확 재질의 필요 확정 답변을 제공하지 않고 추천 질의, 조건 선택, 추가 입력을 유도한다.
결과 생성 중 신뢰도 기준 충족 결과 제공 AI 요약 답변, 검색 결과 카드, 출처, 추천 질의, 후속 업무 경로를 제공한다.
결과 생성 중 검색 결과 없음 무결과 유사 검색어, 관련 메뉴, 상담 전환 경로를 제공한다.
무결과 로그를 저장한다.
결과 생성 중 답변 근거 부족 저신뢰 결과 확정형 답변은 제한하고 후보 결과 또는 재질의 유도 중심으로 제공한다.
결과 제공 고객 결과 선택 후속 업무 연결 선택한 결과가 업무 처리형이면 해당 후속 업무 프로세스로 이동한다.
후속 업무 진입 전 필요한 인증, 동의, 고지를 확인한다.
결과 제공 고객 재질의 질의 접수 기존 검색 세션과 이전 질의 컨텍스트를 유지한 상태로 추가 질의를 접수한다.

나. 정책 상세

작성 기준
정책 상세는 실제 의사결정값을 확정하는 장이다.
AI 검색은 고객에게 답변과 업무 경로를 직접 제공하므로, 답변 가능 범위와 제한 기준을 명확히 작성해야 한다.
정책값이 확정되지 않은 항목은 TBD로 두되, 결정 주체와 결정 필요 사유를 함께 남긴다.

1) 검색 입력 정책 (PG-AIS-SCH-001)

• 허용 입력 방식 (PI-AIS-SCH-001-01)
- AI 검색은 자연어, 키워드, 음성 입력, 추천 질의 선택을 허용한다.
음성 입력은 텍스트 변환 후 검색 질의로 처리한다.
• 입력 제한 기준 (PI-AIS-SCH-001-02)
- 빈 질의, 공백만 있는 질의, 시스템이 처리할 수 없는 특수문자 중심 질의는 검색을 실행하지 않고 재입력을 안내한다.
최대 입력 길이는 출시 기준 500자로 제한한다.
운영 기준에 따라 조정할 수 있다.
• 음성 검색 처리 기준 (PI-AIS-SCH-001-03)
- 음성 인식 결과는 고객에게 텍스트로 표시한 뒤 검색을 수행한다.
음성 변환에 실패하면 음성 재입력 또는 텍스트 입력을 안내한다.
• 비정상 입력 처리 기준 (PI-AIS-SCH-001-04)
- 반복 입력, 자동화 의심 입력, 과도한 단시간 검색 요청은 보안 정책에 따라 제한할 수 있다.
제한 시 고객에게 일시적 이용 제한 사유를 안내한다.

2) 금칙어·위험 질의 정책 (PG-AIS-SCH-002)

• 금칙어 유형 (PI-AIS-SCH-002-01)
- 욕설, 혐오 표현, 성적 표현, 불법 행위 요청, 개인정보 탈취 유도, 보안 우회, 시스템 공격, 내부 정보 요구는 금칙어 또는 제한 질의로 분류한다.
• 위험 질의 유형 (PI-AIS-SCH-002-02)
- 타인 명의 정보 조회, 인증 우회, 결제 우회, 계정 탈취, 시스템 프롬프트 요구, 내부 운영 기준 노출 요구는 위험 질의로 분류한다.
• 제한 응답 기준 (PI-AIS-SCH-002-03)
- 금칙어 또는 위험 질의는 검색 결과를 제공하지 않는다.
고객이 정상 업무를 수행하려는 것으로 해석 가능한 경우에는 안전한 공식 업무 경로를 안내한다.
• 상담·신고 전환 기준 (PI-AIS-SCH-002-04)
- 보안 위협, 반복적 악성 입력, 개인정보 침해 가능성이 있는 질의는 운영 모니터링 대상으로 저장하고 필요 시 상담 또는 보안 대응 프로세스로 전환한다.

3) 검색 의도 분류 정책 (PG-AIS-INT-001)

• 의도 유형 (PI-AIS-INT-001-01)
- 검색 의도는 정보 확인, 상품·혜택 탐색, 상품 비교, 업무 조회, 업무 신청·변경·해지, 오류 해결, 상담 연결, 기타로 분류한다.
• 업무 도메인 분류 (PI-AIS-INT-001-02)
- 업무 도메인은 요금제, 청구·납부, 부가서비스, 구독, 멤버십, 데이터, 로밍, 결합, 주문·가입, 해지, 인증·개인정보, 고객지원으로 분류한다.
• 모호 질의 처리 (PI-AIS-INT-001-03)
- 고객 질의가 둘 이상의 의도로 해석되거나 업무 도메인이 불명확하면 확정 답변을 제공하지 않고 추천 질의 또는 조건 선택지를 제공한다.
• 의도 재분류 기준 (PI-AIS-INT-001-04)
- 고객이 재질의하거나 조건을 선택하면 기존 검색 세션의 컨텍스트를 유지한 상태로 의도를 재분류한다.

4) 검색 결과 구성 정책 (PG-AIS-RES-001)

• 결과 유형 (PI-AIS-RES-001-01)

• 결과 우선순위 (PI-AIS-RES-001-02)
- 고객 질의가 업무 처리 의도이면 후속 업무 액션을 우선 노출한다.
정보 확인 의도이면 공식 콘텐츠와 FAQ를 우선 노출한다.
상품 비교 의도이면 비교 가능한 상품·혜택 정보를 우선 노출한다.
• AI 요약 답변 제공 기준 (PI-AIS-RES-001-03)
- AI 요약 답변은 출처가 확인되고 답변 신뢰도 기준을 충족하는 경우에만 제공한다.
요금, 약정, 할인, 해지, 납부처럼 고객 피해 가능성이 있는 내용은 확정형 표현을 제한하고 기준 확인 또는 후속 업무 경로를 함께 제공한다.
• 검색 결과 카드 구성 기준 (PI-AIS-RES-001-04)
- 검색 결과 카드는 제목, 요약, 출처, 주요 액션, 후속 경로를 기본 구성으로 한다.
업무 처리형 결과에는 처리 가능 여부 확인 또는 인증 필요 여부를 함께 표시한다.

• 출처 표시 항목 (PI-AIS-RES-002-02)
- 출처는 콘텐츠명, 기준일 또는 업데이트일, 상세 보기 경로를 포함한다.
BSS 조회 결과를 활용한 경우에는 고객별 조회 결과임을 표시한다.
• 출처 없음 처리 (PI-AIS-RES-002-03)
- 출처가 없는 답변은 확정 답변으로 제공하지 않는다.
관련 메뉴, 상담 연결, 재질의 유도 중 하나로 처리한다.
• 기준일 표시 기준 (PI-AIS-RES-002-04)
- 요금, 혜택, 약정, 이벤트, 공지, 장애, 운영시간처럼 변경 가능성이 높은 정보는 기준일 또는 조회 시점을 표시한다.

• 답변 신뢰도 기준 (PI-AIS-REL-001-02)
- 답변 신뢰도는 출처 유무, 출처 최신성, 고객 상태 일치 여부, 정책 기준 충족 여부로 판단한다.
출처가 없거나 고객 상태와 불일치하면 확정 답변을 제공하지 않는다.
• 저신뢰 결과 처리 (PI-AIS-REL-001-03)
- 저신뢰 결과는 확정형 답변을 제공하지 않고 후보 결과, 재질의, 조건 선택, 상담 연결 중 하나로 처리한다.
• 무결과 처리 (PI-AIS-REL-001-04)
- 무결과 상태에서는 유사 검색어, 추천 질의, 관련 메뉴, 상담 연결을 제공한다.
무결과 질의는 운영 검토 대상으로 저장한다.

7) 개인화 적용 정책 (PG-AIS-PRS-001)

• 개인화 적용 조건 (PI-AIS-PRS-001-01)
- 개인화 검색은 고객이 로그인했고, 필요한 고객 정보 활용 동의와 권한 기준을 충족한 경우에만 적용한다.
• 비로그인 결과 기준 (PI-AIS-PRS-001-02)
- 비로그인 상태에서는 공통 콘텐츠, 공통 메뉴, 공통 상품·혜택, 일반 안내만 제공한다.
고객별 요금, 청구, 약정, 가입 상품, 회선 상태, 혜택 보유 여부는 제공하지 않는다.
• 동의 필요 기준 (PI-AIS-PRS-001-03)
- 고객 정보, 이용 이력, 위치, 최근 검색 이력, 보유 상품 정보를 검색 결과 조정에 활용하려면 관련 동의 기준을 충족해야 한다.
• 개인화 제외 기준 (PI-AIS-PRS-001-04)
- 동의가 없거나, 고객 상태가 불명확하거나, 대리 처리 권한이 확인되지 않은 경우 개인화 결과를 제공하지 않고 비개인화 결과로 전환한다.

8) 권한 기반 노출 정책 (PG-AIS-AUT-001)

• 권한 확인 기준 (PI-AIS-AUT-001-01)
- 고객별 후속 업무 액션은 명의, 회선 상태, 인증 상태, 법정대리인 여부, 대리 권한 기준을 확인한 뒤 노출한다.
• 명의·회선 기준 (PI-AIS-AUT-001-02)
- 본인 명의 회선은 해당 업무 정책 기준에 따라 후속 업무 액션을 제공한다.
타인 명의 또는 가족 회선은 대리 권한이 확인된 경우에만 처리형 액션을 제공한다.
• 미성년자 기준 (PI-AIS-AUT-001-03)
- 미성년자 관련 업무는 법정대리인 동의 또는 인증이 필요한 경우 직접 처리 액션을 제한하고, 해당 업무의 법정대리인 처리 기준을 따른다.
• 대리 처리 기준 (PI-AIS-AUT-001-04)
- 대리 처리가 가능한 업무만 검색 결과에서 대리 처리 경로를 제공한다.
대리 처리 가능 여부가 불명확하면 상담 또는 안내 경로로 전환한다.

9) 후속 업무 연결 정책 (PG-AIS-ACT-001)

• 연결 가능 업무 (PI-AIS-ACT-001-01)
- 조회, 신청, 변경, 해지, 납부, 인증, 상담 업무는 각 업무 정책서에서 허용한 조건을 충족한 경우 AI 검색 결과에서 후속 업무로 연결할 수 있다.
• 연결 제한 업무 (PI-AIS-ACT-001-02)
- 본인확인, 법정대리인 동의, 서류 확인, 상담원 확인이 필수인 업무는 AI 검색에서 즉시 실행하지 않고 확인 절차 또는 상담 경로로 연결한다.
• 후속 업무 진입 조건 (PI-AIS-ACT-001-03)
- 후속 업무 진입 전 고객 상태, 업무 가능 여부, 인증 필요 여부, 동의 필요 여부, 영향도 고지 필요 여부를 확인한다.
• 세션 전달 기준 (PI-AIS-ACT-001-04)
- 후속 업무로 이동할 때 검색 질의, 선택 결과, 업무 유형, 고객 상태 확인 결과를 세션 정보로 전달할 수 있다.
민감정보는 전달하지 않는다.

10) 고위험 업무 보호 정책 (PG-AIS-RSK-001)

• 고위험 업무 유형 (PI-AIS-RSK-001-01)
- 요금제 변경, 회선 해지, 번호 해지, 자동납부 변경, 결제·납부, 개인정보 변경, 인증수단 변경, 약정 변경, 구독 해지, 부가서비스 해지는 고위험 업무로 관리한다.
• 직접 실행 제한 (PI-AIS-RSK-001-02)
- 고위험 업무는 AI 검색 결과에서 즉시 실행하지 않는다.
업무 상세 화면 또는 해당 업무 프로세스로 이동한 뒤 인증, 영향도 고지, 최종 확인을 거쳐 처리한다.
• 추가 확인 기준 (PI-AIS-RSK-001-03)
- 요금, 혜택, 약정, 청구, 구독, 결합, 멤버십 등 고객 영향이 있는 업무는 변경 전 예상 영향도를 제공하고 고객 확인을 받아야 한다.
• 상담 전환 기준 (PI-AIS-RSK-001-04)
- 고객 상태가 불명확하거나, 정책상 셀프 처리가 불가하거나, 법적·보안 리스크가 있는 경우 상담 전환으로 처리한다.

11) 인증·동의·고지 정책 (PG-AIS-CFM-001)

• 본인인증 기준 (PI-AIS-CFM-001-01)
- AI 검색 자체는 본인인증 없이 이용할 수 있다.
다만 고객별 정보 조회 또는 후속 업무 처리가 필요한 경우 해당 업무의 인증 기준을 따른다.
• 동의 필요 기준 (PI-AIS-CFM-001-02)
- 고객 정보 활용, 개인화 결과 제공, 위치 기반 검색, 마케팅성 추천, 제휴 혜택 추천은 관련 동의 상태를 확인한 뒤 제공한다.
• 영향도 고지 기준 (PI-AIS-CFM-001-03)
- 고객 비용, 혜택, 약정, 서비스 이용 가능 여부가 바뀌는 업무는 후속 업무 진입 전 영향도를 고지한다.
• 최종 확인 기준 (PI-AIS-CFM-001-04)
- 신청, 변경, 해지, 결제, 개인정보 변경은 AI 검색 결과에서 바로 확정하지 않고 해당 업무 화면에서 고객의 최종 확인을 받아야 한다.

12) 재질의 및 결과 보정 정책 (PG-AIS-REF-001)

• 재질의 유도 기준 (PI-AIS-REF-001-01)
- 검색 의도가 모호하거나, 답변 근거가 부족하거나, 조건이 누락된 경우 재질의를 유도한다.
• 보정 조건 제공 기준 (PI-AIS-REF-001-02)
- 보정 조건은 업무 도메인, 고객 상태, 상품 유형, 처리 목적, 기간, 지역, 혜택 유형 등을 기준으로 제공한다.
• 추천 질의 기준 (PI-AIS-REF-001-03)
- 추천 질의는 고객의 현재 질의와 관련 있는 범위에서 제공한다.
고객이 동의하지 않은 개인정보 또는 민감정보를 근거로 추천 질의를 생성하지 않는다.
• 검색 컨텍스트 유지 기준 (PI-AIS-REF-001-04)
- 동일 검색 세션 안에서는 이전 질의와 선택 조건을 유지할 수 있다.
검색 세션 종료 후에는 검색 이력 활용 동의와 보관 기준에 따라 처리한다.

13) 검색 이력·로그 정책 (PG-AIS-LOG-001)

• 저장 대상 (PI-AIS-LOG-001-01)
- 검색 질의, 검색 의도, 결과 유형, 결과 선택, 후속 업무 진입, 무결과, 저신뢰 결과, 피드백, 오류 정보를 저장한다.
• 활용 목적 (PI-AIS-LOG-001-02)
- 검색 이력과 로그는 검색 품질 개선, 오류 분석, 운영 모니터링, 통계 산출, 고객 불편 개선에 한해 활용한다.
• 보관 기간 (PI-AIS-LOG-001-03)
- 고객 식별이 가능한 검색 이력의 보관 기간은 개인정보 보관 정책을 따른다.
운영 통계용 로그는 비식별 처리 후 별도 보관할 수 있다.
정확한 보관 기간은 개인정보 정책과 법무 검토 기준으로 확정한다.
• 삭제 기준 (PI-AIS-LOG-001-04)
- 고객이 검색 이력 삭제를 요청하거나 보관 기간이 만료된 경우 식별 가능한 검색 이력을 삭제한다.
법적 보관 대상은 관련 기준을 따른다.

14) 개인정보 보호 정책 (PG-AIS-PRV-001)

• 민감정보 제한 (PI-AIS-PRV-001-01)
- 주민등록번호, 계좌번호, 카드번호 전체, 인증번호, 비밀번호, 상세 상담 내용, 내부 식별키는 AI 검색 답변과 결과 카드에 직접 노출하지 않는다.
• 마스킹 기준 (PI-AIS-PRV-001-02)
- 고객 식별정보, 결제정보, 회선정보, 주소, 연락처는 업무상 필요한 최소 범위만 표시하고 나머지는 마스킹한다.
• 로그 비식별 기준 (PI-AIS-PRV-001-03)
- 검색 로그는 운영 분석 전에 고객 식별 가능 정보를 제거하거나 비식별 처리한다.
민감정보가 포함된 질의는 별도 보호 대상으로 분류한다.
• 운영자 열람 제한 (PI-AIS-PRV-001-04)
- 운영자는 검색 품질 개선에 필요한 최소 정보만 열람할 수 있다.
고객 식별정보 열람은 권한과 이력 관리 대상이다.
• AI 학습 활용 제한 (PI-AIS-PRV-001-05)
- 고객 식별 가능 검색 이력, 민감정보 포함 질의, 상담성 질의는 명시된 목적과 동의 범위를 벗어나 AI 모델 학습에 활용하지 않는다.
품질 개선 목적의 분석은 비식별·집계 기준으로 수행한다.

15) 검색 품질 피드백 정책 (PG-AIS-FDB-001)

• 피드백 수집 항목 (PI-AIS-FDB-001-01)
- 도움 여부, 정확성, 결과 적합성, 부정확 답변 신고, 자유 의견을 수집할 수 있다.
• 피드백 분류 (PI-AIS-FDB-001-02)
- 피드백은 부정확 답변, 무결과, 부적합 결과, 불만, 개선 요청, 긍정 피드백으로 분류한다.
• 운영 검토 대상 (PI-AIS-FDB-001-03)
- 부정확 답변, 반복 무결과, 고위험 업무 오연결, 금칙어 우회, 개인정보 노출 가능성은 운영 검토 대상으로 생성한다.
• 품질 지표 반영 (PI-AIS-FDB-001-04)
- 피드백 결과는 검색 품질 지표와 개선 과제 도출에 반영한다.
개별 고객 불만은 필요 시 상담 또는 VOC 처리 기준과 연결한다.

16) 검색 운영 관리 정책 (PG-AIS-OPR-001)

• 지식 베이스 관리 (PI-AIS-OPR-001-01)
- 지식 베이스는 등록, 검수, 승인, 배포, 비활성화 상태로 관리한다.
정책성 콘텐츠는 담당 부서 검수 후 배포한다.
• 금칙어 관리 (PI-AIS-OPR-001-02)
- 금칙어와 위험 질의 유형은 운영자가 등록·수정할 수 있다.
보안 관련 금칙어는 보안 담당 조직의 기준을 따른다.
• 추천 질의 관리 (PI-AIS-OPR-001-03)
- 추천 질의는 고객 이용 빈도, 상담 유입, 무결과 로그, 주요 시즌성 업무를 기준으로 운영자가 관리한다.
• 운영 변경 이력 (PI-AIS-OPR-001-04)
- 지식 베이스, 금칙어, 추천 질의, 결과 보정 기준의 변경 이력은 변경자, 변경일, 변경 사유, 변경 전후 값으로 저장한다.

17) 검색 품질 관리 정책 (PG-AIS-QUL-001)

• 품질 지표 (PI-AIS-QUL-001-01)
- AI 검색 품질은 무결과율, 저신뢰 결과율, 후속 업무 진입률, 상담 전환율, 피드백 만족도, 부정확 답변 신고율로 관리한다.
• 모니터링 기준 (PI-AIS-QUL-001-02)
- 검색 품질 지표는 일·주·월 단위로 모니터링한다.
고위험 업무 오연결, 개인정보 노출 가능성, 부정확 답변 급증은 즉시 검토 대상으로 분류한다.
• 개선 과제 생성 (PI-AIS-QUL-001-03)
- 무결과 반복, 상담 전환 과다, 후속 업무 진입 실패, 고객 불만 반복 항목은 검색 개선 과제로 등록한다.
• 개선 효과 추적 (PI-AIS-QUL-001-04)
- 운영 보정 후 무결과율, 상담 전환율, 후속 업무 진입률, 피드백 만족도 변화를 비교해 개선 효과를 추적한다.

18) 연계 오류·장애 처리 정책 (PG-AIS-ERR-001)

• BSS 조회 실패 처리 (PI-AIS-ERR-001-01)
- BSS 조회가 실패하면 고객별 요금, 청구, 약정, 가입 상품, 회선 상태, 혜택 보유 여부를 기반으로 한 개인화 결과와 처리형 후속 액션을 제공하지 않는다.
공통 정보 검색 결과와 상담 또는 메뉴 경로를 우선 제공한다.
• 지식 베이스 오류 처리 (PI-AIS-ERR-001-02)
- 지식 베이스 조회가 실패하거나 출처 유효성이 확인되지 않으면 AI 요약 답변을 제공하지 않는다.
관련 메뉴, 고객센터 안내, 재시도 안내 중 하나로 처리한다.
• 후속 업무 연결 실패 처리 (PI-AIS-ERR-001-03)
- 후속 업무 시스템 연결이 실패하면 고객에게 업무 처리 실패가 아니라 연결 실패임을 안내한다.
재시도, 관련 메뉴 이동, 상담 전환 중 하나를 제공하고 실패 로그를 저장한다.
• 장애 안내 및 상담 전환 (PI-AIS-ERR-001-04)
- 연계 장애가 반복되거나 고객 피해 가능성이 있는 업무에서 장애가 발생하면 상담 전환을 우선 제공한다.
장애 유형, 질의, 선택 결과, 실패 시점은 운영 검토 대상으로 저장한다.
최종 점검 기준
1. 범위 정합성
AI 검색이 다루는 범위와 제외 범위가 명확한지 확인한다.
검색 모델 상세 알고리즘, 프롬프트 상세, API 필드, DB 컬럼, 운영자 화면 상세처럼 후속 산출물에서 다룰 내용이 과도하게 포함되지 않았는지 확인한다.
반대로 검색 결과 노출, 개인화, 신뢰도, 제한 업무, 고위험 업무 보호처럼 SKT가 결정해야 하는 정책 기준이 빠지지 않았는지 확인한다.
2. 고객 완결성
고객이 검색어를 정확히 몰라도 자연어, 키워드, 음성, 추천 질의로 원하는 업무를 찾을 수 있는 구조인지 확인한다.
검색 결과가 단순 안내에 그치지 않고, 처리 가능한 업무는 조회·신청·변경·해지·납부·상담 등 후속 업무로 연결되는지 확인한다.
무결과, 저신뢰 결과, 모호 질의에서도 고객이 다음 행동을 할 수 있도록 재질의, 후보 결과, 관련 메뉴, 상담 전환 기준이 정의되어 있는지 확인한다.
3. AI 답변 안전성
AI 요약 답변이 출처와 기준일을 기반으로 제공되는지 확인한다.
출처가 없거나 답변 근거가 부족한 경우 확정 답변을 제공하지 않는 기준이 정의되어 있는지 확인한다.
요금, 약정, 할인, 해지, 납부, 개인정보 변경처럼 고객 피해 가능성이 큰 내용은 단정형 답변이 제한되고 후속 업무 화면에서 최종 확인하도록 되어 있는지 확인한다.
4. 개인화·권한 보호
비로그인 상태에서 고객별 요금, 청구, 약정, 가입 상품, 회선 상태, 혜택 보유 여부가 노출되지 않도록 정책이 정의되어 있는지 확인한다.
로그인 고객이라도 고객 정보 활용 동의, 권한, 명의, 회선 상태, 미성년자·대리 처리 조건을 확인한 뒤 개인화 결과와 후속 업무 액션을 제공하도록 정의되어 있는지 확인한다.
5. 후속 업무 연결성
검색 결과에서 후속 업무로 이동할 때 고객 상태, 업무 가능 여부, 인증 필요 여부, 동의 필요 여부, 영향도 고지 필요 여부를 확인하도록 정의되어 있는지 확인한다.
고위험 업무는 AI 검색 결과에서 즉시 실행하지 않고, 해당 업무 프로세스에서 인증·동의·영향도 고지·최종 확인을 거치도록 정의되어 있는지 확인한다.
6. BSS·연계 시스템 포함성
AI 검색 결과가 채널 콘텐츠만 기준으로 생성되지 않고, 고객 상태, 가입 상품, 청구, 납부, 약정, 혜택, 구독, 멤버십 등 BSS 및 연계 시스템 정보를 고려하도록 정의되어 있는지 확인한다.
연계 시스템이 제공해야 하는 근거 정보, 고객 상태, 업무 가능 여부, 후속 업무 경로가 기능과 정책에 반영되어 있는지 확인한다.
7. 검색 운영 가능성
검색 운영자가 지식 베이스, 금칙어, 추천 질의, 결과 보정 기준, 출처, 검색 품질 지표를 관리할 수 있도록 프로세스와 기능이 정의되어 있는지 확인한다.
운영 변경 이력, 변경자, 변경 사유, 변경 전후 값이 저장되도록 정책이 정의되어 있는지 확인한다.
무결과, 저신뢰 결과, 부정확 답변, 상담 전환 과다, 후속 업무 진입 실패가 개선 과제로 이어지는 구조인지 확인한다.
8. 로그·개인정보 보호
검색 이력과 로그의 저장 대상, 활용 목적, 보관 기간, 삭제 기준이 정의되어 있는지 확인한다.
주민등록번호, 계좌번호, 카드번호 전체, 인증번호, 비밀번호, 상세 상담 내용, 내부 식별키 등 민감정보가 AI 검색 답변, 결과 카드, 운영 화면, 로그에 직접 노출되지 않도록 정의되어 있는지 확인한다.
9. 연결성
유즈케이스 → 상태 → 프로세스 → 기능 → 정책이 누락 없이 연결되어 있는지 확인한다.
사람 액터인 고객과 검색 운영자의 유즈케이스는 프로세스 정의 대상 Y로 관리되어 있는지 확인한다.
AI 검색 시스템과 연계 시스템의 보조 유즈케이스는 관련 프로세스의 기능·정책으로 반영되어 있는지 확인한다.

AI 검색 관련 기준 요구사항이 프로세스, 기능, 정책 중 하나 이상에 모두 반영되어 있는지 확인한다.
