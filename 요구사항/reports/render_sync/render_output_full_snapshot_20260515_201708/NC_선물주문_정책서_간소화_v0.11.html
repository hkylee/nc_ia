1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 상품, 이용권, 쿠폰, 데이터성 자산, 단말 등 선물 가능한 대상을 보내고 받는 업무의 처리 기준을 정의한다.

• 포함 범위는 선물 가능 상품 확인, 수신자 지정, 즉시·예약 발송, 메시지 전달, 수락·거절·만료, 재전송·취소·회수, 상태 조회, 정산·자산 반영, 운영 예외 처리다.

• 일반 주문, 결제 승인, 물류 배송, 교환·반품·환불 상세 정책은 인접 정책서를 우선 적용하되 선물 상태와 고객 고지는 이 문서에서 연결한다.

• 대상 채널은 앱·웹 FO이며, 발신 고객과 수신 고객이 각각 자신의 처리 가능 시점과 책임을 이해할 수 있어야 한다.

• BSS와 연계 시스템이 수행하는 자격 판정, 이용권·쿠폰·자산 발급, 정산 상태 반영, 이력 저장, 알림 결과 회신은 정책 판단 기준에 포함한다.

• API 필드, DB 컬럼, 알림 템플릿 문안 전체, 운영자 화면 상세, 결제·배송 상세 설계는 후속 산출물에서 상세화한다.

• 채널 업무 시스템은 발신·수신 고객 안내, 수락·거절·만료 상태, 선물함·쿠폰함·이용권함 귀속 경로를 책임지고 선물 연계 시스템은 선물 가능성, 수신 자격, 결제·환불, BSS·상품·혜택 반영 결과를 책임진다.

• 고객 표시 상태는 발송 가능, 수락 대기, 수락 완료, 미수락 만료, 취소·환불, 귀속 보류로 안내하고 내부 상태는 수신 자격, 상품·혜택 원장 반영, 결제 취소, 알림 발송 이력으로 분리한다.

• 상세 요구사항 29건은 발신자·수신자 책임, 선물 가능성, 수락·거절·만료, 취소·환불, 귀속·알림 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했다.

• 핵심 정책 질문은 누가 선물을 보낼 수 있는가, 언제 수신자가 수락·거절할 수 있는가, 몇 번까지 재전송·취소할 수 있는가, 어떤 조건이면 만료·회수·환불되는가, 실패하면 어떤 상태와 알림을 남기는가이다.

• 중복 요청과 멱등 기준은 동일 발신자·동일 수신자·동일 상품·동일 결제 요청이 5분 안에 반복되면 기존 선물 주문 ID를 기준으로 결과를 회신하고, 결제 상태가 달라진 경우에만 새 요청으로 처리하는 것이다.

• 통합 리스크는 발신 채널, 수신 채널, 선물함, 쿠폰함, 이용권함, BSS 원장 반영 시점이 어긋나는 경우이며 고객에게는 수락 대기, 귀속 보류, 복구 진행 상태 중 하나로 일관되게 표시한다.

나. 설계 원칙

• 양면 고객 기준: 발신자와 수신자의 권리, 책임, 상태, 고지를 각각 분리하되 하나의 선물 상태 흐름으로 연결한다.

• 수락 전 불확실성 축소: 수신자는 선물명, 발신자, 유효기간, 수락 효력, 거절·만료 결과를 한 화면에서 판단할 수 있어야 한다.

• 상태 기반 운영: 발송 준비, 전달 완료, 열람, 수락 대기, 수락 완료, 거절, 만료, 취소, 회수, 발송 실패를 상태로 관리한다.

• 예외 선처리: 오발송, 링크 미수신, 반복 재전송, 자격 미충족, 수락 오류, 만료 임박은 고객이 재전송·취소·상담 전환을 선택할 수 있게 한다.

• 정산·자산 일관성: 환불, 포인트 복원, 쿠폰 재발급, 이용권 발급, 데이터 차감·복원은 선물 상태와 동일한 이력으로 추적한다.

• 개인정보 최소 노출: 발신자와 수신자의 이름, 연락처, 배송 정보는 단계별 목적에 필요한 범위로만 마스킹해 표시한다.

2) 상태 전이 기준

현재 상태전이 이벤트다음 상태처리 기준 및 후속 처리
선물 주문 전선물 가능 상품과 수신자 지정수신자 확인 중발신자가 선물 가능 상품을 선택하면 수신자 입력과 수취 가능성 검증을 시작한다.
수신자 확인 중선물 가능 상품과 수신자 지정발송 준비수신자 정보, 선물 가능 대상, 오발송 위험 검증이 통과하면 메시지와 발송 방식을 선택한다.
발송 준비선물 주문 확정과 발송 관리예약 발송 대기발신자가 예약 발송을 선택하면 발송 예정 시점까지 수정·취소 가능 기준을 유지한다.
발송 준비선물 주문 확정과 발송 관리발송 완료발신자가 즉시 발송을 확정하고 결제·정산 조건이 통과하면 수신자에게 링크 또는 알림을 발송한다.
예약 발송 대기선물 주문 확정과 발송 관리발송 완료예약 발송 시점이 도래하고 상품·수신자·정산 조건이 유지되면 발송한다.
발송 완료선물 알림·딥링크 전달열람 완료수신자가 링크, 알림톡, 푸시, 앱 내 알림 중 하나로 진입하면 열람 상태로 전환한다.
열람 완료선물 확인과 수락·거절 판단수락 대기수신자가 선물 내용을 확인했지만 수락 또는 거절을 확정하지 않으면 수락 대기 상태로 유지한다.
수락 대기선물 확인과 수락·거절 판단수락 완료수신자가 본인확인과 자격 검증을 통과하고 수락을 선택하면 수락 완료로 전환한다.
수락 완료선물 등록·사용으로 이어가기등록·사용 가능선물 유형에 따라 선물함, 쿠폰함, 사용처, 후속 신청 화면으로 자산 또는 이용 흐름을 연결한다.
수락 대기선물 확인과 수락·거절 판단거절수신자가 거절을 선택하면 발신자에게 결과를 안내하고 정산 대기 상태를 생성한다.
수락 대기선물 상태 조회와 예외 복구만료 임박수락 가능 기한이 임박하면 발신자와 수신자에게 남은 기한과 대체 행동을 안내한다.
만료 임박선물 상태 조회와 예외 복구만료유효기간이 경과하면 정책에 따라 미수락 만료 또는 사용 불가 상태로 전환한다.
발송 완료선물 상태 조회와 예외 복구발송 실패수신자에게 링크나 알림이 전달되지 않았거나 실패 응답이 확인되면 재전송 가능 상태로 전환한다.
발송 실패선물 상태 조회와 예외 복구발송 완료재전송 횟수와 전달 방식 기준을 충족하면 다시 발송한다.
발송 완료선물 주문 확정과 발송 관리취소·회수수락 전이며 정책상 취소·회수 가능 기한 안이면 발신자 취소 또는 회수를 허용한다.
거절선물 자산·정산 반영정산 대기거절 결과에 따라 환불, 포인트 복원, 쿠폰 재발급, 자산 복원 중 하나를 판정한다.
만료선물 자산·정산 반영정산 대기만료 후 처리 방식에 따라 주문 취소, 환불, 포인트 복원, 쿠폰 재발급 여부를 판정한다.
취소·회수선물 자산·정산 반영정산 대기취소·회수 결과에 따라 수신자 접근을 차단하고 발신자 정산 복원을 진행한다.

발송 실패선물 주문 모니터링과 예외 처리운영 예외반복 재전송 실패, 오발송 문의, 분쟁, 수동 보정 필요 건은 운영 예외로 전환한다.
수신자 확인 중미성년·대리 선물 권한 확인발송 준비대리·법정대리인 관계와 동의가 필요한 선물은 권한 확인이 통과한 경우에만 발송 준비로 전환한다.

1) 선물주문 범위 정책 (PG-GFT-SCOPE-001)

• 직접 범위 (PI-GFT-SCOPE-001-001)
- 선물주문은 선물 가능 상품 확인, 수신자 지정, 메시지·발송 방식 선택, 수락·거절·만료, 재전송·취소·회수, 선물 상태 조회, 정산·자산 반영을 직접 범위로 둔다.
• 인접 정책서 경계 (PI-GFT-SCOPE-001-002)
- 일반 주문, 결제 승인, 물류 배송, 교환·반품·환불 상세는 인접 정책서를 우선 적용한다.
- 다만 선물 상태, 발신자·수신자 고지, 정산 결과 연결은 본 문서에서 관리한다.
• 선물 가능 유형 (PI-GFT-SCOPE-001-003)
- 선물 가능 유형은 상품, 이용권, 쿠폰, 데이터성 자산, 단말로 구분한다.
- 각 유형은 수락 가능 기한, 사용 가능 기한, 정산 방식이 다르면 별도 정책 항목으로 관리한다.
• 단말 선물 범위 (PI-GFT-SCOPE-001-004)
- 고객이 구매한 단말을 다른 명의 사용자에게 선물하는 경우 수령 대상, 선물 방식, 처리 완료 상태를 채널에서 확인할 수 있어야 한다.
- 단말 배송·개통 상세는 관련 정책서와 연결한다.
• 보유 혜택 전송 범위 (PI-GFT-SCOPE-001-005)
- 고객이 보유한 혜택을 선물로 전송하는 경우 양도 가능 여부, 잔여 수량, 사용 가능 기간, 수신자 조건을 발송 전 확인한다.
- 양도 불가 또는 일부 사용으로 전송이 제한되는 혜택은 선물 대상에서 제외하고 제한 사유를 표시한다.

2) 선물 가능성 판정 정책 (PG-GFT-ELIG-001)

• 발신 가능 조건 (PI-GFT-ELIG-001-001)
- 발신자는 본인확인, 결제·보유 자산, 상품 선물 가능 여부, 발송 제한 횟수, 부정 사용 제한을 모두 충족한 경우에만 선물을 생성할 수 있다.
• 수신 가능 조건 (PI-GFT-ELIG-001-002)
- 수신자는 회원 상태, 수신 가능 상품 조건, 연령·권한, 보유 한도, 중복 수령 제한을 충족한 경우에만 수락할 수 있다.
• 상품 제한 (PI-GFT-ELIG-001-003)
- 선물 불가 상품은 발송 전에 차단하고, 사유와 대체 가능한 처리 경로를 제공한다.
- 제한 사유는 상품 판매 상태, 약관 제한, 수신 자격, 정산 불가 중 하나로 분류한다.
• 데이터 선물 제한 (PI-GFT-ELIG-001-004)
- 데이터 선물은 회선, 요금제, 가족 결합, 잔여 데이터, 선물 가능 용량, 선물 횟수, 청소년 제한, 법정대리인 동의 예외 기준을 함께 판정한다.
• 상업적 이용 제한 (PI-GFT-ELIG-001-005)
- 상업적 목적 또는 금전 거래 목적의 선물은 제한 대상으로 분류한다.
- 반복 발송, 다수 수신자, 비정상 회수 요청은 운영 예외로 기록한다.

3) 수신자 지정·권한 정책 (PG-GFT-RECIPIENT-001)

• 수신자 지정 (PI-GFT-RECIPIENT-001-001)
- 수신자 지정은 연락처, 회원 식별, 수령 가능 상태를 기준으로 검증한다.
- 발신자는 발송 전 수신자 식별 정보를 마스킹된 형태로 재확인해야 한다.
• 비회원 수신 (PI-GFT-RECIPIENT-001-002)
- 비회원 수신자가 선물을 받는 경우 회원가입 또는 최소 본인확인 후 이용 가능 상태로 전환한다.
- 비회원이 기한 내 절차를 완료하지 않으면 만료 기준을 적용한다.
• 미성년·대리 수신 (PI-GFT-RECIPIENT-001-003)
- 미성년 자녀 또는 대리 수신은 가족 관계, 법정대리인 동의, 위임 권한 중 필요한 조건을 충족한 경우에만 허용한다.
• 오발송 방지 (PI-GFT-RECIPIENT-001-004)
- 오발송 위험은 수신자 이름 일부, 연락처 일부, 최근 수신 여부, 동일 수신자 반복 여부를 기준으로 사전 경고한다.
- 발송 후 수락 전에는 정책 범위 내에서 회수 가능성을 안내한다.
• 법정대리인·대리 권한 (PI-GFT-RECIPIENT-001-005)
- 미성년자, 가족 회선, 대리 수신 또는 대리 발송은 관계 확인과 필요한 동의가 완료된 경우에만 선물 발송 또는 수락을 허용한다.
- 권한이 만료되거나 거절되면 수신자 확인 중 상태로 유지하고 사유를 안내한다.

4) 메시지·전달 방식 정책 (PG-GFT-MESSAGE-001)

• 메시지 구성 (PI-GFT-MESSAGE-001-001)
- 메시지는 기본 카드, 짧은 메시지, 링크 전달 문구로 구성할 수 있다.
- 상품명, 발신자 표시, 수락 기한, 주요 유의사항은 자동 포함한다.
• 전달 방식 (PI-GFT-MESSAGE-001-002)
- 전달 방식은 문자, 알림톡, 앱 푸시, 앱 내 알림, 딥링크를 지원한다.
- 수신자의 앱 설치와 로그인 상태에 따라 표준 진입 흐름을 분기한다.
• 부적절 문구 제한 (PI-GFT-MESSAGE-001-003)
- 고객 입력 메시지는 금칙어, 개인정보, 상업적 홍보 문구를 제한할 수 있다.
- 제한 시 고객에게 수정 요청 사유를 안내한다.
• 템플릿 적용 (PI-GFT-MESSAGE-001-004)
- 운영자가 등록한 메시지 템플릿은 상품 유형, 선물 상태, 발송 채널별로 적용한다.
- 템플릿 변경은 적용 시점과 대상 상품을 함께 관리한다.

5) 발송·예약 정책 (PG-GFT-SEND-001)

• 즉시 발송 (PI-GFT-SEND-001-001)
- 즉시 발송은 발신 가능 조건, 수신 가능 조건, 결제·정산 조건, 개인정보 고지 확인이 모두 통과한 경우에만 실행한다.
• 예약 발송 (PI-GFT-SEND-001-002)
- 예약 발송은 발송 예정 시점, 수정 가능 시점, 취소 가능 시점을 발신자에게 안내한 뒤 등록한다.
- 예약 시점 도래 전 상품·수신자 조건이 바뀌면 재판정한다.
• 예약 수정 (PI-GFT-SEND-001-003)
- 예약 발송의 메시지, 전달 방식, 발송 예정 시점은 수락 전이면서 발송 예정 시점 이전에만 수정할 수 있다.
• 예약 취소 (PI-GFT-SEND-001-004)
- 예약 발송 취소는 발송 전까지만 허용한다.
- 이미 발송된 선물은 취소·회수 정책을 따른다.

6) 링크·딥링크 진입 정책 (PG-GFT-LINK-001)

• 링크 진입 (PI-GFT-LINK-001-001)
- 수신 링크는 선물 ID, 유효기간, 수신자 식별 상태를 기준으로 유효성을 확인한다.
- 링크가 유효하지 않으면 사유와 상담 경로를 안내한다.
• 앱 설치 분기 (PI-GFT-LINK-001-002)
- 앱 미설치 수신자는 설치 또는 웹 수락 가능 경로를 안내한다.
- 설치 후 복귀 시 선물 컨텍스트가 유지되어야 한다.
• 로그인 상태 분기 (PI-GFT-LINK-001-003)
- 수신자가 로그인 상태가 아니면 선물 내용의 민감 정보는 제한하고, 수락 또는 등록 전에 필요한 인증을 요구한다.
• 비정상 링크 제한 (PI-GFT-LINK-001-004)
- 선물하기 기능 외 방법으로 임의 전달된 이용권 또는 변조 링크는 회사 책임 범위 밖으로 안내하고 사용을 제한한다.

7) 수락·거절 정책 (PG-GFT-ACCEPT-001)

• 수락 전 안내 (PI-GFT-ACCEPT-001-001)
- 수신자는 선물명, 발신자 정보, 메시지, 수락 가능 기한, 수락 시 효력, 거절 시 결과를 수락 전에 확인해야 한다.
• 수락 기준 (PI-GFT-ACCEPT-001-002)
- 수락은 수신자 본인확인, 수령 자격, 중복 수락 여부, 유효기간이 모두 유효한 경우에만 완료한다.
• 거절 기준 (PI-GFT-ACCEPT-001-003)
- 수신자가 거절하면 선물 상태를 거절로 전환하고 발신자에게 결과를 안내한다.
- 거절 이후 재수락은 정책상 허용된 경우에만 가능하다.
• 나중에 보기 (PI-GFT-ACCEPT-001-004)
- 나중에 보기는 수락 가능 기한 안에서만 제공한다.
- 고객에게 남은 기한과 만료 후 결과를 함께 안내한다.

8) 유효기간·만료 정책 (PG-GFT-VALIDITY-001)

• 수락 가능 기한 (PI-GFT-VALIDITY-001-001)
- 수락 가능 기한은 발송 시점 또는 수신 시점 중 선물 유형별 정책 기준으로 산정한다.
- 기한 기준은 발신자와 수신자 모두에게 같은 의미로 표시한다.
• 사용 가능 기한 (PI-GFT-VALIDITY-001-002)
- 사용 가능 기한은 수락 또는 등록 이후 실제 사용이 가능한 기간이다.
- 선물하기 이용권은 이용권에 명시된 상품과 기한 안에서만 사용할 수 있다.
• 만료 임박 (PI-GFT-VALIDITY-001-003)
- 만료 임박 알림은 남은 기간, 만료 후 결과, 가능한 대체 행동을 포함한다.
- 발신자에게는 재전송·취소 가능 여부를, 수신자에게는 수락·등록 가능 여부를 안내한다.
• 만료 후 처리 (PI-GFT-VALIDITY-001-004)
- 유효기간이 경과하면 미수락 만료, 사용 불가, 주문 취소, 환불, 포인트 복원, 쿠폰 재발급 중 선물 유형별 후처리를 자동 수행한다.
• 유효기간 연장 (PI-GFT-VALIDITY-001-005)
- 유효기간 연장은 상품 약관과 운영 정책에서 허용한 범위 안에서만 제공한다.
- 연장 가능 여부와 최대 기한은 고객에게 사전에 안내한다.

9) 알림·리마인드 정책 (PG-GFT-NOTI-001)

• 발송 알림 (PI-GFT-NOTI-001-001)
- 발송 알림에는 선물명, 발신자 표시, 수락 가능 기한, 진입 링크, 고객이 해야 할 다음 행동을 포함한다.
• 수락·거절 알림 (PI-GFT-NOTI-001-002)
- 수락, 거절, 등록, 사용 가능 상태가 변경되면 발신자와 수신자에게 필요한 범위로 결과를 안내한다.
• 만료 임박 알림 (PI-GFT-NOTI-001-003)
- 만료 임박 알림은 최소 1회 이상 제공하고, 재전송 또는 수락 같은 즉시 행동과 연결한다.
• 재전송 알림 (PI-GFT-NOTI-001-004)
- 재전송 알림은 기존 선물 상태를 유지한 채 새 링크 또는 알림을 제공한다.
- 재전송 횟수 초과 시 상담 또는 다른 전달 방식을 안내한다.
• 정보 구분 (PI-GFT-NOTI-001-005)
- 알림은 발신자와 수신자에게 필요한 정보만 분리해 제공한다.
- 정산 금액은 발신자에게, 사용 방법은 수신자에게 우선 안내한다.

10) 개인정보 보호·마스킹 정책 (PG-GFT-PRIVACY-001)

• 목적 제한 (PI-GFT-PRIVACY-001-001)
- 선물 주문 개인정보는 선물 발송, 수락, 정산, 고객 문의 처리 목적에 필요한 범위에서만 사용한다.
• 발신자 노출 (PI-GFT-PRIVACY-001-002)
- 수신 전 발신자 정보는 이름 일부 또는 닉네임 등 정책상 허용된 수준으로만 표시한다.
- 익명 선물이 허용되면 수신 전 발신자 신원을 숨길 수 있다.
• 수신자 노출 (PI-GFT-PRIVACY-001-003)
- 발신자에게 수신자의 연락처와 이름은 오발송 방지에 필요한 범위로 마스킹해 보여준다.
- 수락 후에도 불필요한 배송·개인 정보는 노출하지 않는다.
• 익명 선물 (PI-GFT-PRIVACY-001-004)
- 익명 선물은 상품 유형과 분쟁 대응 가능성을 검토해 허용한다.
- 정산, 회수, 고객 문의 처리에는 내부 이력이 반드시 남아야 한다.

11) 선물 상태 정책 (PG-GFT-STATE-001)

• 상태 목록 (PI-GFT-STATE-001-001)
- 선물 상태는 발송 준비, 전달 완료, 열람, 수락 대기, 수락 완료, 거절, 만료, 취소, 회수, 발송 실패, 정산 완료로 관리한다.
• 발신자 조회 (PI-GFT-STATE-001-002)
- 발신자는 발송 준비부터 수락 완료, 거절, 만료, 취소, 회수까지 보낸 선물 상태와 가능 행동을 조회할 수 있어야 한다.
• 수신자 조회 (PI-GFT-STATE-001-003)
- 수신자는 받은 선물의 수락 여부, 등록 여부, 현재 사용 가능 상태, 만료 예정일을 조회하고 즉시 등록 또는 사용 단계로 이동할 수 있어야 한다.
• 상태 사유 (PI-GFT-STATE-001-004)
- 사용 가능, 오픈 예정, 조건 미충족, 사용 완료, 만료, 회수 등 상태에는 고객이 이해할 수 있는 사유를 함께 표시한다.

12) 재전송·취소·회수 정책 (PG-GFT-RECOVERY-001)

• 재전송 가능 (PI-GFT-RECOVERY-001-001)
- 재전송은 수신자가 수락하지 않았고 재전송 횟수 안에 있을 때만 허용한다.
- 링크 분실, 알림 미수신, 만료 임박 사유는 재전송 가능 사유로 보되, 이미 수락 완료 또는 정산 완료된 선물은 재전송하지 않는다.
• 취소 가능 (PI-GFT-RECOVERY-001-002)
- 발신자 취소는 발송 전 또는 수락 전 상태에서만 허용한다.
- 수락 완료 이후에는 취소보다 환불·회수·상담 정책을 우선 적용한다.
• 회수 가능 (PI-GFT-RECOVERY-001-003)
- 회수는 오발송, 수신자 미열람, 운영 예외 등 정책상 허용된 경우에만 제공한다.
- 회수 요청 시 수신자 접근 제한과 정산 복원을 함께 처리한다.
• 되돌리기 제한 (PI-GFT-RECOVERY-001-004)
- 선물 상태 되돌리기는 정산 또는 자산 사용이 발생하기 전까지만 허용한다.
- 사용 또는 환불 완료 후에는 운영자 보정 이력으로만 처리한다.

13) 예외·실패 대응 정책 (PG-GFT-EXCEPTION-001)

• 발송 실패 (PI-GFT-EXCEPTION-001-001)
- 발송 실패 시 실패 채널, 실패 사유, 재전송 가능 여부를 안내한다.
- 반복 실패는 다른 전달 방식 또는 상담 연결로 전환한다.
• 수락 오류 (PI-GFT-EXCEPTION-001-002)
- 수락 오류는 인증 실패, 자격 미충족, 중복 수락, 만료, 시스템 오류로 분류한다.
- 고객은 상태를 유지한 채 재시도하거나 상담으로 전환할 수 있어야 한다.
• 등록 실패 (PI-GFT-EXCEPTION-001-003)
- 선물함, 쿠폰함, 이용권함, 데이터 자산 등록 실패는 자산 반영 전 상태로 보류하고 원인과 재처리 가능 여부를 안내한다.
• 오발송 문의 (PI-GFT-EXCEPTION-001-004)
- 오발송 문의는 수락 전 회수 가능성, 수신자 열람 여부, 정산 영향, 개인정보 노출 범위를 함께 확인해 처리한다.
• 교환 예외 (PI-GFT-EXCEPTION-001-005)
- 선물 교환 또는 대체 상품 전환은 상품 정책에서 허용한 경우에만 제공한다.
- 교환 결과는 수신자 자산과 발신자 정산 이력에 함께 남긴다.

14) 정산·복원 정책 (PG-GFT-SETTLE-001)

• 환불 기준 (PI-GFT-SETTLE-001-001)
- 거절, 만료, 취소, 회수 이후 환불 가능 여부는 결제 수단, 사용 여부, 수락 상태, 유효기간 경과 여부를 기준으로 판정한다.
• 포인트 복원 (PI-GFT-SETTLE-001-002)
- 포인트로 구매한 선물이 거절, 만료, 취소된 경우 복원 가능 시점과 복원 대상 포인트를 발신자에게 안내한다.
• 쿠폰 재발급 (PI-GFT-SETTLE-001-003)
- 쿠폰형 선물의 재발급은 미사용, 미수락, 시스템 오류, 운영자 승인 조건을 충족한 경우에만 허용한다.
• 자산 복원 (PI-GFT-SETTLE-001-004)
- 데이터성 자산 또는 이용권 자산은 수락 전 취소·만료·회수와 수락 후 사용 여부에 따라 차감 복원 또는 사용 불가 처리를 구분한다.
• 정산 안내 (PI-GFT-SETTLE-001-005)
- 정산 결과는 발신자에게 환불·복원·재발급 상태로, 수신자에게 사용 가능·사용 불가·만료 결과로 구분해 안내한다.

15) 선물 자산 등록·사용 정책 (PG-GFT-ASSET-001)

• 선물함 등록 (PI-GFT-ASSET-001-001)
- 수락한 선물은 선물함에 등록하고 받은 선물과 보낸 선물을 한 화면에서 구분해 조회할 수 있게 한다.
• 쿠폰·이용권 등록 (PI-GFT-ASSET-001-002)
- 쿠폰과 이용권은 쿠폰함 또는 이용권함에 등록하고 사용 가능, 조건 미충족, 사용 완료, 만료, 회수 상태를 표시한다.
• 데이터 자산 반영 (PI-GFT-ASSET-001-003)
- 데이터 선물은 기본 제공량, 선물 수신량, 리필·충전량, 공유 데이터, 로밍 데이터와 구분해 잔량과 소진 우선순위를 표시한다.
• 사용처 연결 (PI-GFT-ASSET-001-004)
- 수락 완료 후 고객은 사용처, 후속 신청 화면, 상품 이용 화면, 상담 연결 중 필요한 다음 행동으로 이동할 수 있어야 한다.
- 다음 행동은 선물 유형, 등록 상태, 사용 가능 기간, 수신자 자격에 따라 제한하거나 우선순위를 다르게 적용한다.
• 수락 후 인계 기준 (PI-GFT-ASSET-001-005)
- 수락 완료 시 선물 유형별로 선물함, 쿠폰함, 이용권함, 데이터 잔량, 단말·서비스 신청 화면 중 하나로 귀속 위치를 확정한다.
- 귀속 위치가 확정되지 않으면 등록 보류 상태로 두고 재처리 또는 상담 연결 기준을 안내한다.

16) 정기결제 전환 정책 (PG-GFT-SUBSCRIPTION-001)

• 정기결제 선택 (PI-GFT-SUBSCRIPTION-001-001)
- 선물받은 구독형 이용권은 수신자가 정기결제 전환 여부를 선택할 수 있어야 한다.
- 전환하지 않으면 이용권 기간 종료 후 자동 결제를 하지 않는다.
• 결제수단 등록 (PI-GFT-SUBSCRIPTION-001-002)
- 정기결제를 선택한 경우 이용권 기간 종료 후 등록한 결제수단으로 결제가 진행될 수 있음을 명확히 고지한다.
• 마지막 이용월 해지 (PI-GFT-SUBSCRIPTION-001-003)
- 추가 이용을 원하지 않는 고객은 마지막 이용월 안에 해지 신청을 해야 하며, 해지 가능 시점과 경로를 안내받아야 한다.
• 전환 고지 (PI-GFT-SUBSCRIPTION-001-004)
- 정기결제 전환, 결제 예정일, 해지 가능 시점은 수락 완료 직후와 만료 전 알림에서 반복 고지한다.

17) 운영 관리 정책 (PG-GFT-OPER-001)

• 상품 관리 (PI-GFT-OPER-001-001)
- 운영자는 선물 가능 상품 유형, 상품별 예외 조건, 수신 가능 대상, 단말 선물 가능 여부를 관리할 수 있어야 한다.
• 템플릿 관리 (PI-GFT-OPER-001-002)
- 운영자는 선물 발송, 수락, 거절, 만료 임박, 만료 완료 상태별 알림 템플릿과 발송 규칙을 관리한다.
• 정책 버전 (PI-GFT-OPER-001-003)
- 유효기간, 재전송 횟수, 거절·만료 처리 기준 변경 시 적용 시작일과 이미 발송된 선물의 적용 기준을 구분한다.
• 상태 모니터링 (PI-GFT-OPER-001-004)
- 운영자는 발송, 열람, 수락, 거절, 만료, 재전송, 취소 현황을 조회하고 예외 상태를 식별할 수 있어야 한다.
• 품질 지표 (PI-GFT-OPER-001-005)
- 운영자는 발송 성공률, 수락률, 만료율, 재전송률, 실패 사유 Top, 오발송 문의를 집계해 개선 대상을 관리한다.

18) 이력·증적 정책 (PG-GFT-HISTORY-001)

• 상태 이력 (PI-GFT-HISTORY-001-001)
- 선물 상태 변경은 발신자, 수신자, 상태, 변경 시각, 변경 사유, 처리 주체를 포함해 저장한다.
• 알림 이력 (PI-GFT-HISTORY-001-002)
- 알림 이력은 발송 채널, 발송 시각, 성공·실패 결과, 재전송 횟수, 고객 진입 여부를 포함한다.
• 정산 이력 (PI-GFT-HISTORY-001-003)
- 정산 이력은 환불, 포인트 복원, 쿠폰 재발급, 자산 복원, 정산 실패와 재처리 결과를 포함한다.
• 운영 조치 이력 (PI-GFT-HISTORY-001-004)
- 운영자 예외 처리는 조치 사유, 조치 전후 상태, 고객 안내 여부, 승인자, 처리 시각을 기록한다.

최종 점검 기준

선물주문 정책서 제출 전 점검
1. 범위 정합성
선물주문이 일반 주문, 결제, 배송, 환불 상세를 침범하지 않고 선물 상태와 고객 고지를 연결했는지 확인한다.
2. 양면 고객 완결성
발신자와 수신자가 각자의 책임, 처리 가능 시점, 결과를 독립적으로 이해할 수 있는지 확인한다.
3. 유즈케이스 적정성
발신, 수신, 상태조회/복구, 운영 유즈케이스가 너무 세부 단계로 쪼개지지 않았는지 확인한다.
4. 상태 전이 정합성
발송 준비부터 정산 완료까지 상태 코드와 전이 기준이 일치하는지 확인한다.
5. 프로세스-기능-정책 연결성
각 프로세스가 실제 기능과 정책 그룹에 연결되어 개발/QA가 추적 가능한지 확인한다.
6. 수락·거절·만료 기준
수신자 수락, 거절, 만료, 나중에 보기, 등록·사용 가능 상태의 기준이 명확한지 확인한다.
7. 정산·복원 기준
거절, 만료, 취소, 회수 이후 환불, 포인트 복원, 쿠폰 재발급, 자산 복원 기준이 구분되어 있는지 확인한다.
8. 개인정보 보호
발신자와 수신자의 연락처, 이름, 배송 정보가 목적 범위에 맞게 마스킹되는지 확인한다.
9. 운영 예외 대응
전달 실패, 오발송, 반복 재전송, 수락 오류, 등록 실패를 운영자가 확인하고 조치할 수 있는지 확인한다.
10. 요구사항 반영
PM-15 상세 요구사항 29건이 유즈케이스, 프로세스, 기능, 정책, 정책 항목 중 하나 이상에 추적되는지 확인한다.
11. 요구사항 추적성
상세 요구사항 29건은 발신자·수신자 책임, 선물 가능성, 수락·거절·만료, 재전송·취소·회수, 귀속·정산·알림 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했는지 확인한다.
