1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 자신의 회선, 계약, 상품, 요금제, 부가서비스, 구독, 결합, 혜택, 제한 상태를 이해하고 다음 행동을 선택하는 가입정보 관리 기준을 정의한다.

• 포함 범위는 가입정보 통합 홈, 회선·서비스 상세, 계약 조건, 약정·할인·프로그램 상태, 부가서비스·구독·제휴서비스 상태, 결합·그룹 관계, 관리 대상 회선, 긴급 조치, 증빙·이력, 개인화와 운영 기준을 포함한다.

• 대상 채널은 앱·웹 FO이며 고객이 직접 상태를 확인하고 관련 업무로 이동하는 셀프 처리 흐름을 우선한다.

• 요금제 변경 실행, 구독 플랜 변경 실행, 결합 가입·해지 실행, 결제·정산, 상담 화면 상세는 인접 정책서에서 다루고 본 문서는 가입정보에서의 판단과 연결 기준에 집중한다.

• API 필드, DB 컬럼, 내부 배치, 운영자 화면 상세는 후속 상세 설계에서 다루며 본 문서는 개발·QA가 테스트 가능한 정책 판단 기준만 포함한다.

• 채널 업무 시스템은 가입정보 홈, 상태 안내, 다음 행동 연결, 고객 표시 상태를 책임지고 가입정보 연계 시스템은 회선·계약·혜택·청구·제휴 상태와 BSS 기준 시점을 책임진다.

• 고객 표시 상태는 정상 이용 중, 변경 가능, 제한·보류, 혜택 확인 필요, 계약 영향 확인 필요, 연계 응답 지연으로 안내하고 내부 상태는 원장 조회, 관계 매핑, 권한 판정, 이력 저장으로 분리한다.

• 중복 요청과 멱등 기준은 동일 고객·동일 회선·동일 조회 범위의 반복 조회가 5분 안에 발생하면 기존 조회 결과와 기준 시점을 재사용하고, 고객 상태나 BSS 응답 버전이 바뀐 경우에만 새 조회로 처리하는 것이다.

• 상세 요구사항 83건은 가입정보 통합 홈, 회선·계약·약정·혜택 상태, 부가서비스·구독·결합 관계, 관리 대상 회선, 증빙·이력, 다음 행동 연결 기준으로 재구성해 유즈케이스·프로세스·기능·정책 중 하나 이상에 매핑했다.

나. 설계 원칙

• 관리 허브 중심: 가입정보는 단순 목록이 아니라 고객이 현재 상태와 관리 필요성을 이해하는 대표 진입점으로 구성한다.

• 상태 기반 다음 행동: 고객에게 보이는 상태마다 가능한 행동, 제한 사유, 상담 전환 기준을 함께 제공한다.

• 계약·혜택 영향 연결: 회선, 계약, 부가서비스, 구독, 결합, 약정, 할인의 관계를 비용·혜택 영향과 연결한다.

• BSS 기준 시점 투명화: 고객 표시 상태는 BSS·연계 응답 기준 시점과 지연 가능성을 함께 안내한다.

• 권한·개인정보 우선: 명의자, 실사용자, 관리 대상 회선, 법정대리 권한에 따라 표시 범위와 처리 가능 업무를 제한한다.

• 운영 가능한 품질: 상태코드 매핑, 우선 노출, 정합성 모니터링, 증빙 발급 기준을 운영자가 관리할 수 있게 한다.

2) 상태 전이 기준

현재 상태전이 이벤트다음 상태처리 기준 및 후속 처리
진입 전가입정보 통합 홈에서 보유 서비스 상태 확인본인·권한 확인 필요고객 식별 또는 세션 권한이 없으면 본인확인으로 전환한다.
본인·권한 확인 필요가입·계약·혜택 상태 제공가입정보 조회 가능BSS가 조회 가능 회선과 계약 범위를 회신하면 가입정보 조회를 허용한다.
가입정보 조회 가능가입정보 통합 홈에서 보유 서비스 상태 확인통합 홈 표시회선, 상품, 계약, 부가서비스, 결합, 혜택 요약을 통합 홈에 표시한다.
통합 홈 표시회선·서비스·계약 상세와 변경 가능 업무 선택회선 선택 필요보유 회선이 2개 이상이거나 관리 대상 회선이 있으면 회선 선택을 요구한다.
회선 선택 필요회선·서비스·계약 상세와 변경 가능 업무 선택상세 확인 중고객이 회선 또는 가입 항목을 선택하면 계약 조건과 변경 가능 업무를 조회한다.
상세 확인 중약정·할인·프로그램·혜택 만료 영향 확인관리 필요 있음만료 임박, 할인 종료, 변경 처리 중, 미사용 혜택, 제한 상태가 있으면 관리 필요로 표시한다.
상세 확인 중회선·서비스·계약 상세와 변경 가능 업무 선택변경 가능권한과 계약 상태가 충족되면 변경, 해지, 차단, 증빙 발급 등 후속 업무로 이동할 수 있다.
상세 확인 중회선·서비스·계약 상세와 변경 가능 업무 선택변경 제한미납, 정지, 제휴 제한, 대표자 불일치, 미성년 권한 제한이면 직접 처리를 제한한다.
통합 홈 표시결합·그룹·관리 대상 회선 권한 확인관리 대상 등록 대기가족 또는 대리 회선을 추가하려면 대상 회선 인증·동의 절차로 전환한다.
관리 대상 등록 대기관리 대상 회선 등록·조회 권한 확인관리 대상 등록 완료인증·동의가 성공하면 조회 범위와 직접 처리 가능한 업무를 저장한다.
관리 대상 등록 대기관리 대상 회선 등록·조회 권한 확인변경 제한동의 거절, 권한 불일치, 법정대리 확인 실패 시 등록을 제한한다.
상세 확인 중부가서비스·구독·제휴서비스 보유 상태 확인증빙 조회 가능가입일, 이용기간, 만료일, 구매확인, 영수증, 가입경위 이력이 있으면 증빙 조회를 허용한다.
상세 확인 중긴급·제한·불일치 상황에서 조치 경로 선택긴급 조치 필요분실·도난, 로밍 장애, 미납, 정지, USIM/eSIM 재발급 필요 상태가 확인되면 긴급 조치로 전환한다.
긴급 조치 필요긴급·제한·불일치 상황에서 조치 경로 선택긴급 처리 진행고객이 정지, 납부, 재발급, 장애 조회, 상담 연결 중 하나를 선택하면 요청을 접수한다.
긴급 처리 진행가입·계약·혜택 상태 제공이력 보관 완료BSS 또는 연계 시스템이 처리 결과를 회신하면 이력과 고객 표시 상태를 저장한다.
상세 확인 중긴급·제한·불일치 상황에서 조치 경로 선택정보 불일치실제 이용 상태와 조회 정보가 다르거나 연결 정보가 누락되면 불일치로 분류한다.
정보 불일치긴급·제한·불일치 상황에서 조치 경로 선택정정 요청 접수고객이 정정 요청을 제출하면 접수번호, 처리 예상 시점, 추가 증빙 필요 여부를 저장한다.
정정 요청 접수가입정보 표시·동기화 기준 운영반영 지연운영자 또는 BSS 재동기화가 필요한 경우 반영 지연 상태로 표시한다.
반영 지연가입·계약·혜택 상태 제공통합 홈 표시재동기화 결과가 정상 반영되면 기준 시점과 함께 통합 홈을 갱신한다.
통합 홈 표시내 서비스 카드 개인화와 우선 노출 관리개인화 적용 완료고객이 카드 순서, 숨김, 즐겨찾기, 퀵메뉴 설정을 저장하면 개인화 상태로 전환한다.
변경 제한가입정보 문맥 기반 상담 지원상담 전환 필요제한 사유가 고객 셀프 처리로 해소되지 않으면 상담 문맥을 포함해 상담으로 연결한다.
증빙 조회 가능증빙·신청·변경 이력 확인이력 보관 완료증빙 발급 또는 이력 조회가 완료되면 발급 이력과 조회 이력을 저장한다.
상담 전환 필요가입정보 문맥 기반 상담 지원이력 보관 완료상담 접수와 안내 결과를 가입정보 이력에 저장한다.

- 권한이 필요한 변경, 해지, 선물, 자녀 관리 업무는 인증 필요 사실과 인증 후 재진입 경로를 먼저 안내한다.
• 해외 접속 정지 해제 (PI-MYI-AUTH-001-04)
- 해외 접속 상태에서 정지 해제를 시도하면 해외 인증 가능 여부, 대체 인증, 상담 전환 조건을 함께 표시한다.
• 보호자 자녀 관리 메뉴 (PI-MYI-AUTH-001-05)
- 보호자 권한이 확인된 경우 자녀 회선의 요금제, 데이터, OTT, 로밍 관련 상태를 가입정보 홈에서 묶어 표시한다.
• 권한제한·예외 안내 (PI-MYI-AUTH-001-06)
- 회원 상태, 가입·전환·권한 변경 조건에 따라 처리 가능 범위와 제한 사유를 표시하고 예외 처리는 상담 또는 운영 확인으로 전환한다.

12) 다음 행동 연결 정책 (PG-MYI-ACTION-001)

• 상태별 다음 행동 (PI-MYI-ACTION-001-01)
- 가입 상태별로 변경, 해지, 증빙 발급, 상담 연결, 납부, 재발급, 장애 조회 중 가능한 행동만 표시한다.
• 관련 업무 바로가기 (PI-MYI-ACTION-001-02)
- 요금제 상세, 부가서비스 관리, 결합 관리, 단말·클럽 관리, 증빙 발급은 현재 회선과 선택 항목을 유지해 이동한다.
• 빠른 행동 우선순위 (PI-MYI-ACTION-001-03)
- 긴급 조치, 반영 대기, 만료 임박, 제한 해소, 미사용 유료상품 점검 순으로 빠른 행동 우선순위를 둔다.
• 불가 사유 안내 (PI-MYI-ACTION-001-04)
- 직접 처리 불가 시 권한 부족, 계약 제한, BSS 반영 대기, 제휴 응답 실패 등 제한 사유와 해소 조건을 표시한다.
• 환불/취소/철회 안내 및 진입 (PI-MYI-ACTION-001-05)
- 상품 또는 혜택 상세에서 환불, 취소, 철회 가능 여부를 표시하고 가능 상태일 때만 신청 또는 상담 경로로 이동한다.

13) 상태 표시·반영 시점 정책 (PG-MYI-STATUS-001)

• 고객 표시 상태 (PI-MYI-STATUS-001-01)
- 가입, 변경, 정지, 해지, 반영 대기, 이상 상태는 고객 이해 문구와 내부 상태코드를 분리해 표시한다.
• 신청·변경 진행 상태 (PI-MYI-STATUS-001-02)
- 신청 중, 변경 중, 개통 대기 상태는 현재 단계, 반영 예정 시점, 추가 조치 필요 여부를 함께 표시한다.
• 반영 지연 안내 (PI-MYI-STATUS-001-03)
- 실시간 반영이 아닌 항목은 기준 시점, 마지막 동기화 시각, 예상 지연 가능성을 표시한다.
• 정보 불일치 정정 (PI-MYI-STATUS-001-04)
- 고객이 실제 이용 상태와 조회 정보의 차이를 제기하면 정정 요청, 문의, 재동기화 중 가능한 경로를 안내한다.

14) 긴급·예외 대응 정책 (PG-MYI-EXC-001)

• 긴급 상황 경로 (PI-MYI-EXC-001-01)
- 분실·도난, 로밍 장애, 정지, 미납, USIM/eSIM 재발급 상황은 가입정보 상세에서 즉시 조치 경로를 제공한다.
• 재시도 기준 (PI-MYI-EXC-001-02)
- 가입정보 조회 실패 또는 연계 응답 지연은 최대 2회 자동 재조회 후 기준 시점과 상담 전환을 안내한다.
• 정정 요청 (PI-MYI-EXC-001-03)
- 정보 불일치 정정 요청은 접수번호, 처리 예상 시점, 추가 증빙 필요 여부를 저장하고 고객에게 회신한다.
• 상담 전환 조건 (PI-MYI-EXC-001-04)
- 권한 불일치, 원장 불일치, 제휴 응답 실패, 긴급 업무 실패가 2회 이상 반복되면 상담 전환을 기본 경로로 둔다.
• 이전 로밍 이력 기반 재신청 (PI-MYI-EXC-001-05)
- 로밍 장애 또는 재신청 맥락에서는 과거 로밍 상품 이용 이력을 표시하고 동일·유사 상품 재신청 가능 여부를 안내한다.

15) 이력·증빙 정책 (PG-MYI-HISTORY-001)

• 신청·변경 이력 (PI-MYI-HISTORY-001-01)
- 가입, 변경, 개통, 일시정지, 재개, 해지예약 신청 이력은 상태, 접수일, 반영 예정일, 완료일 기준으로 조회한다.
• 증빙 발급 (PI-MYI-HISTORY-001-02)
- 가입사실 확인서, 구매확인서, 영수증, 거래내역은 본인 또는 권한 확인 후 발급하고 발급 이력을 저장한다.
• 가입경위 타임라인 (PI-MYI-HISTORY-001-03)
- 부가서비스 가입경위는 가입 채널, 시각, 인증 또는 동의 단계, 변경 이력을 타임라인으로 제공한다.
• 이력 보관 (PI-MYI-HISTORY-001-04)
- 증빙 발급, 정정 요청, 상담 전환, 차단 설정 변경 이력은 고객 확인과 분쟁 대응을 위해 최소 1년 이상 보관한다.

16) MY 개인화·메뉴 정비 정책 (PG-MYI-PERS-001)

• 카드 개인화 (PI-MYI-PERS-001-01)
- 고객은 내 서비스 카드의 노출 순서, 숨김, 즐겨찾기를 설정할 수 있고 설정 변경은 즉시 MY 홈에 반영한다.
• 최근 이용 퀵메뉴 (PI-MYI-PERS-001-02)
- 최근 이용 메뉴와 개인별 퀵메뉴는 고객이 최근 30일 안에 사용한 업무와 관리 필요 상태를 함께 고려해 노출한다.
• 중복 MY 통합 (PI-MYI-PERS-001-03)
- 동일 정보를 제공하는 MY와 이용현황 메뉴는 대표 진입을 1개로 지정하고 중복 노출을 최소화한다.
• 서비스 소속성 (PI-MYI-PERS-001-04)
- 통합 서비스 내 개별 서비스로 진입한 경우 홈, MY, 메뉴의 소속 서비스를 명확히 표시한다.

17) BSS·연계 동기화 정책 (PG-MYI-BSS-001)

• BSS 상태 제공 (PI-MYI-BSS-001-01)
- BSS는 회선, 계약, 요금·혜택, 제한 여부, 청구 종속, 반영 상태를 채널에 제공하고 응답 기준 시각을 함께 회신한다.
• 원장 종속 관계 (PI-MYI-BSS-001-02)
- 옵션 계약, 구독, 제휴 부가서비스는 메인 계약 또는 청구 원장 종속 관계를 표시하고 원장 변경 시 영향 항목을 재산정한다.
• 연계 실패 처리 (PI-MYI-BSS-001-03)
- BSS 또는 외부 연계 조회가 실패하면 기존 표시값을 최신값으로 확정하지 않고 기준 시점과 재조회 가능 시간을 안내한다.
• 재동기화·보정 (PI-MYI-BSS-001-04)
- 동기화 지연, 누락, 불일치가 감지되면 운영자는 재조회, 재동기화, 수동 보정, 장애 공지 중 하나를 선택한다.

18) 데이터 보호·표시 정책 (PG-MYI-DATA-001)

• 입력 데이터 (PI-MYI-DATA-001-01)
- 가입정보 조회는 고객 식별, 회선, 계약, 상품, 서비스, 결합, 구독, 혜택, 청구 상태를 입력 데이터로 사용한다.
• 마스킹 기준 (PI-MYI-DATA-001-02)
- 설치 주소, 계정 식별자, 대표번호 외 보조번호, 증빙 식별값은 고객 확인에 필요한 최소 범위로 마스킹한다.
• 출력 데이터 (PI-MYI-DATA-001-03)
- 출력 데이터는 고객 표시 상태, 계약 조건, 제한 사유, 다음 행동, 반영 시점, 증빙 가능 여부, 이력 요약으로 제한한다.
• 저장·조회 기준 (PI-MYI-DATA-001-04)
- 개인화 설정, 발급 이력, 정정 요청, 상담 전환 이력은 고객 요청과 감사 대응 기준에 따라 저장·조회 권한을 분리한다.

19) 운영 기준 관리 정책 (PG-MYI-OPER-001)

• 요약·우선 노출 관리 (PI-MYI-OPER-001-01)
- 운영자는 고객군, 회선 상태, 계약 유형별로 요약 카드, 경고, 관리 필요 항목, 빠른 행동 노출 기준을 등록한다.
• 상태코드 매핑 (PI-MYI-OPER-001-02)
- 백엔드 상태코드는 고객 문구, 유의사항, 가능한 다음 행동, 상담 전환 여부와 1:1 또는 1:N으로 매핑해 관리한다.
• 정합성 모니터링 (PI-MYI-OPER-001-03)
- 운영자는 기준 시점, 연계 시스템별 동기화 상태, 지연, 누락, 불일치 건수를 매일 모니터링한다.
• 증빙 정책 관리 (PI-MYI-OPER-001-04)
- 증빙 종류, 서식, 마스킹, 발급 권한, 재발급 제한, 발급 이력 조회 기준은 운영 승인 후 배포한다.

20) 상담 전환·문맥 전달 정책 (PG-MYI-CS-001)

• 상담 문맥 전달 (PI-MYI-CS-001-01)
- 상담 전환 시 고객 식별, 선택 회선, 계약 상태, 제한 사유, 최근 변경, 정정 요청, 오류 단계 정보를 상담사에게 전달한다.
• 제한 사유 설명 (PI-MYI-CS-001-02)
- 상담사는 권한, 계약, 미납, 정지, 제휴 실패, 원장 불일치 중 제한 사유를 고객 문구로 설명한다.

- 셀프 처리 불가 예외는 상담사가 정정 요청, 운영 확인, 긴급 처리, 후속 연락 중 하나로 접수한다.
• 분쟁 확인 (PI-MYI-CS-001-04)
- 증빙 발급, 가입경위, 변경 이력, 상담 접수 이력은 고객 문의나 분쟁 발생 시 확인 기준으로 사용한다.
