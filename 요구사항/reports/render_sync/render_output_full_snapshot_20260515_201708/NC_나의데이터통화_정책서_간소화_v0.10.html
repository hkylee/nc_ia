1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 데이터·통화·문자 자산을 이해하고 부족 위험을 해소하며 충전·리필·선물·차단·로밍 옵션을 직접 실행하는 업무 기준을 정의한다.

• 대상 업무는 사용량·잔여량 조회, 구성별 잔량·소진 순서 확인, 예상 소진 안내, 알림·딥링크, 충전·리필·쿠폰 사용, 선물·공유, 자동 선물·충전, 가족·다회선 위임, 로밍 잔량·쿠폰, 실패 복구를 포함한다.

• 대상 채널은 앱·웹에서 고객이 직접 조회·설정·신청·취소·재시도·상담 전환을 수행하는 통합채널이며, 사용량 연계 시스템과 외부 연계 시스템의 결과 회신을 정책 판단 기준으로 포함한다.

• 요금제 가입·변경 자체, 외부 제휴 화면 상세, API 필드, DB 컬럼, 통신망 과금 원장 상세는 제외하되 고객 판단에 영향을 주는 제한·권한·한도·반영 기준은 포함한다.

• 후속 산출물에서는 BSS 인터페이스, 외부 제휴 연동, 통화내역 상세 마스킹, 알림 템플릿, 테스트 데이터와 운영 대시보드를 상세화한다.

• 상세 요구사항 70건은 사용량 조회, 소진 위험, 충전·리필, 선물·공유, 자동 설정, 가족·위임, 로밍, 실패 복구 기준 중 하나 이상에 매핑한다.

나. 설계 원칙

• 자산 상태 중심: 조회 화면은 단순 잔여량이 아니라 고객 표시 상태(충분, 위험, 소진, 충전 가능, 선물 가능)와 내부 처리 상태(사용량 집계 중, 반영 지연, 외부 결과 대조)를 구분해 다음 행동을 함께 제공한다.

• 즉시 실행 연결: 부족 위험이나 소진 상태에서는 리필, 충전, 선물 요청, 요금제 변경, 차단 설정으로 바로 이동할 수 있게 한다.

• 조건 사전 판정: 선물·충전·쿠폰 사용은 실행 후 실패보다 실행 전 한도, 권한, 대상, 유효기간, 취소 불가 조건을 먼저 판정한다.

• 가족·위임 최소 노출: 가족·보호자 권한은 업무에 필요한 범위만 허용하고 민감 통화·사용 정보는 최소 노출한다.

• 실시간성 투명성: 실시간처럼 보이는 정보도 기준 시각, 갱신 주기, 지연 가능성, 미반영 항목을 명확히 고지한다.

• 실패 복구 우선: 조회 실패, 반영 지연, 선물·충전 실패는 단순 오류로 끝내지 않고 재시도, 대체 행동, 상담 전환을 제공한다.

나. 정책 상세

1) 데이터·통화 범위·경계 정책 (PG-DTC-SCOPE-001)

• 대상 업무 범위 (PI-DTC-SCOPE-001-01)
- 본 문서는 데이터·통화·문자 사용량 조회, 소진 위험 판단, 충전·리필·쿠폰 사용, 선물·공유, 자동 설정, 가족·위임, 로밍, 실패 복구를 포함한다.
• 인접 업무 경계 (PI-DTC-SCOPE-001-02)
- 요금제 가입·변경 자체는 상품 변경 정책에서 다루고, 본 문서는 사용량 부족을 해소하기 위한 자산 조회와 실행 기준을 정의한다.
• 자산 상태 중심 (PI-DTC-SCOPE-001-03)
- 기능은 화면 단위가 아니라 잔여량 충분, 소진 위험, 충전 가능, 선물 가능, 반영 지연 같은 자산 상태 기준으로 동작한다.
• 후속 산출물 경계 (PI-DTC-SCOPE-001-04)
- 앱 화면 상세, 외부 제휴 화면, BSS API 필드, 정산 로직은 후속 산출물에서 상세화한다.

2) 사용량·잔여량 표시 정책 (PG-DTC-USAGE-001)

• 기준 시점 표시 (PI-DTC-USAGE-001-01)
- 사용량과 잔여량은 기준 시각, 집계 지연 가능성, 조회 기간을 함께 표시해야 한다.
• 구성별 잔량 분리 (PI-DTC-USAGE-001-02)
- 기본 제공량, 선물 수신량, 리필/충전량, 공유 데이터, 로밍 데이터는 같은 합계 안에서도 구분 표시한다.
• 단위 표준화 (PI-DTC-USAGE-001-03)
- 데이터 잔량은 GB 기준을 기본으로 하고 1GB 미만은 MB를 병기한다.
- 통화와 문자는 분·건 단위로 표시한다.
• 소진 순서 표시 (PI-DTC-USAGE-001-04)
- 기본 제공량, 선물 수신량, 리필/충전량, 로밍 데이터 등 차감 우선순위를 고객이 볼 수 있게 표시한다.

3) 실시간성·집계 지연 정책 (PG-DTC-REAL-001)

• 갱신 주기 고지 (PI-DTC-REAL-001-01)
- 실시간성 정보는 갱신 주기와 마지막 갱신 시각을 함께 제공한다.
• 지연 상태 표시 (PI-DTC-REAL-001-02)
- 집계 지연이 발생하면 고객 표시 상태는 처리 중 또는 반영 지연으로 안내하고, 내부 상태는 사용량 집계 중·외부 결과 대조·운영 보정 필요로 구분해 저장한다.
• 미반영 항목 안내 (PI-DTC-REAL-001-03)
- 로밍, 외부 제휴, 공유 데이터처럼 반영이 늦을 수 있는 항목은 미반영 가능성을 별도로 안내한다.
• 경량 조회 기준 (PI-DTC-REAL-001-04)
- 저속/해외 환경용 경량 조회 화면은 잔여량, 소진 위험, 충전 CTA, 기준 시각만 우선 제공하고 상세 그래프는 네트워크가 안정된 뒤 조회하게 한다.

4) 사용 패턴·소진 예측 정책 (PG-DTC-PRED-001)

• 추이 산정 기간 (PI-DTC-PRED-001-01)
- 예상 소진 시점은 최근 7일 사용 패턴을 기본으로 산정하고 데이터가 부족하면 이번 달 평균을 보조 기준으로 사용한다.
• 급증 감지 기준 (PI-DTC-PRED-001-02)
- 일 사용량이 최근 7일 평균의 150% 이상이면 급증 상태로 표시하고 주요 통화·데이터 사용 패턴 정보와 함께 원인 확인 경로를 제공한다.
• 예상 소진 산정 (PI-DTC-PRED-001-03)
- 예상 소진일은 잔여량과 최근 사용량을 기준으로 산정하되 오차 가능성을 함께 안내한다.
• 대안 우선순위 (PI-DTC-PRED-001-04)
- 부족 위험 시 보유 쿠폰 사용, 리필, 선물 요청, 충전, 요금제 변경 순으로 고객 비용과 즉시성을 고려해 제안한다.

5) 알림·딥링크 정책 (PG-DTC-ALERT-001)

• 알림 발송 조건 (PI-DTC-ALERT-001-01)
- 데이터·통화·문자 잔여량 20%, 10%, 소진, 로밍 소진 임박, 선물 완료, 충전 실패, 자동 신청 예정은 알림 발송 후보로 분류한다.
• 수신 채널 선택 (PI-DTC-ALERT-001-02)
- 고객은 Push, 문자, 인앱 알림 중 수신 채널과 임계치를 선택할 수 있다.
• 딥링크 연결 (PI-DTC-ALERT-001-03)
- 소진 또는 로밍 부족 알림은 데이터 선물하기, 데이터 충전하기, 리필 쿠폰, 요금제 변경 화면으로 바로 연결하는 딥링크를 포함한다.
• 알림 이력 저장 (PI-DTC-ALERT-001-04)
- 알림 발송 대상, 발송 시각, 이벤트 유형, 진입 결과는 이력으로 저장한다.

6) 초과 사용 차단 정책 (PG-DTC-LIMIT-001)

• 차단 설정 조건 (PI-DTC-LIMIT-001-01)
- 초과 사용 차단은 대상 회선, 고객 권한, 현재 상품 상태, 이미 진행 중인 변경 여부를 확인한 뒤 허용한다.
• 차단 해제 조건 (PI-DTC-LIMIT-001-02)
- 차단 해제는 본인 또는 위임 권한 확인 후 허용하며 미성년·보호자 설정은 권한 범위에 따른다.

- 자녀 회선 차단은 보호자 권한이 확인된 경우에만 허용하고 자녀 회선에 적용 결과를 안내한다.
• 차단 결과 고지 (PI-DTC-LIMIT-001-04)
- 차단 설정 또는 해제 완료 시 적용 시각, 대상 회선, 제한 범위, 되돌리기 가능 여부를 안내한다.

• 선물 이력 조회 (PI-DTC-GIFT-001-04)

• 쿠폰 유효기간 (PI-DTC-CHARGE-001-02)
- 데이터·통화 쿠폰은 만료일과 소멸 예정 상태를 표시하고 소멸 임박 시 우선 노출한다.
• 취소 불가 고지 (PI-DTC-CHARGE-001-03)
- 충전·쿠폰 사용 후 취소가 불가능한 경우 최종 실행 전 확인 단계에서 명확히 고지한다.
• 적용 결과 안내 (PI-DTC-CHARGE-001-04)
- 충전 완료 후 적용 유형, 적용 시각, 증가한 잔여량, 남은 쿠폰 수, 다음 행동을 안내한다.

• 자동 실행 예정 (PI-DTC-AUTO-001-02)
- 자동 신청 예정 또는 자동 선물 실행 전 고객에게 예정일, 대상, 용량, 해제 가능 경로를 알림으로 제공한다.
• 자동 해제 감지 (PI-DTC-AUTO-001-03)
- 요금제 변경, 권한 변경, 대상 조건 미충족으로 자동 설정이 해제되면 원인과 재등록 경로를 안내한다.

10) 가족·위임 권한 정책 (PG-DTC-FAMILY-001)

• 위임 권한 등록 (PI-DTC-FAMILY-001-01)
- 가족·보호자 위임은 대상 회선, 권한 범위, 유효 기간, 해지 조건을 확인한 뒤 등록한다.
• 정보 노출 범위 (PI-DTC-FAMILY-001-02)
- 보호자·가족 대표에게는 필요한 잔량과 위험 상태만 노출하고 상세 통화내역 등 민감 정보는 제한한다.
• 대리 선물 인증 (PI-DTC-FAMILY-001-03)
- 미성년 자녀 회선 대리 선물은 가족 관계 또는 법정대리인 기준의 대리 인증을 통과해야 한다.
• 가족 잔량 비교 (PI-DTC-FAMILY-001-04)
- 가족·다회선 비교는 회선별 잔량, 소진 위험, 선물 가능성, 차단 상태를 한 화면에서 제공한다.
• 셰어링 USIM 상태·해지 (PI-DTC-FAMILY-001-05)
- T 데이터 셰어링 USIM 상태·해지는 공유 데이터 차감, 부가회선 사용 가능 여부, 해지 완료·진행 상태, 상담·매장 연결 필요 여부를 함께 안내해야 한다.

11) 로밍 데이터·쿠폰 정책 (PG-DTC-ROAM-001)

• 로밍 잔량 표시 (PI-DTC-ROAM-001-01)
- 로밍 잔량은 국내 데이터와 분리하고 기준 시각, 국가/권역, 상품별 사용 가능 여부를 표시한다.
• 로밍 충전 연결 (PI-DTC-ROAM-001-02)
- 로밍 소진 알림과 위젯은 로밍 쿠폰 등록 또는 추가 충전 화면으로 연결한다.
• 로밍 쿠폰 등록 (PI-DTC-ROAM-001-03)
- 로밍 쿠폰 자동 등록은 구매 시 고객 선택을 받아 처리하고 등록 결과를 알림으로 안내한다.
• 로밍 제한 안내 (PI-DTC-ROAM-001-04)
- 로밍 중 국내 데이터 선물 수신 가능 여부와 로밍 데이터 선물 제한 조건을 선물 화면에서 안내한다.

12) 이력·되돌리기 정책 (PG-DTC-HIST-001)

• 변경 이력 조회 (PI-DTC-HIST-001-01)
- 선물, 충전, 차단, 자동 설정, 위임 권한 변경 이력은 기간과 유형별로 조회할 수 있다.
• 되돌리기 허용 (PI-DTC-HIST-001-02)
- 되돌리기는 정책상 취소 가능한 설정 변경에 한해 허용하고 이미 적용된 충전·쿠폰 사용은 취소 불가로 안내한다.
• 발송 진행률 (PI-DTC-HIST-001-03)
- 대용량 또는 다수 선물은 수신자별 진행률과 완료/실패 상태를 표시한다.
• 처리 결과 저장 (PI-DTC-HIST-001-04)
- 조회·변경·선물·충전·알림 결과는 고객 확인과 상담 문맥 전달을 위해 이력으로 저장한다.

13) 예외·실패 복구 정책 (PG-DTC-EXC-001)

• 재시도 기준 (PI-DTC-EXC-001-01)
- 조회 실패, 충전 실패, 선물 실패는 동일 세션에서 최대 3회까지 재시도를 허용하고 초과 시 대체 경로를 안내한다.
• 실패 사유 안내 (PI-DTC-EXC-001-02)
- 실패 안내는 대상 조건, 한도, 기간, 권한, 연계 상태 중 어떤 조건이 원인인지 표시해야 한다.
• 보정 요청 (PI-DTC-EXC-001-03)
- 반영 지연 또는 집계 오류가 기준 시간을 초과하면 운영 보정 요청으로 분류한다.
• 상담 전환 조건 (PI-DTC-EXC-001-04)
- 반복 실패, 권한 불일치, 외부 연계 지연, 보정 필요 상태는 상담 문맥을 포함해 전환한다.

14) BSS·연계 반영 정책 (PG-DTC-BSS-001)

• 사용량 원장 회신 (PI-DTC-BSS-001-01)
- 사용량 연계 시스템은 사용량, 잔여량, 기준 시각, 차감 순서, 반영 상태를 회신한다.
• 충전·선물 반영 (PI-DTC-BSS-001-02)
- 충전·선물 적용 결과는 발신자 차감, 수신자 증가, 쿠폰 차감, 적용 시각을 함께 저장한다.
• 외부 결과 대조 (PI-DTC-BSS-001-03)
- 외부 제휴·로밍 결과와 BSS 반영 결과가 다르면 고객에게 완료로 표시하지 않고 반영 지연 상태로 둔다.
• 정합성 불일치 처리 (PI-DTC-BSS-001-04)
- 사용량 또는 잔여량 불일치가 감지되면 완료로 표시하지 않고 운영자 확인 대상으로 분류하며, 통합 장애가 알림·충전·선물·로밍 조회 등 여러 채널 행동에 확대될 수 있음을 고객 안내와 운영 보정 기준에 반영한다.

15) 개인정보·통화내역 보호 정책 (PG-DTC-PRIV-001)

• 본인확인 기준 (PI-DTC-PRIV-001-01)
- 통화내역, 타 회선 정보, 선물·차단·위임 설정은 본인확인 또는 위임 권한 확인을 통과해야 한다.
• 마스킹 기준 (PI-DTC-PRIV-001-02)
- 통화 상대, 전화번호, 가족 회선 정보, 제휴 식별자는 최소 식별 가능 범위만 노출한다.
• 대리 권한 제한 (PI-DTC-PRIV-001-03)
- 보호자·대리 권한은 조회·차단·선물 가능 범위를 분리하고 민감 통화내역 조회는 제한한다.
• 민감 이력 보호 (PI-DTC-PRIV-001-04)
- 조회와 설정 변경 이력은 조회자, 시각, 대상 회선, 처리 목적을 저장하고 권한 없는 접근을 차단한다.

16) 운영 정책·모니터링 정책 (PG-DTC-OPS-001)

• 운영 임계치 관리 (PI-DTC-OPS-001-01)
- 운영자는 상품 유형, 고객군, 회선 유형별 알림 임계치와 문구를 관리할 수 있다.
• 이상치 모니터링 (PI-DTC-OPS-001-02)
- 사용량 급증, 집계 누락, 알림 미발송, 자동 리필 실패, 반복 실패 고객군을 모니터링한다.
• 정책 변경 영향 (PI-DTC-OPS-001-03)
- 선물 한도, 충전 조건, 자동 설정 기준 변경 시 대상 고객과 기존 설정 영향 범위를 확인한다.
• 운영 보정 이력 (PI-DTC-OPS-001-04)
- 운영 보정은 사유, 대상 회선, 보정 전후 상태, 승인자, 고객 고지 여부를 이력으로 저장한다.

최종 점검 기준

나의데이터통화 정책서 제출 전 점검
1. 데이터·통화 범위가 잔여량 조회에 머물지 않고 충전·리필·선물·차단·로밍·실패 복구까지 연결되어 있는가.
2. 유즈케이스가 고객 또는 가족 대표·운영자의 완결 업무 단위로 정의되었는가.
3. 프로세스가 사용량 이해, 조건 판정, 실행, 결과 반영, 실패 복구 순서로 연결되어 있는가.
4. 기능 세부 구성이 대상 식별, 상태 조회, 조건 판정, 결과 반영 같은 실제 처리 단위로 작성되어 있는가.
5. 상태 전이표의 모든 상태가 상태 코드 목록에 존재하고 처리 기준 및 후속 처리가 비어 있지 않은가.
6. 선물·충전·쿠폰 사용 정책에 한도, 대상, 유효기간, 취소 불가, 실패 재시도 기준이 있는가.
7. 가족·보호자 위임과 개인정보 최소 노출 기준이 포함되어 있는가.
8. BSS·사용량 연계와 외부 제휴·로밍 결과의 반영 지연, 정합성 불일치, 통합 장애 영향 확대 기준이 포함되어 있는가.
9. 요구사항 70건이 프로세스, 기능, 정책 중 하나 이상에 매핑되어 있는가.
10. 정책 상세에 실제 조건, 시간, 상태, 제한, 고지, 이력 기준이 포함되어 있는가.
11. 요구사항 추적성
상세 요구사항은 유즈케이스, 상태, 프로세스, 기능, 정책 또는 정책 항목 중 하나 이상으로 추적 가능해야 한다.
12. 개발·QA 전환성
각 정책 항목은 테스트 조건, 기대 결과, 실패·복구 경로로 전환 가능해야 한다.
13. 책임 경계 정합성
FO, BSS, 연계 시스템, 외부 주체의 판단·저장·회신 책임이 혼재되지 않아야 한다.
