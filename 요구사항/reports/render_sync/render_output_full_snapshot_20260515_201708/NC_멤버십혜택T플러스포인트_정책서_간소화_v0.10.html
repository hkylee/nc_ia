1. 개요

가. 범위

• 본 정책서는 통합채널에서 제공하는 멤버십 혜택, 등급, 제휴처 사용, 정기 프로모션, T 플러스포인트 적립·사용·소멸·이력·선물·통합·정정 기준을 정의한다.

• 고객이 혜택을 탐색하고 사용 가능 상태를 판단하며 바코드 또는 매직바코드로 혜택을 사용하고 오류 발생 시 복구 또는 상담 전환할 수 있는 기준을 포함한다.

• 멤버십 등급 산정 기준, 변경 적용 시점, 체감가치, 로열티 배지, 등급 달성 조건, 데이터 최신성 안내와 이력 조회 기준을 포함한다.

• 운영자가 혜택 프로그램, 제휴처 매핑, 등급 로직, 바코드 정책, 포인트 적립·회수·정산, 오류 모니터링, 성과 리포트를 관리하는 기준을 포함한다.

• 외부 쿠폰 코드 재고 상세, 결제 승인, 주문·배송, 포인트 회계 정산 세부 로직은 후속 산출물 또는 인접 정책서에서 상세화한다.

• 고객이 내 혜택 한눈에 보기, 쿠폰 일괄 다운로드, 통합 바코드 즉시 호출, 포인트 사용 사전 세팅, 결제 직후 절약 피드백을 통해 현장 사용 경험을 완료하는 기준을 포함한다.

• 브랜드 친밀도, 관심 브랜드 알림, 소셜 티커, AI 소비 분석 리포트, 공유 지갑처럼 리워드 재방문과 관계 기반 경험을 만드는 기준을 포함한다.

• 핵심 정책 질문은 누가 어떤 등급과 상태에서 혜택을 언제 몇 번까지 사용할 수 있는지, 실패하면 어떤 복구와 상담 전환을 제공하는지, 포인트와 혜택 이력을 어디에 남기는지로 정의한다.

• 고객 표시 상태와 시스템 내부 상태는 분리한다.
고객에게는 사용 가능, 조건 미충족, 사용 완료, 정지·회수, 재연동 필요로 안내하고 내부적으로는 BSS 등급 반영, 제휴처 승인, 포인트 정산, 알림 발송 이력을 별도 관리한다.

• FO, BSS, 제휴처, 포인트 시스템, 알림 시스템의 책임 경계를 구분하고 통합 바코드·공유 지갑·포인트 동기화 장애가 다른 채널 사용 가능 상태로 확산되지 않게 제한한다.

• BSS는 고객·회선·상품·납부·가입 상태 기준 정보를 제공하며, 채널은 BSS 기준을 직접 변경하지 않고 멤버십 연계 결과와 정책 판단 결과를 고객에게 안내한다.

• 채널 업무 시스템은 고객 표시 상태, 통합 바코드 호출, 포인트 사전 세팅, 알림과 재진입 경로를 책임지고 멤버십 연계 시스템은 등급·혜택 판정, 포인트 원장 반영, 제휴처 승인·오류 회신을 책임진다.

• API 필드, DB 컬럼, 운영자 화면 상세, 배치 설계는 제외하되 고객 경험과 정책 판단에 필요한 조건, 상태, 제한, 고지, 이력 기준은 포함한다.

나. 설계 원칙

• 통합 리워드 관점: 고객이 할인, 포인트, 쿠폰, 이용권, 등급 혜택을 하나의 리워드 체계로 이해하게 한다.

• 지금 쓸 수 있는 가치 우선: 조건보다 고객이 받을 가치와 즉시 사용 가능 여부를 먼저 보여준다.

• 현장 사용 최소 동선: 통합 바코드, 0 Depth 호출, 포인트 사전 세팅, 사용 실패 복구를 통해 매장 현장에서 고객 조작과 구두 설명을 줄인다.

• 셀프 사용·복구 완결: 바코드, 포인트, 혜택 사용 실패 시 고객이 재시도, 대체수단, 상담 전환을 스스로 선택하게 한다.

• 관계 기반 리텐션: 브랜드 친밀도, 관심 브랜드, 소셜 티커, 절약 리포트는 고객에게 혜택 사용 이유와 다음 행동을 설명하는 기준으로만 사용한다.

• 등급·포인트 신뢰성: 등급 산정 기준, 포인트 잔액, 소멸, 이력의 최신성과 추정·확정 구분을 명확히 한다.

• 운영 통제와 실험 가능성: 운영자는 프로그램 편성, 제휴처 정책, 바코드 예외, 성과 측정을 변경 이력과 함께 관리한다.

• 인접 정책 경계 유지: 외부 쿠폰, 결제 할인, 이벤트 미션의 세부 정책은 해당 정책서와 연결하되 본 문서는 멤버십 리워드 실행 기준에 집중한다.

1) 멤버십 리워드 범위·용어 정책 (PG-MBP-SCOPE-001)

• 리워드 범위 (PI-MBP-SCOPE-001-01)
- 본 문서의 멤버십 리워드는 등급 혜택, 제휴처 할인, 쿠폰, 이용권, 응모권, T 플러스포인트, 정기 프로모션을 포함한다.
- 외부 쿠폰의 코드·재고 운영 상세와 결제 할인 산정은 인접 정책서에서 다룬다.
• 표준 유형 (PI-MBP-SCOPE-001-02)
- 혜택 유형은 할인, 적립, 무료 제공, 쿠폰, 이용권, 응모권, 포인트, 배지로 구분한다.
- 고객 노출 라벨은 8개 유형 중 하나를 필수로 사용한다.
• 고객 문구 분리 (PI-MBP-SCOPE-001-03)
- 내부 관리 용어와 고객 노출 용어는 분리한다.
- 운영 용어가 고객 화면에 노출되는 경우 검수에서 보류하고 고객 이해 문구로 변환한다.

2) 통합 혜택 허브 전시 정책 (PG-MBP-HUB-001)

• 허브 우선순위 (PI-MBP-HUB-001-01)
- 혜택 허브는 지금 사용 가능, 내 주변, 자주 쓰는 혜택, 마감 임박, 높은 체감가치 순으로 우선 노출한다.
- 고객별 개인화 결과가 없으면 등급 대표 혜택과 정기 프로모션을 대체 노출한다.
• 로그인 전 가치 (PI-MBP-HUB-001-02)
- 로그인 전에는 대표 혜택, 예상 절감 예시, 로그인 후 가능한 행동을 먼저 제공한다.
- 개인화 혜택과 잔여 한도는 로그인 이후에만 표시한다.
• 조건 분리 (PI-MBP-HUB-001-03)
- 혜택 상세는 핵심 가치와 사용 행동을 먼저 노출하고 조건, 주의사항, 제한 사유는 별도 영역으로 분리한다.
- 단, 사용 불가 조건은 접기 영역이 아니라 즉시 보이는 상태 문구로 안내한다.

- 고객에게 사유, 영향, 이의제기 경로를 안내한다.
• 상담 fallback (PI-MBP-FAIL-001-03)
- 가입 상태 불일치, 제휴처 사용 불가, 본인확인 실패, 재발급 예외는 현재 멤버십 문맥을 유지해 상담 또는 문의 채널로 연결한다.
• 중복 요청·멱등 처리 (PI-MBP-FAIL-001-04)
- 바코드 재호출, 포인트 사용 사전 세팅, 쿠폰 일괄 다운로드, 공유 지갑 동기화 요청은 동일 고객·동일 혜택·동일 거래 기준으로 중복 요청을 식별한다.
- 이미 성공한 요청은 멱등 처리하고 고객에게 기존 결과를 재표시하며, 실패 요청은 최대 1회 자동 재시도 후 대체 행동을 안내한다.

12) 멤버십 알림·재방문 정책 (PG-MBP-NOTI-001)

• 수신 설정 (PI-MBP-NOTI-001-01)
- 고객은 등급 변경, 소멸 예정 포인트, 신규 혜택, 기간 한정 혜택 알림의 수신 여부와 채널, 빈도를 설정할 수 있다.
• 발송 트리거 (PI-MBP-NOTI-001-02)
- 알림은 사전 예고, 오픈 직후, 마감 임박, 수령 완료, 사용 만료 임박, 가입 완료 조건을 트리거로 발송한다.
- 동일 항목 중복 발송은 제한한다.
• 재연동 안내 (PI-MBP-NOTI-001-03)
- 번호이동, 요금제 변경, 기기변경으로 멤버십 재연동이 필요한 경우 단계별 복구 절차와 재연동 완료 기준을 알림과 화면에서 안내한다.
• 관심 브랜드 알림 (PI-MBP-NOTI-001-04)
- 세그먼트 기반 맞춤 혜택 노출 및 관심 브랜드 알림은 고객이 저장한 브랜드, 나이, 지역, 소비 습관, 수신 동의를 충족한 경우에만 발송한다.
- 관심 브랜드 알림은 시작 알림, 마감 임박 알림, 혜택 변경 알림으로 구분하고 수신 거부 시 즉시 중단한다.

13) 혜택 프로그램 운영 정책 (PG-MBP-OPS-001)

• 프로그램 편성 (PI-MBP-OPS-001-01)
- 운영자는 혜택 프로그램의 노출 기간, 대상 조건, 혜택 구성, 우선순위, 중복 노출 제한, 연계 프로모션을 편성 캘린더로 관리한다.
• 제휴처 매핑 (PI-MBP-OPS-001-02)
- 제휴 브랜드, 가맹점, 카테고리별로 연결되는 혜택 프로그램을 관리한다.
- 동일 제휴처가 복수 프로그램에 참여하면 우선 적용 프로그램과 예외 규칙을 설정한다.
• 개인화 우선 노출 (PI-MBP-OPS-001-03)
- 최근 사용, 자주 사용, 보유 상품, 위치, 시간대 기준으로 개인화 우선 노출을 적용한다.
- 고객이 비선호한 혜택은 후순위로 배치한다.
• 세그먼트 기반 맞춤 혜택 (PI-MBP-OPS-001-04)
- 세그먼트 기반 맞춤 혜택은 나이, 지역, 소비 습관, 최근 사용, 관심 브랜드를 기준으로 산정한다.
- 운영자는 세그먼트별 노출 우선순위, 제외 대상, 성과 지표, 변경 이력을 관리하고 고객에게 불리한 제한 조건으로 사용하지 않는다.

14) 바코드·제휴처 사용 정책 운영 (PG-MBP-BAROPS-001)

• 카드 유형 정책 (PI-MBP-BAROPS-001-01)
- 운영자는 일반 카드 바코드, 매직바코드, 카드번호의 사용 가능 제휴처, 적용 조건, 제외 조건, 유효시간을 관리한다.
• 매직바코드 채번 (PI-MBP-BAROPS-001-02)
- 매직바코드 채번은 발급 대상, 발급 조건, 유효시간, 재발급 가능 여부, 사용 완료 처리 기준을 충족해야 한다.
- 정책 변경 시 영향 대상과 적용 시점을 확인한다.
• 실패 예외 정책 (PI-MBP-BAROPS-001-03)
- 운영자는 바코드 호출 실패, 스캔 실패, 유효시간 만료, 네트워크 오류, 제휴처 사용 불가 유형별 안내 문구와 대체 수단 제공 기준을 관리한다.
• 브랜드 Identity 반영 (PI-MBP-BAROPS-001-04)
- 바코드 크게 보기 팝업 내 브랜드별 Identity 반영은 제휴사별 브랜드 컬러·로고가 운영 검수와 승인 상태를 통과한 경우에만 노출한다.
- 브랜드 리소스가 누락되거나 만료되면 기본 통합 바코드 디자인으로 대체하고 오류 이력을 남긴다.

15) 포인트·혜택 정산·모니터링 정책 (PG-MBP-SETTLE-001)

• 포인트 운영 (PI-MBP-SETTLE-001-01)
- 운영자는 포인트 적립 규모, 조건, 한도, 지급 예정일을 설정하고 잘못 지급된 포인트는 회수 또는 정정 처리한다.
• 정산 기준 (PI-MBP-SETTLE-001-02)
- 포인트 사용과 제휴처 혜택 사용은 사용 완료, 취소, 회수, 오류 보정 상태를 기준으로 정산한다.
- 불일치 건은 정산 보류로 분리한다.
• 오류 모니터링 (PI-MBP-SETTLE-001-03)
- 운영자는 카드 호출 수, 사용 성공률, 실패 유형, 제휴처별 오류, 매직바코드 사용량, 재발급 요청량을 조회하고 반복 오류 제휴처를 식별한다.

- FO는 고객 표시 상태, 바코드·포인트 사용 요청, 안내 문구와 재시도 경로를 책임진다.
- BSS와 포인트 시스템은 등급·포인트 원장, 사용·회수·정산 반영을 책임지고, 제휴처 시스템은 사용 승인·거절·취소 결과를 회신한다.

• 통합 장애 영향 제한 (PI-MBP-SETTLE-001-05)
- 통합 바코드, 공유 지갑, 포인트 동기화, 제휴처 승인 연동 장애가 발생하면 장애 범위와 영향을 사용수단별로 분리한다.
- 일반 카드 사용이 가능한 경우 매직바코드 또는 포인트 기능 장애가 전체 멤버십 사용 불가로 확산되지 않게 대체 수단을 우선 안내한다.
