1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 쿠폰, 이용권, 포인트, 선물성 혜택 자산을 한곳에서 확인하고 사용할 수 있는 통합 자산함의 업무 기준을 정의한다.

• 대상 업무는 자산함 진입, 목록 정렬, 자산 원천별 조회, 상태 구분, 수령·발급 상태 관리, 제휴처 인증·계정 연동, 사용 진입, 만료·소멸 아카이브, 변화 이력, 베네핏 가치 요약, 공유 카드, 누락·불일치 정정 요청을 포함한다.

• 운영 범위는 자산 유형·탭·상태·표시 우선순위, 개인 혜택 메시지·배너 우선 노출, 쿠폰 기간 상태 자동 산출, 사용기간 유형, 쿠폰 유형별 입력 항목, 선택항목 기반 일괄 처리 제어 기준을 포함한다.

• 대상 채널은 앱·웹 FO이며, 혜택 원장 시스템과 제휴사 시스템의 판정·결과 회신은 고객 표시와 정책 판단 기준으로 포함한다.

• 외부 쿠폰 발급 조건, 상품 가입·변경, 멤버십 포인트 적립 상세, 외부 BP 계약 관리는 별도 정책서 범위로 두되 통합 자산함 표시와 사용 가능성에 영향을 주는 결과 기준은 포함한다.

• 화면 UI 상세, API 필드, DB 컬럼, 제휴 계약 세부 조건은 제외하고 고객 행동, 상태, 사용 가능성, 이력, 원장 반영 기준을 정책으로 정의한다.

• 핵심 정책 질문은 누가 이 업무를 시작할 수 있는지, 어떤 조건에서 제한되는지, 중복 요청을 어떻게 처리하는지, 실패 시 어떤 상태와 후속 조치로 복구하는지를 기준으로 둔다.

• 고객 표시 상태는 고객이 이해해야 하는 진행·완료·제한 상태로, 시스템 내부 상태는 BSS·연계 시스템의 판정·저장·회신 상태로 구분한다.

• 개발·QA는 유즈케이스 ID, 프로세스 ID, 기능 ID, 정책/정책항목 ID를 기준으로 정상·제한·실패·복구 테스트 조건과 기대 결과를 도출한다.

• 요구사항 추적 기준은 최신 상세 요구사항명·상세 요구사항 설명 31건을 기준으로 하며, 각 요구사항은 통합 자산함 조회, 상태 구분, 사용 진입, 가치 산정, 정정 요청 중 하나 이상의 구조에 연결한다.

나. 설계 원칙

• 고객 자산함 중심: 쿠폰과 이용권을 원천별 목록이 아니라 고객이 지금 사용할 수 있는 자산과 다음 행동 중심으로 구성한다.

• 상태 기반 행동 연결: 사용 가능, 등록 필요, 오픈 예정, 조건 미충족, 만료 예정 상태마다 고객에게 다른 행동과 안내를 제공한다.

• 가치 산정 투명성: 절감액과 누적 베네핏은 실제 반영된 항목과 추정 항목을 구분하고 산정 근거를 함께 제공한다.

• 원장·제휴 결과 우선: 자산 보유와 사용 결과는 BSS 원장과 제휴사 응답을 기준으로 확정하고 충돌 시 정정 경로로 전환한다.

• 손실 예방과 복구 보장: 만료 예정과 놓친 가치를 먼저 알려주고 누락·불일치가 발생하면 재동기화와 상담 전환을 제공한다.

• 공유 안전성: 내 베네핏 공유는 개인정보와 과장 표시를 막기 위해 고객 선택형 노출 수준과 상태 재확인 기준을 적용한다.

1) BSS·제휴 연계 반영 정책 (PG-CPN-BSS-001)

• 원장 우선 기준 (PI-CPN-BSS-001-001)
- 자산 보유 여부, 사용 완료 여부, 유효기간, 남은 횟수는 BSS와 혜택 원장 값을 우선한다.
- 채널 캐시는 고객 표시 속도 개선 목적으로만 사용한다.
• 제휴 응답 반영 (PI-CPN-BSS-001-002)
- 제휴사 응답은 사용처, 제시수단, 사용 성공·실패 결과에 한해 반영한다.
- 제휴 응답과 원장이 충돌하면 동기화 불일치로 전환한다.
• 장애 시 상태 기준 (PI-CPN-BSS-001-003)
- BSS 또는 제휴 연계 장애 시 신규 사용 요청은 제한하고 기존 보유 목록은 마지막 정상 조회 시점과 함께 표시한다.

2) 선택항목 일괄 처리 제어 정책 (PG-CPN-BULK-001)

• 선택 기준 (PI-CPN-BULK-001-001)
- 삭제, 사용중지, 배포 같은 일괄 처리는 1건 이상 선택된 경우에만 버튼을 활성화한다.
• 상태 혼합 제한 (PI-CPN-BULK-001-002)
- 선택 항목에 배포 완료와 검수 대기 상태가 혼합되면 삭제와 배포 버튼을 비활성화하고 가능한 처리만 안내한다.
• 처리 이력 (PI-CPN-BULK-001-003)
- 일괄 처리 결과는 성공, 실패, 일부 실패로 구분하고 선택 건수, 처리 건수, 실패 사유, 처리자를 이력으로 저장한다.

3) 항목 카드 요약 표시 정책 (PG-CPN-CARD-001)

• 필수 요약 필드 (PI-CPN-CARD-001-001)
- 항목 카드는 자산명, 유형, 상태, 유효기간 또는 다음 결제일, 사용처, 남은 횟수·한도, 수령·이행 방식, 판매·제휴사를 필수 또는 조건부 필드로 표시한다.
• CTA 표시 기준 (PI-CPN-CARD-001-002)
- CTA는 사용 가능 상태에는 사용하기를 우선 제공하고, 등록 필요 상태에는 등록·수령, 조건 미충족 상태에는 조건 확인, 만료 예정 상태에는 우선 사용 또는 리마인드로 제한한다.
• 판매·제휴 정보 표시 (PI-CPN-CARD-001-003)
- 판매·제휴 정보는 고객이 사용 책임 경계를 이해해야 하는 경우 표시한다.
- 제휴사명은 고객 노출 명칭으로 표시하고 내부 계약명은 노출하지 않는다.

4) 만료·소멸·리마인드 정책 (PG-CPN-EXP-001)

• 만료 예정 기준 (PI-CPN-EXP-001-001)
- 만료 예정은 기본 7일 이내로 표시하고, 유효기간이 24시간 이내인 자산은 당일 만료 배지를 추가한다.
• 소멸 사유 저장 (PI-CPN-EXP-001-002)
- 소멸 사유는 유효기간 경과, 조건 해제, 회수, 제휴 종료, 원장 정정 중 하나로 저장한다.
- 회원별 만료 혜택 ID 목록 조회 결과는 고객 표시용 아카이브와 운영 정산 확인용 이력으로 분리해 보관한다.
- 고객에게는 고객 이해 가능한 문구로 변환해 표시한다.
• 리마인드 발송 기준 (PI-CPN-EXP-001-003)
- 리마인드는 고객 동의와 알림 수신 설정이 허용된 경우에만 발송한다.
- 같은 자산의 만료 리마인드는 하루 1회, 최대 3회로 제한한다.

5) 사용·변화 이력 정책 (PG-CPN-HIST-001)

• 이력 대상 범위 (PI-CPN-HIST-001-001)
- 이력 대상은 쿠폰 발급, 사용, 회수, 만료, 선물 수락·전송, 포인트 적립·차감, 이용권 등록·해지, 정정 요청 결과를 포함한다.
• 기간 필터 기준 (PI-CPN-HIST-001-002)
- 이력 필터는 기본 3개월, 6개월, 1년, 직접 지정 기간을 제공하고 유형, 원천, 제휴처, 상태 기준 필터를 함께 제공한다.
• 재사용 연결 기준 (PI-CPN-HIST-001-003)
- 사용 이력에서 동일 유형의 재사용 가능 자산이 있으면 상세 이동 또는 사용 진입을 제공한다.
- 만료·소멸 이력은 재사용 CTA를 제공하지 않는다.

6) 제휴처 인증·계정 연동 정책 (PG-CPN-LINK-001)

• 연동 필요 표시 (PI-CPN-LINK-001-001)
- 제휴처 인증 또는 계정 연동이 필요한 혜택은 연동 필요 또는 인증 필요 배지를 표시하고 사용 CTA보다 인증 이동을 우선한다.
• 인증 이동 기준 (PI-CPN-LINK-001-002)
- 인증 이동 전 제휴처명, 전달되는 최소 정보, 복귀 가능 경로를 안내한다.
- 인증 미완료 복귀 시 기존 자산함 상태를 유지한다.
• 연동 실패 처리 (PI-CPN-LINK-001-003)
- 제휴처 연동 실패는 조건 미충족 또는 발급실패와 구분한다.
- 2회 연속 실패하면 상담 또는 제휴사 안내 경로를 제공한다.

7) 운영 매핑·검수·배포 정책 (PG-CPN-OPS-001)

• 분류 매핑 승인 (PI-CPN-OPS-001-001)
- 운영자가 변경한 유형 분류, 상태값, 배지, 정렬 우선순위는 검수 승인 후 배포한다.
- 긴급 회수는 승인자 1인 이상 확인 후 즉시 반영할 수 있다.
• 산정 로직 검수 (PI-CPN-OPS-001-002)
- 가치 산정 로직은 배포 전 샘플 고객군 10건 이상으로 중복 합산, 제외 기준, 표시 문구를 검증한다.
• 변경 이력 기준 (PI-CPN-OPS-001-003)
- 운영 변경 이력은 변경자, 변경 사유, 적용 채널, 적용 시작일, 롤백 가능 여부를 저장한다.
- 변경 이력은 최소 1년 보관한다.

8) 쿠폰 기간·상태 자동 산출 정책 (PG-CPN-PERIOD-001)

• 발행기간 상태 산출 (PI-CPN-PERIOD-001-001)
- 쿠폰 발행기간 전에는 오픈 예정, 발행기간 중에는 등록 가능, 발행기간 종료 후 미등록 건은 종료 또는 등록 불가로 산출한다.
• 사용기간 유형 (PI-CPN-PERIOD-001-002)
- 사용기간은 절대기간과 발행일 기준 기간으로 구분한다.
- 발행일 기준 기간은 고객 수령 또는 발급 완료일을 기준일로 계산한다.
• 만료 자동 전환 (PI-CPN-PERIOD-001-003)
- 사용기간 종료 시 미사용 쿠폰은 만료·소멸 상태로 전환하고 소멸 사유와 전환 시각을 이력으로 저장한다.

9) 개인 혜택 메시지·배너 노출 정책 (PG-CPN-PERSONAL-001)

• 노출 조건 (PI-CPN-PERSONAL-001-001)
- 개인 혜택 메시지는 고객군, 등급, 회선 상태, 보유 상품, 만료 임박 여부 중 하나 이상의 조건이 충족될 때 노출한다.
• 우선순위 기준 (PI-CPN-PERSONAL-001-002)
- 만료 임박, 사용 가능, 미수령, 조건 미충족, 추천 행동 순으로 우선순위를 적용한다.
- 동일 우선순위는 최근 지급일과 고객 행동 이력을 함께 본다.
• 과다 노출 제한 (PI-CPN-PERSONAL-001-003)
- 같은 혜택의 경고 또는 배너는 하루 1회, 7일 3회 이내로 제한한다.
- 고객이 닫은 배너는 24시간 재노출하지 않는다.

10) 제시수단·사용처 안내 정책 (PG-CPN-PRESENT-001)

• 온라인 사용 기준 (PI-CPN-PRESENT-001-001)
- 온라인 사용은 서비스 내 적용형, 코드 입력형, 제휴 이동형으로 구분하고 고객에게 적용 전 예상 혜택과 적용 결과를 모두 보여준다.
• 오프라인 제시 기준 (PI-CPN-PRESENT-001-002)
- 오프라인 사용은 바코드, QR, 쿠폰 번호 중 운영자가 지정한 제시 수단을 제공한다.
- 유효 시간형 바코드는 기본 10분 단위로 갱신한다.
• 제휴사 이동 기준 (PI-CPN-PRESENT-001-003)
- 제휴사 이동형 자산은 이동 전 제휴사명, 필요 계정, 이용 제한, 복귀 경로를 안내한다.
- 제휴 응답 실패 시 원래 자산 상세로 복귀한다.

11) 개인정보·민감정보 노출 정책 (PG-CPN-PRIV-001)

• 금액·식별정보 최소 노출 (PI-CPN-PRIV-001-001)
- 자산함 목록에는 고객 식별번호, 내부 계약번호, 원장 키를 노출하지 않는다.
- 고객에게 필요한 경우 마지막 4자리 또는 별칭만 표시한다.
• 마스킹 기준 (PI-CPN-PRIV-001-002)
- 공유 카드와 외부 이동 화면에는 이름, 회선번호, 상세 금액, 쿠폰 번호 전체를 기본 노출하지 않는다.
- 고객이 선택한 경우에도 민감 식별정보는 마스킹한다.
• 공유 카드 보호 기준 (PI-CPN-PRIV-001-003)
- 제휴사 이동 시 전달 정보는 사용 검증에 필요한 최소 항목으로 제한한다.
- 전달 이력과 동의 기준은 고객 요청 시 확인 가능한 수준으로 저장한다.

12) 혜택 수령·등록·재발급 정책 (PG-CPN-RECEIVE-001)

• 수령 상태값 (PI-CPN-RECEIVE-001-001)
- 수령 상태는 수령완료, 미수령, 발급중, 발급실패로 구분한다.
- 발급중 상태가 3분을 초과하면 재조회 가능 안내를 제공한다.
• 재조회·재발급 기준 (PI-CPN-RECEIVE-001-002)
- 발급실패는 원장 또는 제휴사 실패 사유가 회신된 경우에 표시한다.
- 동일 쿠폰의 재발급은 실패 확정 후 1회 허용하고, 중복 발급 가능성이 있으면 상담 전환한다.
• 나의쇼핑 쿠폰 등록 (PI-CPN-RECEIVE-001-003)
- 로그인 고객은 쿠폰 탭에서 쿠폰을 등록하고 내 쿠폰 리스트에서 확인할 수 있다.
- 이미 등록된 쿠폰 번호는 중복 등록을 차단하고 기존 상태를 안내한다.

13) 통합 자산함 범위·경계 정책 (PG-CPN-SCOPE-001)

• 대상 자산 범위 (PI-CPN-SCOPE-001-001)
- 통합 자산함은 고객이 보유한 쿠폰, 이용권, 포인트, 선물성 자산의 조회, 사용 진입, 상태 확인, 만료·소멸 확인, 정정 요청을 포함한다.
• 인접 정책서 경계 (PI-CPN-SCOPE-001-002)
- 외부 쿠폰 발급 조건, 상품 가입 조건, 멤버십 포인트 적립 상세는 별도 정책서 기준을 따르되 통합 자산함 표시와 사용 가능성에 영향을 주는 결과값은 본 문서에 반영한다.
• 원천 구분 기준 (PI-CPN-SCOPE-001-003)
- 각 자산은 멤버십, 이벤트, 구독, 요금제, 결합, 제휴처, 선물 중 하나 이상의 원천을 가진다.
- 원천이 2개 이상이면 고객 표시에는 대표 원천 1개와 상세 원천 목록을 함께 제공한다.

14) 내 베네핏 공유 정책 (PG-CPN-SHARE-001)

• 공유 대상 기준 (PI-CPN-SHARE-001-001)
- 공유 대상은 고객이 선택한 누적 베네핏 요약, 대표 혜택 성과, 만료 예방 성과로 제한한다.
- 타인 식별 가능 정보와 원장 상세는 공유 대상에서 제외한다.
• 노출 수준 선택 (PI-CPN-SHARE-001-002)
- 고객은 금액 전체 표시, 금액 범위 표시, 금액 숨김 중 하나를 선택할 수 있다.
- 기본값은 금액 숨김으로 둔다.
• 공유 제한 기준 (PI-CPN-SHARE-001-003)
- 현재 상태가 조건 미충족, 만료·소멸, 동기화 불일치인 자산은 공유 카드에 성과로 표시하지 않는다.
- 공유 시점 기준일을 카드에 표시한다.

15) 정렬·우선순위 정책 (PG-CPN-SORT-001)

• 즉시 사용 우선 (PI-CPN-SORT-001-001)
- 기본 정렬은 현재 사용 가능한 혜택·권한, 만료 임박, 최근 지급, 등록 필요, 오픈 예정, 조건 미충족, 사용 완료, 만료·소멸 순으로 적용한다.
• 만료 임박 우선 (PI-CPN-SORT-001-002)
- 만료 임박 기준은 기본 7일 이내로 하되 자산 원천별 운영 기준이 있으면 1일 이상 30일 이하 범위에서 조정할 수 있다.
• 최근 지급 보정 (PI-CPN-SORT-001-003)
- 최근 지급 자산은 지급일 기준 7일 동안 상단 보정할 수 있다.
- 단, 즉시 사용 가능 자산과 만료 임박 자산보다 우선하지 않는다.

16) 자산 상태·탭 분류 정책 (PG-CPN-STATE-001)

• 상태 분류 값 (PI-CPN-STATE-001-001)
- 자산 상태는 사용 가능, 등록 필요, 오픈 예정, 조건 미충족, 사용 완료, 만료 예정, 만료·소멸, 동기화 불일치로 구분한다.
• 미수령·예정 구분 (PI-CPN-STATE-001-002)
- 미수령 자산은 등록 필요로, 시작일이 도래하지 않은 자산은 오픈 예정으로 표시한다.
- 사용 시작일과 종료일이 모두 있는 경우 시작일 전에는 오픈 예정, 종료 7일 전부터는 만료 예정으로 표시한다.
• 상태 의미 안내 (PI-CPN-STATE-001-003)
- 상태 의미는 고객 문구로 설명한다.
- 조건 미충족은 필요한 가입상품, 등급, 회선 상태, 동의, 제휴 연결 중 충족되지 않은 항목을 1개 이상 안내한다.

17) 누락·동기화·정정 정책 (PG-CPN-SYNC-001)

• 불일치 판정 (PI-CPN-SYNC-001-001)
- 불일치는 고객 표시 상태와 BSS 원장 또는 제휴사 응답의 상태, 유효기간, 사용 결과가 서로 다를 때 판정한다.
• 자동 재동기화 기준 (PI-CPN-SYNC-001-002)
- 자동 재동기화는 원장 재조회 1회와 제휴 응답 재확인 1회를 기본으로 수행한다.
- 두 조회가 3초 이내 완료되지 않으면 정정 요청 접수로 전환한다.
• 상담 전환 기준 (PI-CPN-SYNC-001-003)
- 자동 복구가 불가하거나 고객 증빙이 필요한 경우 상담 전환 또는 운영 확인 필요 상태로 전환한다.
- 접수 번호와 예상 처리 기준 시간을 고객에게 안내한다.

18) 쿠폰 유형·입력 항목 제어 정책 (PG-CPN-TYPE-001)

• 유형별 필수 항목 (PI-CPN-TYPE-001-001)
- 금액형, 비율형, 횟수형, 교환권형 쿠폰은 각각 금액, 할인율, 사용 횟수, 교환 대상 항목을 필수값으로 둔다.
• 입력 항목 제한 (PI-CPN-TYPE-001-002)
- 쿠폰 유형에서 사용하지 않는 항목은 입력과 저장을 차단한다.
- 이미 배포된 쿠폰의 유형 변경은 운영 검수 후 새 버전으로만 허용한다.
• 저장 전 검증 (PI-CPN-TYPE-001-003)
- 사용기간, 발행기간, 사용처, 적용 대상, 제외 조건 중 필수 조합이 누락되면 저장을 차단하고 누락 항목을 안내한다.

19) 사용 가능성·사용 진입 정책 (PG-CPN-USE-001)

• 사용 가능 판정 (PI-CPN-USE-001-001)
- 사용 가능 판정은 고객 자격, 자산 상태, 유효기간, 남은 횟수, 사용처, 제휴 응답이 모두 허용일 때만 성공으로 둔다.
• 사용 불가 사유 (PI-CPN-USE-001-002)
- 사용 불가 사유는 조건 미충족, 유효기간 만료, 남은 횟수 0회, 제휴 응답 실패, 원장 불일치 중 하나 이상으로 제공한다.
• 중복 사용 방지 (PI-CPN-USE-001-003)
- 사용 처리 중 동일 자산의 중복 사용 요청은 1건만 허용한다.
- 5초 이내 같은 요청이 반복되면 기존 요청 상태를 조회해 안내한다.

20) 베네핏 가치 산정 정책 (PG-CPN-VALUE-001)

• 절감액 산정 기준 (PI-CPN-VALUE-001-001)
- 이번 달 절감액은 실제 적용 완료된 할인, 쿠폰 사용, 포인트 사용, 제휴 혜택 중 고객 청구액 또는 사용 가치에 반영된 금액만 합산한다.
• 놓친 가치 산정 기준 (PI-CPN-VALUE-001-002)
- 놓친 가치는 만료·소멸되었거나 미사용으로 종료된 자산 중 고객이 사용할 수 있었던 조건이 확인된 항목만 추정한다.
- 조건 미충족 자산은 놓친 가치에서 제외한다.
• 산정 근거 표시 (PI-CPN-VALUE-001-003)
- 산정 근거는 원천, 적용일, 혜택 유형, 금액 또는 횟수, 제외 사유를 항목별로 제공한다.
- 추정값은 추정 표시와 산정 기준일을 함께 제공한다.
