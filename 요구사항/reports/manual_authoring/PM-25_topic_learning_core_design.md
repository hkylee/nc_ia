# PM-25 상품·서비스 혜택 이용/공유 Topic Learning & Core Design

- 혜택을 보유 목록이 아니라 사용·공유·만료·복구까지 이어지는 실행 업무로 정의했다.
- 요구사항 30건은 상세 요구사항명과 상세 요구사항 설명을 기준으로 프로세스, 기능, 정책 항목에 재구성했다.
- 액터는 고객, 공유 수신자, 운영자, 상담사, 혜택 연계 시스템, 제휴사 시스템, 채널 업무 시스템으로 제한했다.
- 모든 사람 액터 유즈케이스는 2개 이상의 프로세스로 분해했고, 상태 전이와 정책 판단 기준을 연결했다.
- 미구독·로그아웃 고객의 빈 상태는 별도 유즈케이스로 과분화하지 않고, 혜택 이용 가이드·퀵메뉴와 첫 실행 CTA 프로세스 및 빈 상태 퀵메뉴 정책 항목으로 반영했다.
- 채널 업무 시스템은 화면·알림·딥링크·퀵메뉴 문맥을 구성하는 시스템 유즈케이스로 두고, 고객 표시 상태를 바꾸는 상태 전이에 함께 연결했다.
