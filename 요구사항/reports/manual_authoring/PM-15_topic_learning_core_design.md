# PM-15 선물주문 Topic Learning & Core Design

## 작성 목표
- 선물주문은 발신자와 수신자의 권한, 수락, 거절, 만료, 취소가 함께 움직이는 양면 고객 업무로 정의한다.
- 발신자는 선물 가능 여부와 수신자 조건을 사전에 알고, 수신자는 선물의 효력과 다음 행동을 한 화면에서 판단할 수 있어야 한다.
- 선물 상태, 알림, 재전송, 회수, 정산·복원 기준을 상태 기반으로 연결한다.

## Core Design Map
- 액터: 발신 고객, 수신 고객, 대리 권한자, 운영자, 채널 업무 시스템, BSS/연계 시스템.
- 유즈케이스: 발신 준비, 발송 관리, 수신 판단, 등록·사용, 상태조회/예외복구, 운영 관리, 시스템 판정/알림/정산.
- 상태: 발송 준비, 예약, 발송 완료, 열람, 수락 대기, 수락 완료, 거절, 만료, 취소·회수, 정산 완료, 운영 예외.
- 정책 질문: 누가 선물할 수 있는가, 누가 받을 수 있는가, 수락 후 어떤 자산·사용 흐름으로 이어지는가, 언제 취소·회수할 수 있는가, 만료되면 무엇이 복원되는가, 어떤 정보가 누구에게 보이는가.

## 작성 금지
- 일반 주문, 결제, 배송, 환불 상세를 본 문서에서 과도하게 확정하지 않는다.
- 수신자 상태를 액터로 세분화하지 않는다.
- 요구사항 문장을 그대로 복사하지 않고 고객 행동과 정책 판단 기준으로 재구성한다.
