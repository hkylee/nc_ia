# PM-04 외부 BP 서비스 관리 체계 Topic Learning & Core Design

## 1. 요구사항 해석 기준

- 상세 요구사항명/설명 18건을 기준으로 작성했다.
- 6BPS-H01은 외부 BP 진입 전 안내, 호환 환경·권한 점검, 제휴사/운영주체/보안·동의 안내로 해석했다.
- 6BPS-H02는 인증·세션·고객 컨텍스트, 미니앱 진입·종료, 결과 동기화, 지연·보류, 이어하기, 실패 대체 경로로 해석했다.
- 6BPS-H03은 제휴 서비스 마스터, 호출·인증·전달 정보, 노출·랜딩·복귀 규칙, 테스트, 단계적 오픈, 차단·롤백, 장애 모니터링으로 해석했다.

## 2. 문서 지향점

- 외부 BP 서비스는 외부 링크 모음이 아니라 고객 경험 책임이 Next Channel 밖으로 확장되는 연계 업무로 정의한다.
- 프로세스 이노베이션 관점에서는 진입 조건, 권한 전달, 외부 처리 상태, 실패·지연·복귀, 이력 기준을 표준화한다.
- 고객 경험 혁신 관점에서는 고객이 외부 서비스로 이동해도 기대 결과, 처리 주체, 복귀 경로, 문제 해결 방법을 명확히 알게 한다.

## 3. Core Design Map

- 고객 유즈케이스는 `이용 가능성 확인`, `외부 BP 이용 및 NC 복귀 완료`, `지연·실패 후 복구 경로 선택`의 상위 업무 목표로 묶는다.
- 고객 프로세스는 진입 전 안내 → 호환·권한 점검 → 외부 BP 이용 → 결과 확인·복귀 → 지연·보류/이어하기 → 실패 대체 경로로 분해한다.
- 운영 유즈케이스는 `연동 기준 관리`, `노출·검증·오픈 관리`, `장애·차단·롤백 운영`의 상위 업무 목표로 묶는다.
- 운영 프로세스는 마스터 등록 → 호출·인증·전달 정보 설정 → 노출·랜딩·복귀 규칙 설정 → 테스트·오픈 승인 → 모니터링·차단·롤백 → 복구 검증·재오픈 판단으로 분해한다.
- 시스템 흐름: 채널 업무 시스템은 고객 컨텍스트와 세션을 제공하고, 외부 BP 시스템은 결과와 상태를 회신한다.

## 4. 작성 금지 경계

- 제휴사 내부 화면, API 필드, DB 컬럼, 계약·정산 세부 조건은 작성하지 않는다.
- 외부 쿠폰, 상품·서비스 혜택, 주문 상태/사후 관리의 전체 절차는 이 문서에서 작성하지 않고 연계 조건으로만 다룬다.
- 요구사항 문구를 그대로 복사하지 않고 고객 과업과 정책 판단 기준으로 재구성한다.
