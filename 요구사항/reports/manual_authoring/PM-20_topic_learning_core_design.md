# PM-20 나의 가입 정보 Topic Learning & Core Design

## 작성 지향점
- 나의 가입 정보는 조회 화면이 아니라 고객이 자신의 계약, 혜택, 제한 상태를 이해하고 다음 행동을 선택하는 관리 허브로 정의한다.
- 회선, 요금제, 약정, 결합, 부가서비스, 구독, 제한 상태를 변경 가능 업무와 연결하는 셀프 관리 프로세스로 설계한다.
- 고객이 상담 없이 현재 상태의 의미, 변경 가능성, 비용·혜택 영향을 이해하고 바로 실행할 수 있게 한다.

## 요구사항 해석
- 최신 상세 요구사항 83건은 ID가 아니라 상세 요구사항명과 설명 기준으로 읽었다.
- 요구사항 원문은 본문에 복사하지 않고 가입정보 홈, 계약 상세, 권한, 상태, 다음 행동, 증빙·이력, 운영 기준으로 재구성했다.
- 구독, 결합, 멤버십, 회선 변경 등 인접 주제는 실행 상세가 아니라 가입정보에서의 표시·판단·이동 기준으로 제한했다.
- 약정 자동 연장 설정은 약정 실행 상세가 아니라 가입정보에서 고객이 현재 설정, 다음 적용 예정 회선, 적용 예정일, 변경 가능 여부를 이해하는 정책 판단축으로 보강했다.

## Core Design Map
- 액터: 고객, 대리 권한자, 상담사, 가입정보 운영자, 채널 업무 시스템, BSS/연계 시스템.
- 유즈케이스: 고객 완결 업무 8개, 대리 관리 1개, 운영 1개, 상담 1개, 시스템 상태 제공 1개로 구성했다.
- 프로세스: 유즈케이스별 3~4개 절차로 분해하고 시스템 내부 조회·저장은 기능으로 연결했다.
- 기능: 각 프로세스별 조회·판정·안내·이력 저장 역량을 정의했다.
- 정책: 21개 정책 그룹과 91개 정책 항목으로 권한, 상태, BSS, 이력, 데이터 보호, 운영 기준을 세분화했다.

## 경계
- 요금제 변경, 구독 플랜 변경, 결합 가입·해지, 결제·정산은 인접 정책서의 실행 업무로 남겼다.
- PM-20은 해당 업무로 이동하기 전 고객이 현재 상태와 처리 가능성을 이해하는 기준에 집중한다.
- 자동 Inspector와 Health Check 외에 Codex가 직접 복합 액터명, 시스템 유즈케이스 상태 전이 연결, 요구사항 Trace 83건 반영 여부를 검수했다.
