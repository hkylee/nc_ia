# PM-33 고객센터_FAQ/공지/이용안내 Topic Learning & Core Design

## 1. 요구사항 확인

- 기준 데이터: `reports/evidence/requirements.db`
- 정책 모듈: `PM-33`
- Depth 4: `고객센터_FAQ/공지/이용안내`
- 상세 요구사항 수: 42건
- 검토 기준: 상세 요구사항 ID가 아니라 `상세 요구사항명`, `상세 요구사항 설명`을 중심으로 업무 의미를 재구성했다.

## 2. 작성 목표

고객센터_FAQ/공지/이용안내는 단순 콘텐츠 목록이 아니라 고객이 문제를 이해하고 해결 행동까지 이어갈 수 있게 하는 도움 콘텐츠 운영 체계로 정의한다.

고객은 FAQ, 이용안내, 가이드, 공지, 장애·점검 정보를 목적 기준으로 탐색하고, 안내 확인 후 관련 업무 화면, 문의, 상담, 매장찾기 등 다음 행동으로 이동할 수 있어야 한다.

운영자는 콘텐츠 등록, 검수, 게시, 노출순서, 삭제, 최신성, 버전, 통합 운영 전환 기준을 관리해야 한다.

## 3. 핵심 범위

- FAQ, 공지, 가이드, 이용안내 통합 검색과 목적별 탐색
- FAQ 카테고리, Top FAQ, 문제 유형별 추천
- FAQ 상세의 단계별 해결 가이드와 관련 업무 바로가기
- 미해결, 검색 실패, 불만족 시 문의·상담 연결
- 장애·점검, 긴급 공지, 정책 변경 요약, 영향 범위 안내
- 기준 시점, 최종 업데이트 일시, 콘텐츠 최신성 표시
- 다국어, 접근성, 외국인 고객, 로밍, 분실 후 보안 조치 안내
- 공통유의사항, 콘텐츠 등록·검수·노출순서·삭제·프론트 연동 기준

## 4. Core Design Map

### 액터

- 고객: FAQ, 공지, 이용안내를 탐색하고 해결 가이드와 관련 업무 화면으로 이동하는 주체
- 운영자: 도움 콘텐츠, 공지, 공통유의사항, 노출순서, 최신성, 통합 운영 기준을 관리하는 주체
- BSS: 고객 상태, 회선·상품·주문·지역 조건을 기준으로 안내 대상과 처리 가능 여부를 판정하는 시스템
- 채널 업무 시스템: 콘텐츠 검색, 추천, 노출, 피드백, 바로가기, 이력 정보를 처리하고 화면에 회신하는 시스템
- 연계 시스템: 상담, 문의, 매장찾기, 알림, 외부 도움 경로의 연결 결과를 제공하는 시스템

## 6. Agent 개선 인사이트

PM-33은 PM-32 고객센터 통합허브와 인접하지만 상담 접수·처리 자체가 아니라 도움 콘텐츠의 탐색, 해결 가이드, 공지 영향, 상황별 안내, 콘텐츠 운영 생명주기가 중심이다. `고객센터`라는 단어만 보고 상담 허브 프로세스를 재사용하지 않는다.

`인증기관`은 FAQ/공지/이용안내의 독립 액터가 아니다. 인증 관련 안내는 상황별 콘텐츠와 정책 조건으로 설명하고, 실제 인증 수행은 담당 도메인 정책서 또는 연계 경로로 연결한다.

## 7. 최종 검증 결과

- Schema/JSON Gate: 통과

- Requirement Trace: 42/42 자동 매칭
- Health Check: 100/100, 우수, 필수 게이트 통과
- Asset/Rendering Check: 통과
- Codex 직접 Inspector: 통과

PM-33은 상담 접수·처리 상태를 재사용하지 않고 FAQ, 공지, 이용안내의 고유 업무 축으로 재작성했다.
검색 실패는 종료가 아니라 연관 카테고리, 추천 FAQ, 대표 문의 유형, 매장찾기, 상담 연결 중 하나 이상의 후속 경로로 이어지도록 보완했다.
콘텐츠 최신성은 기준 시점, 최종 업데이트, 만료·회수, 대체 콘텐츠 연결, 변경 이력 기준까지 포함했다.
