# PM-06 추천 Topic Learning / Blueprint / Core Design

## 1. Topic Learning

### 주제 정의

추천 정책서는 추천 슬롯을 채우는 전시 정책이 아니라, 고객 상태와 맥락을 기준으로 지금 선택할 만한 상품·혜택·콘텐츠·다음 행동을 제안하고 고객이 납득해 실행 또는 거절할 수 있게 하는 의사결정 지원 기준을 정의한다.

### 핵심 고객 과업

- 고객은 추천 후보가 왜 제안되었는지, 현재 조건에서 어떤 가치와 제한이 있는지 이해한다.
- 고객은 추천 결과를 비교, 상세, 가입·변경·혜택 사용, 상담 연결 같은 실행 경로로 이어간다.
- 고객은 원치 않는 추천을 숨기거나 관심 없음으로 처리하고, 개인화 적용 여부와 이력을 확인한다.
- 고객은 추천이 없거나 부적합할 때 검색, 필터, 비교, 상담 같은 대체 경로로 흐름을 이어간다.

### 운영 과업

- 추천/AI 운영자는 슬롯, 추천 전략, 개인화 실패 폴백, 비즈니스 룰, 제약 룰, 우선순위를 등록·검증·배포·회수한다.
- 상품/콘텐츠 운영자는 추천 가능한 자산, 라벨, CTA, 비교 기준, 비노출 기준을 관리한다.
- 운영자는 추천 노출, 수용, 실행, 완료, 숨김, 관심 없음, 오탐 신고, 상담 전환을 분석해 개선안을 승인한다.

### 핵심 요구사항 묶음

- 추천 정보 통합 제공, 추천 슬롯 표준화, 추천 전시 유형·UI 구성, 추천 영역 간 충돌·우선순위 관리.
- 홈 추천, 상세 연관 추천, 모아보기/더보기 랜딩, 큐레이션 피드, 컨시어지 카드, Next Action 오퍼링.
- AI PICK 캡션, 소셜 프루프, 추천 사유, 추천 제한 사유를 고객이 이해할 수 있는 설명 기준으로 제공.
- 스크린 어웨어 맥락, 위치·생활권, 소비패턴, 라이프스타일, 소비 성향 온보딩 결과를 추천 후보와 근거 산정에 활용.
- 무료 혜택 이용 현황 기반 유료 전환 추천, 구독 중 상품 기반 선물 추천, 관심상품 재방문 추천, 이탈 방지 혜택 추천.
- 도메인 간 맥락 기반 크로스셀 넛지, 컨시어지 카드, 다음 행동 연결 기준.
- 개인화 추천 제어 설정 진입, 추천 카드 관심 없음/숨기기, 개인화 제어·학습 피드백.
- 추천 근거 생성, AI 개인화 콘텐츠 라벨, 개인화 데이터 활용 범위와 투명성 고지.
- AI 추천 비즈니스/수익성 룰, 추천 제약 룰, 세그먼트별 룰 분기, 추천 결과 사전 검증, 추천 보류·대안 응답.
- 로그인·선택동의 Case 분기, Cold Start 추천, 개인화 컨텍스트·항목·그룹 운영.
- 추천 성과 분석, 추천 품질·전환 지표, 적용군·대조군 비교, 캠페인 결과 분석, 개선안 자동 제안과 운영자 승인.

### 적용 근거

- 1순위 요구사항 상세명/설명 78건을 기준으로 작성한다.
- TK 핵심 지향점은 "내 마음을 읽어주는 기술, 나만의 정답을 찾아주는 AI 추천"과 "수익성 가드레일 기반 AI 추천 체계"를 현재 주제에 맞게 해석한다.
- 전시/관리 기능, 상품 목록, AI 검색은 인접 주제로 두고, 추천 문서에서는 추천 결과와 실행 연결 기준만 다룬다.

### Evidence Gap

- 개별 추천 알고리즘, 모델 피처, 랭킹 수식, UI 컴포넌트 세부 스펙은 정책서 범위가 아니다.
- 개인화 데이터별 보관 기간과 비식별 고객 이력 결합 세부 기준은 개인정보·보안 정책과 정합성 확인이 필요하다.

## 2. Blueprint / Core Design

### Document Strategy

추천 정책서는 고객에게 "왜 이것을 추천하는지", "내 조건에서 무엇이 달라지는지", "싫으면 어떻게 제어하는지", "실행하면 어떤 상태로 이어지는지"를 설명할 수 있어야 한다.
운영 관점에서는 추천 결과가 고객 적합도, 사업 룰, 제한 조건, 개인화 동의, 성과 피드백을 기준으로 통제 가능해야 한다.

### Core Policy Questions

- 어떤 고객 상태와 맥락에서 개인화 추천을 적용하는가.
- 개인화가 불가하거나 신뢰도가 낮을 때 어떤 폴백을 제공하는가.
- 로그인·선택동의·Cold Start Case별 추천 노출 범위는 어떻게 달라지는가.
- 위치·생활권·소비패턴·라이프스타일 데이터는 어떤 동의와 제한 안에서 추천 근거로 쓸 수 있는가.
- 추천 근거는 어떤 수준까지 고객에게 설명하는가.
- AI PICK 캡션과 소셜 프루프는 어떤 조건에서 노출하고 어떤 표현은 제한하는가.
- 추천 노출 빈도와 반복 노출은 언제 제한하는가.
- 고객의 숨김·관심 없음·피드백은 언제, 어디에 반영하는가.
- 비즈니스 룰과 고객 불이익 방지 기준이 충돌하면 무엇을 우선하는가.
- 추천 보류 시 고객에게 어떤 대체 경로를 제공하는가.
- 무료 혜택 이용, 구독 보유, 관심상품, 이탈 징후를 실행 가능한 추천·혜택·선물 제안으로 연결하는 기준은 무엇인가.
- 도메인 간 크로스셀 넛지와 스크린 어웨어 Next Action은 언제 고객 경험 개선으로 인정되는가.
- BSS/연계 시스템은 어떤 고객 상태, 자격, 보유 관계, 제한 조건을 회신하는가.
- 추천 노출, 수용, 실행, 완료, 실패, 숨김 이력은 어떤 기준으로 저장하는가.

### Hierarchy Map

- 액터: 고객, 추천/AI 운영자, 상품/콘텐츠 운영자, 추천 엔진, BSS/연계 시스템, 채널 업무 시스템.
- 유즈케이스: 추천 후보 이해·비교, 추천 기반 실행 연결, 추천 제어·피드백 관리, 추천 슬롯·룰 운영, 추천 품질 개선, 추천 대상 자산 관리, 시스템 추천 후보 산정, BSS 판정, 이력 저장.
- 상태: 추천 미진입, 후보 산정 중, 확인 가능, 보류, 비교·대체 탐색, 실행 연결 대기, 실행 완료, 숨김, 제한, 피드백 반영 대기, 운영 검수 대기, 배포 완료, 회수·롤백.
- 프로세스: 고객 추천 확인·비교·실행·피드백, 운영자 슬롯·룰 등록·검증·배포·회수, 성과 분석·개선 승인, 자산 기준 관리.
- 기능: 슬롯 관리, 고객 맥락 수집, 후보 생성·폴백, AI PICK·소셜 프루프 근거 생성, 비교·대체 연결, 실행 CTA, 숨김·관심 없음, 개인화 컨텍스트·항목·그룹 운영, 비즈니스 룰, 사전 검증, 배포·회수, 성과 지표, 캠페인 분석, 이력 저장.
- 정책: 추천 범위, 개인화·폴백, AI PICK·소셜 프루프, 우선순위·충돌, 노출 빈도, 실행 연결, 고객 제어, 데이터 동의·제한, 비즈니스 룰, 사전 검증, 운영 배포, 성과 개선, BSS 판정, 실패 대체, 이력·감사.

### Must Include

- 개인화 추천과 비개인화 폴백을 모두 포함한다.
- 로그인·선택동의·Cold Start Case별 추천 노출·제한 기준을 포함한다.
- 추천 이유, 추천 제한 사유, 추천 보류 사유, 대체 경로를 정책 항목으로 둔다.
- AI PICK 캡션, 소셜 프루프, 스크린 어웨어 맥락, 위치·소비패턴 근거는 고객 설명 가능성과 오해 방지 기준을 함께 둔다.
- 무료 혜택 이용 기반 유료 전환, 구독 기반 선물 추천, 관심상품 재방문, 이탈 방지 혜택은 실행 연결 정책으로 다룬다.
- 고객 제어 항목은 현재 지면 제외, 차단시간, 이력 확인, 초기화 기준까지 포함한다.
- 운영 룰은 등록, 검증, 충돌 해결, 배포, 회수, 롤백, 감사 기준으로 분해한다.
- 추천 성과는 노출, 수용, 실행, 완료, 숨김, 관심 없음, 상담 전환, 적용군·대조군 비교까지 이어지는 전환 퍼널로 관리한다.

### Must Not

- 추천을 전시 슬롯 설정 화면 설명으로 축소하지 않는다.
- AI 모델 내부 알고리즘, 프롬프트 원문, API/DB 필드, 화면 컴포넌트 스펙을 본문에 쓰지 않는다.
- 상품 상세, AI 검색, 전시 관리, 데이터 트래킹의 전체 절차를 현재 문서에서 다시 작성하지 않는다.
- 요구사항 문구를 그대로 복사하지 않고 추천 업무 흐름과 정책 판단 기준으로 재구성한다.

## 3. 장별 작성 지침

- 개요: 추천을 고객 의사결정 지원과 운영 품질 루프로 정의한다.
- 용어: 개인화 추천, 비개인화 폴백, 추천 슬롯, 추천 근거, 관심 없음, 숨김, 비즈니스 룰, 추천 보류를 포함한다.
- 유즈케이스: 고객·운영자의 완결 업무 목표로 묶고, 홈 추천·상세 추천·숨김 같은 세부 항목은 프로세스로 내린다.
- 상태: 추천 가능 여부와 고객 후속 행동이 달라지는 상태만 코드화한다.
- 프로세스: 각 Y 유즈케이스가 최소 시작, 판단, 처리, 결과, 예외 중 2개 이상의 업무 경계를 갖게 한다.
- 기능: 세부 기능 구성은 설명문이 아니라 짧은 하위 처리명으로 작성한다.
- 정책: 추천 항목마다 허용, 제한, 횟수·기간, 상태, 예외, 고지, 이력, BSS 기준 중 하나 이상을 포함하고, AI PICK·소셜 프루프·위치/소비패턴 활용은 고객 보호와 동의 조건을 함께 둔다.
