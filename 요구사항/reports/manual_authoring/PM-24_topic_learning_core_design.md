# PM-24 나의 데이터·통화 Topic Learning / Core Design

## 작성 지향점
- 데이터·통화 관리는 잔여량 조회가 아니라 사용량 이해, 충전, 선물, 제한, 로밍 옵션 실행을 연결하는 통신 자산 관리 업무로 정의한다.
- 프로세스 이노베이션 관점에서는 실시간성, 차감·충전, 선물 가능 조건, 만료, 알림, 실패 복구 기준을 자산 상태 기반으로 관리한다.
- 고객 경험 혁신 관점에서는 고객이 남은 자산과 부족 위험을 이해하고 필요한 조치를 즉시 실행해 사용 중단 불안을 줄이게 한다.

## 요구사항 학습 요약
- 총 상세 요구사항: 70건
- 최신 요구사항 DB 기준으로 실시간 사용량 조회, 데이터 선물하기, 충전·리필·쿠폰, 자동선물, T 데이터 셰어링 USIM, 가족·위임, 로밍, 예외 복구 요구사항을 확인했다.
- 요구사항은 상세 요구사항명과 상세 요구사항 설명을 기준으로 읽고, 원문을 복사하지 않고 고객 자산 관리 흐름과 정책 판단 기준으로 재구성한다.

## Core Design Map
- 액터는 고객, 가족 대표, 운영자, 상담사, 채널 업무 시스템, 사용량 연계 시스템, 외부 연계 시스템으로 제한했다.
- 가족 대표는 보호자·가족·대리 권한의 고객 측 책임을 대표하고, 권한 차이는 정책 조건으로 내린다.
- 사용량 연계 시스템과 외부 연계 시스템은 세부 BSS·제휴·로밍 시스템을 과도하게 나누지 않고 상태 회신과 결과 대조 책임으로 통합한다.
- 유즈케이스는 조회 단계를 과도하게 쪼개지 않고 자산 이해, 소진 위험 관리, 충전·리필, 선물·공유, 자동 설정, 가족·위임, 로밍, 실패 복구로 묶었다.
- 프로세스는 각 유즈케이스 안에서 조건 판정과 실행 단위를 2~3개로 분해해 기능과 정책으로 이어지게 했다.
- 상태는 고객 안내와 후속 처리 기준이 달라지는 자산 상태만 승격했다.
- 정책은 표시, 실시간성, 예측, 알림, 차단, 선물, 충전, 자동 설정, 가족·위임, 로밍, 이력, 예외, BSS, 개인정보, 운영 기준으로 나눴다.

## 2026-05-10 보완 기록
- 데이터 선물하기, 데이터 충전하기, 자동선물 설정 조회, 선물 수단 구분, 공유데이터·개인데이터 구분, 저속/해외 경량 조회, 주요 통화·데이터 사용 패턴 요구사항이 산출물 표면에서 추적되도록 기능명·정책 항목을 보강했다.
- T 데이터 셰어링 USIM 상태·해지 영향 확인을 가족·다회선 위임 관리 유즈케이스의 프로세스와 기능으로 추가했다.
- 시스템 유즈케이스는 독립 프로세스로 만들지 않고 상태 전이 usecase_ids에 함께 연결해 BSS·외부 회신이 고객 표시 상태를 바꾸는 지점을 추적하게 했다.
