# PM-26 통합 쿠폰/이용권함 Topic Learning & Core Design

## 작성 기준
- 최신 requirements.db의 Depth 4 `통합 쿠폰/이용권함` 상세 요구사항명/상세 요구사항 설명 31건을 1순위 근거로 사용했다.
- TK 지향점은 쿠폰·이용권·포인트·선물성 혜택 자산을 고객이 한곳에서 이해하고 사용할 수 있는 혜택 자산함으로 정의하는 방향으로 반영했다.
- 기존 19건 기준 초안에서 누락되던 수령·발급 상태, 제휴처 인증·계정 연동, 개인 혜택 메시지·배너, 쿠폰 기간·유형 운영, 선택항목 일괄 처리 제어를 보강했다.
- 외부 쿠폰 발급 조건, 상품 가입 조건, 멤버십 포인트 적립 상세, 외부 BP 계약 운영은 인접 정책서 범위로 두고, 자산함 표시와 사용 가능성에 필요한 결과값만 포함했다.

## Core Design Map
- 액터: 고객, 운영자, 상담사, 채널 업무 시스템, 혜택 원장 시스템, 제휴사 시스템.
- 고객 유즈케이스: 자산함 확인, 상태 구분 확인, 사용 진입, 베네핏 가치 확인, 만료·이력 조회, 정정 요청, 공유 카드 생성, 쿠폰 등록·수령 상태 관리.
- 운영 유즈케이스: 자산 상태·노출 우선순위 관리, 가치 산정·표시 기준 관리, 쿠폰 기간·유형·일괄 처리 기준 관리.
- 상태 축: 사용 가능, 등록 필요, 발급 중, 발급 실패, 연동 필요, 조건 미충족, 오픈 예정, 만료 예정, 사용 처리 중, 동기화 불일치, 정정 요청 중, 운영 확인 필요.
- 정책 축: 카드 요약, 상태 분류, 정렬, 사용 가능성, 제시수단, 수령·등록·재발급, 제휴처 인증·연동, 가치 산정, 이력, 만료·소멸, 동기화 정정, 공유, 개인 혜택 메시지, 쿠폰 기간·상태, 쿠폰 유형·입력 항목, 선택항목 일괄 처리, 운영, 원장·제휴, 개인정보.
