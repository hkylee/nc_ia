# PM-31 설정 Topic Learning / Core Design

## 1. 작성 목표

- 설정은 고객이 개인화, 홈 구성, 알림 환경, 언어·접근성, 보안·세션, 시스템 권한, 기록 삭제와 기본값 복원을 직접 제어하는 고객 제어면으로 정의한다.
- 실제 약관 원장, 알림 발송 정책, 추천 알고리즘 자체를 다시 작성하지 않고, 고객이 설정에서 확인·변경·초기화·복구할 수 있는 범위와 제한 기준을 정리한다.
- 고객이 설정 변경의 영향, 제한되는 기능, 되돌리기 가능 여부, 변경 이력을 이해하고 다음 행동을 선택할 수 있게 한다.

## 2. 근거 및 요구사항 범위

- 요구사항 DB 기준 `normalized_depth4=설정` 92건을 확인했다.
- 핵심 상세 요구사항은 개인화 서비스 제공 동의/설정 진입, 개인화 초기화, AI·개인화 기록 관리, 추천 제어, 홈 우선 영역·기본 탭·퀵액션, 대표 회선/프로필 기본값, 쉬운모드, 언어, 글자 크기·고대비·화면 밀도, 다크모드·테마, 외부 앱·브라우저·미디어 설정, 자동 로그아웃·앱 잠금·생체인증, 시스템 권한 상태 조회, 알림 환경설정 연동, 동의 이력 진입, 데이터 절약·자동재생 기준이다.
- 요구사항 문구는 그대로 복사하지 않고 설정의 고객 제어, 적용 범위, 변경 이력, 실패·대체 경로 기준으로 재구성했다.

## 3. Core Design Map

### 액터

- 고객: 설정 상태 확인, 개인화·알림·접근성 제어, 초기화·삭제·기본값 복원을 수행한다.
- 운영자: 설정 기준값, 기본값, 예외 조건, 품질 지표, 배포·회수 이력을 관리한다.
- BSS: 고객 상태, 회선/프로필 기본값, 권한·보안 조건, 설정 적용 가능 여부를 판정한다.
- 인증기관: 보안·민감 설정 변경에 필요한 본인확인 또는 생체인증 결과를 제공한다.
- 연계 시스템: OS 권한, 외부 앱, 브라우저, 미디어 재생, 알림 권한 같은 외부 설정 상태를 제공한다.

### 유즈케이스

- 설정 상태·권한 관리: 현재 설정 상태, 시스템 권한, 보안·세션, 접근성 적용 범위를 확인한다.
- 설정 개인화·알림 관리: 개인화, 홈 구성, 알림 환경, 언어·접근성, 데이터 절약 설정을 변경한다.
- 설정 초기화·삭제 관리: 설정 초기화, 기본값 복원, 기록 삭제, 변경 이력, 실패·대체 경로를 처리한다.
- 설정 운영 기준 및 품질 관리: 운영자가 설정 기준과 기본값, 예외 기준, 품질 지표를 관리한다.

### 주요 프로세스

- 설정 진입 및 상태 확인
- 권한·보안 조건 판정
- 설정 경로 및 대체 안내
- 설정 목적 선택
- 적용 범위 확인
- 설정값 변경 접수
- 설정 결과 반영
- 변경 이력 안내
- 초기화 대상 선택
- 영향 범위 고지
- 초기화·삭제 요청 접수
- 처리 결과 및 이력 안내

### 주요 정책

- 개인화·접근성 설정 정책
- 알림 설정 정책
- 동의·권한 설정 정책
- 보안·세션 설정 정책
- 기록·초기화 정책
- 설정 가능 여부 정책
- 운영 변경 이력 관리 정책

## 4. 이번 작성 전 보강한 Agent 개선점

- 설정 주제의 유즈케이스 설명이 상세 요구사항 제목 나열로 흐르지 않도록 `상태·권한`, `개인화·알림`, `초기화·삭제`별 설명 초점을 고정했다.
- 약관, 알림, 추천, 홈, 보안, OS 권한 같은 인접 주제 표현은 해당 원장 업무가 아니라 설정 화면에서의 고객 제어·진입·영향 안내 기준으로만 해석하도록 했다.
