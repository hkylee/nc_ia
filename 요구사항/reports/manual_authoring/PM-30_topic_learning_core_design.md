# PM-30 통합 약관 Topic Learning / Core Design

## 1. 작성 목표

- 통합 약관은 약관 원문 열람, 필수/선택 동의, 개인정보 제공·위탁 고지, 거부권, 개정 고지, 철회·재동의, 증적 관리를 하나의 고객 권리 경험으로 정리한다.
- 단순 약관 목록이나 동의 체크 UI가 아니라 고객이 어떤 약관이 왜 필요한지 이해하고, 거부·철회·재동의 시 업무 영향과 후속 행동을 알 수 있게 한다.
- BSS와 채널은 동의 상태, 약관 버전, 적용 서비스, 고지 확인, 철회 이력을 기준으로 동일한 판단을 수행해야 한다.

## 2. 근거 및 요구사항 범위

- 요구사항 DB 기준 `normalized_depth4=통합약관` 83건을 확인했다.
- 핵심 상세 요구사항은 미동의/비활성 고객 기본 경험, 필수·선택 동의 구분, 약관 동의 상태 조회, 선택 동의 철회·재동의, 위치기반서비스·개인위치정보 동의, 제3자 제공동의 철회, 동의 이력 조회, 개인정보 제공·위탁 고지, 운영·관리 기준이다.
- 요구사항 문구는 그대로 복사하지 않고 약관 업무 판단 기준과 후속 처리 기준으로 재구성했다.

## 3. Core Design Map

### 액터

- 고객: 약관 열람, 동의, 철회, 재동의, 미동의 영향 확인을 수행한다.
- 운영자: 약관 버전, 시행일, 고지 내용, 예외 기준, 배포·회수 이력을 관리한다.
- BSS: 고객별 동의 상태, 필수 동의 충족 여부, 약관 버전 정합성, 업무 제한 여부를 판정한다.
- 본인확인 시스템: 약관 동의, 철회, 재동의, 법정대리인 확인에 필요한 본인확인 결과를 제공한다.
- 연계 시스템: 위치정보, 제3자 제공, 위탁, 쿠키 등 외부 기준 또는 처리 결과를 제공한다.

### 유즈케이스

- 통합 약관 권리 관리: 약관 원문·요약, 적용 범위, 개인정보 제공·위탁·쿠키 고지, 거부권 기준을 확인한다.
- 통합 약관 필수·선택 동의 관리: 업무 유형별 필수/선택 동의와 미동의 제한 범위를 판정하고 동의를 처리한다.
- 통합 약관 변경·철회 관리: 약관 개정, 철회, 재동의, 버전·시행일, 증빙·감사 이력을 관리한다.
- 통합 약관 운영 기준 및 품질 관리: 운영자가 약관 기준을 등록, 승인, 배포, 회수하고 품질 지표를 관리한다.

### 주요 프로세스

- 약관 원문·요약 열람
- 개인정보 제공·위탁 고지 확인
- 거부권 및 미동의 영향 확인
- 업무 유형별 동의 항목 확인
- 필수·선택 동의 구분
- 미동의 제한 범위 판정
- 동의 처리 및 증적 저장
- 변경·철회 대상 선택
- 철회 가능 여부 판정
- 변경·철회 요청 접수
- 운영 기준 확인, 기준·예외 등록, 승인·배포 및 이력 관리

### 주요 정책

- 약관 열람·고지 정책
- 필수·선택 동의 정책
- 철회·재동의 정책
- 개인정보·제3자 고지 정책
- 약관 버전·증적 정책
- 예외 동의·분기 정책
- 운영 변경 이력 관리 정책

## 4. 이번 작성 전 보강한 Agent 개선점

- `통합 약관` 주제에서 요구사항 축을 단일 `약관·동의`로 묶지 않고 `약관 열람·고지`, `필수·선택 동의`, `철회·재동의`, `개인정보·제3자 고지`, `버전·증적`, `예외 동의·분기`로 분리했다.
- 개인정보 정정·삭제, 제3자 제공, 위탁, 쿠키 고지 요구사항이 `해지·취소` 같은 인접 주제로 오분류되지 않도록 보정했다.
- 유즈케이스 설명에서 조사 오류가 반복되지 않도록 목적어 조사를 자동 선택하는 helper를 추가했다.
