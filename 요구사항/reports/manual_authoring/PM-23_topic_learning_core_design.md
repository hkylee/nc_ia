# PM-23 청구 및 수납 관리 Topic Learning / Core Design

## 1. 주제 정의
청구 및 수납 관리는 고객이 복잡한 요금 구조를 이해하고 납부를 완료하며, 실패·미납·증빙·포인트 문제까지 셀프로 해결할 수 있게 하는 신뢰 기반 금전 업무다.

## 2. 작성 지향점
- 청구·수납은 복잡한 요금 구조를 고객 언어로 재구성하고 납부를 셀프로 완료하게 하는 신뢰 기반 금전 업무로 정의한다.
- 청구 분리, 납부수단, 미납, 영수증, 실시간 요금, 납부 실패와 재처리를 하나의 수납 프로세스로 관리한다.
- 고객이 왜 이 금액이 청구됐는지, 언제 어떻게 납부해야 하는지, 실패 시 무엇을 해야 하는지 이해하게 한다.

## 3. Core Design Map
- 액터: 고객, 운영자, 상담사, 채널 업무 시스템, 수납 연계 시스템, 결제 금융기관.
- 유즈케이스: 청구 요약·상세 이해, 수신·증빙 관리, 납부수단·자동납부 관리, 즉시·예약·부분 납부, 미납·실패 복구, 통합청구 관리, 실시간 요금·한도 관리, 포인트 확인, 운영 예외 관리, 상담 문맥 지원.
- 핵심 상태: 청구 확정, 납부 예정, 자동납부 대기, 납부 진행, 수납 반영 대기, 납부 완료, 납부 실패, 미납, 연체, 부분 수납 가능, 청구 이의·보정 필요, 증빙 발급 가능, 통합청구 구성 중, 실시간 요금 산정 중, 포인트 보류·지연.
- 정책 후보: 청구 안내, 수신·증빙, 납부 실행, 납부수단·자동납부, 미납·실패 복구, 통합청구, 실시간 요금·한도, 포인트, 운영 보정, BSS 반영, 개인정보·보안.

## 4. 요구사항 묶음
- BILL-H01~H04: 청구 요약, 상세, 수신·증빙, 통합청구, 운영 보정.
- PAY-H01~H05: 즉시 납부, 자동납부, 납부수단, 계약 연장 결제, 실패·미납 복구.
- RTB-H01~H02: 실시간 이용요금, 콘텐츠 이용료, 한도·차단, 할인 영향.
- MYPNT-H01~H02: 통합 포인트 잔액, 소멸, 반영 지연, 운영 관리.
- 무ID 보강: 청구 알림, 카드 만료, 우편 발송, 이전 통신사 요금, 복지 자격 등은 관련 정책 축에 편입했다.

- 모든 상태 전이에 고객/운영자 유즈케이스와 상태 변경을 유발하는 시스템 유즈케이스를 함께 연결했다.
