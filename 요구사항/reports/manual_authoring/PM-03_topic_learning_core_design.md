# PM-03 상품 목록 Topic Learning / Core Design

## 1. Topic Learning

상품 목록은 단순 카탈로그가 아니라 고객이 자신의 목적에 맞는 후보를 좁히고 비교를 시작하는 탐색 허브로 정의한다. 요구사항 72건은 카테고리 진입, 목록 카드, 필터·정렬, 비교·찜·최근본, 가입 가능 상태, 개인화 목록, 운영 관리로 묶인다.

## 2. 핵심 고객 과업

- 고객은 상품군과 카테고리 경로를 이해하고 원하는 목록으로 진입한다.
- 고객은 필터, 정렬, 검색, 빠른 조건을 이용해 후보를 좁힌다.
- 고객은 목록 단계에서 가격, 혜택, 가입 가능성, 제한 사유, 현재 상품 대비 차이를 확인한다.
- 고객은 비교 후보, 찜, 최근본으로 탐색을 이어간다.
- 고객은 결과 없음이나 오류가 발생했을 때 조건 초기화, 검색, 상담, 추천 카테고리로 이동한다.

## 3. Core Design Map

- 액터: 고객, 상품 운영자, 상품 기획자, 채널 업무 시스템, 상품 마스터, BSS.
- 유즈케이스: 목록 후보 탐색, 조건 조정, 비교·보관, 가입 가능 확인, 폴백 선택, 카테고리·전시 단위 관리, 필터·카드 기준 관리, 품질 검수·배포, 상품 속성 기준 제공.
- 운영 프로세스: 필터·정렬 기준 등록과 목록 카드·상태 배지 검증을 분리하고, 품질 검수와 예약 배포·회수·복구 처리를 분리한다.
- 상품 기획 프로세스: 상품 속성 기준 제공과 전시 그룹 매핑 기준 검증을 분리해 목록 카테고리와 상품 속성의 연결성을 확인한다.
- 상태: 탐색 전, 카테고리 진입, 목록 조회 완료, 조건 적용, 후보 선별 완료, 가입 가능 확인, 제한 사유 확인, 비교·보관 중, 후속 진입, 결과 없음, 오류·복구 필요, 운영 검수 대기, 운영 배포 완료, 운영 회수 필요.
- 정책 축: 범위·경계, 카테고리, 목록 카드, 필터·정렬, 우선순위, 개인화, 비교, 찜·최근본, 가입 가능, BSS 연계, 폴백, 후속 CTA, 맥락 유지, 운영, 품질 Gate, 이력·감사, 개인정보.

## 4. 범위 경계

상품 상세, 장바구니, 주문·계약, 결제, 할인 확정 계산은 후속 정책서 범위다. PM-03은 해당 업무로 넘길 상품 후보, 고객 조건, 목록 맥락, 가입 가능 결과를 정의한다.

## 5. 검수 결과

- 세부 기능 구성은 설명문이 아니라 짧은 하위 처리명으로 작성했다.
- 프로세스의 관련 기능과 정책은 순수 ID로 저장했다.
- 상태 전이표에는 관련 유즈케이스 컬럼을 추가하지 않고 전이 이벤트에 업무 사건을 작성했다.
- 요구사항 문구를 직접 복사하지 않고 업무 흐름과 정책 판단 기준으로 재구성했다.
- 운영자·상품 기획자 유즈케이스가 프로세스 1개로 끝나던 부분을 기준 등록, 검증, 배포·회수, 매핑 검증 단계로 보완했다.

## 보완 기록
- 유즈케이스를 목록 탐색 단계가 아니라 상품 후보 선별, 비교·저장·가입 가능성 확인, 결과 없음 복구, 운영 기준 관리의 완결 업무 목표로 재정리했다.
