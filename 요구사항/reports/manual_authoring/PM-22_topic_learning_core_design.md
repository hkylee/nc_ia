# PM-22 멤버십 카드 관리 Topic Learning / Core Design

## 1. 주제 정의
멤버십 카드 관리는 카드 이미지를 보여주는 업무가 아니라 고객의 멤버십 자격, 등급, 사용 가능 상태, 제한 사유, 발급·재발급, 해지 유예·확정, 대표 회선 기준을 상태 기반으로 관리하는 업무다.

## 2. 작성 지향점
- 멤버십 카드는 자격, 등급, 사용 가능 상태, 제한 사유를 증명하는 혜택 접근 수단으로 정의한다.
- 발급, 재발급, 대표 회선, 해지 유예, 복구 불가 삭제, 사용 제한은 상태 기반으로 관리한다.
- 고객이 현장에서 바로 사용할 수 있는 카드 상태와 문제 해결 방법을 빠르게 확인하도록 한다.

## 3. 핵심 고객 과업
- 지금 멤버십 카드를 사용할 수 있는지 확인한다.
- 바코드, QR, 카드번호를 안전하게 제시하거나 복사한다.
- 실물 카드 발급, 재발급, 분실·비활성 복구를 요청한다.
- 가입, 탈퇴, 재가입, 번호이동이 카드와 혜택에 미치는 영향을 확인한다.
- 대표 회선과 카드 처리 이력을 조회·변경한다.

## 4. Core Design Map
- 액터: 고객, 멤버십 운영자, 상담사, 채널 업무 시스템, 멤버십 연계 시스템, 외부 사용처.
- 유즈케이스: 카드 사용 가능성 확인, 디지털 카드 제시, 실물 카드 발급·재발급, 상태 변경 영향 확인, 대표 회선/이력 관리, 운영 예외 관리, 상담 문맥 지원, 시스템 동기화.
- 상태: 자격 확인 전, 사용 가능, 사용 제한, 디지털 카드 표시 중, 제시 완료, 발급 접수/처리/완료, 재발급 필요, 상태 변경 영향 확인 중, 해지 유예, 해지 확정, 복구 불가 삭제, 운영 확인 필요, 상담 전환 필요.
- 정책 후보: 자격·상태 판정, 디지털 카드 제시, 번호 복사, 실물 발급, 재발급, 상태 변경 영향, 해지 유예·삭제, 대표 회선, 운영 예외, BSS 반영, 실패·상담 전환, 보안.

## 5. 작성 경계
- 혜택 상세 사용, 제휴처별 포인트 적립, 쿠폰/이용권은 인접 정책서에 둔다.
- 본 문서는 멤버십 카드가 사용 가능한 상태인지, 어떤 수단을 제시할 수 있는지, 실패하면 어떻게 복구하는지에 집중한다.

## 6. 2026-05-10 보완 기록
- 최신 요구사항은 14건으로 확인했으며 Trace는 14/14로 재생성했다.
- 멤버십 자격·카드 정보 동기화, 카드 제시 결과 회신, 카드 화면·후속 행동 구성은 `process_target=N`이지만 고객 표시 상태와 복구 경로를 바꾸므로 상태 전이 `usecase_ids`에 함께 연결했다.
- 시스템 액터도 슬래시 복합명 대신 `멤버십 연계 시스템`, `외부 사용처`로 정리해 책임 경계를 명확히 했다.
