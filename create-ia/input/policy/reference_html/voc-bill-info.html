VOC analysis readout

청구서 정보 VoC 분석

청구서 정보 VoC의 핵심은 단일 기능 부족보다 고객이 처리 조건, 영향, 결과를 스스로 확인하지 못해 상담으로 전환되는 데 있다.
가장 큰 병목은 청구서 수신 방법 변경 · 주소 변경 경로 불명이며, 통합채널은 이를 탐색, 판단, 실행, 확인, 복구가 연결된 업무 경험으로 재설계해야 한다

1. 핵심 신호

청구서 정보 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

청구서 수신 방법 변경 · 주소 변경 경로 불명

" 번거로운 절차 없이 빠르게 처리하고 싶다 " → 단순 설정 변경을 앱에서 즉시 처리하고 싶은 편의 · 효율 욕구

02

청구서 금액 구성 이해 불가 및 요금 변동 혼란

" 내가 내는 금액이 정확한지 확인하고 싶다 " → 청구서 금액 구성을 투명하게 이해하고 싶은 신뢰 · 정확성 욕구

03

통합청구 · 자동이체 변경 시 채널 간 프로세스 단절

" 한 번에 모두 처리하고 싶다 " → 복합 처리 시 미납 · 서비스 중단 방지 욕구

04

청구서 미수신 시 납부 방법 불명

"제때 납부하지 못할까 봐 불안 " → 납부 기한 내 정확히 납부하고 싶은 리스크 회피 욕구

05

청구서 발송 오류 및 시스템 미수신

" 청구서를 안정적으로 수신하고 싶다 " → 시스템 오류로 인한 미수신 불안 해소 욕구

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01이사 · 근무지 변경 후 청구서 수신 주소 변경 필요 — 전 기간 1 위 Trigger
  2. 02납부 완료 후 미납 포함 청구서 문자 재수신 — 3/3 이후 1 위로 급부상 (약 15~32 건 / 일)
  3. 03청구서 미수신 (우편 / 문자 / 이메일) 후 납부 방법 불명확 — 전 기간 2~3 위
  4. 04청구서 수신 방법 변경 희망 (우편 ↔ 문자 ↔ 이메일 ↔ 지로 전환) — 전 기간 지속
  5. 05요금제 변경 후 청구서 금액이 예상과 달라 확인 요청 — 전 기간 지속
  6. 06이메일 · 문자 청구서 미수신 후 재발행 요청 — 전 기간 지속
  7. 07가족 · 자녀 회선 청구서 분리 · 통합 신청 필요 — 전 기간 지속
  8. 08카드 분실 · 재발급 후 연결 서비스 결제 정보 일괄 변경 필요 — 전 기간 지속
  9. 09법인 청구서 재발행 · 이메일 발송 · 파일 요청 — 전 기간 지속
  10. 10자동이체 설정 후 청구서 이중 납부 우려 발생 — 전 기간 지속
  11. 11통합청구 해지 후 개별 자동이체 등록 필요 — 전 기간 지속
  12. 12로밍 청구서 이메일 발송 요청 (업무 · 법인 카드 처리 목적) — 전 기간 지속
  13. 13선택약정 할인 적용 시점 및 예상 청구금액 확인 필요 — 전 기간 지속
  14. 14번호이동 · 신규 가입 · 매장 변경 후 청구서 수신 설정 미이전 — 전 기간 지속
  15. 15이메일 청구서 반송 문자 수신 후 재신청 경로 불명 — 전 기간 지속
  16. 16우편 청구서 설정 후 반복 자동 변경 → 재문의 — 전 기간 지속
  17. 17개명 후 청구서 명의 불일치 발견 — 전 기간 지속
  18. 18야간 · 주말 상담 불가로 처리 불가 — 공휴일 · 주말 집중
  19. 19과거 청구서 소급 발송 요청 (법원 제출 · 경비처리 · 증빙용) — 전 기간 지속
  20. 20대리점 처리 후 청구서 재발행 자동화 미흡으로 반복 전화 — 전 기간 지속

5. Quick Win

즉시 개선 가능한 항목은 진입점 노출, 완료 안내, 상태 조회, 권한 처리, 실패 복구를 중심으로 정리된다.

6 청구서 재발행 셀프 신청 기능 직관적 개선 + 발송 상태 실시간 조회 + 재발행 후 발송 완료 알림 18 일 전체 재발행 관련 재문의 전화 감소
7 요금제 변경 완료 후 " 다음 달 예상 청구금액 미리보기 " 팝업 자동 제공 18 일 전체 요금 변동 혼란 전화 6~9 건 / 일 감소
8 번호이동 · 신규 가입 · 매장 변경 완료 후 청구서 수신 방법 설정 안내 팝업 자동 제공 18 일 전체 이벤트 후 미수신 전화 감소

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 청구서 정보 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
진입점

고객은 어디에서 업무를 시작할 수 있는가?

통합채널의 홈, 마이, 요금, 상품 영역에서 같은 업무 진입점을 제공하고, 현재 상태에서 가능한 다음 행동을 명확히 보여줘야 한다

사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

효율

반복 입력과 재문의를 어떻게 줄이는가?

반복 입력, 회선별 재처리, 동일 문의 재접수를 줄이도록 일괄 처리와 저장된 정보 재사용 기준을 설계해야 한다

상태·오류

실패와 오류를 어떻게 복구하는가?

접수, 검증, 반영, 실패, 취소 상태를 고객에게 보여주고 재시도, 대체 경로, 상담 전환 기준을 연결해야 한다

권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

청구서 정보 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
