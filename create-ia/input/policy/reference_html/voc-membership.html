1. 핵심 신호

멤버십 VoC는 고객이 업무를 시작하고, 조건을 이해하고, 결과를 확인하고, 실패를 복구하는 전체 흐름을 정책 기준으로 재설계해야 함을 보여준다.

Core signals
01

멤버십 등급 산정 기준 불명 및 요금제 변경 시 등급 변동 미인지

" 내가 낸 요금만큼 혜택을 제대로 받고 싶다 " · " 오래 썼는데 왜 VIP 가 아닌지 납득하고 싶다 " → 등급 산정 기준 투명성 욕구 + 혜택 손실 회피 욕구

02

멤버십 미가입 · 미발급 상태에서 혜택 사용 불가 및 가입 경로 불명

" 지금 당장 혜택을 받고 싶다 " · " 혜택을 놓치고 있다는 불안감 " → 즉시 가입 · 즉시 혜택 사용 욕구

03

로밍 신청 후 클럽 티로밍 혜택 신청 경로 불명

" 로밍 신청과 무료충전을 한 번에 처리하고 싶다 " · " 혜택을 놓치고 싶지 않다 " → 혜택 손실 공포 + 즉시 처리 욕구

04

공휴일 · 야간 멤버십 상담 운영 공백

" 원하는 시간에 바로 처리하고 싶다 " → 24 시간 셀프 처리 채널 욕구

05

미성년자 멤버십 가입 절차 복잡

" 복잡한 절차 없이 자녀 멤버십을 간단하게 가입시키고 싶다 " → 간편성 욕구

3. Hidden Needs

표면 요청 뒤에는 고객이 직접 판단하고 처리 결과를 신뢰하고 싶은 요구가 숨어 있다.

Hidden needs
안심

등급 투명성 욕구 (1 위): " 내가 낸 요금만큼 혜택을 제대로 받고 싶다 / 등급 기준을 납득하고 싶다 "

등급 산정 근거 시각화 + VIP 달성 시뮬레이터 대응 필요

통제

즉시 혜택 사용 욕구 (2 위): " 지금 당장 혜택을 받고 싶다 / 가입 즉시 사용하고 싶다 "

미가입 감지 팝업 + 원스톱 가입 + 바코드 즉시 발급 대응 필요

즉시

미성년자 온라인 가입 프로세스 구축 대응 필요

4. Trigger Top 20

아래 계기는 고객이 셀프 채널을 이탈해 상담으로 전환하는 실제 출발점이다.

Triggers
  1. 01가맹점에서 멤버십 할인 시도 중 미가입 상태 발견 — 전 기간 최다 빈도, 전 기간 1 위
  2. 02요금제 변경 후 VIP→ 골드 / 실버 등급 하락 확인 — 전 기간 2 위
  3. 03로밍 데이터 소진 문자 수신 후 무료충전 방법 모름 — 주말 · 공휴일 1 위
  4. 04앱에서 바코드 또는 쿠폰 미표시 확인 — 전 기간 지속
  5. 05기기 교체 · 번호이동 후 멤버십 재연동 필요 인지 — 전 기간 지속
  6. 06VIP 혜택 (영화 무료 등) 사용 시도 중 등급 미달 확인 — 전 기간 지속
  7. 07주말 · 야간에 멤버십 문의 전화했으나 운영 제한으로 안내만 받음 — 공휴일 · 주말 집중
  8. 08미성년자 자녀 멤버십 가입 시도 중 법정대리인 서류 제한 발견 — 전 기간 지속
  9. 09로밍 신청 후 클럽 티로밍 혜택 신청 경로 불명 — 주말 · 공휴일 집중
  10. 10멤버십 등급 변경 안내 문자 수신 후 원인 확인 불가 — 전 기간 지속
  11. 11제 3 자 앱 · 사이트에서 멤버십 인증 오류 발생 — 전 기간 지속
  12. 12멤버십 포인트 0 원 표시 → 소멸 여부 불안 — 전 기간 지속
  13. 13요금제 변경 후 VIP 혜택 유지 기간 확인 필요 — 전 기간 지속
  14. 14통신사 이동 (KT·LGU+→SKT) 후 멤버십 재가입 필요성 미인지 — 전 기간 지속
  15. 15VIP 픽 영화 예매 경로를 앱에서 찾지 못함 — 전 기간 지속
  16. 16메가커피 등 제휴사 혜택 변경 사전 안내 없이 할인 미적용 인지 — 3 월 초 급증
  17. 17분실 신고 후 정지 · 위치 조회 셀프 처리 방법 모름 — 주말 · 야간 집중
  18. 18구독 서비스 (넷플릭스 등) 결제 문자 수신 후 해지 경로 미발견 — 전 기간 지속
  19. 19대표회선 미등록 오류로 멤버십 가입 불가 — 전 기간 지속
  20. 20타사 이동 후 재가입 시 등급 복구 불가 (대리점 안내 불일치) — 전 기간 지속

5. Quick Win

즉시 개선 가능한 항목은 진입점 노출, 완료 안내, 상태 조회, 권한 처리, 실패 복구를 중심으로 정리된다.

6. 요구사항·정책서 전환 질문

VoC 분석 결과는 멤버십 정책서에서 아래 질문에 답할 수 있는 기능과 정책 항목으로 전환되어야 한다.

Policy conversion
사전 고지

무엇을 처리 전에 알려야 하는가?

요금, 혜택, 적용 시점, 제한 조건, 후속 영향을 업무 실행 전에 이해 가능한 문장과 비교 정보로 제공해야 한다

처리 가능 범위

언제 셀프 처리할 수 있고 언제 제한되는가?

평일, 야간, 주말, 공휴일, 청구 마감, 개통·해지 시점별 처리 가능 범위와 대체 행동을 정의해야 한다

권한

누가 직접 처리하거나 대신 처리할 수 있는가?

본인, 가족, 법정대리인, 대리인의 처리 가능 범위와 인증, 동의, 위임 조건을 정책 기준으로 분리해야 한다

연계

어떤 업무를 한 번에 묶어야 하는가?

상담, 앱, BSS, 외부 기관에서 끊어진 처리를 고객 과업 기준의 단일 흐름으로 재구성해야 한다

상태

처리 상태를 어디까지 보여줘야 하는가?

멤버십 업무의 접수, 진행, 완료, 실패, 취소 상태를 고객이 직접 확인하도록 정의해야 한다
