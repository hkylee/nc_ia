1. 개요

가. 범위

• 본 정책서는 통합채널에서 고객이 보유 회선의 변경, 정지, 정지해제, 분실·도난 차단, 해지·해지예약, 계약자 변경과 구성원 권한 관리를 셀프로 처리하기 위한 기준을 정의한다.

• 대상 채널은 앱·웹 FO이며, 고객이 직접 처리할 수 없는 대리·사망·해외·서류 필요 업무는 운영 검토 또는 상담 전환 경로까지 포함한다.

• BSS와 연계 시스템이 수행하는 회선 상태 조회, 권한 판정, 영향도 산정, 원장 반영, 상태 변경, 이력 저장, 결과 회신은 정책 판단 기준에 포함한다.

• 화면 UI 상세, API 필드, DB 컬럼, 상담 스크립트 원문, 외부기관 내부 절차는 제외하되 고객 처리 가능 여부와 후속 행동에 영향을 주는 기준은 정책 항목으로 남긴다.

• 상품 가입, 결제, 배송, 환불, 혜택 사용 자체의 상세 기준은 인접 정책서에서 다루며 본 문서는 회선 변경과 관리가 해당 업무에 미치는 영향 기준만 정의한다.

• 포함 범위는 회선 선택, 권한 확인, 일시정지·해제, 분실·도난 차단, 해지·예약, 계약자 변경, 특수상황 증빙, 신청 이력과 복구 행동이다.

• 제외 범위는 상품 가입, 결제, 배송, 환불, 혜택 사용의 상세 실행 기준이며, 본 문서는 해당 업무가 회선 변경에 주는 영향과 제한 기준만 다룬다.

• 핵심 정책 질문은 누가 어떤 회선을 변경할 수 있는지, 언제 셀프 처리가 제한되는지, 어떤 증빙과 이력이 필요한지, BSS 원장 반영 실패 시 어떻게 복구하는지다.

• FO는 회선 선택과 고객 안내를 담당하고, BSS는 회선 상태·명의·약정·결합 원장 판정과 반영을 담당하며, 외부 인증·서류 검증은 회선 연계 시스템 회신 기준으로 관리한다.

• 상세 요구사항 71건은 회선 권한, 상태 전이, 프로세스, 기능, 정책 또는 정책 항목 중 하나 이상에 매핑한다.

• 회선 변경은 정지, 해지, 대표 회선, 결합, 납부, 혜택 표시와 연결되므로 BSS 반영 지연이나 장애가 발생하면 채널 간 상태 불일치, 중복 신청, 잘못된 고객 안내로 확대될 수 있다.

• 동일 고객·동일 회선·동일 업무의 반복 요청은 기존 접수번호와 처리 상태를 우선 안내하고, BSS 회신 전 중복 반영이 발생하지 않도록 멱등 기준을 적용한다.

나. 설계 원칙

• 회선 권한 분리: 계정 로그인 성공과 특정 회선 변경 권한을 별도로 판단하고, 권한이 없는 회선은 조회 범위와 변경 행위를 제한한다.

• 영향도 사전 고지: 정지, 해지, 계약자 변경, 분실 차단 전에 요금, 약정, 결합, 혜택, 수신 방법, 후속 서비스 영향이 고객에게 먼저 제시되어야 한다.

• 셀프 처리 우선: 고객이 앱·웹에서 완료할 수 있는 회선 관리 업무는 상담 전환보다 셀프 처리 경로를 우선 제공하되, 법적·권한·증빙 예외는 명확히 전환한다.

• BSS 상태 일관성: FO 표시 상태, BSS 회선 상태, 예약 상태, 신청 이력은 동일 기준으로 관리하고 불일치 시 고객에게 보류 사유와 다음 행동을 안내한다.

• 긴급 차단 우선: 분실·도난 상황은 고객 피해 방지를 위해 인증과 권한을 충족하는 범위에서 차단 요청을 우선 처리하고 후속 복구 경로를 함께 제시한다.

• 증빙 최소화와 추적성: 대리·사망·해외·특수 상황은 필요한 증빙만 요구하고 OCR, 전자서명, 접수번호, 이력 저장으로 후속 검증과 감사가 가능해야 한다.

- 상품 신규 가입, 결제, 배송, 환불, 멤버십 혜택 사용 자체는 인접 정책서로 분리하고, 본 문서는 회선 변경이 해당 업무에 미치는 영향만 정의한다.
• 회선·계정 분리 (PI-LCM-SCOPE-001-03)
- 계정 로그인 성공만으로 회선 변경을 허용하지 않는다.
- 선택 회선의 명의, 위임, 구성원 권한이 확인된 경우에만 변경을 허용한다.
• 채널 일관성 (PI-LCM-SCOPE-001-04)
- 앱·웹·상담에서 동일 회선 상태와 제한 사유를 사용한다.
- 채널별 안내 문구가 다르더라도 처리 가능 여부 기준은 동일하게 관리한다.
• 유선 설치 정보 확인 (PI-LCM-SCOPE-001-05)
- 유선 서비스 해지 또는 중지 전에는 설치 주소와 연락처를 확인한다.
- 장비 회수, 처리 결과 안내, 현장 방문 필요 여부에 영향을 주는 경우 고객 확인을 필수로 둔다.

2) 회선 권한·본인확인 정책 (PG-LCM-AUTH-001)

• 본인확인 기준 (PI-LCM-AUTH-001-01)
- 정지, 해지, 계약자 변경, 분실 신고 취소, PUK 조회는 본인확인 또는 승인된 대리 권한이 확인된 경우에만 허용한다.
• 대리 권한 기준 (PI-LCM-AUTH-001-02)
- 대리 권한자은 가족관계, 위임장, 사망 증빙, 보호자 증빙 중 업무별 필수 증빙이 충족된 경우 해당 범위 안에서만 처리할 수 있다.
• 해외 인증 기준 (PI-LCM-AUTH-001-03)
- 해외 접속 고객은 휴대폰 본인확인이 불가한 경우 이메일 OTP, 아이핀, 공동인증 등 대체 인증을 허용한다.
- 인증 성공 시점과 방식은 이력으로 저장한다.
• 민감 업무 재인증 (PI-LCM-AUTH-001-04)
- 해지, 분실 신고 취소, 유심 잠금 해제, PUK 조회는 인증 완료 후 10분 이내에만 제출할 수 있다.
- 시간이 지나면 재인증을 요구한다.
• 권한 실패 안내 (PI-LCM-AUTH-001-05)
- 권한이 부족한 경우 권한 유형, 부족한 증빙, 가능한 대체 경로를 함께 안내한다.
- 명의자 개인정보는 마스킹한다.
• 납부정보 인증 힌트 (PI-LCM-AUTH-001-06)
- 납부정보 기반 인증이 필요한 경우 카드사명, 은행명, 일부 마스킹 정보처럼 고객이 본인 정보를 식별할 수 있는 힌트를 제공한다.
- 전체 번호는 노출하지 않는다.

3) 처리 가능 여부·제한 정책 (PG-LCM-ELIG-001)

• 진행 중 신청 제한 (PI-LCM-ELIG-001-01)
- 동일 회선에 정지, 해지, 명의 변경, 분실 차단 신청이 진행 중이면 동일 유형의 신규 신청은 제한하고 기존 접수번호를 안내한다.
• 상태별 허용 (PI-LCM-ELIG-001-02)
- 정상 회선은 정지, 해지, 계약자 변경을 신청할 수 있다.
- 해지 완료 회선은 복구 불가로 분류하고 조회와 증빙 발급만 허용한다.
• 미납·법인·특수 회선 제한 (PI-LCM-ELIG-001-03)
- 미납, 법인, 특수 계약, 사법기관 제한, 임대단말 미반납 조건이 있는 경우 셀프 처리를 제한하고 상담 또는 운영 검토로 전환한다.
• 예약 충돌 제한 (PI-LCM-ELIG-001-04)
- 해지 예약과 정지 예약이 동일 시점에 충돌하면 고객에게 우선 처리 대상 선택을 요구한다.
- 선택 전에는 BSS 반영 요청을 전송하지 않는다.

4) 변경 영향도·대체 경로 고지 정책 (PG-LCM-IMPACT-001)

• 비용 영향 고지 (PI-LCM-IMPACT-001-01)
- 해지 또는 정지 전 약정 잔여일수, 위약금, 할부 잔액, 정지 기간 요금, 미납액을 합산해 고객에게 사전 표시한다.
• 결합·가족 영향 고지 (PI-LCM-IMPACT-001-02)
- 대표 회선 변경, 그룹 탈퇴, 해지, 양도 전 가족 할인, 결합 할인, 구성원 권한 영향과 소멸 시점을 고지한다.
• 수신·인증 영향 고지 (PI-LCM-IMPACT-001-03)
- 정지 또는 분실 차단 전 문자 수신, 인증번호 수신, 금융 알림, 로밍 이용 가능 여부를 안내하고 대체 수신 방법을 제시한다.
• 대체 경로 제시 (PI-LCM-IMPACT-001-04)
- 고객에게 불리한 영향이 큰 경우 즉시 처리, 예약 처리, 상담 전환, 철회 중 선택 가능한 대체 경로를 제공한다.

5) 일시정지·정지해제 정책 (PG-LCM-SUSPEND-001)

• 정지 유형 (PI-LCM-SUSPEND-001-01)
- 일시정지는 고객 요청 정지, 분실 정지, 계약 정지, 해외 장기 정지로 구분한다.
- 각 유형은 기간, 요금, 수신 가능 범위를 별도로 관리한다.
• 기간·예약 기준 (PI-LCM-SUSPEND-001-02)
- 일시정지와 정지해제는 즉시 처리와 예약 처리를 모두 허용한다.
- 예약은 현재 시점보다 미래인 날짜만 허용하고 예약 변경 이력을 저장한다.
• 정지해제 조건 (PI-LCM-SUSPEND-001-03)
- 정지해제는 본인확인, 미납 해소, 분실 취소 또는 복구 가능 상태가 충족된 경우에 허용한다.
- 조건 미충족 시 보완 항목을 안내한다.
• 다회선 처리 (PI-LCM-SUSPEND-001-04)
- 다회선 일괄 해제는 고객이 관리 권한을 가진 회선에만 적용한다.
- 일부 회선 실패 시 성공·실패 회선을 분리해 결과를 제공한다.

6) 분실·도난 긴급 차단 정책 (PG-LCM-LOSS-001)

• 긴급 차단 우선 (PI-LCM-LOSS-001-01)
- 분실·도난 신고는 권한이 확인되면 요금 영향 설명보다 차단 요청을 우선 접수한다.
- 차단 후 비용·후속 서비스 안내를 제공한다.
• 기기 유형 기준 (PI-LCM-LOSS-001-02)
- 휴대폰, 워치, 태블릿, 유심 분실은 동일 접수 흐름을 사용하되 차단 대상과 후속 서비스는 기기 유형별로 구분한다.
• 접수번호 자동 조회 (PI-LCM-LOSS-001-03)
- 기존 분실 신고가 있으면 접수번호를 자동 조회해 중복 접수를 제한한다.
- 기존 접수의 취소 또는 복구 경로를 우선 안내한다.
• 신고 취소 기준 (PI-LCM-LOSS-001-04)
- 분실 신고 취소는 신고자 본인 또는 권한 보유자가 재인증한 경우에 허용한다.
- 취소 시 차단 해제 가능 여부를 별도 판정한다.

7) 분실 복구·후속 서비스 정책 (PG-LCM-LOSSREC-001)

• 후속 서비스 연결 (PI-LCM-LOSSREC-001-01)
- 분실 신고 후 위치조회, 보험, 보상센터, 임대폰, 금융 차단, 로밍 차단 중 고객 상황에 필요한 경로를 체크리스트로 제공한다.
• PUK·유심 기준 (PI-LCM-LOSSREC-001-02)
- PUK 조회와 유심 잠금 해제는 재인증 후 1회성 노출로 제공한다.
- 노출 시각과 조회 주체를 감사 이력으로 저장한다.
• 임대폰·보상 연계 (PI-LCM-LOSSREC-001-03)
- 보험 가입 여부와 임대폰 가능 여부는 회선 상태와 상품 조건을 조회해 표시한다.
- 신청은 해당 후속 서비스 경로로 연결한다.
• 복구 완료 안내 (PI-LCM-LOSSREC-001-04)
- 분실 복구가 완료되면 차단 해제, 유심 상태, 보험·임대폰 진행 상태, 금융·로밍 차단 후속 확인을 고객에게 안내한다.
• 보상센터 원스톱 연계 (PI-LCM-LOSSREC-001-05)
- 보험 가입 현황이 확인된 고객은 보상센터 이동 경로와 필요 서류를 분실 완료 화면에서 바로 확인할 수 있다.
- 미가입 고객에게는 대체 후속 경로를 안내한다.
• 유심 잠금 해제 지원 (PI-LCM-LOSSREC-001-06)
- 유심 잠금 상태는 본인확인 후 진단 결과를 제공하고, PIN 초기화, PUK 조회, 지점 방문 중 가능한 해제 경로를 안내한다.
• PUK 코드 조회 (PI-LCM-LOSSREC-001-07)
- PUK 코드 조회는 재인증 성공 후 제한 시간 안에 1회성으로 제공한다.
- 조회 이력, 조회 주체, 노출 시각은 감사 로그로 저장한다.
• 임대 단말 신청 (PI-LCM-LOSSREC-001-08)
- 분실 또는 파손 조건을 충족한 고객은 임대 단말 신청 가능 여부를 확인할 수 있다.
- 불가 사유가 있으면 보증, 재고, 회선 상태 기준으로 사전 안내한다.

8) 회선 해지·해지예약 정책 (PG-LCM-TERMINATION-001)

• 해지 가능 회선 조회 (PI-LCM-TERM-001-01)
- 해지 신청 전 고객 명의 또는 승인된 대리 권한이 있는 회선만 해지 가능 후보로 표시한다.
- 이미 해지 예약된 회선은 기존 예약을 우선 안내한다.
• 해지 영향 확인 (PI-LCM-TERM-001-02)
- 고객이 위약금, 할부, 결합, 번호이동, 가족 할인, 임대단말 반납 영향을 확인해야 해지 신청 버튼을 활성화한다.
• 해지 예약 처리 (PI-LCM-TERM-001-03)
- 해지 예약은 평일 자동 처리 가능 일자와 고객 지정 일자 중 선택한다.
- 예약 시점 전에는 철회를 허용하고 철회 이력을 저장한다.
• 번호이동 후속 처리 (PI-LCM-TERM-001-04)
- 번호이동 후 타사 해지가 완료되지 않은 경우 미완료 상태와 고객 후속 조치를 표시한다.
- 완료 전 중복 해지 요청은 제한한다.
• 모바일 해지 진입 기준 (PI-LCM-TERM-001-05)
- 앱 홈과 나의 가입정보에서 번호 해지 메뉴 바로가기를 제공하되, 진입 후에는 본인확인, 영향 확인, 전자서명, 예약 또는 즉시 처리 단계로 이어지게 한다.
• 해지 절차 단계 안내 (PI-LCM-TERM-001-06)
- 번호 해지 전 본인확인, 서류 제출, 해지 영향 확인, 운영 검토, 완료까지의 단계와 예상 소요 시간을 고객에게 순서대로 안내한다.
• 해지 총 부담금 시뮬레이션 (PI-LCM-TERM-001-07)
- 해지 총 부담금 시뮬레이션은 위약금, 단말 할부금, 미납액, 잔여 청구 가능 금액을 항목별로 표시하고 합계 금액을 해지 신청 전에 확인받는다.

9) 서류·전자서명·증빙 정책 (PG-LCM-DOCUMENT-001)

• 필수 서류 사전 안내 (PI-LCM-DOC-001-01)
- 해지, 사망자 회선, 대리 신청, 해외 장기 정지, 계약자 변경은 신청 전 필수 서류와 준비사항을 업무별로 안내한다.
• 서류 업로드·OCR (PI-LCM-DOC-001-02)
- 업로드 서류는 OCR로 문서 종류, 성명, 생년월일, 유효기간을 사전 점검한다.
- OCR 실패 시 수기 확인이 아닌 운영 검토로 전환한다.
• 전자서명 증적 (PI-LCM-DOC-001-03)
- 전자서명은 신청자, 회선, 문서 버전, 서명 시각, IP 또는 인증 수단을 함께 저장한다.
- 서명 후 문서 내용 변경은 새 서명을 요구한다.
• 보완 제출 기한 (PI-LCM-DOC-001-04)
- 서류 누락 또는 불명확 판정 시 보완 제출 기한을 7일로 안내한다.
- 기한 초과 시 신청은 반려하고 재신청 경로를 제공한다.
• 해외 장기 정지 서류 (PI-LCM-DOC-001-05)
- 해외 장기 정지 또는 재정지 신청은 신분증, 항공권, 체류 증빙 등 업무별 필수 서류를 앱에서 업로드하도록 한다.
- 서류 누락은 보완 필요 상태로 분류한다.

10) 계약자 변경·양도 정책 (PG-LCM-TRANSFER-001)

• 계약자 변경 허용 (PI-LCM-TRN-001-01)
- 계약자 변경은 양도인과 양수인의 권한, 미납 여부, 보증 조건, 약정 승계 동의가 충족된 경우에 허용한다.
• 임대상품 교체 (PI-LCM-TRN-001-02)
- 임대상품 교체는 회선 상태가 정상이고 반납 또는 교체 가능 조건이 충족된 경우에 접수한다.
- 미반납 조건은 제한 사유로 안내한다.
• 양도 영향 고지 (PI-LCM-TRN-001-03)
- 양도 전 약정, 할인, 결합, 멤버십, 청구, 납부 정보 변경 영향을 양도인과 양수인에게 각각 고지한다.
• 권한 갱신 결과 (PI-LCM-TRN-001-04)
- 계약자 변경 완료 후 기존 권한은 즉시 회수하고 신규 계약자의 조회·변경 권한을 BSS와 채널에 반영한다.

11) 대표 회선·구성원·결합 영향 정책 (PG-LCM-GROUP-001)

• 구성원 검증 (PI-LCM-GRP-001-01)
- 구성원 추가·변경은 성명, 생년월일, 관계, 동의 여부가 확인된 경우에만 접수한다.
- 정보 불일치 시 보완을 요구한다.
• 대표 회선 영향 (PI-LCM-GRP-001-02)
- 대표 회선 변경 전 결합, 가족 할인, 알림 수신, 권한 위임 영향이 있는 구성원을 표시하고 고객 확인을 요구한다.
• 그룹 탈퇴 기준 (PI-LCM-GRP-001-03)
- 가족·결합 그룹 탈퇴는 할인 소멸, 잔여 구성원 영향, 재가입 제한 여부를 고지한 뒤 접수한다.
• 권한 범위 (PI-LCM-GRP-001-04)
- 구성원 권한은 조회, 변경 요청, 결합 관리, 해지 요청 범위로 구분한다.
- 권한 범위 변경은 이력으로 저장한다.

12) 해외·사망·대리 특수상황 정책 (PG-LCM-SPECIAL-001)

• 사망자 회선 처리 (PI-LCM-SPC-001-01)
- 사망자 회선 해지는 사망 증빙과 신청자 관계 증빙을 필수로 요구한다.
- 서류 충족 시 운영 검토 후 해지 또는 보완으로 확정한다.
• 해외 신청 처리 (PI-LCM-SPC-001-02)
- 해외 고객은 해외 접속 인증과 대체 서류 제출을 허용한다.
- 귀국 예정일이 있는 경우 귀국 전용 정지해제 예약을 제공한다.
• 대리 신청 범위 (PI-LCM-SPC-001-03)
- 대리 신청은 위임 범위 안의 업무만 허용한다.
- 해지, 명의 변경, 분실 취소는 별도 명시 위임이 있는 경우에만 허용한다.
• 특수 상황 상담 전환 (PI-LCM-SPC-001-04)
- 거동 불편, 서류 판독 불가, 해외 인증 실패, 법인/특수 회선은 운영 검토 또는 상담 전환으로 안내한다.

13) 신청 이력·상태 추적 정책 (PG-LCM-HISTORY-001)

• 접수번호 부여 (PI-LCM-HIS-001-01)
- 정지, 해지, 분실, 계약자 변경, 서류 제출, 운영 검토 요청은 접수번호를 부여하고 고객·상담·운영에서 동일 번호로 조회한다.
• 상태 이력 저장 (PI-LCM-HIS-001-02)
- 상태 변경, 예약 등록, 예약 취소, 반려, 보완, BSS 반영 실패, 운영 승인 결과는 변경 전후 상태와 시각을 저장한다.
• 감사 로그 (PI-LCM-HIS-001-03)
- PUK 조회, 서류 열람, 대리 권한 승인, 사망자 회선 처리는 처리 주체, 근거, 접근 시각을 감사 로그로 저장한다.
• 고객 조회 범위 (PI-LCM-HIS-001-04)
- 고객은 본인 또는 승인된 대리 권한이 있는 회선의 신청 내역만 조회할 수 있다.
- 만료된 신청은 요약 이력만 제공한다.

14) BSS 반영·원장 동기화 정책 (PG-LCM-BSS-001)

• 사전 검증 회신 (PI-LCM-BSS-001-01)
- BSS는 회선 상태, 명의, 납부, 약정, 결합, 임대상품, 진행 중 예약 정보를 사전 검증 결과로 회신한다.
• 상태 변경 반영 (PI-LCM-BSS-001-02)
- 정지, 정지해제, 해지, 명의 변경, 권한 갱신은 BSS 원장 반영 성공 후 고객 완료 상태로 표시한다.
• 연계 실패 처리 (PI-LCM-BSS-001-03)
- BSS 반영 실패 시 고객 화면은 처리 중 또는 운영 확인 필요 상태로 표시하고, 동일 요청 자동 재시도는 최대 2회로 제한한다.
- 운영 확인 전에는 완료 상태로 확정하지 않는다.
• 결과 회신 기준 (PI-LCM-BSS-001-04)
- BSS 회신에는 성공·실패 상태, 실패 사유, 재처리 가능 여부, 반영 시각을 포함한다.
- 고객에게는 내부 오류 코드를 노출하지 않는다.

15) 고객 고지·fallback 정책 (PG-LCM-NOTICE-001)

• 실패 사유 안내 (PI-LCM-NOT-001-01)
- 실패, 제한, 반려, 보류 상태에는 고객이 이해할 수 있는 사유와 다음 행동을 함께 표시한다.
• 사유별 fallback (PI-LCM-NOT-001-02)
- 셀프 처리가 불가한 경우 상담 전환, 운영 검토, 서류 보완, 나중에 다시 시도 중 하나의 fallback 경로를 제공한다.
• 처리 결과 고지 (PI-LCM-NOT-001-03)
- 처리 완료 시 변경된 회선 상태, 적용 시점, 접수번호, 후속 체크리스트를 앱·웹과 알림으로 제공한다.
• 고객 용어 기준 (PI-LCM-NOT-001-04)
- 고객 안내 문구에는 내부 시스템명과 약어를 제외하고 회선 상태, 제한 사유, 필요한 행동을 쉬운 표현으로 작성한다.
• 채널 간 해지 기준 일관화 (PI-LCM-NOT-001-05)
- 채널 간 해지 기준 일관화는 앱, 웹, 상담, 오프라인 채널의 해지 기준, 제출 서류 기준, 처리 상태 표현을 동일 정책 버전으로 관리하는 것을 의미한다.

16) 예외 승인·운영 모니터링 정책 (PG-LCM-OPERATION-001)

• 운영 확인 SLA (PI-LCM-OPR-001-01)
- 운영 검토 대상은 접수 후 1영업일 안에 승인, 반려, 보완 중 하나로 분류한다.
- SLA 초과 건은 모니터링 목록에 노출한다.
• 예외 승인 기준 (PI-LCM-OPR-001-02)
- 예외 승인은 증빙 충족, 고객 피해 가능성, BSS 처리 가능성, 법적 제한 여부를 모두 확인한 경우에만 허용한다.
• 예약 충돌 관리 (PI-LCM-OPR-001-03)
- 동일 회선의 예약 충돌은 고객 선택이 필요한 충돌과 시스템 재처리 충돌로 분류한다.
- 고객 선택 충돌은 자동 반영하지 않는다.
• 안내문 관리 (PI-LCM-OPR-001-04)
- 사유별 안내문과 fallback 정책은 운영자가 버전, 적용일, 승인자를 관리한다.
- 변경 이력은 최소 1년 보관한다.

17) 개인정보·증빙 보관 정책 (PG-LCM-PRIVACY-001)

• 민감 증빙 최소화 (PI-LCM-PRV-001-01)
- 서류 제출은 업무에 필요한 항목만 수집한다.
- 주민등록번호 뒷자리, 계좌번호, 타인 연락처는 표시와 저장 단계에서 마스킹한다.
• 위치조회 고지 (PI-LCM-PRV-001-02)
- 분실 후 위치조회 연계는 고객 동의와 서비스 가입 여부가 확인된 경우에만 안내한다.
- 위치정보는 본 정책서에서 저장하지 않는다.
• PUK 보안 (PI-LCM-PRV-001-03)
- PUK와 유심 잠금 해제 정보는 재인증 후 제한 시간 안에만 노출한다.
- 화면 캡처 방지와 조회 이력 저장 기준을 적용한다.
• 보관 기간 (PI-LCM-PRV-001-04)
- 서류와 전자서명 증적은 업무 완료 후 법정·분쟁 대응 기간 기준으로 보관하고, 기간 만료 시 파기 대상에 포함한다.

18) 중복 요청·예약 충돌 정책 (PG-LCM-IDEMPOTENCY-001)

• 중복 요청 제한 (PI-LCM-IDM-001-01)
- 동일 회선, 동일 업무, 동일 신청자가 5분 이내 같은 요청을 제출하면 중복 요청으로 분류하고 기존 접수번호를 안내한다.
• 예약 우선순위 (PI-LCM-IDM-001-02)
- 해지 예약, 정지 예약, 정지해제 예약이 충돌하면 고객 피해가 큰 해지와 분실 차단을 우선 판단하고 나머지는 고객 선택을 요구한다.
• 재시도 기준 (PI-LCM-IDM-001-03)
- BSS 일시 실패는 최대 2회 자동 재시도한다.
- 2회 실패 후에는 운영 확인 필요 상태로 전환하고 고객에게 처리 중 안내와 예상 확인 경로를 제공한다.
• 멱등 처리 (PI-LCM-IDM-001-04)
- 재전송된 동일 접수번호 요청은 새 신청을 만들지 않고 기존 신청의 최신 상태를 반환한다.

최종 점검 기준

회선변경관리 정책서 제출 전 점검
1. 회선 권한과 고객 계정 인증이 분리되어 있는지 확인한다.
2. 정지, 해지, 분실, 계약자 변경 전 고객 영향도와 대체 경로가 제시되는지 확인한다.
3. 유즈케이스가 고객 또는 운영자의 완결 업무 단위이며, 프로세스가 해당 업무를 완료하는 절차인지 확인한다.
4. 각 프로세스가 실제 기능 ID와 정책 ID에 연결되어 있는지 확인한다.
5. 상태 전이표의 현재 상태와 다음 상태가 상태 코드 목록과 일치하는지 확인한다.
6. BSS 원장 반영, 연계 실패, 재처리, 이력 저장 기준이 정책 항목으로 남아 있는지 확인한다.
7. 대리, 사망, 해외, 분실 등 특수 상황의 인증·증빙·개인정보 보호 기준이 있는지 확인한다.
8. 정책 항목에 실제 조건, 기간, 횟수, 상태, 제한, 고지, 이력 기준이 포함되어 있는지 확인한다.
9. 요구사항 73건이 유즈케이스, 프로세스, 기능, 정책 중 하나 이상에 연결되는지 확인한다.
10. 요구사항 추적성
상세 요구사항은 유즈케이스, 상태, 프로세스, 기능, 정책 또는 정책 항목 중 하나 이상으로 추적 가능해야 한다.
11. 개발·QA 전환성
각 정책 항목은 테스트 조건, 기대 결과, 실패·복구 경로로 전환 가능해야 한다.
12. 책임 경계 정합성
FO, BSS, 연계 시스템, 외부 주체의 판단·저장·회신 책임이 혼재되지 않아야 한다.
