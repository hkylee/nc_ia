1. 개요

가. 범위

• 본 정책서는 통합채널에서 제공하는 외부 제휴처 쿠폰의 탐색, 상세 확인, 발급, 다운로드, 사용, 쿠폰함 저장, 상태·이력 확인, 운영 관리 기준을 정의한다.

• 고객이 발급 전 혜택 내용, 사용처, 사용 방식, 사용 가능 시간·요일, 제외 조건, 중복 사용 가능 여부, 온라인·오프라인 사용 범위를 판단할 수 있는 기준을 포함한다.

• 외부 쿠폰의 선택형 구조, 추가 인증·동의, 제휴처 연결, 발급 실패, 사용 불가, 오픈 알림, 쿠폰함 재진입, 사용 후 결과 확인과 복구 기준을 포함한다.

• 운영자가 쿠폰 기준정보, 발급 정책, 대상 조건, 코드 업로드, 재고, 상태 동기화, 발급 중지, 회수, 재발급, 실적, 정산, 승인·감사로그를 관리하는 기준을 포함한다.

• 결제 할인 산정, 멤버십 혜택 자체 사용, 포인트 적립·차감, 이벤트 보상 상세, 제휴 계약 정산 상세는 인접 정책서 또는 후속 산출물에서 상세화한다.

• API 필드, DB 컬럼, 운영자 화면 상세, 배치 설계, 제휴처 내부 처리 로직은 제외하되 정책 판단에 필요한 상태, 조건, 제한, 이력 기준은 포함한다.

• 책임 경계는 통합채널 FO가 탐색·발급·사용·쿠폰함 상태 표시를 담당하고, 쿠폰 연계 시스템이 제휴처 재고·사용 승인·상태 동기화와 쿠폰함 상태 제공을 담당하며, BSS는 고객·회선·등급·상품 보유 기준 정보를 제공하는 구조로 구분한다.

• 고객 표시 상태와 시스템 내부 상태는 분리한다.
고객에게는 발급 가능, 기보유, 사용 가능, 사용 완료, 만료, 회수, 발급 실패를 보여주고, 내부적으로는 조건 판정, 코드 매핑, 제휴처 승인, 정산 보류, 재발급, 감사로그 상태를 관리한다.

• 고객이 앱·웹에서 직접 발급, 다운로드, 사용 경로 이동, 재다운로드, 양도 회수, 실패 복구 요청을 완료하게 하고, 운영자 수작업은 승인·보정·정산 예외로 제한한다.

나. 설계 원칙

• 사용 가능성 사전 판단: 고객이 쿠폰을 발급받기 전에 지금 사용할 수 있는지, 왜 불가한지, 언제 가능해지는지 알 수 있게 한다.

• 발급 후 행동 연결: 발급 완료 후 즉시 사용, 쿠폰함 이동, 재다운로드, 다른 쿠폰 보기 중 고객에게 필요한 다음 행동을 끊김 없이 제공한다.

• 제휴처 연동 신뢰성: 외부 제휴처 재고, 사용 완료, 회수, 만료, 오류 상태를 동기화하고 불일치 시 운영 확인과 고객 안내 기준을 둔다.

• 통합 쿠폰함 중심 관리: 고객은 모든 외부 쿠폰의 상태와 이력을 통합 쿠폰함에서 확인하고, 운영자는 동일 상태 체계로 회수·재발급·정산을 관리한다.

• 운영 통제 내재화: 쿠폰 등록, 복제, 발급 조건 변경, 코드 업로드, 배포, 중지, 회수, 정산 확정은 승인과 감사 이력을 남긴다.

• 인접 정책 경계 유지: 쿠폰이 연결되는 혜택·결제·포인트·이벤트의 본 정책값은 해당 정책서에서 다루고 본 문서는 외부 쿠폰 실행과 운영 기준에 집중한다.

1) 외부 쿠폰 범위·라벨 정책 (PG-CPN-SCOPE-001)

• 쿠폰 범위 (PI-CPN-SCOPE-001-01)
- 외부 쿠폰은 통합채널과 연계된 외부 제휴처가 제공하는 할인권, 이용권, 교환권, 바코드형 쿠폰, 링크형 쿠폰을 포함한다.
- 내부 포인트, 멤버십 혜택 자체 사용 정책은 별도 정책서로 분리한다.
• 표준 라벨 (PI-CPN-SCOPE-001-02)
- 쿠폰 카드와 상세에는 혜택 유형, 사용처, 사용 방식, 유효기간, 대상 등급, 조건 충족 여부를 6개 표준 라벨로 표시한다.
- 라벨명은 고객 문구와 운영 문구를 분리해 저장한다.
• 인접 업무 경계 (PI-CPN-SCOPE-001-03)
- 상품 구매, 멤버십 혜택, 이벤트 보상은 외부 쿠폰 발급·사용 연결이 있는 경우에만 본 정책서에 포함한다.
- 결제 할인 산정과 포인트 정산은 해당 정책서에서 결정한다.

2) 목록·상세 노출 정책 (PG-CPN-DISC-001)

• 목록 상태 구분 (PI-CPN-DISC-001-01)
- 목록은 지금 발급 가능, 오픈 예정, 마감 임박, 조건 미충족, 기보유, 재다운로드 가능, 소진 상태로 구분한다.
- 상태가 2개 이상 겹치면 기보유, 소진, 마감 임박 순서로 우선 표시한다.
• 상세 핵심 정보 (PI-CPN-DISC-001-02)
- 상세 첫 화면에는 혜택 내용, 사용처, 사용 방식, 사용 가능 시간·요일, 주요 제외 조건, 중복 사용 가능 여부, 온라인/오프라인 범위를 모두 노출한다.
- 7개 항목 중 하나라도 없으면 운영 검수에서 배포를 보류한다.
• 필터·정렬 기준 (PI-CPN-DISC-001-03)
- 필터는 사용처, 내 주변, 카테고리, 등급, 사용 방식, 사용 가능 상태를 제공한다.
- 정렬은 오늘 사용 가능, 만료 임박, 최신 발급, 혜택 큰 순을 기본 후보로 둔다.
• 위치 기반 매장 안내 (PI-CPN-DISC-001-04)
- 쿠폰 상세에는 제휴사별 사용 가능 매장과 사용 불가 매장을 구분해 제공한다.
- 현재 위치 사용에 동의한 고객에게는 내 주변 사용 가능 매장 수를 우선 표시하고, 매장 데이터는 최소 1일 1회 검증 또는 제휴처 갱신 이력 기준으로 관리한다.

3) 발급 가능 조건 정책 (PG-CPN-ELIG-001)

• 대상 조건 (PI-CPN-ELIG-001-01)
- 발급 대상은 등급, 상품 보유, 지역, 채널, 기간, 본인확인 상태 중 운영자가 선택한 조건을 모두 충족한 고객으로 제한한다.
- 불충족 조건은 최대 3개까지 고객 문구로 안내한다.
• 수량·횟수 한도 (PI-CPN-ELIG-001-02)
- 발급 수량은 전체 재고, 고객 1인당 한도, 일 단위 한도, 기간 단위 한도로 판정한다.
- 한도 초과 시 발급 버튼은 노출하지 않고 기발급 또는 한도 초과 상태를 표시한다.
• 시간·요일 제한 (PI-CPN-ELIG-001-03)
- 사용 가능 시간·요일이 있는 쿠폰은 발급 전과 사용 전 모두 제한 상태를 판정한다.
- 제한 시간에는 사용 불가 사유와 다음 사용 가능 시각을 함께 안내한다.
• 할인 적용 대상 (PI-CPN-ELIG-001-04)
- 할인 쿠폰은 할인금액, 최소 구매금액, 적용 대상 상품·서비스·매장, 중복 사용 가능 여부를 기준으로 적용 여부를 판단한다.
- 적용 대상이 불명확하면 자동 적용하지 않고 사용 전 확인 상태로 둔다.

4) 선택형 쿠폰 정책 (PG-CPN-SELECT-001)

• 선택지 비교 (PI-CPN-SELECT-001-01)
- 선택형 쿠폰은 각 선택지의 혜택, 사용처, 유효기간, 변경 가능 여부, 즉시 사용 가능 여부를 같은 행 기준으로 비교한다.
- 고객이 선택 전에 최소 2개 이상의 차이를 이해할 수 있어야 한다.
• 선택 확정 (PI-CPN-SELECT-001-02)
- 선택 완료 후에는 다운로드, 쿠폰함 이동, 즉시 사용 중 가능한 다음 행동을 1개 이상 제공한다.
- 발급이 완료된 선택지는 동일 기간 내 중복 선택을 제한한다.
• 선택 변경 제한 (PI-CPN-SELECT-001-03)
- 선택 변경은 발급 전 또는 제휴처 코드 미매핑 상태에서만 허용한다.
- 발급 완료, 사용 완료, 회수, 만료 상태에서는 변경을 제한하고 사유를 안내한다.
• 2종 중 택1 상호 배타 (PI-CPN-SELECT-001-04)
- 2종 중 택1 쿠폰은 두 쿠폰을 일괄 지급하더라도 사용 시 하나의 쿠폰만 혜택을 적용한다.
- 한 쿠폰이 사용 완료되면 잔여 쿠폰은 자동 소멸 또는 사용 제한 상태로 전환하고 고객에게 사유를 안내한다.

5) 발급·다운로드 정책 (PG-CPN-ISSUE-001)

• 발급 요청 (PI-CPN-ISSUE-001-01)
- 발급 요청은 로그인, 추가 인증, 필수 동의, 외부 제휴 연결, 재고 판정이 모두 완료된 경우에만 접수한다.
- 발급 처리 중 중복 클릭은 5초 이내 1건으로 멱등 처리한다.
• 완료 후 행동 (PI-CPN-ISSUE-001-02)
- 발급 완료 시 즉시 사용 가능한 쿠폰은 즉시 사용을 1순위 행동으로 제공한다.
- 즉시 사용이 불가하면 쿠폰함 저장 완료와 사용 가능 시각을 안내한다.
• 재다운로드 (PI-CPN-ISSUE-001-03)
- 재다운로드는 발급 완료, 미사용, 유효기간 내, 회수되지 않은 쿠폰에만 허용한다.
- 난수번호 또는 바코드가 재생성되는 경우 기존 코드 폐기 이력을 저장한다.

6) 쿠폰함 저장·상태 정책 (PG-CPN-BOX-001)

• 저장 기준 (PI-CPN-BOX-001-01)
- 발급 완료 쿠폰은 통합 쿠폰함에 1건 이상 저장되어야 하며, 쿠폰 ID, 제휴처, 유효기간, 사용 상태, 발급 시각, 회수 여부를 조회 기준으로 남긴다.
• 상태 표시 (PI-CPN-BOX-001-02)
- 쿠폰함 상태는 사용 가능, 사용 완료, 사용 불가, 만료, 회수, 발급 실패, 재다운로드 가능으로 표시한다.
- 고객 노출 상태와 운영 상태가 다르면 고객 노출 상태를 우선 설명한다.
• 이력 제공 (PI-CPN-BOX-001-03)
- 쿠폰함은 발급, 다운로드, 사용 시도, 사용 완료, 실패, 회수, 재발급 이력을 시간순으로 보관한다.
- 고객에게는 최근 1년 이력을 기본 제공한다.

7) 사용수단·보안 정책 (PG-CPN-USE-001)

• 사용수단 표시 (PI-CPN-USE-001-01)
- 사용수단은 바코드, QR, 난수번호, 제휴 링크, 자동 적용 중 하나로 분류한다.
- 고객 화면에는 사용수단, 유효시간, 갱신 필요 여부, 제휴처 확인 문구를 함께 표시한다.
• 오프라인 제시 (PI-CPN-USE-001-02)
- 오프라인 제시형 쿠폰은 전체화면 보기와 화면 밝기 강화 선택을 제공한다.
- 바코드 또는 QR 유효 시간이 있는 경우 남은 시간을 분 단위로 안내한다.
• 코드 보호 (PI-CPN-USE-001-03)
- 쿠폰코드 복제·공유가 제한되는 쿠폰은 화면 캡처 제한 또는 부분 가림을 적용한다.
- 보호 적용 여부와 공유 불가 사유는 상세 유의사항에 고지한다.
• 결제 자동 적용 (PI-CPN-USE-001-04)
- 매장 결제 또는 바코드 제시 시 보유 쿠폰 중 사용처, 유효기간, 최소 구매금액, 적용 대상이 충족된 쿠폰을 자동 감지한다.
- 자동 적용 결과와 미적용 사유는 결제 직후 고객에게 안내하고, 오적용이 확인되면 취소·복구 이력을 저장한다.

8) 실패·사용불가·복구 정책 (PG-CPN-FAIL-001)

• 실패 사유 (PI-CPN-FAIL-001-01)
- 발급 실패와 사용 불가는 재고 소진, 시간 미도래, 등급 미충족, 제휴처 오류, 코드 중복, 회수, 만료, 인증 실패로 분류한다.
- 사유가 확인되지 않으면 운영 확인 필요 상태로 전환한다.
• 재시도 기준 (PI-CPN-FAIL-001-02)
- 일시 오류는 최대 3회까지 재시도를 허용한다.
- 제휴처 오류가 5분 이상 지속되면 자동 재시도 안내 대신 다른 쿠폰 보기, 쿠폰함 이동, 상담 연결을 제공한다.
• 대체 행동 (PI-CPN-FAIL-001-03)
- 발급 또는 사용이 불가하면 재시도, 다른 외부 쿠폰 보기, 쿠폰함 이동, 제휴처 안내, 상담 연결 중 고객에게 가능한 행동을 1개 이상 제공한다.

9) 운영 기준정보·버전 정책 (PG-CPN-OPS-001)

• 기준정보 등록 (PI-CPN-OPS-001-01)
- 운영자는 쿠폰명, 유형, 제휴처, 사용처, 혜택, 적용 방식, 유효기간, 대표 이미지, 안내 문구를 필수 기준정보로 등록한다.
- 필수 9개 항목 누락 시 저장은 임시저장까지만 허용한다.
• 복제·버전 (PI-CPN-OPS-001-02)
- 기존 쿠폰을 복제할 때는 제휴처, 유효기간, 재고, 대상 조건을 새 버전에서 다시 확인해야 한다.
- 배포 이력이 있는 쿠폰 수정은 신규 버전으로 저장한다.
• 검수·승인 (PI-CPN-OPS-001-03)
- 생성, 조건 변경, 회수, 종료, 상단 고정은 승인 대상 작업으로 관리한다.
- 승인자, 작업 시각, 변경 전후 값, 적용 범위는 감사로그에 저장한다.
• 사용기간 유형 (PI-CPN-OPS-001-04)
- 외부 쿠폰 사용기간은 절대기간과 발행일 기준 기간으로 구분한다.
- 절대기간은 고정 시작·종료일을 따르고, 발행일 기준 기간은 발급일 또는 다운로드일 기준 유효 일수를 적용한다.

10) 코드·재고·연동 정책 (PG-CPN-CODE-001)

• 코드 업로드 (PI-CPN-CODE-001-01)
- 쿠폰 코드 대량 업로드는 쿠폰코드, 시리얼, 유효기간, 상태값을 포함해야 한다.
- 1회 업로드 파일은 운영 한도 내에서 분할 처리하고 오류 행은 정상 행과 분리한다.
• 사전 검증 (PI-CPN-CODE-001-02)
- 업로드 전 중복, 형식 오류, 유효기간 오류, 상태 불일치, 이미 사용된 코드 여부를 검증한다.
- 오류 항목은 행 번호와 사유를 포함한 리포트로 제공한다.
• 상태 동기화 (PI-CPN-CODE-001-03)
- 외부 제휴처 연동은 API 또는 파일 방식으로 수행하며 재고, 발급 가능 상태, 사용 완료, 회수, 만료 상태를 주기적으로 동기화한다.
- 연동 실패는 최대 3회 재시도 후 오류 로그, 수동 재수집, 고객 안내 대상으로 기록한다.
• 쿠폰 구매·적재 내재화 (PI-CPN-CODE-001-04)
- 외부 쿠폰 구매, 코드 적재, 발송 대상 고객과 쿠폰 번호 매핑은 권한 있는 운영자가 승인된 파일 또는 연동 결과 기준으로만 등록한다.
- 고객 정보 업로드·다운로드는 권한 있는 운영자만 수행하고 파일 생성·반출 이력을 감사로그에 남긴다.

11) 중지·회수·재발급 정책 (PG-CPN-CONTROL-001)

• 중지 기준 (PI-CPN-CONTROL-001-01)
- 오류, 제휴처 요청, 조건 오등록, 재고 불일치, 과발급 위험이 확인되면 운영자는 발급 중지를 즉시 적용할 수 있다.
- 중지 상태에서는 신규 발급을 제한하고 기존 보유 쿠폰은 별도 회수 기준을 따른다.
• 회수 기준 (PI-CPN-CONTROL-001-02)
- 회수는 대상 범위, 회수 사유, 고객 고지 여부, 대체 보상 여부가 승인된 경우에만 실행한다.
- 사용 완료 쿠폰은 회수 대상에서 제외하고 정산 보정 대상으로 분리한다.
• 재발급 기준 (PI-CPN-CONTROL-001-03)
- 재발급은 제휴처 오류, 코드 불량, 고객 미사용, 유효기간 내 요청, 회수 사유가 고객 귀책이 아닌 경우에 허용한다.
- 재발급 시 기존 코드 폐기와 신규 코드 발급 이력을 모두 저장한다.
• 양도·안심 회수 (PI-CPN-CONTROL-001-04)
- 쿠폰 양도는 발송자 보유, 미사용, 유효기간 내, 공유 허용 상태에서만 가능하다.
- 수신자가 지정 기간 내 미수령 또는 미사용 상태이면 발송자는 쿠폰을 회수할 수 있고, 회수 후 기존 공유 링크는 즉시 무효화하며 양도·회수 이력을 저장한다.

12) 실적·정산·감사 정책 (PG-CPN-MON-001)

• 실적 집계 (PI-CPN-MON-001-01)
- 운영 실적은 발급 수, 다운로드 수, 사용 수, 미사용 잔량, 만료 수, 실패 수, 제휴처 오류 수를 기간, 제휴처, 유형, 고객군, 채널, 상품 기준으로 집계한다.
• 정산 기준 (PI-CPN-MON-001-02)
- 정산 대상은 사용 완료와 제휴처 사용 승인 상태가 일치한 건으로 한정한다.
- 미차감, 반영 지연, 제휴처 오류 건은 정산 보류 상태로 분리하고 보정 완료 후 확정한다.
• 감사로그 (PI-CPN-MON-001-03)
- 생성, 수정, 승인, 배포, 중지, 회수, 재발급, 정산 확정은 작업자, 승인자, 작업 시각, 변경 전후 값, 적용 범위를 최소 3년 보관한다.

13) 알림·재진입 정책 (PG-CPN-NOTI-001)

• 오픈 알림 (PI-CPN-NOTI-001-01)
- 오픈 예정 또는 선착순 쿠폰은 고객이 알림을 신청할 수 있다.
- 오픈 시점 도래 시 알림은 쿠폰 상세 또는 발급 단계로 바로 재진입하는 링크를 포함한다.
• 만료·중단 알림 (PI-CPN-NOTI-001-02)
- 만료 임박, 사용 중단, 제휴처 정책 변경은 고객에게 사전 안내한다.
- 만료 임박 알림은 최소 1회 제공하며, 긴급 중단은 가능한 즉시 대체 행동을 함께 안내한다.
• 재진입 경로 (PI-CPN-NOTI-001-03)
- 알림, 쿠폰함, 목록, 상세에서 재진입할 때는 고객이 보던 쿠폰 맥락을 유지한다.
