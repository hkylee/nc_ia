1. 개요

가. 범위

• 본 정책서는 통합채널에서 홈, 메뉴, 탭, 배너, 카드, 콘텐츠, 퀵메뉴, 개인화 영역을 고객 여정 기준으로 전시하고 운영하는 기준을 정의한다.

• 포함 범위는 고객 전시 확인, 바로가기 편집, 개인화 숨김, 랜딩·복귀, 운영자 메뉴·페이지·컴포넌트 관리, 검수·승인·배포·롤백, 성과 측정이다.

• 대상 채널은 통합채널 앱·웹 FO이며, 어드민에서 설정한 전시 구조가 채널별 화면에 반영되는 범위까지 포함한다.

• BSS와 채널 업무 시스템이 제공하는 고객 상태, 보유 상품, 혜택, 주문 진행 상태, 제한 조건은 전시 가능 여부와 우선순위 판정 기준으로 사용한다.

• 제외 범위는 개별 화면 와이어프레임, 이미지 제작 상세, API 필드, DB 컬럼, 개별 상품·혜택의 세부 계약 정책, 광고 심의 전문이다.

• 후속 산출물에서는 각 업무 정책서가 전시에서 진입한 후 주문, 변경, 납부, 혜택 사용, 상담 전환을 완료하는 세부 처리 기준을 상세화한다.

나. 설계 원칙

• Journey First Display: 전시는 메뉴 나열이 아니라 고객이 현재 상태를 확인하고 다음 행동을 실행하도록 돕는 여정 단위로 구성한다.

• Customer Control: 고객은 퀵메뉴, 숨김, 관심없음, 일정 기간 비노출을 통해 홈과 주요 지면의 개인화 노출을 제어할 수 있어야 한다.

• Governed Operation: 운영 변경은 작성, 검수, 승인, 예약, 게시, 중지, 롤백 상태를 거치며 권한과 이력을 남긴다.

• Reasoned Personalization: 개인화 전시는 고객 상태, 최근 행동, 보유 관계, 진행 업무 근거를 바탕으로 하며 노출 이유와 제한 사유를 설명할 수 있어야 한다.

• Safe Automation: AI 문구·이미지·배치 제안은 운영자 승인 전까지 자동 게시되지 않으며 금칙어, 접근성, 법무·브랜드 기준을 통과해야 한다.

• Measured Improvement: 전시 변경은 노출, 클릭, 후속 행동, 숨김, 이탈, 실험 결과를 통해 검증하고 성과 이상 시 중지하거나 되돌릴 수 있어야 한다.

1) 전시 범위·경계 정책 (PG-DSP-SCP-001)

• 전시 직접 범위 (PI-DSP-SCP-001)
- 전시/관리 기능은 홈, 메뉴, 탭, 배너, 카드, 콘텐츠, 퀵메뉴, 노출 조건, 배포 상태를 직접 범위로 둔다.
- 주문, 변경, 납부, 혜택 사용의 상세 처리 기준은 해당 업무 정책서로 분리한다.
• Core·채널 특화 구분 (PI-DSP-SCP-002)
- Core 공통 영역은 슬롯, 카드, 비교, 카테고리, 상세 템플릿처럼 2개 이상 채널에서 재사용하는 전시 구조로 둔다.
- 채널 특화 영역은 혜택, 외부 BP, 구독처럼 고유 경험을 유지해야 하는 경우에만 예외로 허용한다.
• 가격·혜택 표기 경계 (PI-DSP-SCP-003)
- 가격, 할인, 혜택은 기간, 조건, 예외, 적용 대상, 예상 금액 여부를 함께 표기한다.
- 확정 금액이 아니면 예상 또는 시뮬레이션 표시를 필수로 둔다.

2) 홈 전시 우선순위 정책 (PG-DSP-HOM-001)

• 홈 우선순위 (PI-DSP-HOM-001)
- 홈 전시 우선순위는 필수 알림, 진행 중 업무, 고객 상태 요약, 즉시 사용 혜택, 최근 이용 기능, 일반 콘텐츠 순으로 적용한다.
- 긴급 공지와 보안 안내는 예외적으로 최상단에 노출할 수 있다.
• 첫 진입 체감 (PI-DSP-HOM-002)
- 첫 화면은 10초 안에 고객이 핵심 변화 포인트와 대표 행동을 인지할 수 있도록 구성한다.
- 첫 10초 내 핵심 액션 진입률, 홈 이탈률, 탐색 시작률을 측정 항목으로 둔다.
• 액션형 요약 (PI-DSP-HOM-003)
- 요금, 데이터, 혜택, 주문 상태, 가입 상품 요약은 조회 정보만 노출하지 않는다.
- 각 요약에는 확인, 변경, 충전, 납부, 사용, 상담 중 1개 이상의 후속 행동을 연결한다.
• 동적 홈 재구성 (PI-DSP-HOM-004)
- 홈 전시는 고객 상태, 최근 행동, 가입·사용 상품, 혜택 만료, 미완료 업무를 기준으로 우선순위를 재산정한다.
- 긴급 공지, 장애, 법정 고지는 개인화보다 우선 노출한다.
• 인포데스크 역할 (PI-DSP-HOM-005)
- 인포데스크 영역은 고객이 지금 확인해야 할 고지, 미완료 업무, 상태 변화, 혜택 만료, 오류 복구 경로를 한곳에서 확인하도록 구성한다.
- 단순 배너성 홍보는 인포데스크 핵심 항목으로 분류하지 않는다.
• 진행 중 업무 이어하기 (PI-DSP-HOM-006)
- 홈은 최근 진행 중인 신청, 변경, 결제, 상담, 혜택 사용 업무를 상태 기반 진입점으로 제공한다.
- 이어하기 진입점은 마지막 저장 시각, 현재 상태, 다음 필요 행동을 함께 표시한다.
• 비교·최적가 퀵진입 (PI-DSP-HOM-007)
- 요금제, 결합, 구독, 로밍처럼 구매·변경 의사결정이 필요한 과업은 비교 또는 최적가 확인 CTA를 홈 또는 핵심 지면에서 1단계 이내로 제공한다.
• 로그인 전 가치 프리뷰 (PI-DSP-HOM-008)
- 비로그인 고객에게도 멤버십, 구독, 상품의 핵심 혜택과 로그인 후 가능한 행동을 미리 보여준다.
- 로그인 또는 가입 후에는 프리뷰에서 선택한 맥락을 유지해 후속 화면으로 연결한다.
• 변경 경로 단순화 (PI-DSP-HOM-009)
- 요금제 변경, 혜택 사용, 구독 변경처럼 반복 과업은 고객이 현재 상태를 확인한 뒤 최소 단계로 변경 화면에 도달하도록 홈·메뉴·검색 진입 경로를 제공한다.

3) 퀵메뉴·바로가기 정책 (PG-DSP-QCK-001)

• 퀵메뉴 후보 (PI-DSP-QCK-001)
- 퀵메뉴 후보는 로그인 여부, 보유 회선, 가입 서비스, 최근 이용 이력, C360 기반 사용 패턴을 기준으로 산정한다.
- 데이터, 요금, 납부, 멤버십, 로밍처럼 고빈도 과업은 기본 후보에 포함한다.
• 고객 고정 우선 (PI-DSP-QCK-002)
- 고객이 직접 고정한 퀵메뉴는 시스템 추천보다 우선한다.
- 고객 고정 항목은 삭제, 순서 변경, 기본 추천값 복원을 허용하고 변경 이력을 저장한다.
• 후보 제한 (PI-DSP-QCK-003)
- 고객 상태나 권한 때문에 실행할 수 없는 퀵메뉴는 비활성 또는 대체 안내로 제공한다.
- 실행 불가 사유와 필요한 다음 행동을 함께 안내한다.

4) 개인화 노출·숨김 정책 (PG-DSP-PER-001)

• 개인화 근거 (PI-DSP-PER-001)
- 개인화 노출은 보유 상품, 최근 행동, 진행 업무, 숨김 이력, 관심없음 이력, 동의 범위 안의 데이터만 사용한다.
- 민감정보는 노출 조건으로 직접 사용하지 않는다.
• 숨김 반영 (PI-DSP-PER-002)
- 고객이 숨김 또는 관심없음을 선택하면 동일 소재는 최소 30일 동안 재노출하지 않는다.
- 동일 주제의 전시는 후순위로 낮추고 대체 후보를 제공한다.
• 비개인화 폴백 (PI-DSP-PER-003)
- 개인화 동의가 없거나 고객 상태 판정이 실패하면 공통 기본 전시를 제공한다.
- 이 경우 개인화 실패 사유를 내부 이력으로 저장하고 고객에게 오류성 문구를 노출하지 않는다.
• BSS·분석 연계 실패 폴백 (PI-DSP-PER-004)
- BSS 또는 분석 연계 시스템이 3초 안에 고객 상태·보유 관계 결과를 회신하지 못하면 고객별 전시를 확정하지 않고 공통 기본 전시로 대체한다.
- 실패 원인, 재시도 여부, 대체 전시 적용 결과는 개인화 판정 이력으로 저장한다.

5) 표준 컴포넌트·자산 정책 (PG-DSP-CMP-001)

• 표준 컴포넌트 (PI-DSP-CMP-001)
- 배너, 카드, 리스트, 캐러셀, 코너 유형은 표준 컴포넌트로 등록된 유형만 게시할 수 있다.
- 신규 유형은 필수 속성, 허용 데이터 구조, 반응형 규칙, 접근성 기준을 먼저 정의한다.
• 미디어 규격 (PI-DSP-CMP-002)
- 이미지와 동영상은 포맷, 용량, 해상도, 대체텍스트, 저작권 확인값을 필수 충족해야 등록할 수 있다.
- 규격 미달 소재는 검수 요청 전 단계에서 등록 불가로 차단한다.
• 사용처 추적 (PI-DSP-CMP-003)
- 동일 미디어 자산이 2개 이상 전시에 사용되면 사용처와 영향 범위를 조회할 수 있어야 한다.
- 교체 시에는 적용 대상과 예약 게시 건을 먼저 안내한다.
• 메뉴·탭 운영 (PI-DSP-CMP-004)
- 메뉴와 탭은 고객 과업 단위로 묶고 동일 업무가 여러 위치에 중복 노출되지 않도록 한다.
- 메뉴명은 서비스 내부 분류보다 고객 행동 목적을 우선한다.
• 어드민 메뉴 설정 (PI-DSP-CMP-005)
- 운영자는 상단, 하단, 햄버거 메뉴 등 전체 메뉴 구조를 생성, 수정, 정렬, 숨김 처리할 수 있다.
- 메뉴 깊이, 레이블, 아이콘, 연결 대상 URL·딥링크·페이지는 승인된 범위 안에서만 변경한다.
• 어드민 탭·서브탭 운영 (PI-DSP-CMP-006)
- 운영자는 화면 내 탭과 서브탭을 추가, 삭제, 순서 변경하고 탭별 연결 페이지, 섹션, 필터 상태를 설정할 수 있다.
- 탭 변경은 고객 과업 흐름을 끊지 않는 범위에서만 허용한다.
• 페이지 빌더 배치 기준 (PI-DSP-PGE-004)
- 페이지 빌더는 홈과 카테고리의 전시 슬롯, 섹션, 그리드, 간격을 운영자가 조정할 수 있게 한다.
- 배치 변경은 표준 컴포넌트 규격과 디바이스별 미리보기 기준을 필수 통과해야 한다.
• 미디어 자산 라이브러리 (PI-DSP-AST-004)
- 이미지, 동영상, 아이콘 등 미디어 자산은 폴더, 태그, 검색, 사용처 기준으로 관리한다.
- 업로드 시 포맷, 용량, 해상도, 썸네일, 리사이징 기준을 검증한다.
• 푸터 고객센터 정보 (PI-DSP-CMP-009)
- 메인 푸터의 고객센터 전화번호, 운영시간, 상담 채널 정보는 운영자가 관리하며 변경 시 즉시 반영할 수 있어야 한다.
- 변경 전후 값과 반영 시각을 이력으로 저장한다.
• 탐색 위치 인디케이터 (PI-DSP-CMP-010)
- 팝업 또는 캐러셀형 전시는 전체 항목 수와 현재 탐색 위치를 인디케이터로 제공한다.
- 좌우 이동 시 위치 정보와 다음·이전 이동 가능 여부를 함께 갱신한다.
• 캐시 입력 범위 (PI-DSP-CMP-011)
- 전시 캐시 적용 시간은 최소 0분, 최대 24시간 범위 안에서 입력하도록 검증한다.
- 긴급 공지, 장애 안내, 가격·혜택 변경 전시는 캐시 제외 또는 5분 이하 기준을 우선 적용한다.
• 사용여부 영향 확인 (PI-DSP-CMP-012)
- 운영자가 전시 사용여부를 변경하면 연결 페이지, 슬롯, 딥링크, 캠페인, 진행 중 예약 게시, 프론트 노출 영향 범위를 먼저 확인한다.
- 영향 범위 확인 전에는 비활성 처리를 허용하지 않는다.

6) 랜딩·딥링크 정책 (PG-DSP-LNK-001)

• 딥링크 실패 (PI-DSP-LNK-001)
- 앱 미설치, 버전 미지원, 링크 만료, 외부 오류가 발생하면 대체 랜딩 또는 고객센터 안내로 전환한다.
- 실패와 대체 경로 선택 결과는 링크 품질 이력으로 저장한다.
• 캠페인 파라미터 (PI-DSP-LNK-002)
- 랜딩 변경 시 기존 캠페인 파라미터와 추적값은 유지한다.
- 추적값이 사라지면 게시 전 오류로 분류하고 운영자 확인 전까지 게시를 제한한다.
• 상담 플로팅 분기 (PI-DSP-LNK-003)
- 상담 플로팅 버튼은 고객의 무선·유선 구분과 상담 가능 채널에 따라 전화상담 또는 채팅상담 진입을 분기한다.
- 분기 기준을 판정하지 못하면 공통 상담 홈으로 연결한다.

7) 랜딩·복귀 맥락 정책 (PG-DSP-LND-001)

• 랜딩 일치 (PI-DSP-LND-001)
- 전시 소재의 핵심 메시지, 대상 고객, 혜택, CTA는 랜딩 화면의 첫 화면 정보와 일치해야 한다.
- 랜딩 불일치가 발견되면 게시 전 차단하고 게시 중이면 중지 후보로 분류한다.
• 맥락 복귀 (PI-DSP-LND-002)
- 랜딩 후 뒤로 가기 또는 복귀 시 고객의 이전 홈, 검색어, 비교 대상, 진행 업무 맥락을 유지한다.
- 민감 입력값은 복귀 맥락에 저장하지 않는다.

8) 노출 조건·우선순위·충돌 정책 (PG-DSP-RUL-001)

• 충돌 우선순위 (PI-DSP-RUL-001)
- 동일 슬롯에서 여러 전시가 조건을 만족하면 필수 알림, 고객 진행 업무, 만료 임박 혜택, 고객 상태 관련성, 사업 우선순위 순으로 노출한다.
- 동순위 충돌은 운영자가 지정한 우선순위를 따른다.
• 노출 빈도 (PI-DSP-RUL-002)
- 동일 고객에게 동일 소재는 하루 최대 3회, 동일 주제 전시는 하루 최대 5회까지 노출한다.
- 고객이 닫기 또는 숨김을 선택하면 해당 일자의 반복 노출을 제한한다.
• 기간 자동 종료 (PI-DSP-RUL-003)
- 시작 시각과 종료 시각이 있는 전시는 종료 시각 도래 즉시 자동 비노출 처리한다.
- 종료 후 재노출하려면 새 버전 또는 재승인 이력을 필수로 남겨야 한다.
• 캐시·반영 방식 (PI-DSP-RUL-004)
- 전시 변경은 즉시 반영, 예약 반영, TTL 반영, 캐시 무효화 대상을 구분한다.
- 가격, 혜택, 가입 가능 여부, 장애 안내처럼 고객 판단에 직접 영향을 주는 항목은 긴급 무효화 대상이다.
• 비선호 노출 제한 (PI-DSP-RUL-005)
- 고객이 숨김, 관심 없음, 수신 거부, 반복 미클릭으로 명시한 항목은 동일 캠페인 또는 유사 추천군에서 노출 빈도를 제한한다.
- 법정 고지와 필수 업무 안내는 비선호 제한보다 우선한다.
• 중복 요청·멱등 처리 (PI-DSP-RUL-006)
- 동일 전시 버전, 동일 슬롯, 동일 고객 조건으로 5분 안에 반복 게시·숨김·캐시 무효화 요청이 들어오면 첫 요청의 처리 결과를 재사용한다.
- 중복 요청은 추가 노출이나 중복 이력을 만들지 않고 동일 요청 ID 기준으로 멱등 처리한다.

9) 검수·승인·게시 정책 (PG-DSP-WFL-001)

• 승인 전 게시 차단 (PI-DSP-WFL-001)
- 작성, 검수, 승인, 배포 권한은 분리한다.
- 승인 완료 전 상태의 전시는 실서비스 게시와 예약 게시를 모두 차단한다.
• 게시 전 자동 검증 (PI-DSP-WFL-002)
- 예약 게시 전에는 링크 유효성, 필수 입력, 이미지 규격, 컴포넌트 호환성, 접근성, 노출 충돌을 자동 검증한다.
- P1 오류가 1건이라도 있으면 게시할 수 없다.
• 미리보기 범위 (PI-DSP-WFL-003)
- 미리보기는 모바일, PC, OS, 앱 버전, 로그인 여부, 고객 세그먼트, 보유 상품 상태를 기준으로 확인할 수 있어야 한다.
- 검수자는 최소 1개 모바일 조건과 1개 PC 조건을 확인한다.
• 디바이스별 미리보기 (PI-DSP-WFL-004)
- 전시 승인 전 미리보기는 모바일, 웹, 태블릿, 접근성 확대 글자, 다크모드 등 주요 표시 조건을 포함한다.
- 핵심 CTA와 가격·혜택 정보가 가려지는 경우 승인할 수 없다.
• 메뉴·탭·필터 버전관리 (PI-DSP-WFL-005)
- 메뉴, 탭, 필터 설정은 버전으로 관리하고 작성자, 변경 사유, 변경점, 이전 버전 비교 정보를 저장한다.
- 배포 후 장애나 오노출이 발생하면 승인된 이전 버전으로 복원할 수 있어야 한다.
• 퇴출 승인 비가역 처리 (PI-DSP-WFL-006)
- 전시 퇴출 요청은 항목별 검토자가 병렬로 검토·승인할 수 있다.
- 퇴출 승인 후에는 원본 삭제가 아니라 비가역 Flag 변경으로 처리하고 수정 불가 상태와 승인 이력을 남긴다.
• 상품군 예외 승인 (PI-DSP-WFL-007)
- 상품군을 분리 관리하거나 예외 전시를 적용할 때는 예외 사유, 대상 상품군, 적용 기간, 프론트 영향 범위, 승인자를 기록해야 한다.
- 승인 전에는 실서비스에 반영하지 않는다.
• 게시 반영 결과 회신 (PI-DSP-WFL-008)
- 게시 또는 캐시 무효화 요청 후 채널 업무 시스템은 채널별 성공, 실패, 지연 상태를 회신해야 한다.
- 실패 채널은 게시 중지 또는 수정 필요 상태로 전환하고, 기준 데이터 버전, 반영 시각, 회신 결과를 운영 이력으로 저장한다.

10) 긴급 중지·롤백 정책 (PG-DSP-RBK-001)

• 긴급 중지 조건 (PI-DSP-RBK-001)
- 오노출, 만료 소재 노출, 잘못된 랜딩, 법무·심의 이슈, 고객 피해 가능성, 핵심 흐름 장애가 확인되면 운영자는 즉시 게시 중지를 실행할 수 있다.
• 롤백 허용 버전 (PI-DSP-RBK-002)
- 롤백은 승인 완료 이력이 있는 이전 버전 또는 기본 전시 버전으로만 허용한다.
- 임시 초안이나 검수 반려 버전으로 복원할 수 없다.
• 대체 전시 (PI-DSP-RBK-003)
- 긴급 중지 시 동일 슬롯에 대체 전시가 등록되어 있으면 자동 전환한다.
- 대체 전시가 없으면 기본 안내 또는 빈 슬롯 정책을 적용한다.

11) AI 운영 보조 정책 (PG-DSP-AI-001)

• AI 제안 승인 (PI-DSP-AI-001)
- AI가 생성한 제목, 서브카피, CTA, 이미지, 배치 초안은 운영자 수정·검수·승인 후에만 게시할 수 있다.
- 자동 게시와 무검수 반영은 허용하지 않는다.
• AI 검증 항목 (PI-DSP-AI-002)
- AI 제안은 금칙어, 법무 문구, 브랜드 가이드, 저작권, 상표, 유해성, 접근성 대체텍스트 검증을 필수 통과해야 한다.
- 하나라도 실패하면 검수 요청 단계로 넘길 수 없다.
• 성과 검증 (PI-DSP-AI-003)
- AI 배치 제안은 클릭률, 전환율, 탐색 성공률, 셀프처리율 중 2개 이상 지표로 효과를 검증한다.
- 저성과 또는 부정 반응 증가는 자동 적용 확대 사유가 될 수 없다.

12) 실험·가드레일 정책 (PG-DSP-ABT-001)

• 실험 사전 정의 (PI-DSP-ABT-001)
- A/B 실험은 시작 전 가설, 대상 고객군, KPI, 기간, 표본수, 유의수준, 중지 기준을 등록해야 한다.
- 정의되지 않은 실험은 노출할 수 없다.
• 핵심 흐름 가드레일 (PI-DSP-ABT-002)
- 결제, 인증, 가입, 변경, 상담 전환 흐름은 실패율, CS 유입, 크래시가 사전 임계치를 초과하면 실험을 중지한다.
- 중지 기준은 실험 등록 시 필수값으로 둔다.
• 결과 반영 (PI-DSP-ABT-003)
- 실험 종료 후 결과, 해석, 리스크, 통계적 신뢰도, 전체 적용 여부를 기록한다.
- 실험 등록 시 지정한 신뢰도 기준을 충족하지 못하면 전체 적용은 불가하며 조기 종료 결과는 전체 적용 근거로 단독 사용하지 않는다.

13) 성과 측정·개선 정책 (PG-DSP-MET-001)

• 성과 지표 (PI-DSP-MET-001)
- 전시 성과는 노출, 클릭, CTR, 랜딩 이탈, 후속 행동, 전환, 닫기, 숨김, 관심없음 중 최소 2개 이상 지표를 함께 본다.
- 클릭률만으로 성공 여부를 판정하지 않는다.
• 피로도 기준 (PI-DSP-MET-002)
- 동일 주제 반복 노출, 숨김 증가, 관심없음 증가, 홈 이탈률 증가가 동시에 나타나면 전시 피로도 위험으로 판정한다.
- 위험 판정 시 후순위, 빈도 제한, 소재 교체 중 하나를 적용한다.
• 개선 반영 (PI-DSP-MET-003)
- 성과 리포트는 지면, 고객군, 메시지 유형, 랜딩 유형, 기간 기준으로 비교한다.
- 개선안은 신규 버전 또는 실험 계획으로 등록해야 운영 반영할 수 있으며, 반영 전 승인 이력을 저장한다.
• 채널·OS·버전별 결과 분해 (PI-DSP-MET-004)
- 전시 실험과 성과 리포트는 채널, OS, 앱 버전, 디바이스 성능 구간별로 분해해 효과와 부작용을 확인한다.
- 특정 환경에서 실패율이나 이탈률이 기준보다 높으면 전체 확대를 보류한다.

14) 권한·감사 이력 정책 (PG-DSP-AUD-001)

• 역할 권한 (PI-DSP-AUD-001)
- 작성, 검수, 승인, 배포, 긴급 중지, 롤백 권한은 역할별로 분리한다.
- 동일 사용자가 작성과 최종 승인을 모두 수행할 수 없도록 제한한다.
• 감사 로그 (PI-DSP-AUD-002)
- 메뉴, 페이지, 컴포넌트, 소재, 노출 규칙, 배포 상태 변경은 변경자, 변경일시, 변경 대상, 변경 사유, 이전·이후 값을 저장한다.
- 감사 로그는 최소 1년 보관한다.
• 고객 제어 이력 (PI-DSP-AUD-003)
- 퀵메뉴 고정, 순서 변경, 숨김, 관심없음, 초기화 이력은 고객 제어 이력으로 저장한다.
- 개인화 판정에는 최신 고객 선택을 우선 적용한다.
• 개인정보·로그 보호 (PI-DSP-AUD-004)
- 전시 로그에는 고객 식별자, 세그먼트, 클릭, 숨김, 관심없음, CTA 결과를 최소 범위로 저장한다.
- 민감정보는 저장하지 않고 분석용 식별자는 마스킹 또는 대체키로 처리한다.
• 운영 이력 보관·파기 (PI-DSP-AUD-005)
- 운영 변경, 승인, 배포, 롤백, 고객 제어 이력은 감사 목적에 따라 최소 1년 보관한다.
- 보관 기간이 지난 로그는 통계 목적 집계값을 제외하고 파기한다.

15) 접근성·표기 가이드 정책 (PG-DSP-ACC-001)

• 핵심 메시지 (PI-DSP-ACC-001)
- 배너와 카드는 상품명보다 고객 가치, 혜택, 사용 가능 시점, 다음 행동을 먼저 표기한다.
- 한 소재 안의 핵심 메시지는 1개로 제한하고 보조 문구는 2개 이하로 둔다.
• 대체텍스트 (PI-DSP-ACC-002)
- 이미지형 전시는 대체텍스트를 필수로 둔다.
- 대체텍스트에는 고객이 알아야 할 혜택, 기간, 제한 조건 중 핵심 1개 이상을 포함한다.
• 가격·혜택 조건 (PI-DSP-ACC-003)
- 가격, 할인, 포인트, 쿠폰, 결합 혜택은 적용 기간, 대상, 제외 조건, 예상·확정 여부를 함께 표기한다.
- 조건이 긴 경우 상세 안내로 연결하되 핵심 제한은 전시 본문에 남긴다.

최종 점검 기준

전시/관리 기능 정책서 제출 전 점검
1. 범위 정합성
홈, 메뉴, 배너, 퀵메뉴, 개인화, 운영 배포·롤백 기준이 포함되고 주문·가입·결제 같은 후속 업무 상세는 제외되어야 한다.
전시는 단순 화면 배치가 아니라 고객 여정의 확인, 이해, 실행, 완료를 연결하는 통합 기준으로 정의되어야 한다.
2. 고객 여정 완결성
고객이 현재 상태, 필요한 정보, 다음 행동, 숨김·복귀 경로를 이해할 수 있어야 한다.
고객이 전시를 통해 조회, 신청, 변경, 사용, 상담 전환 중 어떤 행동으로 이어질 수 있는지 예측 가능해야 한다.
3. 운영 통제성
작성, 검수, 승인, 예약, 게시, 중지, 롤백 상태와 권한이 분리되어야 한다.
운영자 변경은 승인자, 적용 시점, 적용 범위, 회수 조건, 변경 이력이 함께 남아야 한다.
4. 전시 우선순위·충돌 제어
필수 알림, 진행 중 업무, 고객 상태, 혜택, 최근 이용 기능의 우선순위가 정책으로 정의되어야 한다.
동일 슬롯에 2개 이상 후보가 경쟁하면 필수성, 고객 영향도, 마감 임박, 고객 제어 이력 순으로 판정해야 한다.
5. 개인화·권한 보호
고객 숨김, 관심없음, 동의 범위, 비개인화 폴백 기준이 있어야 한다.
민감한 상태나 요금·혜택 정보는 고객 본인 권한과 동의 범위가 확인된 경우에만 개인화 전시에 사용할 수 있어야 한다.
6. 프로세스-기능-정책 연결성
모든 프로세스는 관련 기능과 정책 ID에 연결되어야 한다.
기능 세부 구성은 짧은 하위 처리명으로 작성하고 노출 조건, 승인 조건, 빈도 제한, 이력 기준은 정책 상세로 분리해야 한다.
7. 상태 전이 정합성
검수, 승인, 예약, 게시, 중지, 종료, 롤백, 개인화 제한 상태가 상태 목록과 전이표에서 일치해야 한다.
고객 노출 상태와 운영 내부 상태가 섞이지 않아야 하며 상태별 후속 행동이 정의되어야 한다.
8. 정책 구체성
정책 항목에는 기간, 횟수, 우선순위, 제한, 승인, 이력, 보관 기준 중 하나 이상이 있어야 한다.
